# WRK-1178: Calculation Report Skill

## Context

Engineering calculations across digitalmodel/assetutilities lack a consistent reporting format. Formulas, units, and methodology are embedded in code but never rendered into reviewable documents. This skill creates a **YAML → Markdown → HTML** pipeline that produces professional calculation reports with rich formulas and interactive charts, reusing the workflow-html warm-parchment design system.

## Pipeline

```
calculation.yaml  →  generate-calc-report.py  →  report.md  →  report.html
(source of truth)     (generator script)          (LaTeX math)   (default output)
                                                               →  report.pdf
                                                                  (optional, via md-to-pdf)
```

## YAML Schema Design

```yaml
# config/reporting/calculation-report-schema.yaml
# Every calculation YAML must conform to this structure

metadata:
  title: str           # required
  doc_id: str           # e.g. "CALC-001"
  revision: int         # starts at 1, incremented on changes
  date: str             # ISO date
  author: str
  reviewer: str         # optional
  status: str           # draft | reviewed | approved
  project: str          # optional
  change_log:           # revision history
    - rev: int
      date: str
      description: str

inputs:                 # list of input parameters
  - name: str           # human-readable name
    symbol: str         # LaTeX symbol e.g. "S_{range}"
    value: float|str
    unit: str           # e.g. "MPa", "m", "kN"
    source: str         # optional — where value comes from
    notes: str          # optional

methodology:
  description: str      # prose description of approach
  standard: str         # e.g. "DNV-RP-C203"
  equations:            # numbered equations
    - id: int
      name: str         # e.g. "Palmgren-Miner damage"
      latex: str        # e.g. "D = \\sum_{i=1}^{k} \\frac{n_i}{N_i}"
      description: str  # what the equation computes
      variables:        # optional — define each variable
        - symbol: str
          description: str
          unit: str

outputs:                # list of output parameters
  - name: str
    symbol: str
    value: float|str
    unit: str
    pass_fail: str      # optional: "pass" | "fail" | null
    limit: float        # optional: acceptance threshold
    notes: str

assumptions:            # list of strings
  - str

references:             # list of strings (standards, reports)
  - str

charts:                 # optional — interactive chart definitions
  - id: str             # e.g. "sn_curve"
    title: str
    type: str           # line | bar | scatter | log_log
    x_label: str
    y_label: str
    x_unit: str
    y_unit: str
    datasets:
      - label: str
        data: [[x, y], ...]  # or reference to a CSV/data key
        color: str            # optional
    x_scale: str         # linear | log (default: linear)
    y_scale: str         # linear | log (default: linear)

data_tables:            # optional — tabular data for display
  - id: str
    title: str
    columns:
      - name: str
        unit: str
    rows: [[...], ...]
```

## HTML Output Design

Reuse workflow-html warm-parchment design system with these sections:

1. **Hero header** — doc_id, title, status badge, revision, date, author
2. **Metadata chips** — project, standard, reviewer, status (pill grid like workflow-html)
3. **Inputs card** — table: Name | Symbol | Value | Unit | Source
4. **Methodology card** — prose + numbered equations rendered via KaTeX
5. **Outputs card** — table: Name | Symbol | Value | Unit | Pass/Fail | Limit
   - Pass/fail badges (green/red) from workflow-html badge system
6. **Charts card** — interactive Chart.js charts (inlined, no CDN)
7. **Data Tables card** — damage contributions, histograms, etc.
8. **Assumptions & References** — collapsible lists
9. **Change Log** — revision history table
10. **Footer** — "Generated by calculation-report skill"

### Math Rendering

**KaTeX** (inline bundle, ~300KB minified). Rationale:
- Fast rendering (10× faster than MathJax)
- Can be fully inlined (no CDN dependency)
- Supports all LaTeX math needed for engineering formulas
- Works in `file://` URLs (no server needed)

### Charts

**Chart.js** (inline bundle, ~200KB minified). Rationale:
- Interactive (hover tooltips, zoom via plugin)
- Log-log scale support (critical for S-N curves)
- Canvas-based (fast rendering)
- Single file, no CDN dependency

Both KaTeX and Chart.js will be bundled as inline `<script>` blocks in the HTML, fetched once and cached in `config/reporting/vendor/`.

## Generator Script Design (`generate-calc-report.py`)

```
Usage: uv run --no-project python scripts/reporting/generate-calc-report.py <input.yaml> [--format html|md|pdf] [--output <path>]

Default: --format html
```

**Functions:**
1. `load_and_validate(yaml_path)` — load YAML, validate against schema
2. `render_markdown(calc_data)` — produce markdown with LaTeX math blocks
3. `render_html(calc_data, markdown_str)` — produce HTML with KaTeX + Chart.js
4. `render_pdf(html_path)` — optional: call Chrome headless or md-to-pdf skill
5. `main()` — CLI entry point

**Markdown output structure:**
```markdown
# {title}
**{doc_id}** | Rev {revision} | {date} | {author}

## Inputs
| Name | Symbol | Value | Unit | Source |
|------|--------|-------|------|--------|
| ... | $S_{range}$ | 80 | MPa | DNV-RP-C203 Table 2-1 |

## Methodology
{description}

**Standard:** {standard}

### Equation 1: {name}
$$D = \sum_{i=1}^{k} \frac{n_i}{N_i}$$
{description}

## Outputs
| Name | Symbol | Value | Unit | Status | Limit |
|------|--------|-------|------|--------|-------|
| Total Damage | $D$ | 0.45 | — | PASS | ≤ 1.0 |

## Assumptions
1. ...

## References
1. ...
```

## Example YAML (fatigue-pipeline-girth-weld.yaml)

Serialised from `ExampleResult` returned by `pipeline_girth_weld()`:
- **Inputs:** S-N curve (DNV D, seawater+CP), SCF=1.5, DFF=3.0, design life=20yr, Weibull h=0.9/scale=80MPa
- **Methodology:** Palmgren-Miner linear damage accumulation with rainflow counting
- **Equations:** S-N curve ($\log N = \log a - m \log S$), Miner's rule ($D = \sum n_i/N_i$)
- **Outputs:** total damage, factored damage (D×DFF), life years, pass/fail
- **Charts:** S-N curve (log-log), stress histogram (bar), damage contribution (bar)
- **Data tables:** top 15 damage contributors

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Math rendering | KaTeX (inlined) | Fast, no CDN, supports engineering LaTeX |
| Charts | Chart.js (inlined) | Interactive, log-log support, no CDN |
| Default output | HTML | Interactive, stakeholder-friendly |
| PDF option | Chrome headless --print-to-pdf | Reuses HTML output, high fidelity |
| Design system | Warm-parchment (from workflow-html) | Consistent look across all WRK/report artifacts |
| YAML validation | jsonschema on YAML-loaded dict | Standard approach, schema in YAML |
