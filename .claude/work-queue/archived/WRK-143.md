# LineageGraph HTML/SVG Rendering

## What
Extend `LineageGraph` from WRK-095 to render provenance DAGs as standalone HTML or SVG files, beyond the current DOT text export.

## Why
Current `to_dot()` produces Graphviz DOT text requiring external tools to render. Engineers need one-click viewable output (HTML with embedded SVG) for audit reports and documentation.

## Scope
1. Add `LineageGraph.to_svg()` using graphviz Python package (optional dependency)
2. Add `LineageGraph.to_html()` wrapping SVG in a standalone HTML page
3. Fallback: if graphviz not installed, provide a simple HTML table layout
4. Tests for both rendering paths

## Acceptance Criteria
- [x] `.to_svg()` renders graph as SVG string (requires graphviz)
- [x] `.to_html()` produces standalone viewable HTML file
- [x] Graceful fallback when graphviz is not installed
- [x] Tests for both rendering paths
