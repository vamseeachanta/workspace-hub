# Audit and migrate scheduled jobs to claude code schedule tasks on all workstations

## Mission
**Review all expected cron jobs across every workstation (ace-linux-1, ace-linux-2, acma-ansys05, acma-ws014, gali-linux-compute-1) and convert them into Claude Code scheduled tasks where appropriate; out of scope: changing the logic of existing jobs.**

> Read this at the start of every execution stage. If a task, file change, or decision is
> not directly in service of this mission, stop and capture it as a new WRK item instead.

## What
Audit the current cron/Task Scheduler entries on all workstations, compare against the expected schedule documented in the workstations SKILL.md (`cron_variants`), identify gaps or drift, and convert the canonical jobs into Claude Code schedule tasks (using the Claude Code scheduling API or config) so they are version-controlled, visible, and provider-agnostic.

## Why
Cron jobs are currently defined per-machine with no single source of truth. Migrating to Claude Code scheduled tasks provides visibility in the workspace-hub, unified management, and makes the harness self-describing.

## Acceptance Criteria
- [ ] All workstations audited: ace-linux-1, ace-linux-2, acma-ansys05, acma-ws014, gali-linux-compute-1
- [ ] Inventory document listing every existing cron/Task Scheduler job per machine (schedule, command, owner, purpose)
- [ ] Gap analysis: expected jobs (from workstations SKILL.md `cron_variants`) vs actual jobs found
- [ ] Claude Code schedule task definitions created for all jobs that can run via Claude Code
- [ ] Jobs that cannot migrate (hardware-specific, GUI, license-bound) documented with rationale
- [ ] Version-controlled schedule config committed to workspace-hub
- [ ] Tests: verify schedule task files parse correctly; dry-run each task definition

## Agentic AI Horizon

- This directly boosts agentic AI capability: scheduled tasks drive the nightly harness (comprehensive-learning, repo-sync, AI readiness checks).
- Converting from bare cron to Claude Code scheduled tasks means agents can introspect and modify the schedule — a key capability unlock for self-managing workflows.
- Invest now: the groundwork (inventory + migration) is well-defined and worth doing before the harness grows further.

### What "Claude Code schedule tasks" means here
The existing `claude --dangerously-skip-permissions --skill <skill-name>` invocations
in cron ARE Claude Code scheduled tasks. The deliverable is a canonical
`config/scheduled-tasks/schedule-tasks.yaml` that declares every task (schedule,
machine, command, purpose) and becomes the single source of truth that `setup-cron.sh`
reads, replacing its hardcoded ENTRIES arrays.

### Phase 1 — Create `schedule-tasks.yaml` source of truth
- **File**: `config/scheduled-tasks/schedule-tasks.yaml` (new directory)
- **Schema**: one entry per task — `id`, `label`, `schedule`, `machines[]`, `command`,
  `log`, `is_claude_task`, `description`
- Include all 6 known jobs from ace-linux-1; mark ace-linux-2 scope; flag Windows stubs
- Run: manual file creation

### Phase 2 — Fix `setup-cron.sh` to read from YAML
- **File**: `scripts/cron/setup-cron.sh` (edit)
- Replace hardcoded `ENTRIES=()` arrays with `uv run --no-project python` reader of
  `schedule-tasks.yaml` filtered by `machines` + current hostname
- Keep `--dry-run` flag
- Test: `bash scripts/cron/setup-cron.sh --dry-run` on ace-linux-1

### Phase 3 — Fix script path drift in actual crontabs
- ace-linux-1: update 3 wrong-path entries in live crontab to match
  `schedule-tasks.yaml` canonical paths
- ace-linux-2: add missing repo-sync; remove jobs that should only be on ace-linux-1
  (or promote ace-linux-2 to `full` role and document rationale)
- Create wrapper `scripts/cron/skills-curation.sh` (tiny `exec claude ...` wrapper)

### Phase 4 — Document Windows tasks + tests
- Append Windows Task Scheduler YAML stubs to `schedule-tasks.yaml`
- Write `scripts/cron/validate-schedule.py` — parses `schedule-tasks.yaml` and
  validates cron expressions + required fields (unit test)
- Tests: 1 test per task entry; 1 dry-run smoke test on ace-linux-1
