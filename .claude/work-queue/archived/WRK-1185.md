# Publish assetutilities 0.0.9 to PyPI

## Mission
Publish a new assetutilities release to PyPI so digitalmodel CI can resolve dependencies correctly.

## What
PyPI assetutilities 0.0.8 is missing `assetutilities.common.path_resolver` (and possibly other modules added since last release). digitalmodel Quality Gates CI installs from PyPI via `UV_NO_SOURCES=true` and fails at test time because the import fails.

## Why
WRK-1167 fixed the CI workflow files but the Quality Gates tests still fail because the PyPI package is stale. A new release unblocks green CI on digitalmodel main.
