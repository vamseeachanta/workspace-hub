# feat(doc-intel): batch extraction pipeline — 1M doc queue with cost tracking

## Mission
Create extraction-queue.yaml management scripts, batch runner that processes queue with rate limiting, cost tracking per doc via wrk_cost_report.py integration, queue health dashboard.

## Why
Scaling from single-document extraction to a 1M document corpus requires queue management, rate limiting, cost visibility, and failure recovery.

## Acceptance Criteria
- [ ] extraction-queue.yaml schema for managing document queue
- [ ] Batch runner processes queue entries sequentially with configurable rate limiting
- [ ] Cost tracking per document via wrk_cost_report.py integration
- [ ] Queue health summary: processed/pending/failed counts
- [ ] Failed documents tracked with error reason for retry
- [ ] Resume capability: restart from last successful document
- [ ] CLI: --batch-size, --rate-limit, --resume flags

## TDD Approach
Test queue YAML parsing. Test batch runner with mock queue. Test cost tracking integration. Test resume from checkpoint. Test failure tracking and retry logic.
