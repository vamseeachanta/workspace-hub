## Plan

**Route B — inline plan (rev 3 — addresses cross-review REQUEST_CHANGES rounds 1+2)**

**Phase 1: Shell scripts** (`scripts/work-queue/`)
- Write `new-feature.sh WRK-NNN` (two-pass + QUEUE_ROOT injection):
  - Accepts `WORK_QUEUE_ROOT` env var (default: `$(git rev-parse --show-toplevel)/.claude/work-queue`) for testability
  - Pass 1: parse `## Decomposition` table; allocate/adopt WRK IDs; build `child_key → WRK-ID` map
  - Pass 2: resolve `Depends on` — concrete `WRK-NNN` values pass through unchanged; symbolic `child-key` values resolved via map; unknown keys → hard exit 1 with clear error; skipped adoption → key absent from map → explicit error, not silent omission
  - Adoption idempotency: if `parent:` == current feature → no-op; if `parent:` == different feature → hard exit 1 (integrity error, do not add to children list); if `parent:` absent → insert
  - Exit 1 when no children parsed; warn on failed adoption target lookup; inherit `category`/`subcategory`
- Write `feature-status.sh WRK-NNN`:
  - `WORK_QUEUE_ROOT` injection; robust `children:` parsing (handles inline and block-list YAML)
  - Count only `archived`; output: "N/M archived (X%)"
- Write `feature-close-check.sh WRK-NNN`:
  - Same `WORK_QUEUE_ROOT` injection + robust parsing; exit 0 iff all archived, exit 1 otherwise
- Make all three executable

**Phase 2: dep_graph.py --feature flag**
- Add `WORK_QUEUE_ROOT` override (env or `--queue-root` arg) for testability
- Add `FeatureTreeItem` dataclass — separate from `WRKItem` to preserve compute_graph()
- `--feature WRK-NNN`: search all queue dirs including `archived/`, `done/`, `archive/YYYY-MM/`; render ASCII tree

**Phase 3: generate-index.py — By Feature section**
- `WORK_QUEUE_ROOT` override support for tests
- Scan all queue dirs (including `archived/`, `done/`) for children completion counts
- Render: `| WRK-NNN | title | status | N/M archived (%) |`

**Test strategy (hermetic — `scripts/work-queue/tests/test-wrk1130-feature-tooling.sh`):**
- Create tmp-dir with synthetic queue dirs (`pending/`, `working/`, `archived/`)
- Populate: feature WRK with known Decomposition table; 3 child stubs (1 archived, 2 pending)
- Set `WORK_QUEUE_ROOT=$TMPDIR` for all invocations
- Assertions: `new-feature.sh` exits 0, child files created, `blocked_by` has concrete WRK IDs; `feature-status.sh` prints `1/3 archived (33%)`; `feature-close-check.sh` exits 1; `dep_graph.py --queue-root $TMPDIR --feature` exits 0; `generate-index.py` has `## By Feature`
- Edge case tests: block-list `children:`, adoption when parent matches, adoption when parent differs → hard fail, unresolved `Depends on` → hard fail
- Cleanup on exit (trap)
