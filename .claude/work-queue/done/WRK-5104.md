## Mission

Replace local-only human gates with GitHub issue-based approval. The issue
sections (Stage Progress, Plan, Implementation) become the review surface.
User confirms on GitHub; agent reads confirmation before proceeding.

## What

1. At human gate stages (1, 5, 7, 17), exit_stage.py posts an "awaiting
   approval" comment on the GitHub issue and stops
2. Agent checks the GitHub issue for user confirmation before proceeding
   past a human gate (comment, checkbox tick, or label)
3. enforce-human-gate.sh hook reads from GitHub issue (via `gh`) instead
   of relying on local state the agent can bypass
4. Issue sections (Plan, Implementation, Stage Progress, TDD Results)
   serve as the evidence the user reviews before approving

## Why

Local human gates are bypassed — agents skip start_stage.py/exit_stage.py
and commit directly (observed in WRK-5101, WRK-5102, WRK-5103). GitHub
is external state the agent cannot fake. User gets a single place to
review progress and approve stage transitions.

## Acceptance Criteria

- [x] Human gate stages post "awaiting approval" comment on GitHub issue
- [x] Agent blocks until user confirmation is detected on the issue
- [x] enforce-human-gate.sh reads approval from GitHub (not local files)
- [x] Works with existing issue sections (Plan, Implementation, Stage Progress)
- [x] Offline fallback: local approval still works when gh is unavailable
