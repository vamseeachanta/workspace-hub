# docs(digitalmodel): Map ANSYS .inp FEA Models to Structural/Riser Module Tests

## What

Map the 428 ANSYS `.inp` input files in the index to `specs/data-sources/digitalmodel.yaml`
as `fea_models:` entries, providing ground-truth model parameters for validation.

## FEA Model Types Identified (from index)

### VIV Modal Analysis (`common.inp` pattern)
Multiple copies across projects: Shell West Boreas, BP Macondo containment riser.
- Configurations: BW (Bare Wire), BTW (Bare + Top Weight), BTTW (Bare + Top Tension + Weight)
- Contains: pipe geometry (OD, WT, length), material E/density, boundary conditions
- Expected output: mode shapes and natural frequencies
- These provide ground-truth frequencies for `digitalmodel.structural.viv.modes`

### SCF Local FE Analysis
- `commandOBJ_SCF_WT1125_2.inp`
  - `/mnt/ace/docs/disciplines/production/projects/611_mecor_pipeline_installation_analysis/.../commandOBJ_SCF_WT1125_2.inp`
  - ANSYS local FE model for Stress Concentration Factor at WT=11.25mm
  - Relevant to fatigue S-N + SCF (DNV-RP-C203 / API RP 2A-WSD Sec 5)

### Wave Fatigue Load Cases (`Q_*.inp` pattern)
- `Q_07_7D.inp`, `Q_01_1D.inp` (7-day storm, 1-day operations)
  - `/mnt/ace/docs/disciplines/completion/projects/31245_shell_west_boreas_bop_on_tree_assessment/ANL/WF/`
  - ANSYS wave fatigue run files — define loading spectrum per sea state

## Mapping Plan

Add `fea_models:` section to `specs/data-sources/digitalmodel.yaml`:
```yaml
fea_models:
- type: ansys-modal-viv
  path_pattern: /mnt/ace/docs/disciplines/*/projects/*/ANL/VIV/MODES/*/common.inp

  domain: structural
  standards: [API-RP-2A-WSD-22nd, DNV-RP-F105]
  configurations: [BW, BTW, BTTW]
  host: ace-linux-1

- type: ansys-scf-local
  path: /mnt/ace/docs/.../commandOBJ_SCF_WT1125_2.inp
  description: Local FE SCF model — pipe fitting WT=11.25mm
  domain: structural
  standards: [API-RP-2A-WSD-22nd]
  host: ace-linux-1

- type: ansys-wave-fatigue
  path_pattern: /mnt/ace/docs/disciplines/*/projects/*/ANL/WF/*/Q_*.inp

  domain: structural
  standards: [DNV-OS-E301]
  host: ace-linux-1
```

## Value for Testing

The `common.inp` files contain pipe geometry and boundary conditions as comments.
Reading these gives expected natural frequencies that WRK-571 (VIV modes Python)
must reproduce — enabling automated regression tests.
