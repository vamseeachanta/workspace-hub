# Align resource-intelligence skill with Stage 2 micro-skill contract

## Mission
**Update `.claude/skills/workspace-hub/resource-intelligence/SKILL.md` so its outputs, micro-skill checklist, and exit artifact schema exactly match the Stage 2 contract defined in WRK-1028 — and verify it can serve as a drop-in task_agent micro-skill.**

## What
The `resource-intelligence` skill (v1.0.0) exists but its exit artifact is
`resource-intelligence-summary.md`, while the WRK-1028 Stage 2 contract requires
`evidence/resource-intelligence.yaml` with explicit fields:

- `completion_status` (`continue_to_planning` | `pause_and_revise`)
- `domain.problem`
- `domain.architecture_decision`
- `skills.core_used` (≥3 entries)
- `constraints`

The skill also lacks a micro-skill checklist and the "do NOT start implementation"
stop guard required for task_agent invocation.

## Why
WRK-1028 will generate Stage 2 task agents that invoke this skill. If the skill
output schema doesn't match what the stage contract expects, gate evidence
verification will fail and the stage will report missing artifacts.

## Acceptance Criteria

**Exit artifact schema:**
- [ ] `SKILL.md` documents `evidence/resource-intelligence.yaml` as the primary gate-passing exit artifact with full inline schema (`wrk_id`, `generated_at`, `stage`, `domain.problem`, `domain.architecture_decision`, `existing_files[]`, `skills.core_used[]` ≥3, `constraints[]`, `completion_status`, `top_p1_gaps[]`)
- [ ] `resource-intelligence-summary.md` demoted to optional companion (not removed)
- [ ] `templates/resource-intelligence-template.yaml` created with all fields pre-filled and inline comments

**Mining checklist (9 categories):**
- [ ] `## Resource Mining Checklist` section in SKILL.md — ordered table of 9 categories (Skills, Prior WRKs, Memory, Existing code/scripts, Specs/plans, Workspace docs, Document index, Mounted sources, Online/additive) with `Where to look` and `When relevant` columns

**Category→Mining Map:**
- [ ] `## Category→Mining Map` section — table mapping WRK `category` values (harness, engineering, data, platform, business, maintenance, personal) to which mining categories to prioritize and which to skip

**target_repos→paths map:**
- [ ] `## target_repos→Paths` section — table mapping repo names to the `src/`, `scripts/`, `specs/` paths to inventory during mining

**Micro-skill contract:**
- [ ] `## Micro-Skill Checklist (Stage 2)` — 6-item ordered checklist ending with stop guard
- [ ] Stop guard callout block: "do NOT start implementation — exit after writing evidence/resource-intelligence.yaml"

**Version:**
- [ ] `version: 1.0.0` → `1.1.0` with changelog entry

**Tests:**
- [ ] `verify-gate-evidence.py WRK-1029` passes on Stage 2 evidence already written
- [ ] `validate-resource-pack.sh` runs without error on a minimal test asset dir
