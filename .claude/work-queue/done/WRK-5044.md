# fix(cross-review): Codex output format compliance — structured headings

## Problem

Codex cross-review produces substantive findings but in free-form prose format.
`validate-review-output.sh` classifies the output as `INVALID_OUTPUT` because
it requires specific markdown headings:

- `### Verdict:`
- `### Summary`
- `### Issues Found`
- `### Suggestions`
- `### Questions for Author`

Codex instead uses `Full review comments:` with `- [P1]`/`[P2]` items inline.
This causes the Codex hard gate to fail even when the review content is valid.

## Evidence

Raw output preserved at:
`scripts/review/results/20260313T025945Z-commit-ea942f0bab-implementation-codex.raw.md`

## Possible Fixes

1. **Update `submit-to-codex.sh` prompt** to explicitly require the 5 heading format
2. **Add a format-normalizer** that post-processes Codex output into the expected structure
3. **Relax `validate-review-output.sh`** to accept alternative structured formats

Option 1 is preferred — scripts handle mechanics, prompts handle formatting.

## Plan

1. Read `scripts/review/submit-to-codex.sh` to find the prompt sent to Codex
2. Add explicit output format instructions requiring the 5 headings
3. Test by running a dry cross-review or inspecting the prompt string
4. Commit + push
