## Problem

Cron jobs are not consistently deployed across all workstations. Current state
as of 2026-02-23:

| Machine | session-analysis | model-ids | skills-curation | **comprehensive-learning** |
|---------|-----------------|-----------|-----------------|--------------------------|
| dev-primary | ✓ 3AM daily | ✓ Sun 3:30AM | ✓ Mon 4AM | ❌ MISSING |
| dev-secondary | ❌ NO CRONTAB | ❌ | ❌ | ❌ |
| licensed-win-1 | unknown | unknown | unknown | unknown |
| licensed-win-2 | unknown | unknown | unknown | unknown |

The comprehensive-learning nightly pipeline (v2.1.0) is fully built and tested
but has never been scheduled even on dev-primary, the designated host.

## Fix

### dev-primary (add comprehensive-learning)
```bash
# Add to crontab (run at 2AM daily, before session-analysis at 3AM)
0 2 * * * bash /mnt/local-analysis/workspace-hub/scripts/cron/comprehensive-learning-nightly.sh >> /tmp/comprehensive-learning.log 2>&1
```

### dev-secondary (full cron setup)
SSH in and establish complete crontab matching dev-primary schedule.
Path: `/mnt/workspace-hub` (different from dev-primary).

### Office machines (licensed-win-1, licensed-win-2)
Audit what is running; deploy session-analysis and model-ids as minimum.
comprehensive-learning runs only on dev-primary (single-machine guard in SKILL.md).

Both office machines are Windows workstations (role: `contribute-minimal`). Neither uses
cron — they use Windows Task Scheduler per `crontab-template.sh` and `setup-cron.sh`.

| Machine | Status | Notes |
|---------|--------|-------|
| licensed-win-1 | DNS unreachable from dev-primary | Office LAN (192.168.0.x), Tailscale needed |

Instructions are in `scripts/cron/setup-cron.sh` (run on Windows prints Task Scheduler steps)
and `.claude/docs/new-machine-setup.md` § Windows Task Scheduler.
comprehensive-learning is explicitly excluded from both (dev-primary only).
