# feat(diffraction): mesh generate CLI + GmshMeshBuilder exports

## What
Complete the remaining 15% of WRK-140 by adding a CLI subcommand for mesh generation
and registering GmshMeshBuilder in the diffraction package exports.

## Why
GmshMeshBuilder exists and has 26 passing tests, but is not accessible via CLI or the
diffraction package public API. This blocks downstream consumers (WRK-1249 parametric
studies, WRK-1252 CAD-to-CFD pipeline) from using it programmatically or from the
command line.

## Scope

### Deliverable 1: CLI subcommand
Add a `mesh-generate` Click subcommand to `hydrodynamics/diffraction/cli.py` that:
- Accepts a spec YAML file defining GmshMeshSpec fields (box_barge / cylinder geometry)
- Calls GmshMeshBuilder.build() with user-specified output formats (gdf/dat/msh)
- Outputs mesh files to a configurable directory
- Supports --dry-run for validation-only mode
- Follows existing Click patterns in cli.py (convert_spec, run_orcawave_cmd)

### Deliverable 2: Package exports
Add to `hydrodynamics/diffraction/__init__.py`:
- `GmshMeshBuilder`, `GmshMeshSpec`, `GMSH_AVAILABLE`
- Use try/except import guard matching existing ORCAWAVE_AVAILABLE pattern
- Add to `__all__`

## Acceptance Criteria
- [ ] `digitalmodel mesh-generate --spec spec.yaml --format gdf` works end-to-end
- [ ] `from digitalmodel.hydrodynamics.diffraction import GmshMeshBuilder` works
- [ ] TDD: tests for CLI subcommand (happy path, missing spec, dry-run)
- [ ] TDD: tests for __init__.py import with/without gmsh installed
- [ ] Legal scan: PASS
