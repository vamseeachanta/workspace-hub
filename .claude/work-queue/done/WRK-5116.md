## Mission

Establish a discrepancy tracking system using GitHub issue labels and structured comment conventions for flagging dimension, material, boundary condition, and load conflicts during engineering analysis.

## What

1. **Create GitHub labels** for discrepancy tracking:
   - `discrepancy` — broad catch-all
   - `discrepancy:dims` — dimension conflicts
   - `discrepancy:material` — material/spec conflicts
   - `discrepancy:bc` — boundary condition conflicts
   - `discrepancy:load` — load assumption conflicts

2. **Comment convention** for posting discrepancy notes to issues:
   - Inline tag: `DISC[CAT]: description` (e.g. `DISC[DIM]: tube OD = 1.75" per photo but 1.5" in model`)
   - Optional severity flag: `DISC[CAT|BLOCK]:` for blocking items
   - Parseable by grep/scripts for batch analysis

3. **Document** the convention in `.claude/docs/` or a skill for reuse

## Why

User is writing discrepancy notes to GitHub issues (see #1259 as example). Needs a consistent, searchable convention across issues that supports later bulk analysis and filtering. Labels enable GitHub Projects dashboard views.
