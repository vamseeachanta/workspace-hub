## Mission

Extract concrete input→output numerical test vectors from ALL naval architecture and marine engineering literature — both existing repo documents and document intelligence from drives — to enable TDD across all 15 engineering domain feature WRKs.

## What / Why

The digitalmodel repo contains 1,490+ engineering reference documents (PDFs, markdown, worked examples) across 15+ domains, plus a 1M-record document index on drives. This is an untapped goldmine of verified numerical data — standards worked examples, textbook problems, vendor datasheets, and project calculations — that should feed TDD for every calculation module.

Currently each feature WRK's child-c (implementation) has to hunt for test data ad-hoc. Extracting test vectors systematically up front means:
- Every implementation starts with verified input→output pairs (not self-referencing tests)
- Sources are traceable (standard, section, table, page)
- Fixtures are reusable across modules that share parameters (e.g. Cd/Cm used by both Morison and VIV)

### Tier 2: Document intelligence from drives
- Query document index (`scripts/data/document-index/index.jsonl` — 1,033,789 records)
- Search by engineering domain keywords, standard numbers, and organisations
- Identify high-value documents not yet in repo (standards with worked examples, vendor calc sheets)
- Flag paywalled vs. available documents

### Tier 3: Dark intelligence archives
- `knowledge/dark-intelligence/` — extracted inputs/outputs from legacy calculations
- Entries with `use_as_test: true` are pre-validated test vectors

## Output Structure

```
digitalmodel/tests/fixtures/
  propeller_rudder/        # child-a
  cathodic_protection/     # child-b
  mooring/                 # child-c
  structural/              # child-d
  hydrodynamics/           # child-e
  viv/                     # child-f

  metocean_diffraction/    # child-k
```

Each fixture file: YAML with `source`, `standard_ref`, `input_parameters`, `expected_output`, `tolerance`, `notes`.

- [ ] ≥10 vectors per HIGH-priority domain (CP, mooring, structural, hydrodynamics, VIV)
- [ ] ≥5 vectors per MEDIUM-priority domain
- [ ] Document index queried for ≥5 domains; high-value documents catalogued
- [ ] Dark intelligence entries with `use_as_test: true` incorporated
- [ ] Fixture YAML files parseable by pytest parametrize
- [ ] Cross-domain shared parameters identified (e.g. Cd/Cm, seawater density, yield stress)
