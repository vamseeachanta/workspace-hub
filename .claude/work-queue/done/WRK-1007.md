# fix(review): comprehensively test and troubleshoot submit-to-gemini.sh

## Problem

While fixing initial issues in `submit-to-gemini.sh` (like silent success on failure, unbound variables, and pathing fragilities), we recognized the need to further troubleshoot the script with more inputs to uncover other potential edge cases. 

Additionally, comprehensive test coverage is needed to ensure robustness and increase cross-review success rates across all agents (Gemini, Claude, Codex, etc.). Currently, there is a risk that unhandled edge cases in the submission script will cause the cross-review pipeline to fail intermittently.

## What

1. Troubleshoot `submit-to-gemini.sh` with a wider variety of inputs (e.g., various prompts, empty files, non-existent commit SHAs, missing CLI environments) to surface any remaining bugs.
2. Fix any newly discovered issues in the script to ensure rock-solid stability when invoked by automated agents.
3. Create comprehensive tests (unit and/or integration) for the cross-review submission scripts, focusing primarily on `submit-to-gemini.sh` but ensuring the patterns can be applied across the board.

## Acceptance Criteria

- [x] `submit-to-gemini.sh` is tested against a comprehensive set of edge-case inputs.
- [x] Any bugs discovered during troubleshooting are fixed.
- [x] A test suite (or automated test script) is created and documented for verifying the robustness of the submit scripts.
- [x] Tests confirm that failures return non-zero exit codes, warnings are properly piped to stderr, and valid inputs successfully generate the expected JSON outputs.
- [x] Cross-review execution success rate for Gemini agent is validated.
