## Mission

Rewrite `audit-skill-violations.sh` and `skill-coverage-audit.sh` from per-file
subprocess spawning to single-pass Python for ~100x speedup.

## What

1. Create `scripts/skills/audit-skills.py` combining both checks:
   - README presence, word count, description length, XML tags (from violations)
   - Script-wiring gaps (from coverage)
2. Single `find` + Python loop — no per-file `uv run` subprocess
3. Output YAML (same schema as current scripts for backward compat)
4. Deprecate shell scripts with pointer to new Python script

## Why

- Current scripts spawn `uv run --no-project python -c "..."` per SKILL.md file (404×)
- This makes them too slow to run in ecosystem-eval-report.sh (~10+ min)
- eval-skills.py already proves single-pass Python is fast (<5s for 405 skills)

- [x] `scripts/skills/audit-skills.py` created

- [x] Runs in <10s for full ecosystem (~1.5s vs current 10+ min)
