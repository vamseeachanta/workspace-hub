# Bidirectional SSH key auth between dev-primary and dev-secondary

## What

Set up passwordless SSH in both directions between dev-primary and dev-secondary. This is a
prerequisite for WRK-287 (SSHFS network sharing) and general multi-machine workflow.

## Why

- WRK-287 SSHFS mounts require passwordless SSH for auto-mount on boot
- Manual password entry on every SSH/SCP between machines wastes time
- Bidirectional access needed: both machines must reach each other seamlessly

## Current State

- dev-secondary has SSH key: `/home/vamsee/.ssh/id_ed25519` (created 2026-02-21)
- dev-primary is in dev-secondary's `/etc/hosts` at 10.0.0.1
- dev-secondary is in dev-primary's `/etc/hosts` at 10.0.0.2 (confirm)
- SSH connectivity between machines: **not verified**
- Key exchange: **not done**

## Steps

### From dev-secondary (this machine)
1. Test SSH connectivity: `ssh vamsee@dev-primary hostname`
2. Copy key to dev-primary: `ssh-copy-id vamsee@dev-primary`
3. Verify passwordless: `ssh dev-primary hostname` (should return `dev-primary` with no prompt)

### From dev-primary (remote or physical)
4. Check if SSH key exists: `ls ~/.ssh/id_*`
5. If no key: `ssh-keygen -t ed25519 -C "dev-primary"`
6. Copy key to dev-secondary: `ssh-copy-id vamsee@dev-secondary`
7. Verify passwordless: `ssh dev-secondary hostname`

### Verification
8. From dev-secondary: `ssh dev-primary 'echo OK from dev-primary'`
9. From dev-primary: `ssh dev-secondary 'echo OK from dev-secondary'`

## Acceptance Criteria

- [ ] dev-secondary → dev-primary SSH works without password
- [ ] dev-primary → dev-secondary SSH works without password
- [ ] Both machines have /etc/hosts entries for each other
- [ ] SSH host keys accepted (no MITM warnings on future connects)
