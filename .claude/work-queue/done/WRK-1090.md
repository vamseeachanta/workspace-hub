# Dependency health — uv.lock freshness, CVE advisories, outdated deps

## Mission

Ensure all 5 tier-1 Python repos have healthy, up-to-date, CVE-free dependencies at all times without manual tracking.

## What

Build a dependency health check pipeline:

1. **uv.lock freshness** — detect when `uv.lock` is out of sync with `pyproject.toml`
2. **Outdated package detection** — run `uv pip list --outdated` per repo; surface packages >2 minor versions behind
3. **CVE advisory scan** — run `pip-audit` (or `uv audit` when available) per repo; fail on HIGH/CRITICAL CVEs
4. **Auto-WRK creation** — when CVEs or severely outdated deps found, auto-capture a WRK item (don't just log)
5. **Nightly cron** — wire into `crontab-template.sh` (full role, ace-linux-1)

## Why

Currently no automated check exists. Deps can drift silently — stale uv.lock, known CVEs, deprecated APIs. This is a self-healing gap: the ecosystem should surface and queue remediation automatically.

## Acceptance Criteria

- [ ] `scripts/quality/dep-health.sh` — checks uv.lock freshness, outdated deps, CVEs for all 5 repos
- [ ] Exit code 0 = healthy, 1 = issues found (blocking CVEs)
- [ ] Output: structured YAML report at `logs/quality/dep-health-{date}.yaml`
- [ ] CVE HIGH/CRITICAL → auto-capture WRK item via work queue
- [ ] Nightly cron entry added to `crontab-template.sh`
- [ ] ≥5 TDD tests (freshness check, outdated detection, CVE scan, auto-WRK, report format)
- [ ] Passes `check-all.sh` (ruff + mypy) for any Python components

### Phase 2 — Implement `scripts/quality/dep-health.sh`
- Mirror `check-all.sh` structure: `QUALITY_REPO_ROOT` env override, `declare -A REPO_MAP`, `REPO_ORDER`, per-repo result maps, `pad_label`, Summary line, exit 0/1
- `--repo <name>` flag for single-repo targeted runs (mirrors check-all.sh ergonomics)
- **Freshness**: `cd <repo> && uv lock --check --offline 2>&1` — exits 1 when stale; `--offline` prevents false failures from registry unavailability
- **Outdated**: `cd <repo> && uv run pip list --outdated --format=json 2>/dev/null` — uses repo's activated venv; warn-only (no exit 1); any outdated package = warn (no semver parsing to avoid CalVer edge cases)
- **CVE scan**: `uvx pip-audit --format=json --requirement <(uv export --no-hashes --format requirements-txt 2>/dev/null)` per repo; HIGH/CRITICAL severity → exit 1; MEDIUM/LOW → warn only. No `uv audit` probe (does not exist in uv 0.5.x)
- **YAML report**: write to `logs/quality/dep-health-$(date +%Y-%m-%dT%H%M).yaml`; mkdir -p; timestamped to avoid same-day overwrites; fields: `run_at`, per-repo `freshness`/`outdated`/`cves`/`status`
- **Auto-WRK**: when CVE HIGH/CRITICAL found, `flock --timeout 10 .claude/work-queue/state.yaml`, call `scripts/work-queue/next-id.sh`, write `pending/WRK-NNN.md` atomically via mktemp+mv; log error and continue if flock times out
