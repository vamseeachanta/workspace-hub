# Install Tailscale VPN on dev-secondary

## What

Install and configure Tailscale on dev-secondary to match dev-primary's remote access
capability. Both machines should be reachable via Tailscale IPs from any network.

## Why

- dev-primary already has Tailscale (10.1.0.1/32) — dev-secondary does not
- Without Tailscale, dev-secondary is only reachable on the local LAN (10.0.0.2)
- Remote work, phone SSH, and cross-network access all require VPN
- Parity between the two Linux workstations — either should be usable remotely

## Steps

1. Install Tailscale: `curl -fsSL https://tailscale.com/install.sh | sh`
2. Start and authenticate: `sudo tailscale up`
3. Verify: `tailscale status` — confirm assigned IP
4. Test from dev-primary: `ping <dev-secondary-tailscale-ip>`
5. Update aceengineer-02.md network table with Tailscale interface
6. Update workstations SKILL.md with Tailscale IP

## Acceptance Criteria

- [ ] Tailscale installed and running on dev-secondary
- [ ] dev-secondary visible in Tailscale admin console
- [ ] dev-primary can reach dev-secondary via Tailscale IP
- [ ] aceengineer-02.md updated with Tailscale network info
