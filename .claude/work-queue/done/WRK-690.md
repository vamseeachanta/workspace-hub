# feat(workflow): enforce skill-driven /work gatepass from session start to close

## What

Create an enforceable, test-backed workflow so agents cannot bypass gatepass:
`/session-start` -> `/work` (work-queue orchestration) -> 9-stage verification evidence -> close/archive.

## Why

Current behavior shows drift between canonical process and skill docs, causing occasional plan/gate bypass. We need deterministic skill routing and machine-checkable checks across the full lifecycle.

## Acceptance Criteria

- [x] Create a skill revision matrix and update impacted skills (`work-queue`, `session-start`, `session-end`, related adapters)
- [x] Add explicit required-skill sequence for lifecycle start->finish in docs and validator-backed evidence
- [x] Add/extend gate validator to require 9-stage verification ledger evidence before close
- [x] Add `gate evidence` summary table artifact generation (step/result) for close audit
- [x] Add tests ensuring agents cannot pass close without required gate and integrated/repo test evidence
- [x] Propose and scaffold any new skills needed to prevent drift (minimum one, if justified)
- [x] Regenerate queue index and keep WRK/process docs consistent
