Title
Fitness-for-service module enhancement: wall thickness grid, industry targeting, and asset lifecycle
Status
archived
Route

Orchestrator

Prompt Start Context

Enhance the existing FFS module (renamed from pyintegrity to asset_integrity at src/digitalmodel/asset_integrity/) to support: 1. Wall thickness grid input — accept a grid of wall thickness measurements (e.g. from UT inspection) and produce Level 1 and Level 2 assessment results with acceptance/rejection outcomes 2. Existing examples — catalog and add working examples of FFS assessments before new development proceeds 3. Target industry coverage — explicitly support pipelines, upstream facilities, midstream infrastructure, and downstream equipment with industry-specific parameters and code references 4. Asset lifecycle integration — incorporate remaining life estimation, inspection interval recommendations, and degradation trending across the lifecycle of an asset

Why

The current module (renamed to asset_integrity) has strong API 579 calculation foundations but lacks a streamlined workflow for the common real-world use case: an engineer receives a wall thickness grid from UT inspection and needs quick Level 1/2 acceptance screening. Adding industry-specific templates and lifecycle tracking makes the module practical for production engineering workflows across oil & gas sectors.

Acceptance Criteria

  • [x] Module renamed from pyintegrity to asset_integrity
  • [x] All imports fixed (absolute → relative)
  • [x] Legal compliance scan passed (zero violations)
  • [x] Marketing brochure corrected with honest capability inventory
  • [ ] Wall thickness grid input (CSV, array, or YAML) accepted and mapped to component geometry
  • [ ] Level 1 assessment: minimum measured thickness vs. required thickness → accept/reject
  • [ ] Level 2 assessment: remaining strength factor (RSF) calculation for local thin areas → accept/reject
  • [ ] Clear output report showing grid results, governing locations, and accept/reject per API 579
  • [ ] Existing examples cataloged and verified working (at least 3 example configurations)
  • [ ] Industry-specific parameter sets: pipelines (B31.4/B31.8), upstream vessels (Div 1/2), midstream piping
  • [ ] Asset lifecycle module: corrosion rate trending → remaining life → next inspection date
  • [ ] Tests for each enhancement with at least one known benchmark per assessment level
  • [ ] Documentation of API 579 clause references for all new calculations
