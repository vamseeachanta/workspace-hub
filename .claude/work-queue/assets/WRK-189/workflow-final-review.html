Daily stock analysis and position strategy for long-term portfolios
Status

Prompt Start Context

Build a daily stock analysis workflow for assethold that examines existing long-term portfolio positions and produces a clear, actionable daily strategy: whether to build (add to) or trim (reduce) each position based on fundamental and technical signals.

Live portfolio data lives at: /mnt/local-analysis/workspace-hub/achantas-data/_finance/fidelity

Why

Long-term investors still benefit from systematic daily review. Markets move; valuations drift; new information arrives. A repeatable, automated analysis loop removes emotion from the "add vs. trim" decision and keeps position sizing aligned with conviction and risk tolerance.

Acceptance Criteria

  • [x] Portfolio loader reads Fidelity holdings from achantas-data/_finance/fidelity and returns a structured list of positions (ticker, shares, cost basis)
  • [x] Market data fetcher retrieves current price, volume, P/E, 52-week range, and RSI for each ticker
  • [x] Signal engine scores each position and assigns a Build/Hold/Trim recommendation
  • [x] Daily report is written to assethold/reports/daily-strategy/YYYY-MM-DD.md
  • [x] CLI entry point: python -m assethold.analysis.daily_strategy (or equivalent)
  • [x] Unit tests cover signal engine logic with mock market data (136 tests passing)
  • [x] README section documents how to run and interpret the daily report
  • [x] Insider trend analysis integrated (SEC Form 4 via yfinance, 90-day window)
