Phase 2 — run-benchmarks.sh + baseline tooling

Script scripts/testing/run-benchmarks.sh: - Repo config table (same pattern as run-all-tests.sh from WRK-1054) - Per-repo: uv run python -m pytest tests/benchmarks/ --benchmark-only --benchmark-json=<tmp> -q - Aggregate results to scripts/testing/benchmark-results/benchmark-YYYY-MM-DD.json - --repo <name> flag for single-repo run - --save-baseline writes config/testing/benchmark-baseline.json from latest run - Default compare mode: reads baseline, flags any mean >20% slower as REGRESSION (exit 1) - --no-compare flag skips comparison (for first run / bootstrap)

Helper scripts/testing/parse_benchmark_output.py: - Reads pytest-benchmark JSON; outputs summary with pass/fail per benchmark - Compare logic against baseline; prints regression details
