Title
Real-time EIA and IEA feed ingestion — weekly crude and gas production
Status
done
Route

Prompt Start Context

Set up weekly automated ingestion of EIA API (US crude/gas production, imports, stocks) and IEA data feeds. Store as time-series JSONL. Wire into existing worldenergydata data pipeline. Requires EIA_API_KEY env var.

Why

EIA weekly data is the most timely publicly available US energy production dataset, used across the industry for supply/demand analysis. Automating ingestion keeps worldenergydata current without manual downloads and enables time-series analysis of production trends, inventory levels, and import/export patterns.

Acceptance Criteria

  • [x] EIA API v2 client: weekly petroleum status report series (crude production, stocks, imports, refinery utilisation)
  • [x] EIA API v2 client: natural gas weekly storage report
  • [x] JSONL storage format with schema versioning
  • [x] Incremental ingestion: only fetch new data since last run
  • [x] CLI: worldenergydata eia sync for manual trigger
  • [x] Cron-compatible script for weekly automation (scripts/eia_weekly_sync.sh)
  • [x] EIA_API_KEY loaded from environment variable (validated at startup)
  • [x] Pytest coverage with mocked API responses (41 tests, 100% pass)
