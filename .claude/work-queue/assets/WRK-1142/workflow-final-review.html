1. Flesh out all 20 stage micro-skills

Each stages/stage-NN-*.md becomes a complete reference for that stage: - Entry reads (what to load before starting) - Checklist (what to do, in order) - Scripts-over-LLM audit (where applicable — Stage 4, 10 mandatory) - Exit gate (what artifact proves the stage is done) - Hard-gate flag (STOP + wait vs auto-proceed)

Migrate all relevant content from work-queue-workflow/SKILL.md into the appropriate stage file. Keep SKILL.md as a lightweight index with cross-cutting rules only (terminology, gate policy table, orchestrator pattern).

2. Wire start_stage.py to auto-load stage micro-skill

At stage entry, start_stage.py WRK-NNN N should print the micro-skill path and its content into the context resume block so the agent has the rules in scope for that stage. Pattern:

stage_skill = f".claude/skills/workspace-hub/stages/stage-{N:02d}-*.md"
# resolve glob → print as entry_reads block

3. Stage contracts (stages/stage-NN-*.yaml) stay as machine-readable gates

They are not replaced — they serve verification. The micro-skills are the human+agent readable narrative layer on top.

4. Migrate scripts-over-LLM rule into stage-04 and stage-10

The rules added to SKILL.md in this session (2026-03-12) move into: - .claude/skills/workspace-hub/stages/stage-04-plan-draft.md - .claude/skills/workspace-hub/stages/stage-10-work-execution.md

Remove the duplicates from SKILL.md once migration is confirmed.
