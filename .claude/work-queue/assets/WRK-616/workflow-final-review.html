Prompt Start Context

Standing policy work item: wherever document intelligence (WRK-309) identifies calculation files, workbooks, or structured data on /mnt/ace, create a structured specs/data-sources/ YAML entry and corresponding WRK item to implement that data as a typed data model in the appropriate repo.

This item tracks the meta-process. Specific conversion WRK items are created as sub-tasks.

Why

The document index (data/document-index/index.jsonl, 945K records) is only useful when its findings propagate into the codebase as: - calc_examples entries → test verification data (pattern established by WRK-560) - data_sources → structured readers/parsers - gaps → new implementation WRK items

Without this conversion loop, document intelligence stays a one-off audit rather than a continuous source of engineering capability improvements.

Acceptance Criteria

  • [ ] All identified FEA assets registered in specs/data-sources/digitalmodel.yaml
  • [ ] All identified drilling calc workbooks have WRK items (assetutilities/calculations domain)
  • [ ] Riser/mooring APDL models catalogued in specs/data-sources/digitalmodel.yaml
  • [ ] At least 3 new data-source YAML entries added (beyond WRK-560's pipeline domain)
  • [ ] Each YAML entry has: standard, domain, description, files with path, type, description
