Route B — inline plan

Phase 1: scripts/quality/check-all.sh

  1. Create scripts/quality/ directory and check-all.sh
  2. Define repo map: name → relative path (handle OGManufacturing/ capitalisation)
  3. Arg parsing: --fix, --repo <name>, --ruff-only, --mypy-only
  4. Per-repo ruff: cd <repo> && uv tool run ruff check . [--fix] [--config pyproject.toml if present]
  5. Per-repo mypy: cd <repo> && uv run mypy src/ [--config-file pyproject.toml if present] — uses each repo's own project venv for correct dependency resolution — OGManufacturing fallback: --ignore-missing-imports (no mypy config)
  6. Log ruff + mypy versions at top of report
  7. Collect per-repo exit codes; print violation counts table; exit 0 only if all pass

Phase 2: Pre-commit hook insertion

  1. worldenergydata + OGManufacturing: append minimal ruff-only hook block to existing .pre-commit-config.yaml
  2. assetutilities + assethold: create .pre-commit-config.yaml with ruff-only hook (minimal snippet — NOT a copy of digitalmodel which has unrelated hooks)

Phase 3: Tests + cross-review

  1. Write tests/quality/test_check_all.sh — bash tests using fixture temp dirs (not live repos) covering: help flag, --repo nonexistent exits non-zero, --repo filter runs only that repo, aggregate exit non-zero when any repo fails
  2. Codex cross-review of script + tests
  3. Fix MAJOR findings; document deferred MINORs

Test Strategy

  • All tests use temporary fixture directories (not live repos) — deterministic regardless of workspace state
  • --repo nonexistent → non-zero exit + error message
  • Aggregate exit: one fixture "failing" repo → non-zero aggregate exit code
  • No --structure flag in this WRK (deferred to WRK-1058)
