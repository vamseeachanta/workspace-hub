Prompt Start Context

Download actual BSEE incident investigation data and Incidents of Non-Compliance (INCs) from data.bsee.gov. Re-download the 15 stub Excel files from bsee.gov/stats-facts/offshore-incident-statistics. Verify URL importers work with corrected endpoints.

Why

  • [x] Import data into database
  • [x] Document record counts and date ranges

Source: HSE data acquisition - BSEE incident investigations and INCs data

Plan

Remaining work only. Raw data acquired, acquirers and importers built. On-disk data confirmed. Need to: (1) run importers against database, (2) verify URL-based importers, (3) re-download Excel stubs, (4) validate and document.

Phase 1 — Run Existing Importers Against Database

  1. Create HSE database (if not already created by WRK-067):
  2. data/modules/hse/hse_incidents.db — the SQLite DB shared across all HSE importers
  3. Both bsee_incinv_importer.py and bsee_incs_importer.py produce normalized dicts but do NOT directly persist to DB. They use a load() method returning List[Dict], not the BaseImporter.import_data() pipeline.

  4. Gap: Need a DB persistence wrapper that takes the load() output and feeds it through BaseImporter.validate_data() -> BaseImporter.import_record()

  5. Write DB bridge script: scripts/import_bsee_hse_to_db.py

  6. Instantiates BSEEIncInvImporter(data_dir="data/modules/hse/raw/bsee")
  7. Calls importer.load() -> gets 1,984 records
  8. For each record, validates via BaseImporter.validate_data() and persists via BaseImporter.import_record()
  9. Same for BSEEINCSImporter(data_dir="data/modules/hse/raw/bsee")

  10. Expected: ~477K+ INC violation records (each INC row can produce up to 3 records: warnings, component shut-ins, facility shut-ins)

  11. Data files confirmed on disk:

  12. data/modules/hse/raw/bsee/IncInvRawData/mv_acc_investigations.txt (169KB, 1,984 records)
  13. data/modules/hse/raw/bsee/INCSRawData/mv_incs_by_oper.txt (3.3MB, primary INC file)
  14. data/modules/hse/raw/bsee/INCSRawData/mv_incs_*.txt (14 regional files, ~25MB total)

Phase 2 — Verify and Fix URL-Based Importers

  1. URL importers exist but need verification:
  2. bsee_incidents_importer_url.py — imports incidents from data.bsee.gov API
  3. bsee_penalties_importer_url.py — imports civil penalties from data.bsee.gov API

  4. bsee_statistics_importer_url.py — imports safety statistics from data.bsee.gov API

  5. Test each URL importer in dry-run mode:

  6. Verify endpoints resolve (data.bsee.gov may have changed since initial build)
  7. Compare URL importer output against raw file importer output for overlapping records

  8. Document any endpoint changes needed

  9. Run penalties importer:

  10. bsee_penalties_importer.py maps to HSEIncident + ViolationIncident (penalty amounts, regulation citations)
  11. Data source: BSEE civil penalties database (may need separate download from data.bsee.gov)

Phase 3 — Excel Spreadsheet Re-Download

  1. Acceptance criteria item: "Re-download all 17 Excel spreadsheets (replace stubs)"
  2. The work item notes 15 stub Excel files from bsee.gov/stats-facts/offshore-incident-statistics
  3. These are aggregated statistical reports (not raw data), covering: injuries, fatalities, fires, explosions, blowouts, collisions, spills, crane incidents, etc.

  4. bsee_statistics_importer.py is designed to read these

  5. Download script: Extend bsee_acquirer.py or write scripts/download_bsee_statistics_excel.sh

  6. Target URL: https://www.bsee.gov/stats-facts/offshore-incident-statistics
  7. Save to: data/modules/hse/raw/bsee/statistics/

  8. Verify file sizes (stubs are typically <1KB, real files are 50KB-2MB)

  9. Import statistics:

  10. Use BSEEStatisticsImporter to parse Excel files into SafetyStatistic model
  11. Aggregated records (operator-level quarterly/annual summaries)

Phase 4 — Validation and Documentation

  1. Run data quality validator on all imported BSEE HSE data:
  2. DataQualityValidator(db_session).run_all_validations()

  3. Export: data/modules/hse/quality_report_bsee.json

  4. Document record counts:

  5. Incident investigations: expected ~1,984 records
  6. INCs: expected ~477K+ (may expand to ~1.4M+ as each row produces 1-3 violation records)
  7. Civil penalties: count TBD after import

  8. Statistics: count TBD after Excel re-download

  9. Document date ranges:

  10. Incident investigations: earliest/latest dates from DATE_OCCURRED column
  11. INCs: date range from INC_DATE column
  12. Update data catalog: data/catalog/data-catalog.yml

Phase 5 — Tests

  1. Unit tests: tests/unit/hse/test_bsee_incinv_importer.py
  2. Test _normalize_row() with various accident types (fatality, fire, blowout, etc.)
  3. Test _parse_date() with M/D/YYYY and military time formats

  4. Test _map_accident_type() and _map_severity() mappings

  5. Unit tests: tests/unit/hse/test_bsee_incs_importer.py

  6. Test _normalize_row() emits 0-3 records per row (one per non-zero INC category)
  7. Test _safe_int() edge cases

  8. Test region code mapping (G, A, P)

  9. Integration test: tests/unit/hse/test_bsee_hse_db_import.py

  10. Small fixture data (5-10 rows)
  11. Verify HSEIncident records with correct incident_type and severity

  12. Verify composite IDs (INCINV-{date}-{lease}-{idx}, INC-{date}-{operator}-{type}-{idx})

  13. Path: PYTHONPATH="src:../assetutilities/src" python3 -m pytest tests/unit/hse/ -v --tb=short --noconftest
