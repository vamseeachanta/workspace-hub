Anthropic's "Effective Harnesses for Long-Running Agents" recommends agents run a basic end-to-end test at session startup before selecting work. Currently session.sh init does environment setup but no repo health validation.

Add a --health-check step (or integrate into existing session-start) that: 1. Runs a fast smoke test per tier-1 repo (subset of pytest, <30s total) 2. Reports pass/fail summary before WRK selection 3. If any repo fails, flags it as context for WRK selection (avoid broken repos) 4. Stores result in .claude/state/session-health.yaml

Agents inherit broken state from prior sessions. Without a health check, they start implementing on top of a broken baseline and waste an entire session debugging carry-over issues.

  • [ ] Total health check completes in <60 seconds
  • [ ] Failed repos surfaced in session context before WRK selection
  • [ ] Health results persisted to session state file
  • [ ] Existing session-start flow not slowed when all repos healthy
