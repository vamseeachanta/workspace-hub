Skill upkeep and curation — evals, A/B tests, daily cron integration
Status

Prompt Start Context

Add a structured eval/test harness for workspace-hub skills so that skill quality is continuously validated. Each skill gets capability evals (does it do what it claims?) and procedural evals (does it follow the correct workflow?). A/B variants test alternative skill prompts against each other. All evals run nightly in the comprehensive-learning cron pipeline.

Why

Skills currently have no automated quality checks. Skill regressions (prompt drift, broken tool calls, outdated step sequences) go undetected until a session fails. A daily eval run catches degradation early and provides objective signal for skill improvement.

Acceptance Criteria

  • [ ] Eval framework design documented (specs/skills/skill-eval-framework.md)
  • [ ] Capability eval schema defined: each skill declares expected outputs/behaviors
  • [ ] Procedural eval schema defined: step-sequence fidelity checks
  • [ ] A/B eval scaffold — three comparison modes:
  • With vs without: same task, skill injected vs bare model — measures whether the skill adds value
  • Skill vs skill: two closely related skills run on the same task — identifies overlap, merge candidates, or which variant wins
  • Variant vs variant: two prompt versions of the same skill — picks the better prompt
  • [ ] At least 3 pilot skills have evals written (e.g. work-queue, workflow-gatepass, comprehensive-learning)
  • [ ] scripts/skills/run-skill-evals.sh executes all evals and emits a JSONL report
  • [ ] Comprehensive-learning cron (scripts/cron/comprehensive-learning-nightly.sh) calls the eval script
  • [ ] Eval results surfaced in /daily-report or /reflect output
  • [ ] Failing evals create follow-up WRK items automatically
  • [ ] Retirement signal: when A/B delta (with vs without) falls below a threshold, cron flags the skill as a retirement candidate and creates a WRK suggestion (never auto-deletes)
  • [ ] Auto-enhancement: eval results feed back into comprehensive-learning Phase 7 (code-quality scan) so underperforming skills get enhancement candidates queued
  • [ ] Research-driven updates: cron checks for new model capabilities or published prompting research and proposes targeted skill enhancements as WRK items
