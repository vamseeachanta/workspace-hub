Title
Review, refine, and curate hooks + skills — research best practices from existing workflows
Status
archived
Route
-
Orchestrator
unknown

Prompt Start Context

Audit all existing hooks (git hooks, Claude Code hooks, shell profile hooks) AND skills (.claude/skills/, /commands/), research what additional hooks and skills would improve workflows, and curate a clean, documented set across the workspace. Skills define what can be done; hooks define when it fires — they must be reviewed together.

Why

Both hooks and skills have accumulated organically — some are useful, some are stale, some overlap, and many skills exist that could be automated via hooks but aren't. A systematic review ensures: - No wasted cycles: remove hooks/skills that fire unnecessarily or are never used - Coverage gaps filled: identify workflow steps that should have hooks but don't, and skills that should exist but are missing - Hook-skill alignment: skills that should auto-trigger via hooks instead of requiring manual invocation - Consistency: same patterns across repos (workspace-hub, worldenergydata, digitalmodel) - Discoverability: developers know what hooks and skills exist and what they do

Acceptance Criteria

  • [ ] Complete inventory of all existing hooks across workspace
  • [ ] Complete inventory of all existing skills across workspace
  • [ ] Each hook assessed: keep / remove / fix / replace
  • [ ] Each skill assessed: keep / remove / fix / enhance
  • [ ] Research notes on best practices and opportunities
  • [ ] Stale or broken hooks and skills removed
  • [ ] New hooks and skills implemented where gaps identified
  • [ ] Hook-to-skill alignment documented (which skills auto-trigger via hooks)
  • [ ] Combined hook + skills catalog document created
  • [ ] All hooks and skills tested and verified working
  • [ ] Hooks execute within acceptable time limits (document per-hook targets)
