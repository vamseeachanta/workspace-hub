Title
eval(worldenergydata): evaluate pyWAsP for wind resource assessment integration
Status
pending
Route
-
Orchestrator

Prompt Start Context

Evaluate pyWAsP (https://docs.wasp.dk/pywasp/latest/) — the Python API for the WAsP wind resource assessment flow model (DTU Wind Energy / Ørsted) — for integration into the worldenergydata ecosystem. pyWAsP covers terrain effects (IBZ model), wind climate generalisation/downscaling, and AEP estimation with wake effects (via PyWake). Built on WindKit, which uses xarray data structures natively.

Why

Wind energy is a core domain for the Tier 3 GTM targets (Orsted, Equinor, Dominion CVOW — WRK-148). pyWAsP is the industry-standard tool these companies use daily. Adding wind resource assessment capability to worldenergydata: - Enables AEP → field-development economics pipeline (feeds WRK-476 ESG/carbon module) - Creates direct technical credibility with offshore wind prospects - xarray/WindKit structures align with the planned metocean module (NOAA NDBC) - Pairs with existing Arps decline curve (WRK-318) for hybrid fossil/renewables models

Acceptance Criteria

  • [ ] pip install pywasp tested on ace-linux-1 (licence requirement assessed)
  • [ ] PyWake (open-source companion wake model) installed and smoke-tested separately
  • [ ] WindKit data structures understood — xarray compatibility with worldenergydata confirmed or gaps documented
  • [ ] At least one worked example: wind climate generalisation OR AEP calculation using public wind atlas data (Global Wind Atlas: https://globalwindatlas.info/)
  • [ ] Evaluation report written to specs/data-sources/pywasp-eval.md covering: licence cost, integration complexity, xarray compatibility, AEP pipeline sketch
  • [ ] Decision: integrate (create implementation WRK) | defer | skip
  • [ ] specs/data-sources/worldenergydata.yaml tool entry updated with eval outcome

Plan

  • Phase 1 — Install & smoke test: pip install pywasp pywasp-pywake windkit; check if WAsP licence is required for terrain flow (IBZ) vs. free for wind climate ops
  • Phase 2 — Data compatibility: load a Global Wind Atlas .lib wind climate file; inspect WindKit xarray Dataset structure (point/raster/cuboid spatial formats; TSWC/BWC/WWC/GWC wind climate types; elevation/roughness/land-cover topography); compare to worldenergydata data conventions. Ref: https://docs.wasp.dk/pywasp/latest/getting_started/working_with_xarray_and_windkit.html
  • Phase 3 — AEP example: run minimal AEP calculation (wind climate + power curve) without full terrain model (licence-free path); confirm PyWake wake integration
  • Phase 4 — Eval report: document findings, licence model, and integration roadmap
