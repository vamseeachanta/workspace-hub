Summary

Convert radon's warn-only complexity check into a ratchet gate across all 5 tier-1 repos. Modelled on the existing mypy ratchet (WRK-1092). Complexity can only decrease or hold steady.

Route: A (Simple)

Plan Steps

  1. config/quality/complexity-baseline.yaml — write baseline from live radon data
  2. assetutilities: avg=3.03, high_cc=9, very_high_cc=2
  3. digitalmodel: avg=3.48, high_cc=72, very_high_cc=15 (legacy — ratchet, not absolute block)
  4. worldenergydata: avg=3.55, high_cc=39, very_high_cc=3
  5. assethold: avg=3.07, high_cc=1, very_high_cc=1

  6. OGManufacturing: avg=1.72, high_cc=0, very_high_cc=0

  7. scripts/quality/check_complexity_ratchet.py — ratchet script

  8. --init: capture baseline from radon
  9. Ratchet: FAIL if high_cc_count or very_high_cc_count increases above baseline
  10. Auto-update on improvement (no auto-commit)
  11. SKIP_COMPLEXITY_REASON env bypass (audited)

  12. Exit 0=PASS, 1=FAIL

  13. TDD tests — scripts/quality/tests/test_check_complexity_ratchet.py

  14. test_load_baseline, test_ratchet_pass, test_ratchet_fail_high_cc, test_ratchet_fail_very_high_cc, test_auto_update_baseline

  15. check-all.sh --complexity-ratchet flag — upgrade radon from warn to gate

  16. scripts/hooks/pre-push.sh — add complexity ratchet call

Key Design Decision

CC>20 is tracked as a ratchet (baseline-relative), not an absolute hard block. Reason: digitalmodel already has 15 CC>20 functions (legacy). Absolute block would immediately fail every push. Ratchet enforces: no NEW CC>20 functions can be added.
