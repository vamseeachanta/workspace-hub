Review session logs and WRK evidence from recent items to identify where agent behavior diverged from the canonical 20-stage lifecycle specs. Known issues observed:

  1. Missed staging gates — agents proceeding past hard gates (stages 1, 5, 7, 17) without proper evidence or user confirmation
  2. HTML not opened — Plan HTML and Lifecycle HTML not opened in browser at required stage exits (stages 5, 7, 17)
  3. HTML reopened instead of refreshed — when HTML needs to be shown again, agents open a new browser tab instead of relying on the 30s auto-refresh (WRK-1041) or triggering a regenerate + refresh
  4. Stage entry/exit scripts not called — start_stage.py and exit_stage.py skipped or called inconsistently
Why — The 20-stage lifecycle and its gate evidence system exist to ensure quality and user oversight. When agents silently skip gates or mishandle...

The 20-stage lifecycle and its gate evidence system exist to ensure quality and user oversight. When agents silently skip gates or mishandle HTML artifacts, the workflow loses its value — plans go unreviewed, evidence is missing, and the user loses visibility into progress.
