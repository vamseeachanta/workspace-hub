### Verdict: REQUEST_CHANGES

### Summary
The plan is directionally right on marker registration and runner behavior, but it is not executable as written. The `--noconftest` assumption breaks the per-subdirectory `conftest.py` approach, and `responses` is unlikely to intercept the actual network stack used by current `yfinance` and SEC code. AC coverage is therefore incomplete for deterministic fixtures.

### Issues Found
- Major: the Phase 2 fixture-loading strategy is invalid under `--noconftest`. Pytest disables all `conftest.py` discovery with that flag, not just top-level `tests/conftest.py`, so the proposed per-subdirectory conftests in [wrk-1062-deterministic-fixtures.md](/mnt/local-analysis/workspace-hub/specs/wrk/WRK-1062/wrk-1062-deterministic-fixtures.md#L56) will not load either. This blocks the core implementation path.
- Major: the HTTP mocking choice does not match the actual clients. Current `yfinance` in the repo uses `curl_cffi.requests`, not `requests`, and SEC fetching uses custom `urllib3`/`sec_edgar_downloader` paths. The proposed `responses` interception of only Yahoo chart URLs in [wrk-1062-deterministic-fixtures.md](/mnt/local-analysis/workspace-hub/specs/wrk/WRK-1062/wrk-1062-deterministic-fixtures.md#L59) will not reliably catch those calls. Relevant code paths are in [yfinance/data.py](/mnt/local-analysis/workspace-hub/assethold/.venv/lib/python3.11/site-packages/yfinance/data.py#L6), [yfinance/base.py](/mnt/local-analysis/workspace-hub/assethold/.venv/lib/python3.11/site-packages/yfinance/base.py#L30), and [get_SEC_data.py](/mnt/local-analysis/workspace-hub/assethold/src/assethold/modules/stocks/get_SEC_data.py#L1).
- Major: endpoint coverage is incomplete. The plan only mocks `https://query1.finance.yahoo.com/v8/finance/chart/*` plus a small JSON set in [wrk-1062-deterministic-fixtures.md](/mnt/local-analysis/workspace-hub/specs/wrk/WRK-1062/wrk-1062-deterministic-fixtures.md#L67), but the WRK scope explicitly includes `sec_edgar` traffic and likely additional Yahoo endpoints (`info`, holders, etc.). As written, not all live-data failure roots are covered.
- Medium: the plan is internally inconsistent about which tests remain `live_data`. In [wrk-1062-deterministic-fixtures.md](/mnt/local-analysis/workspace-hub/specs/wrk/WRK-1062/wrk-1062-deterministic-fixtures.md#L82) the example keeps `@pytest.mark.live_data`, then [wrk-1062-deterministic-fixtures.md](/mnt/local-analysis/workspace-hub/specs/wrk/WRK-1062/wrk-1062-deterministic-fixtures.md#L87) says to remove it after refactor, while Phase 3 still expects `12+` live-data tests under `--include-live` in [wrk-1062-deterministic-fixtures.md](/mnt/local-analysis/workspace-hub/specs/wrk/WRK-1062/wrk-1062-deterministic-fixtures.md#L113). The steady-state marker policy is not defined.
- Medium: AC coverage is partial for the mission. The plan only commits to refactoring the 7 breakout tests in [wrk-1062-deterministic-fixtures.md](/mnt/local-analysis/workspace-hub/specs/wrk/WRK-1062/wrk-1062-deterministic-fixtures.md#L79) and leaves the other failing assethold tests effectively skipped. That can satisfy the default-run signal, but it does not fully satisfy the stated mission to replace live-data dependencies with deterministic fixtures for the live-data failures.

### Suggestions
- Replace the Phase 2 loading mechanism with one that works under `--noconftest`: either remove `--noconftest` for the relevant repos, move reusable mocks into importable helper modules and call them directly from tests, or load an explicit pytest plugin via `-p`/runner configuration.
- Do not anchor on `responses` until the transport is verified. Prefer mocking at the library boundary actually used by the code: patch `yfinance.Ticker`/wrapper methods and patch SEC fetch helpers (`ReadURLData.get_response_data`, `get_filing_urls_to_download`, or a repo-owned abstraction) instead of raw HTTP.
- Decide the target state explicitly: either (a) all 12 assethold failures become deterministic and lose `live_data`, or (b) only a subset is fixture-backed and the rest remain intentionally live. Then align Phase 2, expected-failures handling, and AC wording to that choice.
- Expand fixture coverage to every root cause named in Resource Intelligence, including SEC payloads and any non-chart Yahoo calls used by the failing tests.
- Tighten the AC mapping so `Zero unexpected failures in default run` is proven by the final runner path, and separately add an AC for deterministic execution of the converted tests themselves.

### Questions for Author
- Is the intended end state that all 12 assethold failures run offline by default, or only the 7 breakout tests with the rest remaining `live_data`?
- Is `--noconftest` a hard constraint from WRK-1054, or can the runner be adjusted per repo so repo-local conftests/plugins are allowed where needed?
- Do we want to mock HTTP transport directly, or is it acceptable to patch `yfinance` and SEC wrapper functions at the module boundary for determinism and lower maintenance?
