Prompt Start Context

An economic analysis module for batch drilling campaign optimization, building on the cost framework from WRK-019 (drilling/completion/intervention cost data):

Part 1 — Batch Drilling Data Model

class DrillingSite:
    well_id: str
    field_name: str
    location_coords: tuple[float, float]  # lat/lon
    water_depth_ft: float
    planned_md_ft: float                  # measured depth
    well_type: str                        # exploration, development, appraisal
    bore_design: str                      # slim, standard (link to WRK-218)
    mpd_required: bool                    # link to WRK-168

class DrillCampaign:
    campaign_id: str
    rig_name: str
    rig_day_rate_usd: float
    mobilization_cost_usd: float
    wells: list[DrillingSite]
    sequence: list[str]                   # ordered well_ids for batch
    shared_bha_configs: dict              # BHA configs reused across wells
    learning_curve_factor: float          # time reduction per well in batch (0-1)

Part 2 — Campaign Scheduling Optimizer

Core economic calculations for batch sequence optimization:

  1. Mobilization cost amortization — share one-time rig mobilization across N wells in a batch vs. individual campaigns

  2. Learning curve model — time reduction on well N vs. well 1 in the same batch:

  3. Industry learning curve factor: ~10-15% time reduction per doubling of wells
  4. Cumulative average model: T(N) = T(1) × N^(log(LC)/log(2))

  5. Sensitive to formation similarity, crew consistency, BHA design reuse

  6. BHA standardization benefit — reusing the same BHA configuration across multiple wells in a batch reduces trip time and equipment handling costs

  7. Sequence optimization — for N wells, find the ordering that minimizes total rig time:

  8. Geographic clustering: minimize vessel/rig transit between well locations
  9. Formation clustering: group wells by similar geology to maximize BHA reuse

  10. NPV ordering: high-value (producer) wells earlier in sequence (cash flow timing)

  11. Batch vs. standalone NPV comparison:

  12. Batch NPV: amortized mobilization + learning curve gains vs. flat day rate × days
  13. Standalone NPV: full mobilization per well, no learning curve
  14. Break-even analysis: minimum N wells to justify batch commitment

Part 3 — BSEE Campaign History Analysis (worldenergydata)

Mine existing BSEE drilling activity data to identify historical batch campaigns:

  • Batch detection heuristic: same rig drilling wells in the same field/block within a 12-month window → likely batch campaign
  • Batch vs. standalone metrics: compare average well duration for batched vs. standalone wells in the same depth band and era
  • Regional patterns: GoM deepwater batch frequency by water depth, era (2010-2025)
  • Learning curve validation: fit learning curve factor to BSEE time series for detected batch campaigns

Part 4 — Content Visualization

Supporting charts for blog post candidate (see WRK-080): - Batch NPV vs. standalone NPV: savings as function of N wells and day rate - Learning curve chart: cumulative well time vs. batch position - Geographic clustering map: well locations colored by optimal batch assignment

Why

  • Builds on WRK-019: The drilling cost layer (day rates, mobilization costs) provides the input data. Batch optimization is the decision tool that sits on top.
  • Common client question: "Should we batch these wells or run independent campaigns?" is one of the first economic questions in field development planning.
  • Data opportunity: BSEE activity data (WRK-016, WRK-018) contains sequential well drilling history that can validate batch economics against real campaigns.
  • GTM content: A data-backed analysis of GoM batch drilling patterns demonstrates both data access and economic modeling capability.
