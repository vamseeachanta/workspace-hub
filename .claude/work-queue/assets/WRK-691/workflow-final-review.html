Status

Prompt Start Context

Enhance /session-start with an interactive analysis phase that assesses — at session open — whether the key rules, skills, and context files are loaded and acknowledged. If proactive hook-based session logging has not fired, trigger it interactively.

This replaces the current "surface and move on" briefing with a structured load assessment → gap surfacing → user confirmation → log loop.

Why

Agents (Claude, Codex, Gemini) drift from rules because they rely on training knowledge rather than reading the canonical rule files. A session-start inventory that confirms each critical rule is in scope — and logs that confirmation — creates an audit trail and forces the agent to load the file before starting work.

Examples surfaced this session (2026-03-03): - python-runtime.md rule: agents used python3 / .venv/bin/python instead of uv run - File-structure skills: agents placed files without consulting file-taxonomy or repo-structure — because neither was explicitly loaded

Acceptance Criteria

  • [ ] load-assessment.sh runs all 7 checks and prints pass/gap table
  • [ ] Session-start skill calls load-assessment.sh and surfaces gaps interactively
  • [ ] If hooks have not fired, manual log entry written to logs/orchestrator/claude/
  • [ ] /session-interactive-analysis skill can be called standalone mid-session
  • [ ] Load assessment events appear in logs/orchestrator/claude/session_YYYYMMDD.jsonl
  • [ ] comprehensive-learning Phase 1 reads event=load_assessment entries for trend analysis
  • [ ] tests/unit/test_load_assessment.sh passes (7 checks: pass + gap cases)
