Currently, generate-html-review.py can emit separate plan-draft / plan-final / lifecycle files at different stages, and not all stages regenerate every file. This leads to stale views and unclear "which file to open" UX.

Desired end-state:

plan.html

  • Top section: latest approved plan (always kept current).
  • Stage-change log: a collapsible per-stage section showing what changed in the plan at that stage (diff-like narrative or structured table).
  • Stage-progress ring: same coloured stage-circles widget used in lifecycle.html so the user can see how far the item has progressed while reading the plan.
  • Regenerated on every stage exit (stages 1–20).

lifecycle.html

  • Unchanged widget + stage circles, but also regenerated on every stage exit (currently regenerated only on some stages).

Skill integration

  • workflow-html/SKILL.md and workflow-gatepass/SKILL.md should reference these two canonical files.
  • exit_stage.py should call generate-html-review.py WRK-NNN --plan and generate-html-review.py WRK-NNN --lifecycle after every successful stage exit.
Why — - Single refresh URL per document type reduces cognitive overhead. - Plan evolution is visible inline — no need to diff separate files. - St...
  • Single refresh URL per document type reduces cognitive overhead.
  • Plan evolution is visible inline — no need to diff separate files.
  • Stage-circle widget in both HTMLs gives immediate orientation.
