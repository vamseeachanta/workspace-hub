WRK-146 Final Review

Title
Overhaul aceengineer-website: fix positioning, narrative, and social proof
Status
done
Route
-
Orchestrator
claude

Prompt Start Context

Complete content and positioning overhaul of aceengineer-website based on detailed external feedback. The site has a positioning problem — strong engineering substance but generic consulting packaging that undersells the actual story: one senior engineer + AI automation delivering at the scale of a team.

Why

The site currently reads like a generic automation consultancy. The real differentiator — a 15+ year offshore engineer who built 700+ Python modules and now uses AI orchestration to outperform teams — is completely absent. This positioning gap is directly impacting business development.

Acceptance Criteria

  • [x] Homepage messaging rewritten: unique value proposition front and center
  • [x] About page rewritten with Vamsee's full story (experience, modules, AI stack)
  • [x] At least 2 case studies rewritten as narratives (problem → process → outcome)
  • [x] Blog articles surfaced/linked from services and homepage
  • [x] Pricing claims linked to supporting case studies
  • [x] Consistent "solo engineer + AI" positioning throughout
  • [x] Social proof section added (standards bodies, industry experience, project types)
  • [x] Site passes visual review — no broken layouts from content changes

Plan

Target: aceengineer-website/ (HTML/CSS content changes)

Phase 1 — Homepage Rewrite

  • Replace generic tagline with: "AI-native offshore engineering analytics"
  • Add hero section with key numbers: 704+ modules, 221 S-N curves, 15+ years
  • Add "How we work" section: solo engineer + AI orchestration model
  • Link to blog posts as credibility anchors

Phase 2 — About Page Rewrite

  • Rewrite from 4 sentences to full founder story
  • Structure: offshore engineering background → built 700+ Python modules → AI-augmented delivery
  • Include: standards bodies worked with, industry verticals, project types
  • Firm positioning: "A&CE" as company, Vamsee as founder/principal engineer

Phase 3 — Case Study Narratives (2 minimum)

  • Convert existing templated case studies to narrative format
  • Structure: problem (what client faced) → process (how A&CE solved it) → outcome (measurable results)
  • Link case studies to relevant blog posts
  • Add to pricing section as proof points

Phase 4 — Blog Integration

  • Surface top 3-5 technical blog posts on homepage
  • Add "Featured Analysis" section linking blog → services
  • Cross-link from services pages to relevant blog posts

Phase 5 — Social Proof Section

  • Standards bodies: DNV, API, ASME, BS, IIW
  • Industry experience: deepwater, FPSO, pipeline, mooring, drilling
  • Project types: fatigue analysis, wall thickness design, production optimization
  • Add near pricing claims

Phase 6 — Visual Review

  • Test all pages for broken layouts after content changes
  • Verify responsive design (mobile/tablet)
  • Check link integrity

Test Strategy

  • Visual review in browser after each phase
  • Link checker for broken references
  • No automated tests — content changes only

Source: External feedback from detailed site review + A&CE strategy session Feb 15-16. "You have a positioning problem, not a product problem. The engineering work is genuinely strong. But the site currently says 'we're an automation consultancy' when it should say 'I'm a 15+ year offshore engineer who built an AI-powered engineering machine that outperforms teams of 10.'"
