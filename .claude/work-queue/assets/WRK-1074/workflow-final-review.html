  • Each file tests the exact public API surface of assetutilities that the consuming repo depends on: function signatures, return types, exception types, key constants

Why — assetutilities is the most depended-upon repo in the ecosystem. A public API change (renamed function, changed return type, removed paramete...

assetutilities is the most depended-upon repo in the ecosystem. A public API change (renamed function, changed return type, removed parameter) can silently break digitalmodel or worldenergydata. No contract tests means breakage is only discovered when those consumers' unit tests happen to exercise the changed path.

Plan — ### Context - assetutilities v0.0.8; 41 unique import sources in consumers - digitalmodel already has `tests/contracts/` (empty dir) — reuse...

Context

  • assetutilities v0.0.8; 41 unique import sources in consumers
  • digitalmodel already has tests/contracts/ (empty dir) — reuse
  • worldenergydata needs tests/contracts/ dir created
  • All repos use pytest with pythonpath = src tests; PYTHONPATH includes ../assetutilities/src
  • run-all-tests.sh at scripts/testing/run-all-tests.sh — has integration step for extension

Steps

Step 1 — Shared contract helper (tests/contracts/conftest.py) - Both repos get a conftest.py in tests/contracts/ with: - au_version() fixture: returns assetutilities.__version__ - ContractViolation custom exception with __str__ showing symbol + version + test name - pytest_configure hook registering contracts marker

Step 2 — digitalmodel contract tests (≥3 files)

tests/contracts/test_assetutilities_data_contract.py - SaveData: importable, callable with (data, file_path) signature; returns None; no TypeError on valid input - ReadData: importable, callable with (file_path) signature - AttributeDict: importable, instantiatable as dict subclass; __getattr__ works - Transform: importable, callable

tests/contracts/test_assetutilities_yml_contract.py - WorkingWithYAML: importable, has .read_yaml() and .save_yaml() methods; signatures stable - ymlInput: importable as legacy alias; callable

tests/contracts/test_assetutilities_units_contract.py - TrackedQuantity: importable from assetutilities.units.quantity; instantiatable with (value, unit) - unit_checked: importable as decorator; callable - UnitMismatchError: importable, is subclass of Exception

tests/contracts/test_assetutilities_common_contract.py (bonus 4th) - FileManagement: importable, has .check_if_file_exists() method - update_deep_dictionary: importable, callable with (base, update) → returns merged dict - is_file_valid_func, is_dir_valid_func: importable, callable

Step 3 — worldenergydata contract tests (≥2 files)

tests/contracts/test_assetutilities_data_contract.py - Mirror of digitalmodel data contract: SaveData, ReadData, AttributeDict

tests/contracts/test_assetutilities_engine_contract.py - engine: importable from assetutilities.engine.engine; has expected class attributes - FileManagement: importable, has file check method - WorkingWithYAML: importable, has YAML read/write methods

Step 4 — Failure output enrichment - conftest.py adds pytest_runtest_logreport hook: on failure, appends [CONTRACT VIOLATION] symbol=<name> au_version=<ver> test=<node_id> to failure message - Achieved via custom ContractViolation marker and pytest_runtest_makereport hook

Step 5 — run-all-tests.sh integration - Add contracts step after unit tests in REPO_CONFIGS block or as a separate for-loop - Command: cd <repo> && PYTHONPATH="src:../assetutilities/src" uv run python -m pytest tests/contracts/ -v --tb=short - Exit code propagated; failure blocks run-all-tests exit 0

Step 6 — docs/api-contracts.md per consuming repo - digitalmodel/docs/api-contracts.md: table of contracted symbols, module path, stability level (stable|provisional) - worldenergydata/docs/api-contracts.md: same structure - Stability levels: stable = no breaking changes without major version bump; provisional = best-effort

Test Strategy

  • All contract tests use real assetutilities (no mocks) — import is the test
  • Tests are pure unit-speed (no I/O, no DB, no files) — run every commit
  • Add @pytest.mark.contracts to every test function — filterable
  • PYTHONPATH = src:../assetutilities/src for both repos (consistent with existing integration setup)
  • Each test file has module-level AU_VERSION = importlib.metadata.version("assetutilities") for failure context
  • Failed test output auto-includes AU_VERSION via conftest hook

Deferred / Out of Scope

  • assetutilities.common.database contract: requires DB — mark @live_data, defer
  • assetutilities.modules.excel_utilities: file I/O — add fixtures in follow-on WRK
  • constants contract: constants are unlikely to break; add basic smoke test only
  • Engine instantiation tests: heavyweight; defer to integration tests
