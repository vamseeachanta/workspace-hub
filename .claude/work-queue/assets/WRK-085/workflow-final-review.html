Title
Create public sample data access page on website
Status
done
Route

Orchestrator

Prompt Start Context

Create a static page on the website with downloadable sample datasets and embedded dashboard screenshots. Gives prospects a tangible way to evaluate data quality and dashboard capabilities without requiring a sales call.

Why

No demo or trial access exists. Prospects see marketing claims but cannot validate them. A sample data page reduces sales friction and builds trust.

Acceptance Criteria

  • [ ] Static HTML page at demos/ or samples/ path
  • [ ] 2-3 downloadable sample datasets (CSV/Excel) from BSEE public data
  • [ ] Embedded screenshots of production dashboards
  • [ ] Brief descriptions of what each dataset contains
  • [ ] Links to contact.html for full access
  • [ ] Matches site design (Bootstrap + marketing.css)
  • [ ] JSON-LD structured data

Once WRK-075 is complete, prepare 2-3 small CSV samples from BSEE public data:

  1. Sample 1: GoM Production Summary (small, ~50 rows)

  3. Source: worldenergydata BSEE production_raw aggregated by field

  4. Target: aceengineer-website/assets/data/samples/gom-production-summary.csv

  5. Sample 2: Deepwater Well Completions (small, ~30 rows)

  6. Fields: api_well_number, field_name, water_depth_ft, completion_date, well_type, status
  7. Source: worldenergydata BSEE completions data

  8. Target: aceengineer-website/assets/data/samples/deepwater-well-completions.csv

  9. Sample 3: Rig Fleet Summary (small, ~16 rows)

  10. Fields: rig_name, rig_type, water_depth_rating, year_built, status
  11. Source: worldenergydata rig_fleet data (already 16 rigs available)
  12. Target: aceengineer-website/assets/data/samples/rig-fleet-summary.csv

All samples must use only BSEE public data -- no proprietary or client data.
