Title
Establish drilling, completion, and intervention cost data by region and environment
Status

Prompt Start Context

Build a cost data layer on top of the field data pipelines that provides drilling, completion, and intervention cost estimates segmented by: - Environment: offshore vs. onshore - Region: Gulf of Mexico, North Sea, Brazil, Mexico, Texas (RRC), etc. - Activity type: drilling (by well type/depth), completion (by type), intervention (by workover type) - Water depth / well depth bands for offshore

Sources may include public cost indices, IHS/Rystad benchmarks (where available), BSEE/regulatory filings, and derived estimates from duration-based proxies (rig day rates × activity duration).

Why

Cost data transforms activity analysis from descriptive to economically meaningful. Clients need cost context to evaluate field economics, benchmark operations, and make investment decisions. Without cost, activity counts alone don't convey operational burden.

Phase 1 — Cost Data Source Identification and Schema

  1. Define cost data sources — create config/analysis/cost_data/cost_sources.yml:
  2. Direct data: BSEE/regulatory filings (limited public cost data)
  3. Proxy-based: Duration x day rate (drilling days x rig rate, completion days x completion rate)
  4. Public indices: Baker Hughes rig count (proxy for market tightness), EIA drilling productivity report
  5. Regional benchmarks: Published industry averages by water depth band and region

  6. Existing config: Extend economic_assumptions.yml pattern to multiple regions

  7. Define cost schema — create src/worldenergydata/bsee/analysis/cost/models.py: @dataclass class CostEstimate: activity_type: str # drilling, completion, intervention environment: str # offshore, onshore region: str # gom, north_sea, permian, etc. water_depth_band: str # shallow (<500ft), mid (500-5000ft), deep (>5000ft), ultra-deep (>7000ft) well_depth_band: str # shallow (<10Kft), mid (10-20Kft), deep (>20Kft) cost_usd: float cost_per_day_usd: float duration_days: float confidence: str # high (direct), medium (proxy), low (benchmark) source: str year: int

  8. Define cost category taxonomy:

  9. Drilling: tangible (casing, cement, mud) + intangible (rig time, services) + mobilization
  10. Completion: perforating, stimulation, gravel pack, equipment
  11. Intervention: workover rig, coiled tubing, wireline, plug-and-abandon

Phase 2 — Cost Estimation Engine

  1. Create cost engine: src/worldenergydata/bsee/analysis/cost/cost_engine.py
  2. Extends DrillingCompletionProcessor from financial/drilling_completion.py
  3. Adds region-aware cost selection
  4. Adds water depth band classification (from BSEE well data)

  5. Adds well depth band classification

  6. Day-rate database: config/analysis/cost_data/day_rates.yml

  7. Offshore GOM by water depth: shallow/jackup ($80-150K/day), mid-water/semisubmersible ($200-350K/day), deepwater/drillship ($400-600K/day)
  8. Onshore by basin: Permian ($15-25K/day), Eagle Ford ($18-28K/day), Bakken ($20-30K/day)
  9. Completion rates: conventional ($50-100K/day), HPHT ($150-250K/day)
  10. Intervention rates: workover rig ($40-80K/day), coiled tubing ($30-60K/day), wireline ($10-20K/day)

  11. Year-indexed (2020-2025) to capture market cycles

  12. Proxy calculation: cost_engine.py methods:

  13. estimate_drilling_cost(well_data, region, year) — matches well to day rate by depth/region, multiplies by BSEE-reported drilling days
  14. estimate_completion_cost(well_data, region, year) — same approach for completion days
  15. estimate_intervention_cost(intervention_data, region, year) — maps workover type to cost category
  16. estimate_with_confidence(...) — returns CostEstimate with confidence level

Data Constraints

  • 129/133 .bin files are LFS stubs — cost engine must handle gracefully (return empty cost estimate with warning)
  • Only 4 real .bin files: production_raw data and rig_fleet (16 rigs)
  • Drilling days data depends on WAR (Well Activity Reports) — available but may be incomplete for all fields
  • No direct public cost data from BSEE filings — all costs are proxy-based or benchmark-derived
  • Confidence levels: HIGH for proxy (days x rate), MEDIUM for regional benchmark, LOW for extrapolation
