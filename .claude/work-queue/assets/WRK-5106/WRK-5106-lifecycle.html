Stage 4: Plan Draft

Scripts to Create

  • scripts/work-queue/backfill-issues.sh — main batch script (~120 lines)
    • Input: --dry-run, --limit N, --resume-from N
    • Output: updated/skipped/failed counts to stdout

Plan

  1. Phase 1: Fetch all ~1256 GitHub issues via gh issue list --state all --limit 5000
  2. Phase 2: For each issue, extract WRK-NNN from title, find local WRK file, backfill github_issue_ref if missing
  3. Phase 3: Call update-github-issue.py --update for each matched WRK file (rate limited: sleep 0.5s)
  4. Print summary report: updated / skipped / failed / total

Acceptance Criteria

  • All WRK files with matching GitHub issues have github_issue_ref populated
  • All matched issues updated to WRK-5104 template format
  • --dry-run mode previews without API writes
  • --resume-from N skips first N issues for failure recovery
  • Summary report: updated/skipped/failed counts

Pseudocode

1. Parse CLI args (--dry-run, --limit, --resume-from)
2. gh issue list --repo REPO --state all --limit 5000 --json number,title > tmpfile
3. For each (number, title) in tmpfile:
   a. Extract WRK_ID from title via regex /WRK-(\d+)/
   b. If no WRK_ID → skip (SKIPPED_NO_WRK++)
   c. Find local WRK file via find_wrk_file()
   d. If no file → skip (SKIPPED_NO_FILE++)
   e. If WRK file missing github_issue_ref → inject it into frontmatter
   f. If --dry-run → log "would update" (DRY_RUN++)
   g. Else → uv run update-github-issue.py WRK_ID --update
   h. If exit 0 → UPDATED++, else ERRORS++
   i. sleep 0.5
4. Print summary: total/updated/skipped/errors
