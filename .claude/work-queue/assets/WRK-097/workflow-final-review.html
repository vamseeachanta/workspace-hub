Title
Implement three-tier data residence strategy (worldenergydata ↔ digitalmodel)
Status
archived
Route
B
Orchestrator
unknown

Prompt Start Context

Establish formal data governance across worldenergydata and digitalmodel using a three-tier model: Tier 1 (Collection Data) → worldenergydata, Tier 2 (Engineering Reference Data) → digitalmodel, Tier 3 (Project Data) → project repos.

Why

Engineering data is scattered with no formal boundaries. INVENTORY.md incorrectly claims hull geometry was "gathered from digitalmodel". SN curves are hardcoded in Python. No boundary test exists for where data should live. This creates confusion about data ownership and source of truth.
