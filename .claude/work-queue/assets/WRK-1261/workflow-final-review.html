  3. Tier B: skill references scripts/files in body (bash scripts/, file paths)
  4. Tier C: skill is < 200 lines, focused, no script refs needed
  5. Tier D: skill is > 500 words with no script refs (candidate for decomposition)
  6. Add quality_tier computation to eval-skills.py — output tier per skill
  7. Add tier distribution to ecosystem-eval-report.sh collated output
  8. Create scripts/skills/skill-tier-report.py — ranks all skills by tier, flags Tier D for curation (decompose into script + micro-skill)
  9. Document the hierarchy in skill-creator SKILL.md and comprehensive-learning Phase 9 so future curation follows the pattern
Why — - patterns.md already defines the enforcement gradient (prose → micro-skill → script → hook) but skill eval doesn't measure where each skill...
  • patterns.md already defines the enforcement gradient (prose → micro-skill → script → hook) but skill eval doesn't measure where each skill sits on that gradient
  • Without quantification, curation is subjective — "is this skill good enough?" has no measurable answer
  • The 25% repetition rule means any repeated LLM reasoning should become a script; this WRK makes that measurable per-skill
