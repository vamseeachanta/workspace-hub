Prompt Start Context

Query NTSB CAROL (data.ntsb.gov) for mode=marine and export as CSV. Download EPA TRI basic data files (1987-present) from EPA. Build TRI importer.

Plan

Remaining work only. EPA TRI data acquired (51K records, 5 yearly files + combined). NTSB CAROL API unavailable, manual export guide generated. Existing importers ready.

Phase 1 — EPA TRI: Import to Database

  1. Import EPA TRI data using existing EPATRIImporter:
  2. Source files at data/modules/hse/raw/epa_tri/:
    • tri_basic_2020_oil_gas.csv (7.7MB)
    • tri_basic_2021_oil_gas.csv (7.5MB)
    • tri_basic_2022_oil_gas.csv (6.6MB)
    • tri_basic_2023_oil_gas.csv (2.9MB)
    • tri_basic_2024_oil_gas.csv (8.4MB)
    • tri_basic_2020-2024_oil_gas_combined.csv (23.6MB, skip during import to avoid double-counting — importer already checks for "combined" in filename)
  3. Run: PYTHONPATH="src" python3 -m worldenergydata.hse.importers.epa_tri_importer --input-dir data/modules/hse/raw/epa_tri --database-url "sqlite:///data/modules/hse/hse_incidents.db"
  4. Target: ToxicRelease model in hse_incidents.db
  5. Dedup key: tri_facility_id + chemical_name + reporting_year

  6. Expected: ~51,503 records across 2020-2024

  7. Verify import:

  8. Count records per reporting year
  9. Spot-check facility names and chemical releases
  10. Verify NAICS codes are oil & gas (211xxx, 213xxx)

  11. Check geographic distribution (state column)

  12. Add README: Update data/modules/hse/raw/epa_tri/README.md with final import counts

Phase 2 — NTSB CAROL: Manual Export Acquisition

  1. NTSB CAROL API is NOT publicly accessible (confirmed during acquisition attempt). Manual export required.
  2. Manual export guide already generated at: data/modules/marine_safety/raw/ntsb/NTSB_CAROL_MANUAL_EXPORT_GUIDE.md

  3. Steps: Navigate to data.ntsb.gov/carol-main-public/query-builder, select Mode=Marine, export CSV

  4. Process manual export once obtained:

  5. Use NTSBCAROLAcquirer.process_export(input_file) to normalize column names and filter marine mode

  6. Output: data/modules/marine_safety/raw/ntsb/ntsb_marine_investigations.csv

  7. Import via existing NTSBImporter:

  8. Located at: src/worldenergydata/marine_safety/importers/ntsb_importer.py
  9. Uses marine_safety database models (Incident, Location, Vessel) — separate from HSE database
  10. Depends on: marine_safety.constants, marine_safety.database.models, marine_safety.processors.data_cleaner, marine_safety.processors.data_normalizer

  11. Note: This uses a different DB schema than the HSE module (marine_safety has its own models)

  12. Alternative data source check: NTSB also publishes data at data.ntsb.gov/avdata (primarily aviation but may include marine). If CAROL manual export is impractical, investigate the NTSB downloadable datasets.

Phase 3 — Verification and Quality

  1. EPA TRI verification:
  2. Run --dry-run mode first to validate CSV structure
  3. Cross-check total_releases_pounds for reasonableness
  4. Verify coordinate data (latitude/longitude) falls within US boundaries

  5. Check for common chemicals: hydrogen sulfide, benzene, toluene, xylene, methanol

  6. NTSB verification (post manual export):

  7. Verify mode filtering (Marine only)
  8. Check date range coverage (2001-present expected)
  9. Verify vessel types present (cargo, tanker, tug, barge, fishing, passenger)

  10. Check fatality/injury counts are non-negative integers

  11. Document in this WRK file:

  12. EPA TRI: final record count, date range, top 10 chemicals by release volume, state distribution
  13. NTSB: record count (estimated 3K-8K marine investigations), date range, vessel type distribution
