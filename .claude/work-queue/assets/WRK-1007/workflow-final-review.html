WRK-1007 Execution Plan: Harden submit-to-gemini.sh and cross-review scripts

Executive Summary

This plan defines the hardening process for the multi-agent cross-review pipeline scripts (`submit-to-*.sh`). It details creating robust argument parsing, memory bounds for large files to prevent orchestrator OOMs, and macOS-compatible fallbacks for execution timeouts.

Scope & Strategy

  • Investigate Edge Cases: test `submit-to-gemini.sh` with blank variables, large binaries, missing timeouts.
  • Argument parsing: implement strict [[ $# -ge 2 ]] constraints across all scripts.
  • Payload bounds: limit ingestion to 5MB using head -c 5000000 | tr -d '\000'.
  • Platform portability: add a Perl fallback for the GNU timeout binary.
  • Tests: Create comprehensive test suites like `test-submit-to-gemini-robustness.sh`.

Validation

Validation will involve running the newly created robustness test suites for Gemini and Claude natively on the workstation, followed by an end-to-end implementation cross-review of the changes.
