Title
feat(skills): add anti-pattern sections (no-to-do lists) to agent skills
Status

Route

Orchestrator

Prompt Start Context

Review the skill library and add explicit anti-pattern sections — "what NOT to do" — to high-impact skills. Analogous to the human productivity insight that a "no-to-do list" is often more effective than a to-do list: knowing what to avoid removes decision friction faster than knowing what to do.

Why

Agents (LLMs) tend to optimise for plausible-looking outputs, not correct ones. A skill that only says "do X" leaves a wide search space. A skill that also says "never do Y, Z" narrows the space and prevents recurring failure modes. Cross-session observation confirms that agents repeat the same anti-patterns (e.g., hardcoding paths, amending published commits, skipping cross-review, archiving before user confirms) even when positive instructions exist. Explicit prohibition lists address this more directly.

Acceptance Criteria

  • [ ] Audit existing skills: identify the 5-10 highest-value skills that lack anti-pattern sections (start with work-queue, git-workflow, coding-style, cross-review, testing)
  • [ ] Define a standard ## Anti-Patterns (or ## Never Do) section format for skills — concise, bulleted, imperative ("Never archive before user confirms", "Never skip cross-review", "Never hardcode /mnt/... paths")
  • [ ] Add ## Anti-Patterns sections to at least 5 high-traffic skills
  • [ ] Update rules/ files (git-workflow.md, coding-style.md, testing.md) with corresponding anti-pattern bullets where missing
  • [ ] Optionally: add an anti_patterns key to SKILL.md frontmatter schema for machine-readable prohibition lists (future gate integration)
  • [ ] Document the pattern in .claude/docs/skill-authoring-guide.md (or create it)

Plan

  • Phase 1 — Audit: read 10 top skills, extract existing prohibitions, identify gaps
  • Phase 2 — Standard: define ## Anti-Patterns section format + frontmatter key
  • Phase 3 — Apply: add sections to 5+ skills + update rules files
  • Phase 4 — Document: update or create skill-authoring guide
