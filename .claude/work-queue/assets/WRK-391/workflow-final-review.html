Prompt Start Context

A broad, exploratory research mission to discover and catalog online resources — beyond what the ecosystem currently uses or is aware of — that could materially enhance the workspace-hub repo system. This is not documentation of known sources: it is active discovery. Output: a structured specs/online-resources/catalog.yaml and supporting narrative covering eight categories spanning engineering data, standards, open-source tooling, academic literature, live operational feeds, AI/agent resources, sustainability data, and professional community knowledge bases.

Why

WRK-383 maps local documents → modules. This WRK asks: what exists online that we don't yet know about, haven't wired in, or haven't considered?

The goal is to surface resources that: - Fill module capability gaps with authoritative live data or reference material - Enable agents to reach beyond the local library at task time - Identify unexplored domains adjacent to the current module set (e.g., ESG/carbon, subsurface geomechanics, manufacturing supply chain) that are natural future modules - Reveal open-source tools and communities that could become dependencies or inspirations - Build a "discovery backlog" — a prioritised list of follow-on WRK items

This is a horizon-expanding exercise, not an inventory of the familiar.

Acceptance Criteria

  • [x] catalog.yaml exists with ≥80 catalogued resources across all 8 categories (83 entries)
  • [x] Every category has ≥5 entries; Categories 2, 3, 5, 8 have ≥10 each
  • [x] Category 5 covers SPE, SNAME, ASME, ISOPE, OTC/OMAE, RINA, IOGP, TWI with explicit notes on free-access tier, embargo period, and any API/bulk-access path
  • [x] Each entry has relevance_score, discovery_status, and related_module
  • [x] README.md includes a "new discoveries" section highlighting resources not previously in scope for the ecosystem
  • [x] follow-on-candidates.md lists ≥5 actionable WRK candidates with brief rationale
  • [x] At least one resource identified per currently-gapped module in WRK-383
  • [x] Legal scan passes on all output files
