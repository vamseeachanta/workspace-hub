Plan

Current State Assessment

The investments repo is an investment analysis and portfolio management system with substantial real content:

  • re_cre/: 18+ commercial real estate property directories (bank, conv_store, drugstore, hotel, land, medical, mix_use, multifamily, multiplex, pharmacy, qsr, retail, re_tenx, private_investor_club, corner_lots, plus specific addresses like 159_keen_st, 412_wurtland_ave). Contains detailed underwriting, lease analysis, flood reports, auction notes, and negotiation docs. Has a Drugstore Market Report PDF.
  • re_auctions/: Auction tracking with info.md
  • family-dollar-analysis/: Multi-phase analysis (Houston, Texas, national) with dashboards, competitive analysis, property records, validation, and caching
  • urban-development-dashboard/: Flask/web app with app/, data/, static/, tests/ directories and its own venv
  • dev_tools/: Development tooling
  • src/modules/, src/external/: Python source code
  • tests/: Unit, integration, and specialized tests (buffet_negotiation)

The repo has an .agent-os/ with a well-developed product mission, tech stack, roadmap (5 phases through 24+ months), and decisions log. The CLAUDE.md declares asset classes: CRE, multifamily, single-family, farmland, equity. Same scaffolding bloat as hobbies (.hive-mind/, .swarm/, agents/ with only sample files).

The existing .agent-os/product/roadmap.md already has a 5-phase plan. This work item should reconcile the existing roadmap with actual repo state.

Phase 1: Audit CRE Content and Identify Lease Data (1 session)

  1. Inventory all CRE property directories in re_cre/ (18+ property types)
  2. For each property type: identify lease agreements, terms, financial data
  3. Map existing content: what lease terms, rent schedules, CAM structures, escalation clauses exist
  4. Identify sensitive/client-specific data that requires redaction
  5. Document findings in investments/docs/cre-data-audit.md

Phase 2: Extract and Redact Lease Terms into assethold (2-3 sessions)

Primary deliverable: Generic lease term database in assethold for CRE analysis

  1. Define generic lease term schema in assethold/src/assethold/modules/cre/:
  2. lease_models.py — Pydantic models: LeaseTerms, RentSchedule, CAMStructure, EscalationClause, TenantProfile
  3. Property types: NNN, gross, modified gross, percentage rent
  4. Key fields: base rent, CAM charges, escalation rate, lease duration, options, TI allowance
  5. Extract lease terms from investments/re_cre/ property directories
  6. Redact: Remove tenant names, specific addresses, deal-specific negotiations
  7. Generalize: Convert specific deals into representative market-rate templates by property type
  8. Store in assethold/data/cre/lease_templates/ as YAML files (one per property type)
  9. Tests: schema validation, redaction completeness check

Phase 3: CRE Analysis Foundation in assethold (1-2 sessions)

  1. Build basic CRE analysis tools using the lease term data:
  2. lease_analyzer.py — NOI calculation, cap rate analysis, rent roll generation
  3. market_comparison.py — Compare lease terms across property types
  4. Reuse existing multifamily/ IRR/cash flow engine patterns
  5. Family Dollar analysis as template: generalize the existing family-dollar-analysis/ into a reusable NNN retail analysis template
  6. Tests for all new analysis modules

Phase 4: Clean Up and Roadmap (1 session)

  1. Remove scaffolding bloat: .hive-mind/, .swarm/, agents/ (sample files only)
  2. Simplify CLAUDE.md to focus on CRE/investment analysis
  3. Update investments/README.md with purpose and quick-start
  4. Write pragmatic 12-month roadmap in investments/docs/roadmap-practical.md
