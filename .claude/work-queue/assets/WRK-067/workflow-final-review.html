Prompt Start Context

Download OSHA enforcement data (inspections + violations) from enforcedata.dol.gov, OSHA severe injury reports from osha.gov/severe-injury-reports, and OSHA fatality data from osha.gov/fatalities. Filter for oil & gas NAICS codes (211, 213111, 213112). Build importers in the HSE module.

Why

Zero onshore oil & gas incident data currently exists. OSHA data covers 1973-present with daily updates.

Source: HSE data acquisition - OSHA enforcement and fatality data for onshore oil & gas

Plan

Remaining work only. Acquirer (osha_acquirer.py) and importer (osha_importer.py) are already built. Raw data is on disk at data/modules/hse/raw/osha/ (6.6GB, 13 datasets). No filtered files or database exist yet.

Phase 1 — NAICS Filtering (run existing code)

  1. Filter inspection data for oil & gas NAICS (211, 213111, 213112) and SIC (1311, 1381, 1382, 1389) codes.
  2. Run: python3 -c "from worldenergydata.hse.acquirers.osha_acquirer import OSHAAcquirer; a = OSHAAcquirer(); a.filter_oil_and_gas('data/modules/hse/raw/osha/osha_inspection0.csv', 'data/modules/hse/raw/osha/filtered/osha_inspection_oil_gas.csv')"
  3. Note: Raw inspection data is split into 6 chunks (osha_inspection0.csv through osha_inspection5.csv, ~1.5GB total). The existing filter_oil_and_gas() method processes one file at a time. Need to concatenate filtered outputs or loop through all chunks.
  4. Fix needed: The acquirer's filter_oil_and_gas() uses append mode. Create a small wrapper script that iterates over all osha_inspection*.csv chunks and appends to a single filtered output.

  5. Target output: data/modules/hse/raw/osha/filtered/osha_inspection_oil_gas.csv

  6. Filter accident data — same approach for osha_accident.csv (38MB, single file).

  7. Target output: data/modules/hse/raw/osha/filtered/osha_accident_oil_gas.csv

  8. Filter accident_injury data — osha_accident_injury.csv (28MB, single file).

  9. Target output: data/modules/hse/raw/osha/filtered/osha_accident_injury_oil_gas.csv

  10. Write chunked filter script: scripts/filter_osha_oil_gas.sh

  11. Loops over osha_inspection{0..5}.csv, osha_violation{0..13}.csv, osha_accident.csv, osha_accident_injury.csv
  12. Calls OSHAAcquirer.filter_oil_and_gas() per chunk, appending to single output per dataset
  13. Reports record counts post-filter
  14. Uses system python3 with PYTHONPATH="src" (no uv)

Phase 2 — Database Import

  1. Create HSE database: The osha_importer.py already calls Base.metadata.create_all(engine).
  2. DB path: data/modules/hse/hse_incidents.db (SQLite, as configured in DEFAULT_DB_URL)
  3. Run: PYTHONPATH="src" python3 -m worldenergydata.hse.importers.osha_importer --data-dir data/modules/hse/raw/osha

  4. The importer's _resolve_path() automatically prefers filtered/ subdirectory when use_filtered=True (default)

  5. Import flow: inspections -> accidents -> accident_injuries (all via OSHAImporter.import_data())

  6. Deduplication by bsee_incident_id (prefixed OSHA-INSP-, OSHA-ACC-, OSHA-INJ-)

  7. Chunk-based commits (50K rows per chunk) for memory efficiency

  8. Known issue: The osha_importer.py currently has a single _resolve_path() that looks for osha_inspection.csv (not osha_inspection0.csv). Since Phase 1 produces a single concatenated filtered file per dataset, this works. But if using unfiltered data, the chunked files need handling.
