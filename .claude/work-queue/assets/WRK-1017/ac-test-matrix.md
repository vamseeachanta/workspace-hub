Notes:
- `Phase 1A` = pre-rollout delivery slice: canonical checker core algorithm, bootstrap dry-run inventory, `stage5-gate-config.yaml`, timeout/circuit-breaker, concrete locking strategy, first guarded entrypoint (`plan.sh`), and core negative/positive tests. Phase 1A must be green before Phase 1B rollout begins. **Phase 1A is NOT the minimum delivered gate milestone — that is Phase 1B hard-gate delivery** (which adds `cross-review.sh`, `claim-item.sh`, `close-item.sh` guards, blocking test coverage, and the rollback/runbook).
- `Phase 1B` = gate engine delivery, guarded entrypoint rollout (all four callers), and subsequent hardening after Phase 1A is green. **`Phase 1B hard-gate delivery` is the minimum milestone at which the Stage 5 hard gate can be described as delivered.**
- `Phase 2` = required Stage 7 presentation/package work; it does not reopen Stage 5, but it must be complete before Stage 7 review and WRK closure. Phase 1 rollout may still proceed earlier if its own hard-gate bar is green.
- `Contract alignment rg checks` refers to the explicit grep-based verification commands already listed in the WRK plan.
- CI/hook rows identify the intended automation backstop targets; execution must wire the final commands into those automation points.
