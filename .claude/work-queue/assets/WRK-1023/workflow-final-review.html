The daily daily_today.sh cron already shows AI usage stats via ai-usage-summary.sh. This WRK adds a per-provider configuration summary block that reads active settings (model name, effort/thinking level, temperature, context window, or other relevant knobs) from each provider's config and prints them in the daily report.

Example output format:

=== AI Provider Config ===
claude  : model=claude-sonnet-4-6  effort=medium  thinking=off
codex   : model=o4-mini             effort=medium  reasoning=medium
gemini  : model=gemini-2.5-pro      effort=standard

Sources to read from (investigate and confirm during execution): - Claude: ~/.claude/settings.json or config/ai-tools/ — model + effort fields - Codex: ~/.codex/config.json or CODEX_MODEL env — model + reasoning effort - Gemini: ~/.gemini/settings.json or similar — model name
