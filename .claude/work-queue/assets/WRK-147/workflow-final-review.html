Prompt Start Context

Create a private GitHub repo aceengineer-strategy that mirrors the engineering orchestration pattern (workspace-hub → agents → work items → review gates) for business strategy, GTM execution, and pipeline management. Same discipline, different problem domain.

Why

Vamsee already solved the engineering execution problem with agent orchestration, work queues, and review gates. The business side needs the same structure but with a privacy wall. Engineering repos (public) and business repo (private) must stay completely separate — no cross-references, no symlinks.

Acceptance Criteria

  • [x] Private repo created on GitHub (https://github.com/vamseeachanta/aceengineer-strategy)
  • [x] .claude/ directory with CLAUDE.md, 5 agent definitions, 3 workflow definitions
  • [x] strategy/ directory with positioning, ICP, competitors, pricing, GTM documents
  • [x] pipeline/ directory with prospects template and lost-deals template
  • [x] experiments/ directory with TEMPLATE.md and at least 1 active experiment
  • [x] metrics/ directory with weekly scorecard template
  • [x] content/ directory with blog-to-sales-map seeded from existing blog posts
  • [x] All templates populated with real content from the Feb 15-16 strategy session

Source: A&CE Strategy Session Feb 15-16 — "The business strategy isn't a different discipline — it's the same system pointed at a different problem. The repo is the business, the experiments are the sprints, and the scorecard is the test suite."

Target: New private GitHub repo aceengineer-strategy

Phase 2 — CLAUDE.md & Agent Definitions

  • Create .claude/CLAUDE.md with business strategy orchestrator rules
  • Create 5 agent definitions: market-analyst, outbound-agent, pricing-analyst, content-strategist, strategy-reviewer
  • Create 3 workflow definitions: prospect-pipeline, quarterly-review, experiment-cycle

Phase 3 — Strategy Documents

  • Seed strategy/positioning.md from Feb 15-16 session notes (WRK-146 strategic positioning section)
  • Seed strategy/ideal-customer.md from GTM strategy (mid-market offshore, 10-50 engineers)
  • Seed strategy/competitors.md from WRK-148 comparable companies (Cognite, Akselos, Palantir)
  • Seed strategy/pricing-model.md with $120K retainer model
  • Seed strategy/go-to-market.md from WRK-148 priority action matrix

Phase 4 — Pipeline & Experiments

  • Create pipeline/prospects.md with 20 target companies from WRK-148
  • Create pipeline/lost-deals.md template
  • Create experiments/TEMPLATE.md
  • Create first experiment: "LinkedIn content series — AI automation for OrcaFlex"

Phase 5 — Metrics & Content

  • Create metrics/weekly-scorecard.md template (5 numbers)
  • Create content/blog-to-sales-map.md mapping existing blog posts to ICP pain points
