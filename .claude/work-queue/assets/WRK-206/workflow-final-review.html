Prompt Start Context

Create a new engineering/asset-integrity/ skill category with a Fitness-for-Service (FFS) assessment skill. FFS methodology (governed by API 579-1/ASME FFS-1) enables operators to move beyond visual inspections and thickness checks to engineering-based answers on whether corroded or damaged equipment can continue operating safely.

Gap: current engineering skills cover cathodic-protection (prevention) and risk-assessment (probabilistic), but have no FFS/damage-assessment capability bridging inspection data to run-repair-replace decisions.

Why

2H Offshore (LinkedIn, 2024): "modern fitness-for-service assessments help operators move beyond uncertainty, providing clear engineering-based answers on whether assets can continue operating safely." Key driver: visual checks + thickness measurements alone are insufficient for decision-making on corroded offshore critical assets.

FFS is a core workflow in: - Offshore platform life extension decisions - Subsea pipeline integrity management - Pressure vessel and piping re-rating - Late-life asset optimisation (cost vs. safety vs. reliability)

Existing skills cathodic-protection and risk-assessment are prerequisites but do not cover the FFS assessment loop itself.

Acceptance Criteria

  • [ ] engineering/asset-integrity/fitness-for-service/SKILL.md created
  • [ ] Covers all 3 API 579 assessment levels with worked examples
  • [ ] RSF calculation implemented in Python helper (numpy-based, no extra deps)
  • [ ] Remaining life projection function (linear and power-law corrosion models)
  • [ ] Inspection data ingestion: accepts UT grid as CSV, outputs CTP map
  • [ ] Run-repair-replace decision tree documented
  • [ ] see_also: links to cathodic-protection, risk-assessment, structural-analysis
  • [ ] At least 10 unit tests covering RSF edge cases and life calculations
