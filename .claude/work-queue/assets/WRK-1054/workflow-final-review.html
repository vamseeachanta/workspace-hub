Route B — inline plan (revised after Codex/Gemini cross-review)

Architecture

  • scripts/testing/run-all-tests.sh — thin bash orchestrator: iterates repos, invokes pytest, calls Python helper
  • scripts/testing/parse_pytest_output.py — Python helper: reads raw pytest output + exit code → emits JSONL record
  • scripts/testing/expected-failures.txt — one exact pytest node ID per line (initially empty; populate after first run)

Scope

v1 targets exactly 4 repos: assetutilities, digitalmodel, worldenergydata, assethold. ogmanufacturing excluded from v1 (not cloned on ace-linux-1).

Step 1 — run-all-tests.sh

  • Repo config table (hardcoded): name | repo_dir | PYTHONPATH | pytest_extra_args
  • For each repo: run pytest with --tb=short -q; capture stdout+stderr + exit code
  • Pass (exit_code, raw_output) to parse_pytest_output.py via process substitution or temp file
  • Collect JSONL records; print markdown table at end

Step 2 — parse_pytest_output.py

  • Input: exit_code (int), raw_output (stdin), repo_name
  • Handles all pytest exit codes: 0=pass, 1=fail, 2=interrupted, 3=internal-error, 4=cli-error, 5=no-tests
  • Parses summary line for passed/failed/error/skipped counts (regex); falls back to zero counts on missing
  • Loads expected-failures.txt; reads verbose output for failed node IDs; computes unexpected = failed_node_ids - expected_set
  • Output: single JSONL record {"repo":…,"passed":…,"failed":…,"unexpected":…,"expected_fail":…,"skip":…,"exit_code":…,"status":"ok|unexpected_failures|no_tests|error"}

Step 3 — expected-failures.txt

  • Exact node IDs (e.g. tests/test_live_data.py::TestLive::test_fetch_prices)
  • Populated after first run by inspecting failed node IDs in output
  • No wildcards in v1

Step 4 — Output

  • Markdown table: repo | pass | fail | expected-fail | skip | status
  • Aggregate totals row
  • Flags: --repo <name> (single repo), --json-out <file> (JSONL to file)

Step 5 — Exit code

0 iff unexpected_failures == 0 and no error status repos

Step 6 — TDD

  • tests/testing/test_parse_pytest_output.py — unit tests for parse_pytest_output.py: fixtures for: normal pass, normal fail, no-tests (exit 5), collection/import error (exit 2), interrupted/timeout, skipped-only, mixed with expected failures
  • No subprocess needed — feed fixture strings + exit codes directly

Step 7 — Cross-review

Codex hard gate via scripts/review/cross-review.sh (done at plan phase; redo at implementation phase)

Test strategy: - Unit: all pytest exit-code paths + summary-line edge cases - Integration: temp dir with synthetic pass/fail test; assert exit code and JSONL output
