  • Notification log: logs/notifications/YYYY-MM-DD.jsonl — each async job writes a completion event: {"source":"cron|ci|benchmark","job":"<name>","status":"pass|fail","ts":"...","details":"..."}
  • Writer shim: scripts/notify.sh <source> <job> <status> [details] — appended by cron scripts, pre-push hook, and benchmark runner on completion
  • Session-start reader: scripts/hooks/session-start-notify.sh — reads last 24h of notification log; prints a summary banner if any failures: ⚠ 2 async failures since last session: FAIL cron/nightly-tests 2026-03-09 02:14 assethold: 3 new failures FAIL benchmark 2026-03-09 03:01 digitalmodel: +35% regression
  • Retention: notifications older than 7 days purged automatically
  • Integration points: nightly test cron, pre-push CI hook (WRK-1064), benchmark runner (WRK-1071) each call notify.sh on exit
Why — Cron failures and benchmark regressions currently go unnoticed until someone manually checks logs. An agent starting an engineering session ...

Cron failures and benchmark regressions currently go unnoticed until someone manually checks logs. An agent starting an engineering session on a broken baseline produces incorrect results and wastes the entire session. A 3-line session-start banner prevents this.

Route A — simple execution.

  1. scripts/notify.sh — writer shim
  2. Args: <source> <job> <status> [details]
  3. Appends {"source":...,"job":...,"status":...,"ts":"<ISO>","details":"..."} to logs/notifications/$(date +%Y-%m-%d).jsonl
  4. Creates logs/notifications/ if absent

  5. Exit 0 always (non-blocking side-effect)

  6. scripts/hooks/session-start-notify.sh — reader

  7. Reads all logs/notifications/ files from last 24h
  8. Counts FAIL entries; if >0 prints banner to stderr listing each failure

  9. Completely silent (no output) when zero failures

  10. Wire notify.sh into nightly cron (comprehensive-learning-nightly.sh)

  11. Append bash scripts/notify.sh cron nightly-learning "$exit_status" before final exit

  12. 7-day retention — add purge line to crontab-template.sh

  13. find logs/notifications/ -name "*.jsonl" -mtime +7 -delete

  14. Placeholder comments in scripts/notify.sh header for WRK-1064/1071 wiring

  15. Tests (scripts/hooks/tests/test-session-start-notify.sh)

  16. Test 1: no notifications dir → silent
  17. Test 2: only PASS entries → silent
  18. Test 3: one FAIL entry → banner printed to stderr
  19. Test 4: mixed PASS/FAIL → banner lists only FAILs
