analysis(workspace-hub): WRK Queue Triage — Classify 160+ Pending Items

What

The pending/ directory contains 160+ work items. Many are stale, duplicates, or poorly scoped. This volume creates noise in the /work list output and makes it difficult to identify the highest-value next steps.

This task uses Gemini 1.5 Pro to perform a single-pass triage of the entire pending queue.

Why

  • Human triage of 160 items is time-consuming and prone to inconsistent categorization.
  • Gemini's large context window (2M tokens) allows it to see the entire queue state simultaneously, identifying cross-task relationships and duplicates better than sequential processing.
  • Clean queue = clearer signal for all orchestrators.

Acceptance Criteria

  • [x] Every item in pending/ is assigned a triage category (QUICK-WIN, STALE-BLOCKER, MISCLASSIFIED, DUPLICATE, CANCEL, SKIP-MACHINE).
  • [x] A Top-10 priority list is generated based on unblocked, high-leverage items.
  • [x] Stale blockers (items blocked by completed WRKs) are flagged for unblocking.
  • [x] Duplicates are identified and cross-referenced.
  • [x] The triage report is saved to specs/wrk/WRK-663/queue-triage-report.md.

Plan

  • Phase 1 — Snapshot: Capture the frontmatter and first 50 lines of every file in .claude/work-queue/pending/ into a single text artifact.
  • Phase 2 — Gemini Analysis: Process the snapshot to classify each item and rank the backlog.
  • Phase 3 — Remediation: Act on the findings (cancel items, update blockers, re-prioritize).
  • Phase 4 — Validation: Regenerate INDEX.md and confirm the queue state is cleaner.

Agentic AI Horizon

High fit: pure read + classify, no code changes. Gemini excels at bulk document classification with consistent criteria across many items. Output drives immediate queue improvement with zero risk (analysis only until Phase 3 edits are approved).
