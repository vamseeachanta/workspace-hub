1. Audit script (scripts/work-queue/audit-micro-skill-scripts.py)

Read all stages/stage-NN-*.md files, extract checklist items (lines starting with - [ ] or numbered steps), and classify each as: - scriptable: binary check expressible as exit 0/1 (e.g., "verify gate evidence exists") - judgment: requires LLM interpretation (e.g., "assess if plan is complete") - already-scripted: step already calls an existing script

2. Review and prioritize candidates

For each scriptable item: - Estimate script complexity (trivial/small/medium) - Check if an existing script already covers it - Mark high-priority candidates (appear in 3+ stages, or are in hard-gate stages 1/5/7/17)

3. Create WRK items for top candidates

For each high-priority script candidate not already covered: - Create a WRK item to implement the script - Set blocked_by: [WRK-1144]

4. Promote hook candidates

Items that should fire on every session exit or commit → flag for hook promotion (WRK-1141 already covers post-commit; new items go into a new WRK or WRK-1141 extension).
