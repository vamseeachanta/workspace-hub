Step 1 — Write scripts/review/generate-review-input.sh

Single bash script, no external deps beyond standard git + coreutils.

Logic: 1. Parse args: WRK-NNN (required), --phase N (default: 1) 2. Find WRK item: check pending/, then working/; error if not found 3. Extract frontmatter fields via awk (between --- markers): - title, mission text (first ## Mission paragraph), ACs (all - [ ] / - [x] lines), - complexity, route, subcategory, target_repos 4. Build git diff: - If target_repos is non-empty: git diff HEAD -- <repo1> <repo2> ... - Else: git diff HEAD (full workspace diff) - Count lines; if >300 truncate and append summary: ## Diff truncated at 300/N lines 5. Read checkpoint summary: assets/WRK-NNN/checkpoint.yaml → context_summary field 6. Read test snapshot: check scripts/testing/results/ for most-recent markdown; best-effort 7. Generate focus prompts table based on subcategory + complexity 8. Write structured markdown to scripts/review/results/wrk-NNN-phase-N-review-input.md
