Executive Summary

The current 20-stage WRK lifecycle runs as a single long conversation, causing context compaction (hard-stop instructions compressed away), context rot (accumulated noise from earlier stages pollutes later stages), stage jumping (no mechanical barrier between stages), and quality drift (heavy stages get same treatment as light stages).

The fix is a stage-isolation architecture: each stage starts fresh from the previous stage's exit artifacts — exactly the same model as CI/CD job isolation. Autonomous stages run as Task agents (fresh context window). Human-in-loop stages run in the current session (human IS the new context). Each stage has its own micro-skill, YAML contract, and entry/exit scripts.

Delivery order (P1 highest impact → P4):
P1: Hookify backstops (3 human gates) → P2: start/exit-stage.sh + stage-state.yaml → P3: 20 YAML contracts + 20 micro-skills → P4: heavy-stage depth enforcement

Architecture: Stage Isolation Model

Each stage is a stateless function: f(stage_contract, prev_exit_artifacts) → exit_artifacts

Two invocation types:

  • task_agent — spawns a Task agent with a constructed prompt (contract + entry_reads + micro-skill). Fresh context window. Zero context rot. Zero compaction risk.
  • human_session — loads artifacts into current session and emits checklist. Human IS the new context. Natural session boundary. Gates enforced interactively.
  • chained_agent — light stages grouped in one Task agent invocation (e.g. Stages 2+3+4 or Stages 8+9).

Execution Flow

Human(1) → TaskAgent(2,3,4) → Human(5) → TaskAgent(6) → Human(7) → TaskAgent(8,9) → TaskAgent(10) → TaskAgent(11) → TaskAgent(12) → TaskAgent(13,14,15,16) → Human(17) → TaskAgent(19,20)

P2 — Scripts + stage-state.yaml

start-stage.sh WRK-NNN <N>

#!/usr/bin/env bash
# start-stage.sh WRK-NNN N — bootstrap fresh context for stage N
set -euo pipefail
WRK_ID=$1; STAGE_N=$2
REPO_ROOT="$(git rev-parse --show-toplevel)"
CONTRACT="$REPO_ROOT/scripts/work-queue/stages/stage-$(printf '%02d' $STAGE_N)-*.yaml"
# 1. Read contract (entry_reads, invocation, skills_required, context_budget_kb)
# 2. If invocation=task_agent: construct prompt from contract + entry_reads; spawn Task agent
# 3. If invocation=human_session: load entry_reads into current session; emit checklist
# 4. Update stage-state.yaml (current_stage, invocation_model, entered_at)
# 5. Check gate artifacts if human_gate=true in previous stage contract

exit-stage.sh WRK-NNN <N>

#!/usr/bin/env bash
# exit-stage.sh WRK-NNN N — validate exit artifacts, write log, advance
set -euo pipefail
WRK_ID=$1; STAGE_N=$2
# 1. Read stage-N contract exit_artifacts[]
# 2. For each exit artifact: verify file exists; if human_gate=true, check gate field value
# 3. If any missing: print which artifact is missing; exit 1 (do NOT advance)
# 4. Write stage log ($REPO_ROOT/.claude/work-queue/logs/WRK-NNN-stage-N.log)
# 5. Update stage-state.yaml (exit_artifacts_ok: true, exited_at)
# 6. git add + commit stage log
# 7. Print Stage N+1 summary (contract for next stage)
