Deliverable 2 — knowledge/seeds/schema.md

Document the canonical per-entry fields so all domain seed YAMLs are queryable uniformly.

Fields to standardise (from naval-architecture-resources.yaml): - Top-level: category, subcategory, created_at - Per-entry: title, author, year, local_path, source_url, size_mb, topics[], notes - Section keys: textbooks, online_portals, pending_manual (domain-specific sections allowed alongside) - Note: ship_plans-style bulk sections (count + dir, no per-file entries) also acceptable for large batches

Deliverable 3 — .claude/skills/data/doc-research-download/SKILL.md

Skill covering the full repeatable workflow for any domain documentation research WRK:

  1. Fetch seed URLs, extract linked PDFs and further resource links
  2. Web search for additional freely-available resources (OpenCourseWare, archive.org, standards bodies)
  3. Download via scripts/lib/download-helpers.sh — use domain-specific script sourcing the lib
  4. Catalogue into knowledge/seeds/<domain>-resources.yaml following schema.md
  5. Regenerate document index (scripts/data/document-index/)
  6. Record WAF-blocked / borrow-only items in pending_manual: section — never skip silently
  7. No paywalled or DRM-protected content

Include: script template stub, catalogue YAML template, AC checklist.
