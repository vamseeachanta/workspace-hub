Mission

Update work-queue-workflow/SKILL.md Stage 5 (Plan Draft) to require pseudocode during planning and tests/evals specification before the user review gate.

Scope: work-queue-workflow/SKILL.md only. Not in scope: changes to work-queue/SKILL.md or workflow-gatepass/SKILL.md (unless trivially needed for cross-reference).

What

Stage 5 of the work-queue-workflow skill currently produces a plan artifact but does not explicitly require: 1. Pseudocode for non-trivial logic steps in the plan 2. A tests/evals section specifying how the implementation will be verified

Add both as mandatory elements of the Stage 5 plan draft output so that agents produce richer, more testable plans before user review.

Why

Plans without pseudocode often result in implementation drift — the agent interprets ambiguous steps differently than intended. Plans without tests/evals leave the acceptance criteria vague, making Stage 12 (TDD/Eval) harder to execute. Adding these two elements at Stage 5 aligns with TDD-first principles and reduces rework.

Acceptance Criteria

  • [ ] work-queue-workflow/SKILL.md Stage 5 section includes a "Pseudocode" requirement for non-trivial logic (with format guidance)
  • [ ] Stage 5 section includes a "Tests/Evals" subsection specifying what tests must be defined before implementation
  • [ ] Both requirements are reflected in the Stage 5 exit checklist (alongside existing HTML/publish/user-response items)
  • [ ] Language is imperative/blocking (consistent with WRK-1017 Stage 5 hardening style)
  • [ ] At least one example of acceptable pseudocode format is included (inline or referenced)

Agentic AI Horizon

  • Pseudocode + tests/evals at plan time = machine-readable specification that future agents can directly execute; investing now builds the scaffold for fully autonomous WRK execution
  • Shape it as structured YAML-embeddable pseudocode so a future eval harness can parse and auto-generate test stubs
