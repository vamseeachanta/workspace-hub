  • No retry cap — retries continue indefinitely until evidence artifacts heal

The retry loop wastes 6-8 minutes per item, produces noisy log signals without actionable context, and makes gate verification appear unreliable. Users see repeated failures with no way to diagnose or intervene.

  • [ ] verify-gate-evidence.py outputs which specific gate(s) failed on each attempt (not just overall pass/fail)
  • [ ] Retry capped at 3 attempts with exponential backoff (1s, 3s, 9s)
  • [ ] After max retries, exits with clear error listing unmet gates
  • [ ] Log signal includes attempt count: verify_gate_evidence_fail[attempt=2/3]
  • [ ] Delta output between attempts: "gate X changed from MISSING to OK"
