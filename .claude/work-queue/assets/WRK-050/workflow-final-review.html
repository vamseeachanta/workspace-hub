Status

Prompt Start Context

Inventory, assess, and consolidate all available hardware into a coherent plan for repurposing, upgrading, or retiring each device. Build a cross-platform hardware assessment script (Windows + Linux) to automate data collection. Additionally, set up per-machine dev environment readiness: agent CLI health checks, terminal configuration, repo/submodule work assignments, and domain tool maintenance — all automated via shell startup hooks.

Why

Multiple devices are sitting idle or underutilized. A systematic inventory and assessment identifies what can be repurposed (homelab, NAS, dev workstation, GPU compute) vs. what should be retired or recycled. Once machines are assigned roles, each needs a frictionless dev environment: agent CLIs verified, terminal starting in the right place, correct repos cloned, and domain tools up-to-date. Without this, every new terminal session wastes time on setup and troubleshooting.

Step 4b: Dev Environment Check Script

  1. Create scripts/operations/system/dev-env-check.sh (Linux) and dev-env-check.ps1 (Windows)
  2. Script reads the manifest for current hostname and validates:
  3. Agent CLIs (a/): checks claude --version, codex --version, gemini --version; compares against manifest minimums; reports outdated/missing
  4. Terminal config (b/): verifies cwd is workspace-hub; warns if not
  5. Repos (c/): checks each declared repo exists at expected path; reports missing repos with clone instructions
  6. Domain tools (d/): checks each declared tool version; if outdated, generates an upgrade plan (commands to run) and writes to ~/.dev-env-upgrade-plan.md for user review
  7. Output: color-coded summary (green = OK, yellow = outdated, red = missing)
  8. Verbose log to ~/.dev-env-check.log for troubleshooting
  9. Exit 0 always (informational, never blocks terminal startup)
  10. Target: <2 seconds execution time

Step 4c: Shell Profile Integration

  1. Linux: Add one-liner to .bashrc / .zshrc: bash [ -f /path/to/workspace-hub/scripts/operations/system/dev-env-check.sh ] && source /path/to/workspace-hub/scripts/operations/system/dev-env-check.sh
  2. Windows: Add to PowerShell $PROFILE: powershell if (Test-Path "C:\path\to\workspace-hub\scripts\operations\system\dev-env-check.ps1") { & "C:\path\to\workspace-hub\scripts\operations\system\dev-env-check.ps1" }
  3. Set default start directory to workspace-hub in terminal emulator settings (Windows Terminal settings.json, Linux .bashrc cd)
  4. Document setup instructions per OS in specs/modules/hardware-inventory/dev-env-setup.md
