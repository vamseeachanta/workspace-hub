  • Log file: logs/audit/agent-actions-YYYY-MM.jsonl — append-only, one JSON per line: {"ts":"...","session_id":"...","wrk_id":"WRK-NNN","action":"file_write|commit|stage_exit|script_run","target":"<path|hash>","provider":"claude"}
  • Writer: scripts/audit/log-action.sh <action> <target> [--wrk <id>]
  • Called by: session-start hook (session_start), stop hook (session_end), exit_stage.py (stage_exit), commit hook (commit), close-item.sh (wrk_close)
  • Integrity: each entry includes SHA256 of previous entry (chain) so tampering is detectable
  • Query tool: scripts/audit/audit-query.sh [--wrk WRK-NNN] [--session <id>] [--date YYYY-MM-DD]
  • Filters and prints matching entries in human-readable format
  • Retention: monthly rotation; files older than 6 months compressed to .jsonl.gz
  • Chain verification: scripts/audit/verify-chain.sh validates SHA256 chain integrity
Why — The WRK gate-evidence system captures what was intended; the audit trail captures what actually happened. Currently there is no authoritativ...

The WRK gate-evidence system captures what was intended; the audit trail captures what actually happened. Currently there is no authoritative record of which agent wrote which file in which session. This is a governance gap — especially important when multiple agents work concurrently on the same repo.
