Status

Prompt Start Context

Two sets of checks ran synchronously in the commit/session path and caused slowness on Windows/MINGW:

  1. Pre-commit: validate-skills.sh scanned all 480 SKILL.md files on every commit (5+ min on Windows). Blocked commits during WRK-304/307 session.

  2. Stop hook: ensure-readiness.sh (444 lines, 10 checks) ran at every session end. Deleted from Stop hooks in WRK-304 (commit a1854cd), but its 9 cron-suitable checks had no new home.

Root cause: same pattern as WRK-304 — bash subprocess spawning in loops is expensive on Windows. Running analysis at interaction time rather than nightly.
