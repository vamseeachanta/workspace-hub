  • Repeatability: script runs identically every time; LLM loop varies by session
  • Speed: find + wc = milliseconds; Claude scanning 500 skill files = tokens + time
  • Verifiability: exit codes are machine-readable; LLM prose is not
  • Context efficiency: script output is compact; inline scanning fills context
  • Self-improvement: once scripted, violations trigger CI/cron alerts automatically

Phase 1 — Create new scripts (Route B execution depth)

1a. scripts/skills/audit-skill-violations.sh - Input: optional --skill-dir (default .claude/skills/) - Checks per SKILL.md: README.md present, word count ≤5000, description ≤1024 chars, no XML tags in body - Output: YAML to stdout; exit 0 = clean, exit 1 = violations - Tests: happy path (clean skill dir), violation detected (oversized), missing input dir

1b. scripts/skills/skill-coverage-audit.sh - Input: optional --skills-dir, --queue-dir - Maps WRK category values → skill subdirs; counts how many have a script call vs. prose-only - Output: per-skill coverage YAML + gap list; cron-safe (silent when clean) - Tests: full coverage, partial coverage (gap found), empty skills dir

Phase 2 — Wire existing scripts into skills

2a. session-start → call detect-drift.sh --log <session-log> at Step 0 (replace prose drift check) 2b. work-queue → verify queue-status.sh is called (add reference where missing) 2c. resource-intelligence → fix hub-root path alias for validate-resource-pack.sh 2d. skill-eval → add calls to audit-skill-violations.sh + validate-skills.sh 2e. comprehensive-learning → define Phase 9 calling identify-script-candidates.sh + skill-coverage-audit.sh

Phase 3 — TDD / Eval

  • Run each new script's 3 tests (happy path, violation, missing input)
  • Verify wired skills emit correct script calls (grep-check SKILL.md for script names)
  • Run scripts/work-queue/verify-gate-evidence.py WRK-1053
