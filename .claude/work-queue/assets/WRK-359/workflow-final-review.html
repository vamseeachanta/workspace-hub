Phase 1 — Schema design + directory structure

Create under frontierdeepwater/data/processed/vessel-parameters/:

vessel-parameters/
  heavy-lift-parameters.csv        ← per-vessel engineering parameters (heavy lift)
  pipelay-parameters.csv           ← per-vessel engineering parameters (pipelay)
  crane-capacity-curves/
    {IMO}-crane-curve.csv          ← capacity vs. radius table (one file per vessel)
  vessel-rao/
    {IMO}-RAO.csv                  ← placeholder for RAO data (OrcaFlex/AQWA link)
  README.md                        ← schema docs, data sources, update procedure

Phase 2 — Parameter schemas

heavy-lift-parameters.csv

imo_number, vessel_name, contractor,
crane_type,               # revolving | stiff-leg | semi-sub | pedestal
num_cranes,
main_hook_capacity_t,     # at design radius
main_hook_radius_ft,
main_hook_height_wl_ft,   # hook height above waterline
aux_hook_capacity_t, aux_hook_radius_ft,
tandem_lift_capacity_t,
roll_period_s, pitch_period_s, heave_period_s,
gm_operating_m,           # metacentric height
max_deck_load_t_per_m2,
operating_draft_m,
dp_class,                 # DP1 | DP2 | DP3 | None
max_anchor_spread_ft, watch_circle_radius_ft,
max_hs_for_lift_m,        # operability limit
max_hs_for_transit_m, max_wind_speed_knots,
class_society, lift_standard, daf_source,
data_source_url, data_confidence, notes

pipelay-parameters.csv

imo_number, vessel_name, contractor,
lay_method,               # s-lay | j-lay | reel | flex | multi
max_lay_tension_kips,
abandonment_recovery_tension_kips,
stinger_type,             # articulated | rigid | none
stinger_radius_ft, max_departure_angle_deg,
j_lay_tower_angle_range_deg, j_lay_max_water_depth_ft,
reel_diameter_ft, max_pipe_od_reel_in, reel_capacity_ft3,
firing_line_length_ft,
tensioner_count, tensioner_capacity_each_kips,
burial_system,            # jet | plough | cutter | none
max_burial_depth_m, max_burial_dia_in,
roll_period_s, pitch_period_s, heave_period_s,
dp_class, max_anchor_spread_ft, watch_circle_radius_ft,
max_hs_lay_m, max_hs_transit_m, max_current_knots,
class_society, lay_standard,
data_source_url, data_confidence, notes

crane-capacity-curves/{IMO}-crane-curve.csv

imo_number, vessel_name, crane_id, radius_ft, capacity_t, configuration

One row per (radius, configuration) point from the crane load chart.

vessel-rao/{IMO}-RAO.csv

Placeholder files (header only) in OrcaFlex RAO import format:

frequency_hz, heading_deg, surge_rao, sway_rao, heave_rao,
roll_rao_deg_m, pitch_rao_deg_m, yaw_rao_deg_m

Phase 3 — Seed data population

Populate from (priority order): 1. Contractor vessel brochures / datasheets — WebSearch per vessel 2. Class society records (ABS / DNV online, where public) 3. Offshore Magazine capability articles 4. Noble Denton / IMCA guidelines — default DAF values where vessel-specific unavailable 5. IMO / Equasis — draft, GT, regulatory class

Set data_confidence=estimated with basis documented in notes where data is not directly sourced.

First-priority vessels for crane capacity curves: DB 101, THIALF, BALDER, HERMOD, CASTORO OTTO, SOLITAIRE, AUDACIA, LORELAY, OSA GOLIATH, GLOBAL 1200

Phase 4 — README and update procedure

Document in vessel-parameters/README.md: - Schema for each file - Data sources and update procedure - How IMO number links survey → parameters → RAO → OrcaFlex vessel type - Data confidence levels and implications for analysis use
