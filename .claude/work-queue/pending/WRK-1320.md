## Mission

Benchmark Docling on ace-linux-2's NVIDIA T400 4GB GPU (Turing, CUDA 7.5) to determine
if GPU acceleration makes Docling viable for batch processing. Then establish a machine
task routing policy: which PDF extraction tasks run where.

## Context (WRK-1303 findings)

- Docling on ace-linux-1 CPU: **310s/doc** (1570x slower than pdftotext)
- ace-linux-1 GPU: GTX 750 Ti — CUDA 5.0, too old for PyTorch (needs 7.0+)
- ace-linux-2 GPU: **NVIDIA T400 4GB** — CUDA 7.5, meets requirements
- ace-linux-2 driver status: **NOT loaded** (nvidia-smi fails, driver not installed)

## Acceptance Criteria

1. [ ] Install NVIDIA driver on ace-linux-2 (`sudo apt install nvidia-driver-xxx`)
2. [ ] Verify `nvidia-smi` shows T400 with CUDA 7.5
3. [ ] Install Docling + dependencies on ace-linux-2
4. [ ] Run same 6-document benchmark from WRK-1303 on ace-linux-2 GPU
5. [ ] Compare: ace-linux-2 GPU speed vs ace-linux-1 CPU (310s/doc baseline)
6. [ ] If GPU speedup is >10x: propose machine routing policy
7. [ ] Document: which tasks belong on which machine

## Proposed Machine Task Routing (to validate)

| Task | Machine | Rationale |
|------|---------|-----------|
| Batch text extraction (pdftotext) | ace-linux-1 | CPU-bound, proven at scale |
| Readability classification (pypdfium2) | ace-linux-1 | Lightweight, no GPU needed |
| Docling structured extraction | ace-linux-2 | T400 GPU accelerates VLM inference |
| Overnight batch Docling runs | ace-linux-2 | GPU + unattended processing |
| Interactive development | ace-linux-1 | Primary workstation |

## Prerequisites

- SSH access to ace-linux-2: `ssh vamsee@192.168.1.103` (key auth, confirmed working)
- NVIDIA driver installation requires sudo
- T400 has 4GB VRAM — check if Docling's models fit (258M params ~1GB)
