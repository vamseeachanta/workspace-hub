## Mission

Benchmark Docling on dev-secondary's NVIDIA T400 4GB GPU (Turing, CUDA 7.5) to determine
if GPU acceleration makes Docling viable for batch processing. Then establish a machine
task routing policy: which PDF extraction tasks run where.

## Context (WRK-1303 findings)

- Docling on dev-primary CPU: **310s/doc** (1570x slower than pdftotext)
- dev-primary GPU: GTX 750 Ti — CUDA 5.0, too old for PyTorch (needs 7.0+)
- dev-secondary GPU: **NVIDIA T400 4GB** — CUDA 7.5, meets requirements
- dev-secondary driver status: **NOT loaded** (nvidia-smi fails, driver not installed)

## Acceptance Criteria

1. [~] Install NVIDIA driver on dev-secondary — driver-580-open installed, nouveau blacklisted, **reboot required**
2. [ ] Verify `nvidia-smi` shows T400 with CUDA 7.5
3. [ ] Install Docling + dependencies on dev-secondary
4. [ ] Run same 6-document benchmark from WRK-1303 on dev-secondary GPU
5. [ ] Compare: dev-secondary GPU speed vs dev-primary CPU (310s/doc baseline)
6. [ ] If GPU speedup is >10x: propose machine routing policy
7. [ ] Document: which tasks belong on which machine

## Proposed Machine Task Routing (to validate)

| Task | Machine | Rationale |
|------|---------|-----------|
| Batch text extraction (pdftotext) | dev-primary | CPU-bound, proven at scale |
| Readability classification (pypdfium2) | dev-primary | Lightweight, no GPU needed |
| Docling structured extraction | dev-secondary | T400 GPU accelerates VLM inference |
| Overnight batch Docling runs | dev-secondary | GPU + unattended processing |
| Interactive development | dev-primary | Primary workstation |

## Prerequisites

- SSH access to dev-secondary: `ssh vamsee@10.0.0.2` (key auth, confirmed working)
- NVIDIA driver installation requires sudo
- T400 has 4GB VRAM — check if Docling's models fit (258M params ~1GB)
