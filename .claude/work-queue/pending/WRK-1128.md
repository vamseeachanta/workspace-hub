# Observed-Exposure Tracker

## Mission
**Build a script that reads the WRK queue and calculates what % of stage-work is AI-handled vs human-gated per category, producing our own "actual vs theoretical automation" gap report.**

## What
Anthropic's labor market research introduces "observed exposure" — the fraction of tasks that AI *actually* automates vs what it theoretically could. We should compute this for our own work queue:

- For each WRK category (coding, research, data, docs, harness), count stages that were AI-executed vs human gates (stages 1, 5, 7, 17)
- Output table: `category | total_stages | ai_stages | human_stages | observed_exposure_%`
- Compare against theoretical maximum (all non-hard-gate stages = AI-automatable)
- Produce a gap report: which categories have the biggest theory vs practice delta

## Why
Anthropic found Computer & Math jobs have 94% theoretical exposure but only 33% actual automation. We likely have a similar gap. Measuring it shows where to focus next automation efforts.

## Acceptance Criteria
- [ ] Script `scripts/ai/observed-exposure-report.py` reads queue + archive WRK files
- [ ] Classifies each stage as AI or human using `is-human-gate.sh`
- [ ] Outputs Markdown table: category, total stages, AI%, human%, observed_exposure_%
- [ ] Includes theoretical max (100% - hard_gate_fraction) for comparison
- [ ] ≥3 passing TDD tests (empty queue, mixed categories, archived items)
- [ ] Documented in `scripts/ai/README.md`

## Plan

1. **Script structure** — `scripts/ai/observed_exposure_report.py` (snake_case filename matches `wrk_cost_report.py` pattern)
   - `scan_wrk_files()`: glob pending/working/archive .md files, parse YAML frontmatter for id + category
   - `load_stage_evidence()`: read `assets/WRK-NNN/evidence/stage-evidence.yaml`, return stages list
   - `classify_stages()`: binary classification — stages {1,5,7,17} = human, rest = AI (mirrors `is-human-gate.sh`)
   - `aggregate_by_category()`: group + sum stage counts per category
   - `format_table()`: Markdown or CSV output with observed exposure % and theoretical max %
   - CLI via argparse: `--csv`, `--queue-root`

2. **Theoretical max** — computed as `(total_stages - human_stages) / total_stages` per category; for the full 20-stage lifecycle this is 80% (16/20)

3. **TDD tests** — `scripts/ai/tests/test_observed_exposure_report.py`: 9 tests covering empty queue, full lifecycle, mixed categories, partial stages, missing evidence, CSV output, markdown format, archive discovery, human gate constant verification

4. **Documentation** — update `scripts/ai/README.md` with usage examples

5. **Future enhancement** — current binary classification means observed = theoretical for completed WRKs; a future iteration (WRK-5003) should track actual execution method to reveal true automation gaps
