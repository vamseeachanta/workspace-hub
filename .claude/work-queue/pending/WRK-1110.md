# enhance(workstations): Hardware Utility, Future Planning & Upgrade Roadmap

## Mission

The `workstations/SKILL.md` registry is the single source of truth for all machine knowledge —
every workstation knows every other workstation. Extend it with hardware utility analysis,
future acquisition planning, and a structured upgrade roadmap so routing decisions are
grounded in real capacity data, not guesswork.

## Why

`CLAUDE.local.md` (per-machine personal notes) is explicitly rejected: siloed knowledge
creates drift and breaks multi-machine workflows. All workstation knowledge lives in the
shared skill, available to every session on every machine.

The current skill covers registry, routing, and software maps well. It is missing:
- **Utilisation assessment**: are machines being used to their capability ceiling?
- **Future hardware planning**: what to acquire and when?
- **Upgrade roadmap**: spare parts allocation + procurement priorities
- **Network topology**: SSH, Tailscale, and mount-point map for clarity

## What

### Section 1: Utilisation Assessment
For each machine, score current utilisation vs capability ceiling:
- CPU: peak task profile vs core count
- RAM: typical working-set vs installed
- Storage: active data vs capacity
- GPU: current workloads vs VRAM/CUDA profile
- AI CLIs: which tools are actually being invoked per cron variant?

Output: `## Utilisation Snapshot` table in SKILL.md with traffic-light status
(under-utilised / well-matched / at-ceiling) per machine.

### Section 2: Future Hardware Planning
Structured list of hardware candidates to evaluate for acquisition:
- What capability gap each addresses
- Estimated cost tier (budget / mid / high)
- Which workloads become possible or faster
- Dependencies (rack space, power, network)

Seed from current known gaps:
- gali-linux-compute-1 is preliminary — needs procurement decision
- GPU upgrade path for ML/inference (T400 is limited)
- acma-ws014 AI tooling still not set up

### Section 3: Upgrade Roadmap
Ordered list of actionable upgrades across existing machines:
- Spare DDR3 RAM → which machine can actually use it?
- GPU 2 (unidentified) → identify model, assign to best fit
- SSD 500GB spare → consolidation target
- acma-ws014 → AI CLI install plan (WRK scope TBD)
- ace-linux-2 → claude native install (WRK-389)

### Section 4: Network & Mount Topology
Simple reference map:
- SSH access matrix (which machines can reach which, passwordless?)
- Tailscale IPs for each reachable machine
- Key mount points (e.g. `/mnt/local-analysis/` on ace-linux-1)
- Shared storage paths used by cross-machine workflows
