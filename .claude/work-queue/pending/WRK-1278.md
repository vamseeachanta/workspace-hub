# setup-cron.sh --replace mode + deploy canonical crontabs on all workstations

## Mission
**Add a `--replace` flag to `setup-cron.sh` that writes a clean crontab from `schedule-tasks.yaml` (removing stale entries), then deploy canonical crontabs on ace-linux-1, ace-linux-2, and document Windows Task Scheduler steps for acma-ansys05/acma-ws014.**

> Read this at the start of every execution stage. If a task, file change, or decision is
> not directly in service of this mission, stop and capture it as a new WRK item instead.

## What
- Add `--replace` flag to `setup-cron.sh` that replaces the entire crontab with YAML-driven entries (vs current append-only default)
- Remove stale entries from ace-linux-1: `session-analysis` (merged into comprehensive-learning per WRK-1102), hourly `ai-tools-status` (now weekly)
- Add missing entries on ace-linux-1: `dep-health`, `benchmark-regression`, `doc-drift`, `notification-purge`
- Fix ace-linux-2: add missing `repository-sync`, remove jobs beyond `contribute` role (or promote to `full` with documented rationale)
- Standardize log paths: consolidate to `cron.log` or per-job logs under `logs/` instead of scattered `/tmp/` files
- Document Windows Task Scheduler steps for acma-ansys05/acma-ws014 with screenshots or step-by-step

## Why
WRK-1021 built the YAML source of truth and rewired `setup-cron.sh` to read it, but the actual crontabs on machines still have drift (stale entries, wrong paths, missing jobs). A `--replace` mode enables clean deployment.

## Acceptance Criteria
- [ ] `setup-cron.sh --replace` replaces crontab with YAML-only entries (with backup of old crontab)
- [ ] `setup-cron.sh --replace --dry-run` previews the replacement without modifying
- [ ] ace-linux-1 crontab matches `schedule-tasks.yaml` exactly (no stale entries)
- [ ] ace-linux-2 crontab matches its `contribute` role entries
- [ ] Windows Task Scheduler instructions updated in `schedule-tasks.yaml` with concrete paths for acma-ansys05/acma-ws014
- [ ] Tests: `--replace` dry-run test added to `test_validate_schedule.py`
- [ ] Old crontab backed up to `/tmp/crontab-backup-<date>.txt` before replace
- [ ] Per-workstation verification: run `setup-cron.sh --dry-run` on each machine and capture output as evidence
- [ ] Per-workstation smoke test: after `--replace`, verify each cron job runs successfully (log output captured)
- [ ] Evidence files: `assets/WRK-1278/evidence/<hostname>-crontab-verified.yaml` for each machine
