## Mission

Unify skills, scripts, and workflows into co-located nested structures. Eliminate flat `<tool>-*` prefix clusters. Enforce 200-line hard limit on all skill files.

## What

Three concerns are currently split across three directory trees (skills, scripts, workflows). This WRK aligns them so each tool family is self-contained.

### Phase 1: Nest tool families (flat prefix -> nested)

| Family | Current flat count | Target |
|--------|-------------------|--------|
| aqwa-* | 5 skills | `aqwa/` (DONE — exemplar) |
| orcaflex-* | 20+ skills | `orcaflex/` |
| orcawave-* | 6 skills | `orcawave/` |

### Phase 2: Script co-location

Move scripts from `scripts/` into `<skill>/scripts/` where relationship is 1:1. Shared scripts stay in `scripts/lib/`.

### Phase 3: Workflow alignment

Add `workflow.yaml` to skill families with natural sequences.

## Acceptance Criteria

1. Zero flat `<tool>-*` prefix clusters with 3+ siblings in `marine-offshore/`
2. All skill files under 200 lines (hard limit from coding-style.md)
3. Scripts co-located with parent skill where relationship is 1:1
4. `eval-skills.py --skill <tool>` finds full family
5. aqwa-batch-execution split to <200 lines
