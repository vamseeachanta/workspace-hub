## Mission

Batch deep extraction on all 11,741 XLS/XLSX files from ace_project.
Highest yield stratum (93% contain tables, 100% machine-readable, 0% OCR needed).

## Why

WRK-1246 yield assessment identified XLS as the fastest win — every file is
machine-readable and almost all contain tables. This is pure deterministic
extraction with zero LLM cost.

## Plan

### Step 1: Build batch extraction script

Create `scripts/data/doc-intelligence/batch-extract-xlsx.py`:
- Read index.jsonl, filter `source=ace_project` + `ext in (xlsx, xls)`
- Run `extract_document()` → manifest → `deep_extract_manifest()` per file
- 30s timeout per file, skip >20MB
- Write results to `data/doc-intelligence/batch/xlsx/` (one CSV per table)
- Append to `data/doc-intelligence/manifest-index.jsonl`
- Progress log every 100 files

### Step 2: Run batch extraction

```bash
nohup uv run --no-project python scripts/data/doc-intelligence/batch-extract-xlsx.py \
    > logs/enrichment/batch-extract-xlsx-$(date +%Y%m%d).log 2>&1 &
```

### Step 3: Generate summary report

Counts: total extracted, tables per file, error rate, promoted candidates.

## Acceptance Criteria

1. [ ] Batch script created with timeout + resume support
2. [ ] All 11,741 XLS/XLSX files processed (or attempted)
3. [ ] Extracted tables written to `data/doc-intelligence/batch/xlsx/`
4. [ ] Manifest index updated with extraction counts
5. [ ] Summary report YAML with yield vs WRK-1246 sample comparison
