## Mission

Create a top-level orchestration skill that sequences the entire calculation
implementation workflow as a single named pattern. Three skills already exist
and are well-integrated but not documented as a unified workflow:

1. research-literature -> produces research brief with standards, equations, worked examples
2. dark-intelligence-workflow -> extracts methodology from legacy files, legal gate, archives
3. calculation-report -> produces YAML + HTML report from implementation

The orchestration skill connects these into a repeatable end-to-end workflow
that any engineer (human or agent) can follow to implement a new calculation.

## What

Create `.claude/skills/data/calculation-implementation-workflow/SKILL.md` that defines:

### The 10-Step Workflow

1. **Research** -- invoke research-literature skill for the target category/subcategory
2. **Gather** -- search university resources, textbooks, coursework (Step 5 of research skill)
3. **Extract** -- if legacy Excel/files exist, invoke dark-intelligence-workflow
4. **Legal gate** -- run legal-sanity-scan on all extracted content (HARD GATE)
5. **Archive** -- save methodology to knowledge/dark-intelligence/ (if from legacy)
6. **TDD Red** -- write failing tests using worked examples from research brief + archives
7. **TDD Green** -- implement minimal code to pass tests
8. **TDD Refactor** -- clean up while keeping tests green
9. **Calc Report** -- produce calculation-report YAML, validate against schema, generate HTML
10. **Commit** -- commit to appropriate repo + push, update capability map status

### Integration Points
- Input: category, subcategory, target repo/module
- Consumes: research-literature, dark-intelligence-workflow, calculation-report skills
- Produces: implementation code, tests, calc report YAML+HTML, capability map update
- Gates: legal scan (Step 4), schema validation (Step 9), all tests pass (Step 8)

### Enforcement
- Each step has a verification command (script-based, not LLM judgment)
- Steps 4 and 9 are hard gates -- workflow cannot proceed without passing
- The skill references patterns.md enforcement gradient (target L2 -- script)

## Why

- This session implemented 10+ calculations -- each followed this pattern implicitly
- Making it explicit prevents skipped steps (e.g., missing calc reports caught by audit)
- New agents can follow the workflow without understanding three separate skills
- Compounds: every calculation implementation gets faster as the workflow matures

## Acceptance Criteria

1. Skill file at `.claude/skills/data/calculation-implementation-workflow/SKILL.md`
2. References all three constituent skills with integration points
3. 10 steps with verification commands for each
4. Hard gates documented for legal scan and schema validation
5. At least one worked walkthrough showing all 10 steps for a real calculation
6. TDD: tests for skill file structure and required sections
