## Mission

Fix table extraction quality: filter watermarks, set content thresholds, and add
LLM quality rating. Currently 41% of extracted tables are junk (242/664 are IHS
watermark repeats from API RP 2A).

## Why

WRK-1246 audit revealed the table extractor has no quality gate:
- API RP 2A: 242 "tables" = same 3-line IHS license watermark on every page (0% real)
- DNV standards: 80-96% real tables (good quality)
- 28 empty tables across other standards
- No quality scoring — downstream consumers can't filter bad data

Without this fix, WRK-1253/1255 batch extraction will produce millions of junk rows.

## Plan

### Step 1: Deterministic watermark/dedup filter

Add to `table_exporter.py`:
- **Content hash dedup**: hash each table's content; skip if seen before in same doc
- **Watermark pattern list**: regex for known watermarks (IHS Licensee, DNV copyright, etc.)
- **Min-content gate**: skip tables with <3 data rows OR <2 columns with numeric values

### Step 2: LLM quality rating (Haiku)

After deterministic filters, rate surviving tables via Haiku:
- Input: first 10 rows of CSV as text
- Prompt: "Rate this extracted table: usable (clean engineering data) / partial (some useful data, needs cleanup) / junk (headers only, formatting artifacts, no data)"
- Output: quality field in extraction report YAML
- Cost: ~$0.0001/table × ~400 real tables = ~$0.04 total for existing corpus
- For batch runs: rate a sample (every 10th table) to estimate quality without rating all

### Step 3: Retroactive quality audit

Re-score all 664 existing tables:
```bash
uv run --no-project python scripts/data/doc-intelligence/rate-table-quality.py \
    --input data/doc-intelligence/deep/tables/ --output data/doc-intelligence/table-quality-audit.yaml
```

### Step 4: Quality report

Generate per-document and per-domain quality summary:
- Tables: total, real, watermark, empty, usable, partial, junk
- Quality rate = usable / total
- Flag documents with <50% quality rate for parser investigation

## Acceptance Criteria

1. [ ] Dedup filter removes identical repeated tables within a document
2. [ ] Watermark regex catches IHS Licensee + DNV copyright patterns
3. [ ] Min-content gate: ≥3 data rows, ≥2 numeric columns
4. [ ] LLM quality rating: usable/partial/junk per table
5. [ ] Retroactive audit of 664 existing tables with quality YAML
6. [ ] Quality rate improves from 59% to ≥95% (after filtering)
7. [ ] Tests: watermark detection, dedup, min-content, quality rating
