# Custom GitHub CLI bash tools for session-cycle efficiency

## What

Create a set of lightweight `gh` API wrapper scripts for quick GitHub posting and reading mid-session — avoiding context switches and verbose `gh` CLI boilerplate during work cycles.

## Why

During sessions we frequently need to check PR status, read/post review comments, create/update issues, and label PRs. Each time this requires composing `gh api` or `gh pr` commands from scratch. Purpose-built wrappers will let us cycle through GitHub interactions rapidly without leaving the session flow.

## Approach

Review existing session patterns (past conversations, hook logs) to identify the most common GitHub interactions, then build targeted bash scripts.

### Scripts to create (in `scripts/github/`)

1. **gh-pr-status.sh** — Quick PR state + checks summary (pass/fail/pending)
2. **gh-pr-comments.sh** — Fetch and post review comments on a PR
3. **gh-issue-quick.sh** — Create, read, and update issues with minimal args
4. **gh-post.sh** — Generic posting wrapper with common templates (comment, label, review-request)

### Design principles

- Single-purpose scripts, composable via pipes
- Sensible defaults (current repo, current branch) — zero-arg happy path
- Output designed for terminal reading (concise, colored) and machine parsing (--json flag)
- Use `gh api` where it's faster than `gh pr`/`gh issue` subcommands

## Acceptance Criteria

- [ ] `gh-pr-status.sh [PR#]` shows state, checks, reviewers in ≤2 lines
- [ ] `gh-pr-comments.sh read [PR#]` fetches latest review comments
- [ ] `gh-pr-comments.sh post [PR#] "message"` posts a comment
- [ ] `gh-issue-quick.sh create "title" --body "desc"` creates an issue
- [ ] `gh-issue-quick.sh read [ISSUE#]` shows issue details
- [ ] `gh-post.sh label [PR#] "label1,label2"` applies labels
- [ ] All scripts work with zero args (default to current repo/branch/PR)
- [ ] `--json` flag on read commands for machine-parseable output
- [ ] Scripts source a shared `scripts/github/_common.sh` for repo detection and auth check
