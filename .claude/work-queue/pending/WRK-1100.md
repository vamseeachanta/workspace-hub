# fix(whats-next): exclude standing+cadence items from priority list

## Problem

Items with `standing: true` + `cadence:` (e.g. WRK-235 monthly roadmap review) appeared in
`/whats-next` as high-priority items, crowding out genuinely actionable work. These items
self-schedule their own reviews and must not compete with day-to-day priorities.

## Solution Implemented (2026-03-10)

Added a 4-line guard in `scripts/work-queue/whats-next.sh` `process_file()`:

```bash
# Skip periodic-review items (standing + cadence) — they self-schedule, not daily work
local _standing _cadence
_standing=$(get_field "$f" "standing")
_cadence=$(get_field "$f" "cadence")
[[ "$_standing" == "true" && -n "$_cadence" ]] && return
```

**Logic:**
- `standing: true` + `cadence: monthly` → **excluded** (WRK-235 roadmap review)
- `standing: true`, no cadence → **stays** (WRK-234 mission item — always active)
- All other items → **unaffected**
