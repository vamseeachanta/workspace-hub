# WRK-1028 — Stage-aware WRK workflow enforcement

## Mission
**Mechanically enforce the 20-stage WRK lifecycle so agents cannot skip stages or bypass human gates, through per-stage YAML contracts, an advance-stage.sh orchestrator, and hookify backstops.**

> Read this at the start of every execution stage. If a task, file change, or decision is not directly in service of this mission, stop and capture it as a new WRK item instead.

## What

Agents routinely drift from the prescribed 20-stage workflow: jumping past Stage 5 without explicit user approval, missing evidence artifacts with wrong file names or field formats, omitting log entries because wrapper scripts aren't invoked, and losing hard-stop language to context-window compression.

The fix is three mechanical enforcement layers:
1. **Per-stage YAML contracts** — compact (≤15 lines each), machine-readable, one per stage; defines entry criteria, exit artifacts, `human_gate: true/false`, and blocking reason.
2. **`advance-stage.sh WRK-NNN [<N>]`** — validates the current stage's exit artifacts, writes stage logs automatically, updates stage-state.yaml, and emits the next stage contract as a checklist.
3. **Hookify backstops** — pre-Write/Edit hooks that block Stage N+2 artifact creation while Stage N is still pending (specifically: block cross-review artifacts until Stage 5 `user-review-plan-draft.yaml` has `decision: approved`).

## Why

Every enforcement mechanism added so far (SKILL.md hard-stop text, exit checklists, verify-gate-evidence.py at close) is behavioral or terminal. Text instructions are compressed by context windows. Close-time verification catches failures too late. Agents need:
- **Explicit current-stage awareness** at execution time (not inferred from artifacts)
- **Mechanical stage transition validation** (script refuses to advance if exit criteria unmet)
- **Real-time blocking** of premature stage jumps (hookify, not just post-hoc verification)

## Stage Classification

| Type | Stages |
|------|--------|
| Human-in-loop (blocking) | 1 (capture intent), 5 (plan draft review), 7 (plan final review), 17 (implementation review) |
| Autonomous | 2, 3, 4, 6, 8, 9, 10, 11, 12, 13, 14, 15, 16, 18, 19, 20 |

Human-in-loop stages require `human_gate: true` in contract → `advance-stage.sh` emits an explicit STOP message and waits for the agent to receive user response before accepting exit-artifact evidence.

## Acceptance Criteria

- [ ] 20 per-stage YAML contract files created at `scripts/work-queue/stages/stage-NN-<slug>.yaml`
- [ ] Each contract has: `order`, `name`, `human_gate`, `entry_criteria[]`, `exit_artifacts[]`, `blocking_reason`, `log_action`
- [ ] `advance-stage.sh WRK-NNN` validates current stage exit artifacts, writes log, updates `stage-state.yaml`, prints next stage contract
- [ ] `stage-state.yaml` at `.claude/state/stage-state.yaml` tracks: `wrk_id`, `current_stage`, `stage_name`, `entered_at`, `human_gate_pending`
- [ ] Hookify rule: blocks write of cross-review artifacts (review.md, cross-review.yaml, execute.yaml) when Stage 5 user-review-plan-draft.yaml is absent or decision ≠ approved
- [ ] `advance-stage.sh` writes the correct log action (from contract `log_action` field) to `.claude/work-queue/logs/WRK-NNN-<stage>.log`
- [ ] `verify-gate-evidence.py` updated to read `stage-state.yaml` as additional signal (non-blocking warn if absent for legacy WRKs)
- [ ] Stage contracts are the single source of truth for stage names, log actions, and artifact paths — no duplication in SKILL.md prose
- [ ] At least 5 unit tests for `advance-stage.sh` covering: happy path advance, blocked-by-missing-artifact, human-gate stop, log write, stage-state update
- [ ] workflow-gatepass/SKILL.md updated to reference `advance-stage.sh` as canonical stage transition command

## Agentic AI Horizon

- Per-stage YAML contracts are machine-parseable specs — a future eval harness can auto-generate test stubs per stage
- `advance-stage.sh` becomes the runtime backbone for fully autonomous WRK execution: agent calls `advance-stage.sh` → gets exact next-stage checklist → executes → calls again
- Hookify backstops + stage state = audit trail for autonomous session replay without human oversight
- This is the enforcement infrastructure that makes Route A/B/C autonomy safe at scale

## Design Notes (from brainstorming 2026-03-07)

**Root cause analysis — 4 failure categories:**
1. Stage jump: tool-call momentum bypasses interactive stops; no mechanical block exists
2. Format miss: gate artifact contracts (file names, field names, paths) buried in verify-gate-evidence.py, not visible to agents at write time
3. Missing logs: wrapper scripts not invoked → no log signals → agent log gate fails at close
4. Context loss: 161-line monolithic SKILL.md hard-stop language compressed out in long sessions

**Option A (per-stage skills)**: Better than monolith but still behavioral — agent must remember to invoke; doesn't block momentum.
**Option B (stage orchestrator script)**: Machine-enforced transitions; agent can still bypass by writing files directly.
**Option C (hybrid — chosen)**: Script + hookify + compact contracts. Defense in depth. Hookify blocks even if script is bypassed. Script writes logs automatically. Compact contracts (≤15 lines) give fresh per-stage context without truncation.

**Staged delivery:**
- P1: Hookify rule for Stage 5 (immediate, 1 rule, highest-impact failure mode)
- P2: `advance-stage.sh` + `stage-state.yaml` (explicit current-stage tracking + auto log write)
- P3: 20 per-stage YAML contracts (single source of truth for stage definitions)
