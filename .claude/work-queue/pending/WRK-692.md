# feat(session): Interactive Session Analysis — Codex-Orchestrator Variant

## What

Codex-orchestrator parallel of WRK-691. Adapt the session load assessment and interactive
logging skill for sessions where Codex is the orchestrator.

WRK-691 (claude-orchestrator) must be complete before this item. This item wires the same
`load-assessment.sh` and `session-interactive-analysis` skill into Codex's session entry
point and logging path.

## Scope (delta from WRK-691)

### Codex-specific session entry

Codex sessions do not have a `/session-start` hook equivalent. Entry point:
- `AGENTS.md` Dev policy (read at Codex session open)
- `.codex/` settings and prompt files

Adaptation required:
- Add a `## Session Load Assessment` section to `.codex/prompts/session-open.md`
  (or create it if it doesn't exist)
- Prompt Codex to run `bash scripts/session/load-assessment.sh` as the first action
- If output has gaps, Codex reads the referenced file interactively before proceeding

### Logging path for Codex

Codex writes session transcripts to `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl`.
The load assessment log target is the same orchestrator file:
```
logs/orchestrator/codex/session_YYYYMMDD.log
```
(text, not JSONL — consistent with existing Codex log format in `verify-log-presence.sh`)

Append load assessment result:
```
[session_open] ts=<ISO> machine=<hostname> wrk=<active-wrk>
[load_assessment] rules_loaded=python-runtime,git-workflow rules_missing= quota_fresh=true
```

### Codex behavioral note

From WRK-658 agent behavior verdicts (2026-03-02):
> Codex — military discipline — sufficiently rigid, follows rules as specified.
> Primary risk: literal interpretation (rule gaps become loopholes).

This means load-assessment.sh output format must be unambiguous — Codex will follow it
literally. Any gap in the check table is treated as a hard block, not a soft warning.

## Deliverables (Codex-specific, building on WRK-691)

| `.codex/prompts/session-open.md` | CREATE or EDIT — instruct Codex to run load-assessment.sh first |
