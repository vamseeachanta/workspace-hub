# fix(review): setsid hard-fail in submit-to-claude.sh blocks non-Linux agents

## Problem

`scripts/review/submit-to-claude.sh` exits 1 immediately if `setsid` is not found
(lines 8-14). `setsid` is part of `util-linux` — available on Linux but not on macOS
or minimal containers. If Codex (ace-linux-2) or any future macOS agent calls this
script, it hard-fails before attempting the review.

Discovered during agent-call review of `submit-to-claude.sh` (2026-03-03).

## What

Replace the hard-fail guard with a graceful fallback: detect `setsid`, use it when
available, skip it when absent (accepting reduced process-group cleanup on those hosts).

## Fix

```bash
# Lines 8-14 — replace hard exit with conditional
if command -v "${SETSID_CMD:-setsid}" >/dev/null 2>&1; then
  SETSID_CMD="${SETSID_CMD:-setsid}"
else
  echo "WARN: setsid not found — running without process-group isolation" >&2
  SETSID_CMD=""
fi
```

In `run_claude_once()` (line 146), guard the prefix:

```bash
# Before
"$SETSID_CMD" timeout "$CLAUDE_TIMEOUT_SECONDS" claude \

# After
${SETSID_CMD:+"$SETSID_CMD"} timeout "$CLAUDE_TIMEOUT_SECONDS" claude \
```

## Scope

- `scripts/review/submit-to-claude.sh` — primary fix
- Audit `submit-to-codex.sh` and `submit-to-gemini.sh` for the same pattern
