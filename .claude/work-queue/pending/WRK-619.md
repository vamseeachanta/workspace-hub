# Pile Capacity (Phase 2)

## What

Implement pile axial and lateral capacity calculations per API RP 2GEO,
including p-y curves (Matlock soft clay, Reese stiff clay, API sand),
t-z/q-z soil spring curves, and simplified pile driving analysis.

## Why

Pile foundations are ubiquitous in offshore structures (jackets, monopiles,
conductor casings). The vertical_riser module already references p-y models
loaded from Excel — this provides the canonical implementation.

## Scope

### Packages
- openpile (GPL-3.0) — optional extra, cross-validation only
- groundhog (GPL-3.0) — optional extra, cross-validation only

### Standards
- API RP 2GEO (Sec 6-8)
- DNV-RP-C212
- API RP 2A-WSD (legacy reference)

## Deliverables

- `geotechnical/piles/axial_capacity.py` — API RP 2GEO driven piles (clay + sand)
- `geotechnical/piles/lateral_capacity.py` — p-y curves: Matlock, Reese, API sand
- `geotechnical/piles/tz_qz_curves.py` — t-z / q-z soil springs
- `geotechnical/piles/pile_driving.py` — simplified Smith wave equation
- `geotechnical/piles/_openpile_adapter.py` — optional openpile wrapper
- Test suite validated against Reese & Van Impe (2001) p-y examples

## Acceptance Criteria

- [ ] Axial capacity for driven piles in clay (alpha method) and sand (beta method)
- [ ] p-y curves: Matlock soft clay, Reese stiff clay, API sand
- [ ] t-z and q-z soil spring curves per API RP 2GEO
- [ ] Simplified Smith wave equation for pile driving
- [ ] Optional openpile adapter with GPL guard pattern

- [ ] Cross-review passes (Codex hard gate)
