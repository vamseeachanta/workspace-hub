## Problem

`scripts/quality/check-all.sh` now exceeds the 400-line hard limit (currently ~487 lines).
The file grew across three WRKs:
- WRK-1056: ruff + mypy (~274 lines)
- WRK-1081: bandit + radon + vulture (+~155 lines)
- WRK-1059: docs structure + api audit (+~58 lines)

## Goal

Split `check-all.sh` into focused files without changing any external behaviour:

| File | Responsibility |
|------|---------------|
| `scripts/quality/check-all.sh` | Orchestrator: flag parsing, main loop, summary |
| `scripts/quality/lib/ruff-mypy.sh` | run_ruff, run_mypy |
| `scripts/quality/lib/docs-checks.sh` | run_ruff_docs, check_readme_sections, check_docs_dir, check_docs_structure, detect_build_system |
| `scripts/quality/lib/static-checks.sh` | run_bandit, run_radon, run_vulture |
| `scripts/quality/lib/api-checks.sh` | run_api_audit |

Each lib file is sourced by `check-all.sh` via `source "${SCRIPT_DIR}/lib/<file>.sh"`.

## Acceptance Criteria

- `check-all.sh` ≤ 200 lines after split
- All lib files ≤ 150 lines
- All existing tests (T1–T20) still pass with 0 failures
- `bash scripts/quality/check-all.sh --help` still works identically
- No behaviour change — pure structural refactor
