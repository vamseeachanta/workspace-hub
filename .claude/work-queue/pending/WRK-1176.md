## Mission

Formalize the two-agent pattern (initializer + coder) for multi-session WRKs — first session scaffolds environment and verification, subsequent sessions make incremental progress.

## What

Anthropic's harness splits long-running agent work into two distinct roles:
1. **Initializer agent** (first session): creates feature list, setup script, baseline commit, progress file
2. **Coding agent** (subsequent sessions): reads progress, selects next feature, implements, commits, updates progress

Map this to workspace-hub's WRK lifecycle:
- Stages 1-9 = Initializer role (planning, scaffolding, verification setup)
- Stages 10-16 = Coder role (incremental implementation with progress tracking)
- Stages 17-20 = Closure role (review, archive)

Formalize this in session.sh so that multi-session WRKs explicitly track which role the current session is performing, and the coder sessions always start with the session startup protocol (health check → read progress → select next sub-task → verify baseline → implement).

## Why

Without role separation, agents in later sessions repeat scaffolding work or skip the structured startup protocol, leading to context loss and wasted tokens.

## Acceptance Criteria

- [ ] Session role (initializer/coder/closer) tracked in session state
- [ ] Coder sessions enforce startup protocol before implementation
- [ ] Documentation in orchestrator-pattern.md
- [ ] Compatible with existing WRK lifecycle stages
