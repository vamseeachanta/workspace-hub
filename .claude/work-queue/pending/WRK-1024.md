# chore(harness): increase context window limits for Claude and Codex

## Mission
**Identify and apply the maximum available context window settings for both Claude and Codex CLI/API configs across all machines, so long-session work (large codebases, multi-file reviews) does not hit truncation limits prematurely.**

## What
Both Claude and Codex have configurable context limits. This WRK audits the current limits set in each provider's config files and bumps them to the maximum supported values (or project-appropriate values). Applies to:
- Claude: `~/.claude/settings.json` — check `contextWindow`, `maxTokens`, or equivalent fields
- Codex: `~/.codex/config.json` — check `contextLength`, `model`, or equivalent fields
- Verify on ace-linux-1, ace-linux-2, and acma-ansys05

## Why
Long WRK sessions with large codebases (digitalmodel, worldenergydata) hit context limits mid-session, causing truncation of earlier context or forced `/clear` cycles. Maximizing the context window reduces friction and increases session productivity.

## Propagation Strategy
Settings must be consistent across all 3 active workstations: **ace-linux-1**, **ace-linux-2**, **acma-ansys05** (Windows Git Bash).
- ace-linux-1 → ace-linux-2: rsync or SSH copy `~/.claude/settings.json` + `~/.codex/config.json`
- ace-linux-1 → acma-ansys05: manual copy or Git Bash script (no SSH from hub to Windows)
- Consider committing provider-agnostic config fragments to `config/ai-tools/` in workspace-hub repo for version-controlled propagation (investigate if Claude/Codex support repo-local config override)

## Repo-Level Config Note
Investigate whether Claude and Codex support a **repo-local config** that overrides `~/.` user config:
- Claude: `.claude/settings.json` at repo root (already used for hooks/permissions) — check if `contextWindow`/`maxTokens` can be set here
- Codex: check for `.codex/` or `codex.config.json` at repo root
- If supported: commit the context settings to `workspace-hub/.claude/settings.json` (Claude) and equivalent (Codex) so they propagate automatically via git pull on all machines

## Acceptance Criteria
- [ ] Audit current context limit settings for Claude CLI on each machine
- [ ] Audit current context limit settings for Codex CLI on each machine
