## Mission

Document and validate a troubleshooting workflow that uses the OrcaFlex demo version (no licence required) to load and inspect `.dat` / `.sim` input files via GUI or command line on a suitable Windows machine.

## What

- Confirm which OrcaFlex demo build is installed on licensed-win-1 (or specify target machine)
- Document how to load an input file in demo mode:
  - GUI method: `OrcaFlex.exe <inputfile.dat>`
  - Command-line / OrcFxAPI method for batch inspection
- Write a short troubleshooting guide (`docs/orcaflex/demo-troubleshoot.md`) covering:
  - Common load errors and their meaning in demo mode
  - Limitations of the demo (no time-domain simulation, read-only statics, etc.)
  - How to share findings back to the Linux-side analysis pipeline

## Why

The demo version can open and parse input files without a full licence, making it useful for quick sanity-checks and error diagnosis on shared Windows hardware without consuming a licence seat.

## Acceptance Criteria

1. A working command (GUI or CLI) to load an input file in demo mode is documented and tested
2. Known demo-mode limitations are listed so analysts know what checks are valid
3. Guide is committed to `docs/orcaflex/` in workspace-hub
