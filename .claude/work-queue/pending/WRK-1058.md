## Mission
Extend the quality check suite (WRK-1056) with documentation layout enforcement
so docstring coverage and README completeness are measurable before cross-review.

## What
- Add `--docs` flag to `check-all.sh`: runs ruff `D` rules (pydocstyle) per repo
- README check: validate required sections present (installation, usage, examples)
- `docs/` presence: report if missing; warn, do not hard-fail
- Output integrated into existing per-repo violation count table

## Why
Docstring gaps and thin READMEs are consistently flagged in cross-review but
are not caught by ruff/mypy. Surfacing them earlier reduces review rework.

## Acceptance Criteria
- [ ] `--docs` flag runs ruff D rules on all 5 tier-1 repos (warn-only, never raises exit code)
- [ ] README heading check: exact match `grep -qEi "^#+[[:space:]]*${section}$"` for installation/usage/examples
- [ ] Missing README.md → `readme: WARN (missing README.md)`, exit 0
- [ ] docs/ presence reported as PASS/WARN (consistent label with other subchecks)
- [ ] `--docs` is additive: alone = ruff+mypy+docs; with `--ruff-only` = ruff+docs; with `--mypy-only` = mypy+docs
- [ ] Output integrated into check-all.sh report table per repo
- [ ] 14 tests pass (T1–T14), including T13 (ruff D WARN + exit 0) and T14 (missing README WARN)
- [ ] Codex cross-review passes
