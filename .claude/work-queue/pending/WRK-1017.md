# bug: investigate why work-queue-workflow skill Stage 5 interactive planning is missed

## Mission
Identify why agents skip the Stage 5 user-agent interactive planning session and enforce it as a hard gate.

## What
Stage 5 of the 20-stage work-queue lifecycle is defined as an **interactive agent-user plan session** (not a one-way artifact drop). Agents routinely skip the interactive walk-through and jump straight from plan creation to Stage 6 cross-review or Stage 7, bypassing the collaborative review and decision-capture that Stage 5 requires.

## Why
Stage 5 is where scope in/out, acceptance criteria adjustments, risks, and constraints are collaboratively resolved. Skipping it means plans proceed with unvalidated assumptions, leading to implementation drift, rework, and scope creep — exactly the problems the lifecycle was designed to prevent.

## Acceptance Criteria
- [ ] Root cause identified: which part of the skill text is being misread/skipped
- [ ] Fix applied to `work-queue-workflow/SKILL.md` Stage 5 definition to make skipping harder
- [ ] Fix applied to `work-queue/SKILL.md` Stage Contract Stage 5 if needed
- [ ] Verification: a Stage 5 session with the user leaves a `user-review-plan-draft.yaml` evidence artifact
- [ ] Workflow guard or reminder added to prevent jump-from-stage-4-to-stage-6

## Investigation Questions
1. Is Stage 5 clearly marked as a **blocking** interactive gate (not optional)?
2. Does the skill text give a clear signal that Stage 5 requires active user response before Stage 6 fires?
3. Is there a plan-mode (`EnterPlanMode`) flag that causes the agent to rush past Stage 5?
4. Does Stage 5 produce a required evidence artifact (`user-review-plan-draft.yaml`) that gate verification checks?
5. Is there a "show me the HTML" path that bypasses the interactive walk-through?

## Agentic AI Horizon
- Stage 5 skipping is symptomatic of agents treating the lifecycle as a checklist rather than a dialog — a fundamental alignment problem for any multi-stage agentic workflow.
- Strengthening Stage 5 now builds the correct habit before evals (WRK-1009) instrument it.
