# bug: investigate why work-queue-workflow skill Stage 5 interactive planning is missed

## Mission
Identify why agents skip the Stage 5 user-agent interactive planning session and enforce it as a hard gate.

## What
Stage 5 of the 20-stage work-queue lifecycle is defined as an **interactive agent-user plan session** (not a one-way artifact drop). Agents routinely skip the interactive walk-through and jump straight from plan creation to Stage 6 cross-review or Stage 7, bypassing the collaborative review and decision-capture that Stage 5 requires.

## Why
Stage 5 is where scope in/out, acceptance criteria adjustments, risks, and constraints are collaboratively resolved. Skipping it means plans proceed with unvalidated assumptions, leading to implementation drift, rework, and scope creep — exactly the problems the lifecycle was designed to prevent.

## Acceptance Criteria
- [ ] Root cause identified: which part of the skill text is being misread/skipped
- [ ] Fix applied to `work-queue-workflow/SKILL.md` Stage 5 definition to make skipping harder
- [ ] Fix applied to `work-queue/SKILL.md` Stage Contract Stage 5 if needed
- [ ] Verification: a Stage 5 session with the user leaves a `user-review-plan-draft.yaml` evidence artifact
- [ ] Workflow guard or reminder added to prevent jump-from-stage-4-to-stage-6

## Plan
- **Files**: `.claude/skills/workspace-hub/work-queue-workflow/SKILL.md` (required), `.claude/skills/workspace-hub/workflow-gatepass/SKILL.md` (consistency check), `.claude/skills/coordination/workspace/work-queue/SKILL.md` (consistency check if Stage 5 wording must stay aligned)
- **Change**: Rewrite the `User Review - Plan (Draft)` Stage 5 instructions in `work-queue-workflow` so the interactive session is expressed as ordered planning-and-synthesis substeps:
  1. create a common plan draft to seed all model planning sessions
  2. interactive planning with ai agent - claude using the common draft
  3. document and prepare to exit
  4. interactive planning with ai agent - codex using the common draft
  5. document and prepare to exit
  6. interactive planning with ai agent - gemini using the common draft
  7. document and prepare to exit
  8. combine all individual model plans into a single combined plan document
  9. assign a quality rating to each individual model plan during the combine step
  10. seek user review and approval on the combined plan
- **Common draft artifact**:
  - `.claude/work-queue/assets/WRK-1017/common-plan-draft.md`
- **Saved plan artifacts**:
  - `.claude/work-queue/assets/WRK-1017/common-plan-draft.md`
  - `.claude/work-queue/assets/WRK-1017/codex-plan.md`
  - `.claude/work-queue/assets/WRK-1017/claude-plan.md`
  - `.claude/work-queue/assets/WRK-1017/gemini-plan.md`
  - `.claude/work-queue/assets/WRK-1017/combined-plan.md`
- **Plan-quality scoring during combine**:
  - Rate `claude-plan.md`, `codex-plan.md`, and `gemini-plan.md` on a common rubric.
  - Minimum rubric fields: completeness, test-eval quality, execution clarity, risk coverage, and standards/gate alignment.
  - Record per-dimension notes and an overall quality rating for each model plan in `combined-plan.md`.
- **HTML documentation requirement**:
  - Include a `Plan Quality Eval Comparison` section in the work-item plan-review HTML artifacts.
  - At minimum, document the three model plans, their ratings, the combine-step decision, and why the final combined plan kept or rejected key elements from each source plan.
- **Model-plan rule**: Each model-specific plan must start from the same `common-plan-draft.md` structure and preserve shared headings so the combine step is a merge-and-decision pass, not a rewrite from incompatible formats.
- **Canonical execution-plan rule**: For Route B, the saved individual plan files are evidence inputs, not the source of truth for execution. After the combine step, the approved execution plan is normalized back into this WRK item's `## Plan` section, while `combined-plan.md` remains the saved synthesis artifact presented for user review.
- **User-review flow**:
  - Generate the draft HTML from the combined plan state.
  - Walk the combined plan with the user after all three model plans are captured.
  - Review the `Plan Quality Eval Comparison` section with the user as part of the Stage 5 walkthrough.
  - Record approval/revision decisions in `user-review-plan-draft.yaml`.
  - Only after user approval/revision closure may the plan proceed to cross-review/final plan review.
- **Guardrail**: Keep the Stage 5 hard-blocking gate semantics intact; do not weaken the existing requirements for explicit user response, browser-open evidence, publish evidence, and `user-review-plan-draft.yaml`.
- **Consistency Check**: If the new substeps create contract drift with `workflow-gatepass` or `work-queue`, either align those files in the same WRK or document the deliberate single-file scope in the WRK evidence before execution.
- **Verify**: Confirm the ordered substeps and saved-plan artifact paths appear in the skill text, confirm no Stage 5 checklist items were removed, and run targeted grep-based verification plus any existing gate-evidence or skill validation command that applies without inventing new tooling.

## Investigation Questions
1. Is Stage 5 clearly marked as a **blocking** interactive gate (not optional)?
2. Does the skill text give a clear signal that Stage 5 requires active user response before Stage 6 fires?
3. Is there a plan-mode (`EnterPlanMode`) flag that causes the agent to rush past Stage 5?
4. Does Stage 5 produce a required evidence artifact (`user-review-plan-draft.yaml`) that gate verification checks?
5. Is there a "show me the HTML" path that bypasses the interactive walk-through?

## Agentic AI Horizon
- Stage 5 skipping is symptomatic of agents treating the lifecycle as a checklist rather than a dialog — a fundamental alignment problem for any multi-stage agentic workflow.
- Strengthening Stage 5 now builds the correct habit before evals (WRK-1009) instrument it.
