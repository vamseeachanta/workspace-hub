# docs(marketing): Anthropic Research-Blog Format as Marketing Template

## Mission

Capture the structural and rhetorical patterns from Anthropic's "Labor Market Impacts
of AI" blog post as a reusable template for technical/research posts on aceengineer
website and workspace-hub documentation.

## Source Reference

Blog post: https://www.anthropic.com/research/labor-market-impacts
(March 2026, Anthropic — clean example of research-to-marketing translation)

## What Makes This Format Effective

### Structure Pattern ("Problem → Solution → Validation → Evidence")

1. **Key Findings upfront** — 5-bullet executive summary before any methodology
2. **Acknowledge prior failures** — establishes intellectual honesty, deflects skepticism
3. **Define the novel framework** — one clear concept (e.g. "observed exposure") that media can quote
4. **Validate externally** — cross-reference against independent benchmarks (BLS, O*NET)
5. **Present null results honestly** — "no systematic unemployment increase" → paradoxically strengthens credibility
6. **Demographics make it relatable** — who is affected + how = targeted audience engagement
7. **Appendix + PDF download** — signals rigor; specialists can dive deep; casual readers stop at summary

### Visual Strategy

- 6 figures, each combining multiple chart types (not standalone bar charts)
- Descriptive captions with data source and key takeaway in the caption text
- Header graphic reinforces abstract concept (interconnected nodes for "labor network")
- Charts at section boundaries, not floating in text

### Typography and Navigation

- h3 / h4 hierarchy → scannable without reading
- Technical terms italicized on first use
- Superscript footnotes defer detail without breaking flow
- Clean whitespace, minimal decoration

### Tone Balance

- Rigor: mathematical formulas reference appendix, not inline
- Accessibility: every formula has a plain-language interpretation
- Humility: "gives reason for humility" = credible, not promotional

## Deliverables

1. `docs/templates/research-blog-post.md` — Markdown template with section stubs,
   instructions for each section, and the Problem→Solution→Validation→Evidence arc
2. `docs/templates/research-blog-checklist.md` — Pre-publish checklist:
   - [ ] Key findings upfront (≤5 bullets)
   - [ ] Null results included if they exist
   - [ ] At least one external validation reference
   - [ ] Charts have captions with data source
   - [ ] PDF/appendix available for detail-seekers
   - [ ] One novel coined term (the quotable concept)
3. `.claude/skills/workspace-hub/marketing/research-blog/SKILL.md` — micro-skill for
   when writing technical posts: triggers on "write blog post", "research summary",
   "publish findings"
