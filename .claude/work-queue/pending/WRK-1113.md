# feat(doc-intel): Squeeze Maximum Value from Document Corpus

## Mission

Transform document collections (Drilling Riser + CP as Phase 1) into integrated
`digitalmodel` parametric wrappers, publishing lookup charts open-source and running
detailed analysis on-demand — until each source document is fully absorbed and marked
exhausted.

## Why

The document corpus exists to serve one purpose: eliminate the need to re-read source
documents by integrating their data, analysis, and concepts directly into the codebase.
The GTM target is open-source publication and website deployment. This WRK formalises
the extraction pipeline starting with two domains:

- **Phase 1 domains**: Drilling Riser + Cathodic Protection
- **All remaining domains**: tackled iteratively in follow-on WRKs
- **Lookup charts** → baked into code as tabular constants or interpolation functions;
  sufficient for open-source publish
- **Detailed analysis** → on-demand parametric modules invoked per request
- Once a document's key content is absorbed: mark `exhausted: true` in the ledger;
  source document no longer needed for routine work

## What

### Phase 2 — Document-Verified Tests + Gap-Fill

For each selected calculation from the source documents:

1. **Locate the worked example** in the document (input values + expected output)
2. **Map to existing route function** in `src/digitalmodel/cathodic_protection/`
3. **Write a verification test** that replicates the document example exactly:

```python
class TestDocumentVerification:
    def test_current_demand_from_2100_rpt_4005_section_4_2(self):
        """
        Verify against worked example from 2100-RPT-4005-06 §4.2.
        Inputs: area=150 m², current_density=0.025 A/m², coating_factor=0.05
        Expected: 0.1875 A  (document states 0.19 A, rounded)
        """
        result = current_demand(area=150.0, current_density=0.025, coating_factor=0.05)
        assert result == pytest.approx(0.1875, rel=0.01)
```

4. **If the function does not exist** → add it to the appropriate route module
   following existing module conventions (typed params, module constants, `@doc_ref`)
5. **Parametric sweep test**: vary one key input and assert monotonicity / known trend

Test file naming: `tests/cathodic_protection/test_<route>_doc_verified.py`

### Phase 3 — Drilling Riser Route (new module)

All remaining domains → follow-on WRK-1113-phase-2.
