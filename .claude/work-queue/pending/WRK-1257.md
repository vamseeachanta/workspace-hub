## Mission

Extract actual chart/figure images from PDFs — currently 1,153 chart metadata
entries exist but zero images are extracted. Chart data (S-N curves, drag
coefficients, wave spectra) is critical for digitization and test case generation.

## Why

Engineering standards contain design curves that are currently only referenced by
caption text. Without the actual images:
- S-N curves from C203 can't be digitized for fatigue test fixtures
- Drag coefficient charts from C205 can't feed hydrodynamics validation
- Mooring load charts from 2SK can't be used for analysis verification

WRK-1188 established chart metadata capture but skipped image extraction.

## Plan

### Step 1: Image extraction from PDF pages

Use `pdfplumber` or `PyMuPDF (fitz)` to extract embedded images near figure references:
- Match figure references to page numbers from manifest
- Extract all images from those pages (fitz.Page.get_images)
- Filter by size: skip tiny icons (<50x50 px), keep charts (>200x200 px)
- Save as PNG to `data/doc-intelligence/deep/charts/<domain>/<doc>_fig_NNN.png`

### Step 2: Chart-figure matching

Link extracted images to figure metadata:
- Use page proximity: image on same page as "Figure X.Y" reference
- Fallback: sequential matching within sections

### Step 3: Quality rating

LLM-rate chart images (Haiku vision):
- Input: chart image
- Classify: engineering_chart / photograph / diagram / logo / junk
- For engineering_chart: is it digitizable? (has axes, data points, curves)
- Cost: ~$0.002/image × 1153 = ~$2.30

### Step 4: Retroactive extraction on 9 standards

Extract images from the 9 already-processed standards and link to existing metadata.

## Execution Strategy — Phased by Domain

After the extraction capability is built, execution is phased by domain priority
(not batch-all). This keeps focus on pressing engineering work:

| Phase | Domain | Standards | Why first |
|-------|--------|-----------|-----------|
| 1 | fatigue (C203) | 1 | S-N curves needed for assetutilities fatigue validation |
| 2 | cathodic-protection (B401) | 1 | CP design curves for digitalmodel verification |
| 3 | pipeline (F109, RP1111) | 2 | Stability/wall-thickness charts for active calcs |
| 4 | structural (2A) | 1 | Broad structural design — large chart set |
| 5 | marine (C205) | 1 | Wave/current load curves |
| 6 | mooring (2SK) | 1 | Stationkeeping analysis |
| 7+ | batch remaining | per WRK-1255 | After domain priorities are served |

Each phase is a single `/work run` — extract charts for that domain, rate quality,
promote digitizable charts. Don't batch-all until pressing domain needs are met.
