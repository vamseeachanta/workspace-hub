# fix(scripts): replace bare python3 with uv in work-queue scripts

## What

`scripts/work-queue/close-item.sh` (and likely sibling scripts) call `python3` directly,
violating the `uv run` rule in `.claude/rules/python-runtime.md`.

Discovered when closing WRK-689 (session 2026-03-03): `close-item.sh` ran
`verify-gate-evidence.py` with bare `python3` — PyYAML was unavailable, causing
"Integrated test gate: MISSING" even though PyYAML was present under `uv`.

## Files to Fix

```bash
grep -rn "python3\|python " scripts/work-queue/ scripts/agents/ \
  | grep -v "\.pyc\|#"
```

Expected hits to replace:
- `scripts/work-queue/close-item.sh` — `python3 "$VALIDATOR"` and `python3 "${QUEUE_DIR}/scripts/generate-index.py"`
- Any other `*.sh` in `scripts/work-queue/` or `scripts/agents/` calling `python3` directly

## Fix

Replace each occurrence:
```bash
# Before
python3 "$VALIDATOR" "$WRK_ID"
python3 "${QUEUE_DIR}/scripts/generate-index.py"

# After
uv run --no-project python "$VALIDATOR" "$WRK_ID"
uv run --no-project python "${QUEUE_DIR}/scripts/generate-index.py"
```

## Acceptance Criteria

- [ ] `grep -rn "python3" scripts/work-queue/ scripts/agents/` returns 0 hits
- [ ] `bash scripts/work-queue/close-item.sh --help` (or dry-run) runs without PyYAML error
- [ ] `uv run --no-project python scripts/work-queue/verify-gate-evidence.py --help` passes
