# fix(scripts): replace bare python3 with uv in work-queue scripts

## What

`scripts/work-queue/close-item.sh` (and likely sibling scripts) call `python3` directly,
violating the `uv run` rule in `.claude/rules/python-runtime.md`.

Discovered when closing WRK-689 (session 2026-03-03): `close-item.sh` ran
`verify-gate-evidence.py` with bare `python3` — PyYAML was unavailable, causing
"Integrated test gate: MISSING" even though PyYAML was present under `uv`.

## Step 1 — Full Audit (run first)

Scan every shell script under `scripts/` for bare `python3` or `python ` invocations:

```bash
grep -rn --include="*.sh" "python3\|python " scripts/ \
  | grep -v "^\s*#"   # skip comments
```

Triage output into three categories:
- **Fix now**: invoking a hub `.py` script (replace with `uv run --no-project python`)
- **Skip**: string literals, documentation comments, or echo statements
- **Review**: unknown callers — assess before changing

Document the full hit list as evidence before making any edits.

## Step 2 — Known Fixes (from WRK-689 discovery)

Primary target:
- `scripts/work-queue/close-item.sh` — `python3 "$VALIDATOR"` and `python3 ".../generate-index.py"`

Likely additional hits (audit will confirm):
- `scripts/work-queue/` sibling scripts
- `scripts/agents/` wrapper scripts
- `scripts/cron/` nightly scripts
- `scripts/productivity/` section scripts

## Step 3 — Apply Fixes

Replace each confirmed hit:
```bash
# Before
python3 "$VALIDATOR" "$WRK_ID"

# After
uv run --no-project python "$VALIDATOR" "$WRK_ID"
```
