# Keyboard Shortcut Autosuggest — Always-Available Reference + Session Learning

## Mission

Make the terminal/Claude keyboard shortcut reference always available in sessions, and
autosuggest missed shortcuts so the user can learn or relearn them progressively.

## Why

WRK-1022 documented the canonical shortcuts. The next step is surfacing them:
(a) passively as an always-available reference in skills/session-start, and
(b) actively as autosuggest hooks that fire when patterns indicate a missed shortcut.

## What

### Phase 1 — Always-available reference

- Extract the keyboard shortcut table from WRK-1022 into a standalone section in
  `.claude/skills/workspace-hub/workstations/SKILL.md` (already partially done in WRK-1022)
- Add a "Quick shortcuts" callout to `session-start` SKILL.md so it surfaces at session open
- Ensure the reference covers: terminal copy/paste, Claude chat screenshot paste (Ctrl+V),
  AI agent REPL commands (submit, multi-line, interrupt, exit), tmux basics

### Phase 2 — Autosuggest hooks (hookify)

Analyze session history to identify the most-missed shortcuts, then write hookify rules
that detect the pattern and suggest the shortcut inline.

Candidate patterns (from WRK-1022 + session notes):
- User describes taking a screenshot verbally instead of pasting → suggest `Ctrl+V` to paste
- User types `exit` repeatedly when `Ctrl+D` would be faster → suggest `Ctrl+D`
- User asks "how do I paste" in Linux terminal → remind `Ctrl+Shift+V`
- User describes copy-pasting manually → remind `Ctrl+Shift+C` / `Ctrl+Shift+V` on Linux

### Phase 3 — Progressive learning log

Optional: maintain a `.claude/state/shortcut-learning.yaml` that tracks which shortcuts
have been suggested and how many times, to avoid over-suggesting the same hint.

## Acceptance Criteria

- [ ] Phase 1: shortcut reference section in `workstations/SKILL.md` is complete and linked from `session-start`
- [ ] Phase 2: ≥3 hookify rules written for most-missed shortcuts (identified from session history)
- [ ] Phase 2: hooks fire correctly in a test session without false positives
- [ ] Phase 3 (optional): `shortcut-learning.yaml` tracks suggestion frequency per shortcut

## TDD Approach

Hookify rule validation: write expected trigger pattern, verify hook fires on match,
verify it does not fire on non-matching patterns.
