## Mission

Build an end-to-end pipeline script and formatted engineering calculation report consolidating all results, drawings, and comparisons.

## What

1. **Pipeline script**: single entry point that runs the full calculation chain:
   - Drag force calculation (WRK-1362 outputs)
   - 2D frame analysis (WRK-1363 outputs)
   - 3D FEM post-processing (WRK-1364 outputs)
   - OrcaFlex results import (WRK-1365 outputs)
   - 2D vs 3D comparison (WRK-1366 outputs)
   - Member & connection checks (WRK-1367 outputs)
2. **Engineering report** (PDF via Markdown→PDF pipeline):
   - Design basis and references
   - Frame geometry and 2D drawings (from WRK-1361)
   - Parachute drag force calculations with sensitivity analysis
   - Frame analysis results (2D, 3D FEM, OrcaFlex) with comparison
   - Member and connection unity ratios
   - Conclusions and recommendations
3. **Result archive**: all YAML outputs, plots, and drawings in a single output directory

## Why

The engineering report is the final deliverable to the client. The pipeline script ensures reproducibility — any change to inputs re-generates the full report.

## Acceptance Criteria

- [ ] Pipeline script runs end-to-end from raw inputs to final report
- [ ] Engineering report includes all sections listed above
- [ ] Report exported to PDF (printable, professional format)
- [ ] All figures (stress contours, deformation plots, drawings) embedded in report
- [ ] Unity ratio summary table in report with pass/fail per member and connection
- [ ] Pipeline re-runs cleanly after input changes (idempotent)
