# docs(skills): review and beef up file-structure skills with knowledge map and log file locations

## Problem

There are 4 file-structure related skills (`file-taxonomy`, `repo-structure`, `clean-code`,
`infrastructure-layout`) but no skill-level knowledge map linking them, no coverage of log
file locations, and no "where do I look for AI agent session data?" reference.

This session (WRK-687) surfaced native AI session store paths not documented anywhere:
- **Codex native**: `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl`
- **Gemini native**: `~/.gemini/tmp/<project>/chats/session-*.json`
- **Claude orchestrator**: `logs/orchestrator/claude/session_YYYYMMDD.jsonl`

These belong in a canonical "key file locations" reference consumable by all AI agents.

## Deliverables

### 1. Knowledge Map — `file-structure-skills-map.md` (CREATE)

New file at `.claude/docs/file-structure-skills-map.md`:
- Visual skill dependency / coverage map (Markdown table + ASCII diagram)
- Which skill to invoke for which scenario (quick lookup)
- Identifies any gaps (scenarios not covered by existing skills)

Skills to map:
| Skill | Version | Scope |
|-------|---------|-------|
| `file-taxonomy` | 1.6.0 | Output file placement, format guide, gitignore policy |
| `repo-structure` | 1.4.0 | Source layout, root cleanliness, docs classification |
| `clean-code` | 2.1.0 | File/function size limits, God Object detection |
| `infrastructure-layout` | 1.0.0 | infrastructure/ package sub-domains |
