# feat(doc-intel): dedup + multi-source provenance layer for indexes

## Mission
Add a deduplication and multi-source provenance layer between classify and promote stages of the doc-intelligence pipeline, so duplicate records from multiple documents are merged into single canonical entries with all sources tracked, and conflicting values are flagged for resolution.

## Why
The pipeline currently appends every record independently. As more document collections are ingested, the same constants, equations, and definitions will appear from multiple sources (e.g., seawater density appears in PNA, USNA EN400, DNV-RP-C205). Without dedup, promoted code accumulates redundant entries and conflicting values go undetected.

## Trigger
Tackle when conflicts actually arise from ingesting a second corpus or when promoted files become unwieldy with duplicates. Low priority until then.

## Approach

### 1. Dedup keys by content type
| Content type | Dedup key |
|---|---|
| Constants | `(normalized_name, value, units)` |
| Equations | `(normalized_expression)` |
| Requirements | `(standard_ref, clause_id)` |
| Definitions | `(normalized_term)` |
| Tables | `(column_signature, row_count)` fuzzy |
| Procedures | `(normalized_title)` |

### 2. Multi-source provenance
When duplicates match, merge into one canonical record with sources array:
- `authority: primary` for oldest/most authoritative source
- `authority: corroborating` for subsequent matches
- Authority ranking: classification rules > textbooks > course notes

### 3. Conflict detection
When same dedup key yields different values:
- Write to `data/doc-intelligence/conflicts.jsonl`
- Fields: parameter, values[], sources[], resolution (pending/highest_authority/most_specific/user)
- Conflicts block promotion until resolved

### 4. Retirement
- `status: superseded` when newer edition or higher authority contradicts
- Superseded records stay in index (audit trail) but excluded from promote

### 5. Implementation
- New module: `scripts/data/doc_intelligence/dedup.py`
- Runs between `build-doc-intelligence.py` and `promote-to-code.py`
- Or integrated as a post-classify pass in `index_builder.py`
- Script: `scripts/data/doc-intelligence/dedup-indexes.py --verbose`

## Acceptance Criteria
- [ ] Dedup keys defined and tested for all 6 section content types
- [ ] Multi-source provenance merges duplicates with sources array
- [ ] Conflict detection writes conflicts.jsonl for mismatched values
- [ ] Promoted files contain only canonical (deduplicated) records
- [ ] Retirement mechanism excludes superseded records from promotion
- [ ] TDD tests with synthetic duplicate and conflict scenarios
