## Mission
Set up acma-ansys05 (orcaflex-license-machine) so it passes workspace-hub
readiness checks and can participate in the work queue.

## What
Manually execute the workspace-hub setup steps on the Windows/Git Bash machine:
1. Pull latest repo (includes the new `acma-ansys05.yml` manifest)
2. `bash scripts/setup/install-all-hooks.sh`
3. Add bashrc-snippets to `~/.bash_profile`
4. `claude config set statusBarEnabled true`
5. Copy `config/claude/keybindings.json` → `~/.claude/keybindings.json`
6. `bash scripts/setup/verify-setup.sh` — resolve any FAIL items
7. `bash scripts/operations/system/dev-env-check.sh` — confirm manifest validates

## Why
Machine has claude, codex, gemini installed (all confirmed) but no workspace-hub
harness setup. Needed before any WRK items can be routed to it.

## Acceptance Criteria
- [ ] `verify-setup.sh` reports 0 FAIL (WARNs acceptable for cron/SSH)
- [ ] `dev-env-check.sh` finds and reads `acma-ansys05.yml` manifest
- [ ] Git hooks installed (pre-commit, post-merge, post-rewrite)
- [ ] Shell aliases (`ws`, `wrk`, `wh-verify`) available in Git Bash

## Notes
- No SSH access — must be done physically on the machine
- Windows Git Bash (MINGW64) — use `bash`, not PowerShell
- Workspace path: `C:\workspace-hub` (or wherever cloned)
- Cron WARN in verify-setup is expected on Windows — ignore
