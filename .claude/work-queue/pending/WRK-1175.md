## Mission

Add a running progress log (`progress.log`) per WRK item that accumulates session-by-session work summaries — complementing checkpoint.yaml with human-readable chronological context.

## What

Anthropic's harness uses `claude-progress.txt` as a running log that agents read at session start and append to at session end. This differs from checkpoint.yaml (which captures machine-readable resume state) by providing narrative context across sessions.

Implement:
1. `assets/WRK-NNN/progress.log` — append-only text log
2. Auto-append hook: `exit_stage.py` appends a summary line after each stage exit
3. Session-start reads progress.log when resuming a WRK (alongside checkpoint.yaml)
4. Format: `[YYYY-MM-DD HH:MM] Stage N: <one-line summary>`

## Why

checkpoint.yaml captures WHERE to resume but not the narrative of WHAT happened. Agents resuming a WRK in a new session lack context about prior attempts, dead ends, and decisions — leading to repeated mistakes.

## Acceptance Criteria

- [ ] progress.log created automatically at WRK Stage 1
- [ ] exit_stage.py appends summary line per stage transition
- [ ] start_stage.py reads and displays progress.log on resume
- [ ] Log is append-only (no overwrites)
- [ ] Format is human-readable with timestamps
