# feat(workflow): enforce skill-driven /work gatepass from session start to close

## What

Create an enforceable, test-backed workflow so agents cannot bypass gatepass:
`/session-start` -> `/work` (work-queue orchestration) -> 9-stage verification evidence -> close/archive.

## Why

Current behavior shows drift between canonical process and skill docs, causing occasional plan/gate bypass. We need deterministic skill routing and machine-checkable checks across the full lifecycle.

## Acceptance Criteria

- [ ] Create a skill revision matrix and update impacted skills (`work-queue`, `session-start`, `session-end`, related adapters)
- [ ] Add explicit required-skill sequence for lifecycle start->finish in docs and validator-backed evidence
- [ ] Add/extend gate validator to require 9-stage verification ledger evidence before close
- [ ] Add `gate evidence` summary table artifact generation (step/result) for close audit
- [ ] Add tests ensuring agents cannot pass close without required gate and integrated/repo test evidence
- [ ] Propose and scaffold any new skills needed to prevent drift (minimum one, if justified)
- [ ] Regenerate queue index and keep WRK/process docs consistent

## Skill Review Scope

- Existing skills to revise:
  - `coordination/workspace/work-queue`
  - `workspace-hub/session-start`
  - `workspace-hub/session-end`
  - `workspace-hub/work-document-exit`
- Candidate new skills:
  - `workspace-hub/workflow-gatepass` (enforcement + evidence contract)
  - `workspace-hub/wrk-lifecycle-testpack` (start-to-close TDD harness)

## Agentic AI Horizon

As agents gain autonomy, explicit skill-chain enforcement and machine-checkable evidence become mandatory controls rather than documentation hints. This item establishes that control plane.
