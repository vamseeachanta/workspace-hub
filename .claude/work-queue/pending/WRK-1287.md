Build an extraction pipeline that indexes all document files (xlsx, pdf,
docx, csv, etc.) across the repo ecosystem, then uses doc/excel reading
skills to extract key information from each file. Track extraction status
in the index so we know which files have been fully processed and which
still need attention.

## Scope
- Inventory all document files (PDF, XLSX, DOCX, CSV) across repo ecosystem
- For each file, use appropriate skill (pdf reader, excel reader) to extract key data
- Store extracted metadata and key facts in searchable index (JSONL)
- Track extraction status per file: not_started / partial / complete
- Flag files where extraction failed or returned incomplete data
- Integrate with existing document-index infrastructure

## Notes
- Complements WRK-1283 (cross-drive file discovery) — this WRK focuses on content extraction
- Excel skill can read sheets/ranges; PDF skill can read page ranges
- Need to define "key information" per document type (e.g., standards → clauses, data files → column headers + row counts)
- Legal scan required before extracting from any client-originated documents
- Likely Feature WRK — children per document type or per repo
