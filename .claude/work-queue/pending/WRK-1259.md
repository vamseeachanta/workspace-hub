## Mission

Reduce false-positive warnings from eval-skills.py by making required sections
type-aware. Currently all 405 skills are checked against 7 required sections,
producing 2002 section_missing warnings — most are noise.

## What

1. Add a `type:` frontmatter field to eval-skills.py: `workflow`, `guidance`, `tool`, `reference`
2. Define required sections per type (e.g., guidance skills don't need Error Handling)
3. Fix `category_from_path()` to use top-level dir consistently (eliminates 204 category_dir_mismatch)
4. Re-run eval; target: warnings < 500

## Why

- 2002/2276 warnings are section_missing — drowns out real issues
- 204 category_dir_mismatch is a bug in the evaluator, not the data
- Eval must be useful to be run weekly; noisy output = ignored output

## Acceptance Criteria

- [ ] eval-skills.py supports skill type classification
- [ ] Required sections vary by type
- [ ] category_from_path uses top-level category dir
- [ ] Post-fix warnings < 500
- [ ] Existing tests still pass
