# Anchor Capacity (Phase 5)

## What

Implement mooring anchor capacity calculations for drag anchors
(Neubecker-Randolph), suction caissons (DNV-RP-E303), and torpedo/SEPLA
anchors, with an anchor selection decision matrix.

## Why

mooring_analysis/models.py currently requires user-provided holding_capacity.
This phase computes anchor capacity from soil conditions, completing the
soil-to-mooring analysis chain. Suction caisson analysis reuses pile
lateral methods from WRK-619.

## Scope

### Packages
- None — all custom implementations

### Standards
- DNVGL-RP-E301 (fluke/drag anchors)
- DNV-RP-E303 (suction anchors)
- API RP 2SK (station keeping — anchor sections)

## Deliverables

- `geotechnical/anchors/drag_anchor.py` — Neubecker-Randolph method
- `geotechnical/anchors/suction_caisson.py` — DNV-RP-E303 (V/H/inclined)
- `geotechnical/anchors/torpedo_anchor.py` — torpedo/SEPLA simplified capacity
- `geotechnical/anchors/anchor_selection.py` — capacity vs soil type matrix
- Test suite validated against DNV-RP-E303 appendix and Vryhof Anchor Manual

## Acceptance Criteria

- [ ] Drag anchor holding capacity via Neubecker-Randolph method
