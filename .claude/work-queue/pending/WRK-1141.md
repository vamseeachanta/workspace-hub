# chore(git-policy): Adopt Trunk-Based Development for Solo Multi-Workstation Workflow

## Mission

Replace the per-WRK feature-branch habit with a trunk-based policy suited
to a single developer working across multiple machines.

## What

Update `.claude/rules/git-workflow.md` and `CLAUDE.md` to reflect:

**New policy:**
- Commit directly to `main` by default; push immediately after every commit
- Short-lived branches only when work genuinely spans multiple sessions
  or requires cross-machine isolation
- Branches still used for: multi-session complex WRKs, experimental spikes,
  and cross-machine in-progress sync where isolation is needed

**Keep branching for:** any WRK that touches 3+ files and spans >1 session,
or where rollback isolation matters.

## Why

- Feature branches per WRK add overhead (create, switch, merge, delete)
  with no benefit for solo solo-reviewer workflow
- Multi-workstation sync is cleaner via push/pull to `main` than managing
  diverged branches across machines
- Nightly cron and automation (WRK-1140) commits should go straight to main

## Acceptance Criteria

- [ ] `.claude/rules/git-workflow.md` has a `## Solo / Multi-Workstation Policy`
      section documenting the trunk-based default
- [ ] `CLAUDE.md` Quick Reference updated to note "commit to main by default,
      push immediately"
- [ ] `finishing-a-development-branch` usage note added — use only for
      multi-session WRKs
- [ ] No existing branch conventions removed — prefixes/format kept for
      cases where branches are still needed
