## Mission

Classify every document on mounted drives by engineering domain, closing the gap where ~144 SNAME ship plans and other files remain `domain: null` in the 1M-record document index.

## What

Analyze documents across all mounted drives (ace standards, project archives, OG standards) and assign each a domain from the existing 13-domain taxonomy (marine, pipeline, structural, cathodic-protection, cad, materials, etc.). Extend taxonomy if needed for uncovered disciplines.

## Why

The document index has 1,033,789 records classified via phase-c-classify.py, but coverage is incomplete — 144+ SNAME ship plans have `domain: null`, and other mounted sources may have gaps. Full domain classification enables targeted document-intelligence extraction pipelines per discipline.

## Acceptance Criteria

- [ ] AC1: All records in index.jsonl have a non-null `domain` field
- [ ] AC2: Classification accuracy validated on a sample (≥50 docs spot-checked)
- [ ] AC3: Updated domain counts published to registry.yaml
- [ ] AC4: Any new domains added to config.yaml taxonomy with keyword lists
- [ ] AC5: Script exists to re-run classification on new documents incrementally
