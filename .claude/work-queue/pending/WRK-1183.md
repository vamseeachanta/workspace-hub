## Mission

Create a skill that provides a structured workflow for extracting calculation methodology
from legacy Excel spreadsheets and archiving it as clean, client-free engineering knowledge
in the dark intelligence archive. This is the canonical path for porting any legacy
calculation to the 4 public repos while completely avoiding legal/IP issues.

## What

The skill should define a repeatable workflow:

1. **Identify** — locate Excel files containing engineering calculations (from past projects,
   coursework, or reference material)
2. **Extract** — pull out the generic methodology: formulas, input ranges, expected outputs,
   validation cases, unit systems, applicable standards
3. **Sanitize** — run legal-sanity-scan to ensure zero client identifiers, project names,
   or proprietary data remain
4. **Archive** — save as structured YAML in `knowledge/dark-intelligence/<category>/<subcategory>/`:
   - equations (LaTeX)
   - input/output schemas
   - worked examples with known answers
   - standard references
   - methodology notes
5. **Generate TDD test data** — convert worked examples to pytest test cases with concrete
   assertions (known inputs → known outputs)
6. **Implement** — use the extracted methodology + test data to build clean implementations
   in the 4 public repos (digitalmodel, worldenergydata, assethold, assetutilities)
7. **Produce calc report** — generate calculation-report YAML per the calculation-report skill

## Why

- Legacy Excel files contain verified calculations from real projects — years of engineering knowledge
- Carrying original Excel files creates legal/IP risk (client data, project names, proprietary formats)
- Extracting only the methodology strips all risk while preserving 100% of the engineering value
- Worked examples from Excel become TDD test assertions — more reliable than hand-derived values
- University coursework Excel files are also a rich source (no legal risk but same extraction benefit)
- This compounds: every extracted methodology makes the next calculation implementation faster

## Integration

- **research-literature skill**: university_resources and dark intelligence sections feed into this
- **calculation-report skill**: output YAML follows the calc-report schema
- **legal-sanity-scan**: mandatory gate between extraction and archival
- **knowledge/dark-intelligence/**: canonical storage location (gitignored for privacy)

## Acceptance Criteria

1. Skill file at `.claude/skills/data/dark-intelligence-workflow/SKILL.md`

3. Legal scan is a hard gate — extraction cannot proceed without passing
4. YAML schema for dark intelligence archive entries documented
5. Worked example: extract one calculation from an Excel file → archive → TDD test → implementation
6. Integration points with research-literature and calculation-report skills documented
7. TDD: tests for archive YAML schema validation
