## Mission

Split all SKILL.md files exceeding the new 200-line hard limit (coding-style.md) into hub + micro-skill structure.

## Context

- coding-style.md updated: hard limit 400 -> 200, soft target 200 -> 100
- 293 of 408 skills (72%) currently exceed 200 lines
- Many are 1500+ lines (auto-generated API skills)
- WRK-1270 handles nesting/co-location; this WRK handles the size enforcement

## Scope

| Tier | Line count | Count | Action |
|------|-----------|-------|--------|
| Critical | >1000 | ~80 | Split into hub + 3-5 micro-skills |
| High | 500-1000 | ~60 | Split into hub + 2-3 micro-skills |
| Medium | 200-500 | ~150 | Split into hub + 1-2 micro-skills |
| Compliant | <200 | 115 | No action |

## Approach

1. Script to identify and triage all violations: `scripts/skills/audit-line-limits.sh`
2. Process by family first (aligns with WRK-1270 nesting)
3. Auto-generated API skills (slack-api, notion-api, etc.) — extract examples into sub-skills
4. Engineering skills — split by input/output/reference pattern (aqwa exemplar)
5. Batch processing via agent parallelization

## Acceptance Criteria

1. `find .claude/skills -name "SKILL.md" | xargs wc -l | awk '$1 > 200'` returns zero results
2. All split skills pass `eval-skills.py` with no critical failures
3. Cross-references updated in all moved/split files
4. No content lost — diff total line count before/after should be comparable

## Decomposition

This is a feature WRK. Children will be created per skill family/category after Stage 7 approval. Estimated 8-12 child WRKs covering:
- business/* API skills (~15 skills, all 1500+)
- data/* skills (~30 skills)
- engineering/marine-offshore/* (~20 skills, partially done via WRK-1270)
- ai/* skills (~10 skills)
- operations/* skills (~15 skills)
- remaining categories
