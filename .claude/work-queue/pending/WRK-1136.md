## Mission

Enhance the `workspace-hub:repo-sync` skill to output a comprehensive per-repo status
table showing ahead/behind counts, uncommitted changes, and untracked files after every
sync run — and clarify skill invocation scope (all repos vs individual repo vs
workspace-hub hub).

## What

Update `.claude/skills/workspace-hub/repo-sync/SKILL.md` to:

1. **Status summary table** — after Phase 1 bulk pull, emit a markdown table with columns:
   `Repo | Branch | Ahead | Behind | Uncommitted | Untracked | Status`
   - Status values: `In sync`, `Diverged — needs merge`, `Dirty`, `Ahead only`, `Behind only`
   - Uncommitted = count of `M`/`D` lines from `git status --short`
   - Untracked = count of `??` lines from `git status --short`

2. **workspace-hub included** — explicitly document that workspace-hub itself is checked
   (currently the `repository_sync` script only checks submodules/sibling repos, not
   workspace-hub hub). Add a dedicated hub-level git check step.

3. **Scope options** — document invocation variants:
   - `/repo-sync` — all repos (default)
   - `/repo-sync pull <repo-name>` — single named repo
   - `/repo-sync status` — status only (no pull), all repos including hub
   - `/repo-sync push` — push all with unpushed commits

## Why

Current skill only reports pass/fail per repo and does not show ahead/behind counts or
dirty state. The user had to run manual `git rev-list` commands to discover workspace-hub
was diverged. The skill should surface this automatically.

## Acceptance Criteria

- [ ] AC1: After `/repo-sync`, a status table is printed with Ahead/Behind/Uncommitted/Untracked columns for every repo
- [ ] AC2: workspace-hub hub repo is included in the status table (not just submodules)
- [ ] AC3: Skill documents `/repo-sync status` as a status-only mode (no pull)
- [ ] AC4: Skill documents single-repo invocation pattern
- [ ] AC5: Status column uses human-readable labels (In sync / Diverged / Dirty / etc.)
