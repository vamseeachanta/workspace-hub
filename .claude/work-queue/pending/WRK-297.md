# SSHFS mounts on ace-linux-1 for ace-linux-2 drives

## What

Configure SSHFS mounts on ace-linux-1 so it can access ace-linux-2's drives seamlessly.
This is the reverse direction — ace-linux-2 already mounts ace-linux-1's workspace-hub via
SSHFS. Both machines should have full bidirectional drive access.

## Why

- ace-linux-2 already has SSHFS access to ace-linux-1 (workspace-hub mounted at `/mnt/workspace-hub`)
- ace-linux-1 has **no access** to ace-linux-2's drives
- Bidirectional access needed: work from either seat, access any data from either machine
- ace-linux-2 has 2.7 TB bulk storage (`/mnt/dde`) not available from ace-linux-1

## Prerequisites (all met)

- [x] SSH key auth ace-linux-1 → ace-linux-2: working (WRK-295)
- [x] openssh-server on ace-linux-2: installed (this session)
- [x] /etc/hosts: both machines know each other
- [ ] WRK-294 mount paths active on ace-linux-2 (needs reboot)

## Steps

### On ace-linux-1:

1. Install sshfs if not present: `sudo apt install -y sshfs`
2. Create mount points:
   ```bash
   sudo mkdir -p /mnt/remote/ace-linux-2/local-analysis
   sudo mkdir -p /mnt/remote/ace-linux-2/dde
   sudo chown -R vamsee:vamsee /mnt/remote
   ```
3. Test manual mount:
   ```bash
   sshfs vamsee@ace-linux-2:/mnt/local-analysis /mnt/remote/ace-linux-2/local-analysis
   sshfs vamsee@ace-linux-2:/mnt/dde /mnt/remote/ace-linux-2/dde
   ```
4. Verify: `ls /mnt/remote/ace-linux-2/local-analysis` and `ls /mnt/remote/ace-linux-2/dde`
5. Add fstab entries for persistence:
   ```
   vamsee@ace-linux-2:/mnt/local-analysis /mnt/remote/ace-linux-2/local-analysis fuse.sshfs defaults,_netdev,x-systemd.automount,IdentityFile=/home/vamsee/.ssh/id_ed25519,reconnect,ServerAliveInterval=15,ServerAliveCountMax=3,nofail 0 0
   vamsee@ace-linux-2:/mnt/dde /mnt/remote/ace-linux-2/dde fuse.sshfs defaults,_netdev,x-systemd.automount,IdentityFile=/home/vamsee/.ssh/id_ed25519,reconnect,ServerAliveInterval=15,ServerAliveCountMax=3,nofail 0 0
   ```
6. `sudo systemctl daemon-reload && sudo mount -a`

### Also standardize ace-linux-2's existing SSHFS mount

The current mount on ace-linux-2 (`/mnt/workspace-hub` from ace-linux-1) should follow
the same `/mnt/remote/<hostname>/` convention for consistency:
