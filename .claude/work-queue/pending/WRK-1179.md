## Mission

In 15 days (March 14–28, 2026), maximize data completeness and calculation coverage
across the 4 public repos (digitalmodel, worldenergydata, assethold, assetutilities),
capture all remaining gaps as WRK items, and produce vision documents for both streams.

## Streams

### Stream A — Data Completeness (worldenergydata + doc index)

1. **Audit (Days 1–3):** Gap report on 1M doc index content extraction coverage;
   audit all public O&G data sources (ingested vs. not); research new public sources globally
2. **Extract & Ingest (Days 4–10):** Maximize content extraction from docs;
   ingest highest-value missing public O&G data sources into worldenergydata
3. **Capture & Vision (Days 11–15):** Remaining work → WRK items;
   write `worldenergydata/docs/vision/DATA-VISION.md`

### Stream B — Calculation Coverage (digitalmodel + assethold + assetutilities)

1. **Audit (Days 1–3):** Map every calculation that could exist across 4 repos
   vs. what's implemented; include design data + methodology references
2. **Implement (Days 4–12):** Build highest-value calculations following TDD +
   `calculation-report` skill (YAML schema → HTML/MD report); each calc produces
   a worked example in `examples/reporting/`
3. **Capture & Vision (Days 13–15):** Unimplemented calcs → WRK items;
   write `digitalmodel/docs/vision/CALCULATIONS-VISION.md`

## Constraints

- All calculations MUST follow the `calculation-report` skill
  (`.claude/skills/data/calculation-report/SKILL.md`): YAML with metadata, inputs,
  methodology (standard + LaTeX), outputs (pass/fail), assumptions, references;
  validated against `config/reporting/calculation-report-schema.yaml`
- Development in 4 public repos only: digitalmodel, worldenergydata, assethold, assetutilities
- OGManufacturing is client data — never add implementation code there
- Drilling/surveillance calculations go into digitalmodel

## Acceptance Criteria

1. Data audit report exists with: doc index coverage %, public source catalog (ingested/not/new)
2. All feasible content extracted from doc index; new public sources ingested into worldenergydata
3. `worldenergydata/docs/vision/DATA-VISION.md` committed
4. Calculation gap map updated in `specs/capability-map/<repo>.yaml` for all 4 repos
5. Maximum feasible calculations implemented with TDD + calculation-report YAML
6. `digitalmodel/docs/vision/CALCULATIONS-VISION.md` committed
7. All remaining gaps captured as child WRK items with standard refs + acceptance criteria
