# Skill name→path registry for start-stage.sh skill loading

## Mission
**Create a YAML registry mapping logical skill names (e.g. `superpowers/brainstorming`) to their
actual file paths, and make `start-stage.sh` read this registry so stage YAML contracts can use
stable logical names regardless of where skill files live on disk.**

## What

Stage YAML contracts in WRK-1028 declare `skills_required` using logical names:

```yaml
skills_required:
  - workspace-hub/work-queue-workflow
  - superpowers/brainstorming
  - superpowers/systematic-debugging
```

But the actual skill files live at:
- `_internal/workflows/brainstorming/SKILL.md`
- `development/systematic-debugging/SKILL.md`
- `development/tdd-obra/SKILL.md`  ← referenced as `superpowers/test-driven-development`

Without a registry, `start-stage.sh` cannot resolve logical names to file paths.
Skills can never safely be moved or renamed without breaking all stage contracts.

## Why

A name→path registry:
- Decouples stage contracts from skill storage layout (skills can move/rename without editing contracts)
- Single place to update when a skill is relocated
- Enables `start-stage.sh` to validate at boot whether all required skills are resolvable
- Extensible: future skills added to registry rather than hardcoded into scripts

## Acceptance Criteria
- [ ] Registry at `config/skills/skill-registry.yaml` with entries for all skills currently referenced by WRK-1028 stage contracts:
  - `superpowers/brainstorming` → `_internal/workflows/brainstorming/SKILL.md`
  - `superpowers/systematic-debugging` → `development/systematic-debugging/SKILL.md`
  - `superpowers/test-driven-development` → `development/tdd-obra/SKILL.md`
  - `workspace-hub/work-queue-workflow` → `workspace-hub/work-queue-workflow/SKILL.md`
  - `workspace-hub/workflow-html` → `workspace-hub/workflow-html/SKILL.md`
  - `workspace-hub/resource-intelligence` → `workspace-hub/resource-intelligence/SKILL.md`
  - `workspace-hub/workflow-gatepass` → `workspace-hub/workflow-gatepass/SKILL.md`
  - `workspace-hub/work-queue` → `coordination/workspace/work-queue/SKILL.md`
- [ ] `scripts/work-queue/resolve-skill.sh <logical-name>` — reads registry, prints absolute path, exits 1 if not found
- [ ] `start-stage.sh` (WRK-1028 deliverable) calls `resolve-skill.sh` for each `skills_required` entry; aborts with clear error if any skill is unresolvable
- [ ] `scripts/work-queue/validate-skill-registry.sh` — checks every registry entry points to an existing file; intended for CI / nightly cron
- [ ] ≥3 unit tests: happy resolve, missing logical name, broken path (file doesn't exist)

## Agentic AI Horizon

- Registry is the foundation for skill versioning, A/B testing, and deprecation workflows.
- `start-stage.sh` becomes the canonical skill loader for all future autonomous agents.
- Nightly cron can run `validate-skill-registry.sh` to detect skill drift before it breaks a live WRK run.
