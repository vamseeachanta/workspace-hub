# Cross-repo integration test gate

## Mission

Detect cross-repo API breaks before they reach main — assetutilities API changes must not silently break worldenergydata or digitalmodel.

## What

1. **Dependency graph map** — document which repos import from which (assetutilities → worldenergydata, assetutilities → digitalmodel, assetutilities → assethold)
2. **Combined integration run** — `scripts/testing/run-cross-repo-integration.sh`: runs each downstream repo's tests with the current HEAD of its upstream deps installed (not pinned uv.lock version)
3. **Pre-push gate** — extend pre-push hook on assetutilities to trigger cross-repo run for downstream repos
4. **Break detection** — if downstream tests fail after an assetutilities commit, gate blocks and reports which downstream repo is broken
5. **Symbol index integration** — use `scripts/search/find-symbol.sh` (WRK-1085) to identify which downstream files reference changed symbols

## Why

Currently assetutilities can change a public API and the break is only discovered when someone runs the downstream repo's tests manually. This is a self-healing gap: the ecosystem should catch it at commit time.

## Acceptance Criteria

- [ ] `config/deps/cross-repo-graph.yaml` — documents which repos depend on which
- [ ] `scripts/testing/run-cross-repo-integration.sh` — runs combined test suite
- [ ] Pre-push hook on assetutilities triggers cross-repo run
- [ ] Exit code 1 + clear report when downstream breaks detected
- [ ] ≥5 TDD tests (graph loading, combined run logic, break detection, report format)
- [ ] Passes `check-all.sh` for any Python components

## Plan

### Phase 1 — Dependency Graph & Assethold Contracts

**Step 1:** Create `config/deps/cross-repo-graph.yaml`
- Document layer 0 (assetutilities) → layer 1 (digitalmodel, worldenergydata, assethold)
- Include import module references, version constraints, and contract test locations

**Step 2:** Create `assethold/tests/contracts/` contract tests
- Mirror pattern from `digitalmodel/tests/contracts/test_assetutilities_common_contract.py`
- Cover: `FileManagement`, `update_deep_dictionary`, `WorkingWithYAML` (what assethold actually uses)
- Mark with `@pytest.mark.contracts`

### Phase 2 — Cross-Repo Integration Script

**Step 3:** Create `scripts/testing/run-cross-repo-integration.sh`
- Accept `--repo <name>` flag to run a single downstream repo (default: all three)
- Accept `--fail-fast` to exit on first failure
- Set correct PYTHONPATH per repo (reuse `run-all-tests.sh` recipe)
- Run only `@pytest.mark.contracts` tests to keep runtime short
- Emit structured summary: PASS/FAIL per repo + total count
- Exit 1 if any downstream contracts fail; exit 0 if all pass

**Step 4:** Create `scripts/testing/tests/test_cross_repo_integration.py` (TDD)
- `test_graph_yaml_loads_and_has_required_keys` — YAML schema check
- `test_script_exists_and_is_executable` — file existence + permissions
- `test_script_exits_0_when_all_contract_tests_pass` — subprocess integration test (slow, skip by default with marker)
- `test_report_format_contains_repo_names` — output format check
- `test_break_detection_exits_1_on_failure` — inject a failing test, verify exit code
- `test_pythonpath_set_correctly_for_each_repo` — env var validation

### Phase 3 — Pre-Push Hook on Assetutilities (version-controlled)

**Step 5:** Create `scripts/hooks/assetutilities-pre-push.sh` (version-controlled bash script)
- Detect if `assetutilities/src/` changed in the push commits
- Call `run-cross-repo-integration.sh --fail-fast`
- Exit 1 with repo-identifying message on failure
- Log bypass to `logs/hooks/pre-push-bypass.jsonl`
- Bypass: `SKIP_CROSS_REPO_CHECK=1 git push`

**Step 6:** Register in assetutilities `.pre-commit-config.yaml` with `stages: [push]`
- Add hook entry pointing to `../../scripts/hooks/assetutilities-pre-push.sh`
- Version-controls the hook via the existing pre-commit config (assetutilities already has one)
- Document: `git push --no-verify` bypasses all hooks (expected behaviour for emergency)

**Step 7:** Wire into hub `scripts/hooks/pre-push.sh`
- When `ASSETUTILITIES_CHANGED == "true"`, call `run-cross-repo-integration.sh`
- Ensures hub-level push also triggers downstream checks
