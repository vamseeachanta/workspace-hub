## Mission

Systematically fill data needs identified in `digitalmodel/specs/data-needs.yaml`
across all domains. Agents should check this registry when processing documents,
extracting data, or building modules — and update entry status as gaps are filled.

## What

- Maintain `specs/data-needs.yaml` as living registry of data/calculation/analysis gaps
- As document-intelligence pipelines extract data, update status: needed → extracted → integrated
- When agents build new modules, register their data needs in the registry
- Add naval architecture module entry to `specs/module-registry.yaml` and `.claude/capability-manifest.yaml`
- Integrate 6,777 extracted curve CSVs into validated dataframes
- Extract DDG51/FFG-7 hydrostatic tables from SNAME ship plans or EN400 appendix

## Why

Compound value: every future WRK that builds or extends a module can check what data
exists and what's missing. Document-intelligence pipelines can prioritize extraction
toward known gaps. Prevents duplicate extraction work and ensures no data source is
overlooked.

## Acceptance Criteria

- [ ] AC1: `specs/data-needs.yaml` covers all 11 domain groups with at least 2 entries each
- [ ] AC2: Naval architecture added to module-registry.yaml and capability-manifest.yaml
- [ ] AC3: At least 3 needs moved from `needed` to `extracted` or `integrated`
- [ ] AC4: Agent workflows (session-start, work-queue) reference data-needs.yaml for gap awareness
- [ ] AC5: Script to query data-needs by domain/status/priority (`scripts/specs/query-data-needs.py`)
