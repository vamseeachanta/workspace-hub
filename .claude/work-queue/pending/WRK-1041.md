# Lifecycle HTML — auto-refresh (30s meta tag) for lifecycle + plan HTML

## Mission
**Embed `<meta http-equiv="refresh" content="30">` in both HTML generators so lifecycle and plan HTML stay current automatically — no manual reload, no multiple copies.**

## What

Add a 30-second meta refresh tag to both `<head>` sections in `generate-html-review.py`:
1. Lifecycle HTML head (line ~2053, inside `generate_lifecycle()`)
2. Plan/review HTML head (line ~984, inside `render_wrk_html()`)

Scope dropped: `/work html` command and `--open` flag are not needed; auto-refresh covers the requirement.

## Why

Stage-start/end scripts regenerate HTML at each boundary. Without auto-refresh,
the user must manually reload — causing stale views and context-switching errors.
Keeping exactly 1 lifecycle + 1 plan HTML per WRK avoids stale duplicate copies.

## Acceptance Criteria
- [ ] generate-html-review.py `generate_lifecycle()` output includes `<meta http-equiv="refresh" content="30">` in `<head>`
- [ ] generate-html-review.py `render_wrk_html()` output includes the same meta refresh tag
- [ ] Tests verify both: `test_lifecycle_html_has_meta_refresh`, `test_review_html_has_meta_refresh`
- [ ] All existing HTML generation tests pass
