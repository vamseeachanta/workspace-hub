## Findings

### F1: dispatch-run.sh was unwired (fixed in WRK-5099)
command-interface skill had no reference to dispatch-run.sh. Agent browsed
sub-skills instead of calling dispatch. Fixed: added dispatch protocol to
command-interface SKILL.md.

### F2: stage-01 referenced dead lifecycle HTML (fixed in WRK-5100)
Micro-skill and stage YAML still referenced lifecycle HTML generation from
the old local-file workflow. Fixed: removed HTML refs, kept frontmatter
validation only.

### F3: GitHub issue template missing from stage workflow (captured as WRK-5101)
`/work add` should create GitHub issue from template; stages should update it.
start_stage.py had helpers but they weren't connected to capture flow.

### F4: WRK-5101 itself bypassed the stage lifecycle
Parallel session executed WRK-5101 without calling dispatch-run.sh or
stage group runners. No checkpoint.yaml, no assets dir, no evidence.
Went straight from pending → implement → done/ in one commit.
Root cause: the WRK-5099 command-interface fix wasn't loaded by that session
(started before the commit landed). Also suggests the enforcement is still
prose-level (Level 0) — no hook or script prevents skipping stages.
