# feat(doc-intel): run extraction pipeline on naval architecture collection

## Mission
Run the doc-intel pipeline (extract-document.py → build-doc-intelligence.py → query + promote) against the 145 naval architecture PDFs at /mnt/ace/docs/_standards/SNAME/ to produce structured engineering knowledge: constants, equations, tables, curves, procedures, definitions, and worked examples.

## Why
The doc-intel toolchain (WRK-5035–5041) is built but never executed against real documents. The naval architecture collection (WRK-1151) is the ideal first target — 145 curated PDFs covering stability, resistance, hydrostatics, structural design, and classification rules. This validates the pipeline end-to-end and produces immediately useful engineering reference data for digitalmodel.

## Input
- 145 PDFs: /mnt/ace/docs/_standards/SNAME/ (20 textbooks, 13 hydrostatics, 110 ship plans, 2 regulatory)
- Catalogue: knowledge/seeds/naval-architecture-resources.yaml
- Extraction skill: .claude/skills/engineering/doc-extraction/SKILL.md
- Tools: scripts/data/doc-intelligence/extract-document.py, build-doc-intelligence.py, query-doc-intelligence.py, promote-to-code.py

## Execution Steps

### Step 1: Extract — run extract-document.py on all SNAME PDFs
- Process textbooks and hydrostatics first (richest content)
- Ship plans last (mostly drawings — expect thin text extraction)
- Output: manifests in data/doc-intelligence/manifests/naval-architecture/

### Step 2: Index — run build-doc-intelligence.py
- Build federated indexes from naval architecture manifests
- Expected content types: constants (seawater density, steel yield), equations (Holtrop, ITTC, GZ), tables (scantling, classification), curves (GZ, resistance), procedures (stability check, structural assessment), definitions (metacentric height, block coefficient), worked examples

### Step 3: Validate — run query-doc-intelligence.py
- Test queries: --domain naval-architecture --type equations --keyword "resistance"
- Test queries: --type constants --keyword "seawater"
- Test queries: --type procedures --keyword "stability"
- Verify results return with source references to original PDFs

### Step 4: Promote — run promote-to-code.py for highest-value content
- Equations → Python functions in digitalmodel/src/digitalmodel/naval_architecture/
- Constants → Python module with typed variables
- Tables → CSV in data/standards/promoted/naval-architecture/
- Procedures → YAML skills in .claude/skills/engineering/naval-architecture/
- Worked examples → pytest tests

### Step 5: Create naval-architecture doc-extraction sub-skill
- Add .claude/skills/engineering/doc-extraction/naval-architecture/SKILL.md
- Domain heuristics: hull form coefficients (Cb, Cp, Cm, Cwp), stability parameters (GM, GZ, KB, BM), resistance equations, IMO criteria references, classification rule tables

## Acceptance Criteria
- [ ] All 145 SNAME PDFs processed by extract-document.py (manifests created)
- [ ] Federated indexes built with naval architecture content
- [ ] Query returns results for stability, resistance, hydrostatics domains
- [ ] At least 10 equations promoted to Python functions with source citations
- [ ] At least 5 reference tables promoted to CSV
- [ ] At least 3 procedures promoted to YAML skills
- [ ] Naval-architecture doc-extraction sub-skill created
- [ ] Pipeline performance noted (time, token cost, extraction quality)

## TDD Approach
Test promoted Python functions against known textbook values (e.g., Holtrop resistance for standard hull forms, ITTC friction line, IMO weather criterion). Test promoted constants against published values. Test query coverage across content types.
