# Documentation currency — drift detector

## Mission

Detect when documentation has fallen out of sync with the code it describes, and surface those gaps automatically as WRK items.

## What

1. **Public API snapshot** — for each repo, capture the set of public symbols (modules, classes, functions) using the AST symbol index (WRK-1085)
2. **Docs coverage check** — for each public symbol, check if it appears in docs/ or README.md
3. **Drift score** — `(undocumented_public_symbols / total_public_symbols)` per repo; track over time
4. **Staleness detection** — if a symbol's docstring or docs entry hasn't changed but the implementation has (via git blame diff), flag as potentially stale
5. **Nightly report** — `logs/quality/doc-drift-{date}.yaml`; drift score increase → auto-WRK
6. **Integration with `check-all.sh --docs`** — extend existing docs flag with drift score output

## Why

WRK-1058 added docs quality checks (ruff D rules, README section check). WRK-1075 built MkDocs pipeline. But neither detects drift between code changes and existing docs. Public APIs change; docs don't always follow.

## Acceptance Criteria
