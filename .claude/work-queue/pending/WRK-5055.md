# feat(doc-intel): extract ship plan drawings via CAD pipeline on dev-secondary

## Mission
The 110 ship plan PDFs are drawing-heavy (108/110 have zero extractable text). Convert these drawings to structured CAD data using FreeCAD/GMSH on dev-secondary, extracting hull geometry, compartment layouts, and structural arrangements.

## Why
WRK-5054 confirmed ship plans yield 0 sections/0 tables from pdfplumber — they're raster/vector drawings. The engineering value is in the geometry, not text. dev-secondary has FreeCAD and GMSH available for CAD processing.

## Input
- Ship plans index: `data/doc-intelligence/ship-plans-index.yaml` (110 entries, 66 vessel types)
- Source PDFs: `/mnt/ace/docs/_standards/SNAME/ship-plans/`
- CAD skills: `freecad-automation`, `gmsh-meshing`

## Approach
1. PDF to SVG/DXF conversion (pdf2svg or Inkscape CLI on dev-secondary)
2. DXF to FreeCAD import and geometry extraction (hull outlines, frame sections)
3. Build hull geometry index with vessel type, principal dimensions, frame count
4. For key vessels (carriers, battleships, submarines): extract lines plan data for parametric hull modeling

## Acceptance Criteria
- [ ] PDF to DXF conversion pipeline runs on dev-secondary
- [ ] At least 20 plans successfully converted to DXF
- [ ] Hull outline geometry extracted for at least 10 vessels
- [ ] Geometry index YAML created with principal dimensions where extractable
