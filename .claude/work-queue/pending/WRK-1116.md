## Mission

Extract the 5 Claude Code website-building hacks from Nate Herk's video
("Building Beautiful Websites with Claude Code Is Too Easy", Feb 2026, 166K views)
and integrate the most valuable patterns into the workspace-hub skill ecosystem.

## What

The video teaches 5 hacks for building professional websites with Claude Code:

- **Hack #0 — CLAUDE.md as project system prompt**: Create a concise, project-scoped
  CLAUDE.md with design rules and end-goal reminders. Updated iteratively as the project
  evolves. Video provides a downloadable web-design CLAUDE.md template.

- **Hack #1 — Frontend Design Skill**: Load a curated "frontend design skill" file (from
  @trq212 on X: https://x.com/trq212/status/1989061937590837678) that instructs Claude
  to avoid generic AI aesthetics and use professional design patterns. Maps to our
  existing `frontend-design:frontend-design` skill.

- **Hack #2 — Automated Screenshot Loop (Puppeteer)**: CLAUDE.md contains a
  "screenshot workflow" section instructing Claude to install and use Puppeteer to
  auto-take screenshots during builds. Claude creates a `temporary_screenshots/` folder,
  does a two-pass screenshot review, and polishes visually — using its own eyes to check
  output. No manual intervention needed. Setup: ask Claude to set up Puppeteer from
  the CLAUDE.md instructions.

- **Hack #3 — Reference Website Cloning**: Combine the screenshot loop with a reference
  URL (from Dribbble, Godly Website, Awwwards, etc.). Claude screenshots both what it's
  building AND the reference site, compares them visually, and iterates until they match.

- **Hack #4 — Brand Assets Folder**: Create a `brand_assets/` folder containing logo +
  brand guidelines doc (colors, typography, icons). Reference from CLAUDE.md so Claude
  reads them automatically. Enables zero-prompt branding.

- **Iterative CLAUDE.md refinement**: After each iteration, update CLAUDE.md with what
  worked. Have Claude suggest CLAUDE.md improvements itself.

- **Deploy — GitHub + Vercel workflow**: `git init` → push to GitHub → connect Vercel
  for automatic CI/CD deployment (free tier).

## Why

- Our `frontend-design` skill already exists but can be enhanced with @trq212's patterns.
- `examples/claude-code-course/uigen/` (Next.js) would benefit from brand_assets pattern.
- The brand_assets + screenshot reference pattern is a generalizable convention.
- A web-project CLAUDE.md template prevents each user starting from scratch.
- Vercel deploy skill would complete the end-to-end frontend workflow.

## Implementation Constraints

- **Use `skill-creator` skill** for all skill creation and modification work (Hacks #1, #2,
  #3, #4, deploy). Do not hand-author skill files without going through skill-creator.
- **References required before iteration**: for each hack, if no reference content exists
  (e.g. @trq212 skill text, Nate's CLAUDE.md template, a real Puppeteer screenshot script),
  STOP and ask the user to supply it before writing anything. Do not invent content.
