## Mission

Make licensed-win-1 (Windows Git Bash) actually contribute session signals to the
learning pipeline — currently it runs, produces no output, and pushes nothing.

## What

`comprehensive-learning.sh` has a machine guard (line 29): on non-dev-primary
machines it takes a "commit + push state files" path instead of the full
10-phase pipeline. On licensed-win-1 this path fails silently with two root causes.

### Root Cause 1 — `git add` on missing directories

`comprehensive-learning.sh` lines 34-46 do:
```bash
git add .claude/state/candidates/ \
        .claude/state/corrections/ \
        ...
```
`candidates/` doesn't exist on this machine → `fatal: pathspec ... did not
match any files`. `set -uo pipefail` is set but this isn't a pipe, so the
script continues — `git diff --staged --quiet` returns 0 → "No new learning
state to push" → exit 0 with nothing sent to dev-primary.

### Root Cause 2 — No signal producers wired on Windows

The stop hooks (`emit-session-quality-signals.sh`, `capture-corrections.sh`,
`consume-signals.sh`, etc.) are in `.claude/hooks/` but must also be referenced
in Claude's hooks settings to fire. The workspace `settings.json` has only
`permissions:`, no `hooks:` block. Without hooks firing, `.claude/state/` signal
dirs are never created, so there is nothing to push.

### Secondary Issue (improve.sh, for future cross-platform use)

If/when `improve.sh` is ever run directly on Windows via `--quick`:
- `bc` not available → `guard.sh:30` `$(echo "$score < 0.6" | bc -l ... || echo "1")`
  always returns `1` → all improvements silently rejected
- `python3` not in PATH (Windows uses `python`) → `classify.sh` route fails

## Acceptance Criteria

1. `comprehensive-learning.sh` Windows path gracefully skips missing state dirs
   — `git add` only paths that exist; no fatal error; exit 0 with clear message
2. Hooks wiring verified on Windows: confirm `hooks:` block exists in Claude
   settings for licensed-win-1 (or document the required setup step)
3. After wiring, at least `emit-session-quality-signals.sh` and
   `capture-corrections.sh` produce files in `.claude/state/session-signals/`
   during a normal session
4. A second run of `comprehensive-learning.sh` on Windows detects staged files
   and pushes them to origin
5. `bc` dependency removed from `guard.sh` — float comparison via `awk` using
   pattern in `.claude/skills/_core/bash/cross-platform-compat/SKILL.md`
6. `python3` calls in `classify.sh` use platform-aware resolver from
   `cross-platform-compat` skill (`python3 || python || uv run --no-project python`)
7. TDD: tests for AC1 (mock missing dirs), AC5 (mock bc absent), AC6 (mock python3 absent)
8. Cross-platform patterns documented in
   `.claude/skills/_core/bash/cross-platform-compat/SKILL.md` (enables WRK-1118)

## Why

items, but every session's signals are discarded. dev-primary never sees Windows
learnings. Cross-machine learning only works when all machines can produce signals
and push them reliably.
