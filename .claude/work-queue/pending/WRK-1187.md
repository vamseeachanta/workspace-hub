## Mission

Build a robust, code/script-based workflow engine that captures ALL ecosystem workflows
as deterministic, auditable pipelines — more robust than the current WRK stage queue.
Every workflow should be enforced by scripts, not LLM prose.

## Context

The WRK lifecycle stage queue (exit_stage.py, is-human-gate.sh, verify-gate-evidence.py)
works well because it's script-enforced. But new workflows (dark intelligence extraction,
research-literature gathering, calculation implementation, data ingestion) are currently
defined only as skill prose (L0-L1 on the enforcement gradient). These need to be
promoted to L2 (script) or L3 (hook) enforcement.

## Research Phase

Before building, research more robust workflow patterns:

1. **Temporal.io / Prefect / Airflow** — production workflow orchestration patterns:
   - DAG-based task execution
   - Retry/failure handling
   - State persistence
   - Human-in-the-loop gates

2. **GitHub Actions / Dagger.io** — CI/CD workflow patterns:
   - YAML-defined pipelines
   - Step dependencies
   - Artifact passing between steps
   - Conditional execution

3. **State machine patterns** — finite state machines for workflow stages:
   - Explicit transitions with guards
   - No skipping stages
   - Audit trail at every transition
   - Rollback capability

4. **What the ecosystem actually needs**:
   - Lightweight (no external dependencies — pure bash/Python)
   - Stage gates (human and automated)
   - Artifact validation at each stage
   - Audit trail (who did what, when, with what evidence)
   - Composable (workflows can call sub-workflows)
   - Resumable (crash recovery from last completed stage)

## Deliverables

1. **Research report**: evaluate Temporal/Prefect/Airflow/GitHub Actions patterns,
   recommend what to adopt vs. build
2. **Workflow engine spec**: define the architecture (likely: YAML workflow definitions +
   Python/bash stage executors + state persistence)
3. **Migrate existing WRK stage queue** to the new engine (backwards compatible)
4. **Codify these workflows as scripts**:
   - Dark intelligence extraction (WRK-1183)
   - Research-literature gathering (WRK-1181)
   - Calculation implementation (TDD → implement → calc-report → commit)
   - Data source ingestion (loader → tests → schema → commit)
   - Phase B doc extraction (index → extract → classify → ledger → capmap)
5. **Enforcement**: each workflow has a script entry point that validates all gates

## Acceptance Criteria

1. Research report comparing workflow orchestration patterns
2. Workflow engine design spec
3. At least 3 workflows codified as scripts with stage gates
4. Existing WRK stage queue still works (backwards compatible)
5. Every workflow has: defined stages, gate checks, artifact validation, audit trail
6. TDD: tests for workflow engine core (state transitions, gate checks)
