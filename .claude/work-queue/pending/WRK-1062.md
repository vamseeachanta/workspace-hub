## Mission
Replace live-data test dependencies with deterministic fixtures so the 21 assetutilities
and 12 assethold failures disappear from the test signal and TDD is reliable.

## What
- Audit all failing tests: identify which use live API calls, file-system paths, or
  network resources (assethold: 12 live-data; assetutilities: 21 run-time)
- For each failing test category, create a fixture strategy:
  - Record-and-replay: capture real response once, store as JSON fixture in `tests/fixtures/`
  - Stub factory: factory functions generating minimal valid domain objects
  - Skip markers: `@pytest.mark.live_data` for intentionally live tests (excluded by default)
- Implement fixtures for highest-count failure groups first
- Add `pytest -m "not live_data"` as the default command in run-all-tests.sh
- Document fixture update workflow (`scripts/testing/refresh-fixtures.sh`)

## Why
21 + 12 = 33 known failures pollute the test dashboard red. WRK-1054's unified runner
will surface them on every run. Fixing the test signal is prerequisite to meaningful
CI/CD. Also enables TDD without requiring internet/external services.

## Acceptance Criteria
- [ ] Audit report: all live-data failures classified by root cause
- [ ] `@pytest.mark.live_data` marker registered and documented per repo
- [ ] Fixture files in `tests/fixtures/` for top failure groups
- [ ] `run-all-tests.sh` excludes live_data by default; `--include-live` flag re-enables
- [ ] Zero unexpected failures in default run after fixtures applied
- [ ] `scripts/testing/refresh-fixtures.sh` documents how to update fixtures
- [ ] Cross-review (Codex) passes
