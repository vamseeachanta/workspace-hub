## Mission

Implement schema-validated YAML verification checklists for complex/feature WRKs — structured, token-efficient, corruption-resistant alternative to markdown AC lists.

## What

Anthropic's harness guide recommends structured feature lists with: category, description, step-by-step verification steps, and a `passes` boolean. Key insight: agents follow instructions better when format is token-efficient (YAML ~30-40% fewer tokens than JSON) and supports inline comments as co-located rules. Schema validation eliminates YAML's silent corruption weakness.

Create a YAML checklist template and tooling:
1. Template: `config/work-queue/feature-checklist-template.yaml`
2. Schema: `config/work-queue/feature-checklist-schema.yaml` (validates structure, restricts agent edits to `passes` field)
3. Generator: `scripts/work-queue/generate-feature-checklist.py WRK-NNN` — extracts ACs from WRK frontmatter into YAML
4. Validator: `scripts/work-queue/validate-feature-checklist.py WRK-NNN` — schema-checks + confirms all items pass before Stage 17
5. Integration: Stage 12 (TDD/Eval) reads checklist; Stage 14 (Verify Gate) includes checklist status

## Why

Current AC tracking is markdown-based (`ac-test-matrix.md`), which models can inadvertently reformat or corrupt. YAML with schema validation prevents agents from removing or editing test definitions (validator rejects structural changes) while being more token-efficient and supporting inline instruction comments (e.g. `# HARD RULE: never edit description`) that improve agent compliance.

## Format Decision Rule

| Who writes | Format | Why |
|-----------|--------|-----|
| Human curates | YAML | Comments, readability, diff-friendly |
| Agent writes structured data | JSON | Strict schema, less corruption by LLMs |
| Agent writes evidence | YAML | Ecosystem consistency, gate script compat |
| Script generates | JSON | Deterministic, schema-enforceable |

## Acceptance Criteria

- [ ] YAML template with category, description, verification_steps[], passes fields + inline comment instructions
- [ ] Schema file that validates structure and rejects edits to non-`passes` fields
- [ ] Generator script creates checklist from WRK acceptance criteria
- [ ] Validator script schema-checks + confirms all items pass (exit 0/1)
- [ ] Integrated into Stage 12 and Stage 14 workflows
- [ ] Format decision rule documented in work-queue SKILL.md or rules/
