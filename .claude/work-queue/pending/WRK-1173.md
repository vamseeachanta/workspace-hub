## Mission

Implement JSON-based feature verification checklists for complex/feature WRKs — structured, machine-readable, corruption-resistant alternative to markdown AC lists.

## What

Anthropic's harness guide recommends a JSON feature list with: category, description, step-by-step verification steps, and a `passes` boolean. Key insight: **JSON is less likely to be corrupted by models than Markdown**. Agents are restricted to only changing the `passes` field.

Create a `feature-checklist.json` template and tooling:
1. Template: `config/work-queue/feature-checklist-template.json`
2. Generator: `scripts/work-queue/generate-feature-checklist.py WRK-NNN` — extracts ACs from WRK frontmatter into JSON
3. Validator: `scripts/work-queue/validate-feature-checklist.py WRK-NNN` — checks all items pass before Stage 17
4. Integration: Stage 12 (TDD/Eval) reads checklist; Stage 14 (Verify Gate) includes checklist status

## Why

Current AC tracking is markdown-based (`ac-test-matrix.md`), which models can inadvertently reformat or corrupt. JSON with restricted edit fields prevents agents from removing or editing test definitions while still allowing them to mark items as passing.

## Format Decision Rule

| Who writes | Format | Why |
|-----------|--------|-----|
| Human curates | YAML | Comments, readability, diff-friendly |
| Agent writes structured data | JSON | Strict schema, less corruption by LLMs |
| Agent writes evidence | YAML | Ecosystem consistency, gate script compat |
| Script generates | JSON | Deterministic, schema-enforceable |

## Acceptance Criteria

- [ ] JSON template with category, description, verification_steps[], passes fields
- [ ] Generator script creates checklist from WRK acceptance criteria
- [ ] Validator script confirms all items pass (exit 0/1)
- [ ] Agent instructions restrict edits to `passes` field only
- [ ] Integrated into Stage 12 and Stage 14 workflows
- [ ] Format decision rule documented in work-queue SKILL.md or rules/
