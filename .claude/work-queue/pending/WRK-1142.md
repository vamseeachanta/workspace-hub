# feat(workflow): Migrate Stage Rules into Stage Micro-Skills; Auto-Load via start_stage.py

## Mission

Move all stage-specific rules out of the monolithic `work-queue-workflow/SKILL.md`
into the per-stage micro-skill files, and wire `start_stage.py` to load the
relevant micro-skill automatically at stage entry via `entry_reads`.

## Problem

Currently:
- Stage rules live in `work-queue-workflow/SKILL.md` (one big file, ~300 lines)
- The skill is only loaded when explicitly invoked — NOT at stage entry
- Stage micro-skills (`stages/stage-NN-*.md`) exist but are stubs (e.g. stage-04 = 8 lines)
- `start_stage.py` does not load stage micro-skills — so rules are absent at execution time
- Rules added to SKILL.md today (scripts-over-LLM at Stage 4 and Stage 10) won't be
  in context unless the skill happens to be loaded

Result: rules are written but not enforced at the moment they matter.

## What

### 1. Flesh out all 20 stage micro-skills
Each `stages/stage-NN-*.md` becomes a complete reference for that stage:
- Entry reads (what to load before starting)
- Checklist (what to do, in order)
- Scripts-over-LLM audit (where applicable — Stage 4, 10 mandatory)
- Exit gate (what artifact proves the stage is done)
- Hard-gate flag (STOP + wait vs auto-proceed)

Migrate all relevant content from `work-queue-workflow/SKILL.md` into the
appropriate stage file. Keep SKILL.md as a lightweight index with cross-cutting
rules only (terminology, gate policy table, orchestrator pattern).

### 2. Wire `start_stage.py` to auto-load stage micro-skill
At stage entry, `start_stage.py WRK-NNN N` should print the micro-skill path
and its content into the context resume block so the agent has the rules in
scope for that stage. Pattern:

```python
stage_skill = f".claude/skills/workspace-hub/stages/stage-{N:02d}-*.md"
# resolve glob → print as entry_reads block
```

### 3. Stage contracts (`stages/stage-NN-*.yaml`) stay as machine-readable gates
They are not replaced — they serve verification. The micro-skills are the
human+agent readable narrative layer on top.

### 4. Migrate scripts-over-LLM rule into stage-04 and stage-10
The rules added to SKILL.md in this session (2026-03-12) move into:
- `.claude/skills/workspace-hub/stages/stage-04-plan-draft.md`
- `.claude/skills/workspace-hub/stages/stage-10-work-execution.md`

Remove the duplicates from SKILL.md once migration is confirmed.

## Why

- Rules written but not loaded at execution time = rules that don't run
- `start_stage.py` is the canonical entry point — it's the right place to
  inject stage context
- Smaller, focused files load faster and pollute context less than one 300-line skill
- Follows scripts-over-LLM: stage loading should be mechanical, not LLM memory

## Acceptance Criteria

- [ ] All 20 stage micro-skills have complete checklists (≥5 lines each)
- [ ] `start_stage.py` prints stage micro-skill content in the resume block
- [ ] `stage-04-plan-draft.md` includes scripts-over-LLM audit instruction
- [ ] `stage-10-work-execution.md` includes scripts-over-LLM check
- [ ] `work-queue-workflow/SKILL.md` slim — cross-cutting rules only, ≤150 lines
- [ ] Duplicate content removed from SKILL.md after migration confirmed
- [ ] Test: `uv run --no-project python scripts/work-queue/start_stage.py WRK-test 4`
      outputs stage-04 micro-skill content in resume block
- [ ] Existing stage contract YAMLs unchanged
