## Mission

Update verify-gate-evidence.py to accept GitHub issue as an alternative
to HTML lifecycle artifacts, reflecting the WRK-5104 migration.

## What

1. Plan gate: accept `spec_ref` pointing to plan.md as alternative to `plan_html_review_final_ref`
2. User-review HTML-open gate: accept GitHub issue URL as alternative to local HTML open
3. User-review publish gate: accept `github_issue_ref` update as alternative to HTML publish
4. Agent log gate: make optional for single-agent executions
5. Stage evidence gate: accept `stage_evidence_ref` from frontmatter
6. Resource-intelligence gate: accept `completion_status: complete` and `done` (not just specific values)
7. Integrated test gate: change range from 3-5 to 3+ (no upper limit)
8. Future-work gate: accept simple string recommendations (not just objects with disposition)

## Why

WRK-5104 replaced HTML lifecycle with GitHub issues. verify-gate-evidence.py
still checks for HTML-era artifacts, causing stage 14 to fail on all WRKs
using the new GitHub workflow. This blocks the entire pipeline at stage 14.

## Acceptance Criteria

- [ ] verify-gate-evidence.py passes for WRK-5104 with GitHub issue workflow
- [ ] Existing WRKs with HTML workflow still pass (backward compatible)
- [ ] No regression in gate strictness for non-migrated checks
- [ ] Stage 14 pre_exit_hook passes end-to-end
