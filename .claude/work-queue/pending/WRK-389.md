## Context

During WRK-306, discovered that `claude` on ace-linux-2 was installed via `sudo npm install -g`,
requiring `sudo claude update` for upgrades. This is fragile and non-standard.

The `install_method: {claude: npm-sudo}` entry in the workstations SKILL.md documents this
pending issue.

## Goal

Switch Claude CLI install on ace-linux-2 to the native installer (curl-based) or an nvm-managed
npm install (no sudo required). Ensure `claude update` works without sudo.

## Steps

1. SSH into ace-linux-2
2. Check current install: `which claude && claude --version`
3. Remove sudo-npm install: `sudo npm uninstall -g @anthropic-ai/claude-code`
4. Install via native method:
   - Option A: native installer: `curl -fsSL https://claude.ai/install.sh | sh`
   - Option B: nvm-based npm (no sudo): install nvm â†’ `nvm install lts && npm install -g @anthropic-ai/claude-code`
5. Verify: `claude --version && claude update` (no sudo)
6. Update workstations SKILL.md `install_method` entry: `ace-linux-2: {claude: native}`

## Acceptance Criteria

- [ ] `claude --version` works on ace-linux-2 without sudo
- [ ] `claude update` completes without sudo
- [ ] workstations SKILL.md `install_method` updated to reflect new method
