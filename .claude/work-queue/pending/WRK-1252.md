# Full CAD-to-CFD Pipeline

## Mission

Create a single automated pipeline that takes geometry from FreeCAD through gmsh meshing to OpenFOAM simulation to ParaView visualization — driven by a YAML config file, with known-answer validation cases.

## What

This is the integration capstone that connects the deep workflows from WRK-1249 (gmsh), WRK-1250 (OpenFOAM), and WRK-1251 (FreeCAD):

1. **Pipeline orchestrator** — YAML config defines geometry source, mesh parameters, solver setup, post-processing extractions → single digitalmodel cfd run pipeline.yaml command
2. **Stage handoffs** — FreeCAD STEP export → gmsh volume mesh (with boundary layer) → OpenFOAM case directory → solver execution → foamToVTK → ParaView state file
3. **Validation cases with known answers**:
   - Cylinder drag: Cd ≈ 1.0 at Re=100 (2D, laminar)
   - Flat plate friction: Cf within 5% of Blasius solution
   - Box barge wave loading: compare forces with Morison equation / AQWA potential flow
4. **Error propagation** — each stage validates its output before passing to the next; failure at any stage produces diagnostic report (not silent continuation)

## Why

Individual tools (FreeCAD, gmsh, OpenFOAM) are useful but the real engineering value is in the pipeline. Currently an engineer must manually transfer files between tools, check formats, verify mesh quality, set up solver parameters — this is error-prone and slow. An automated pipeline with validation gates enables rapid parametric studies and builds trust through known-answer regression tests.

## Acceptance Criteria

- [ ] Pipeline YAML config schema defined and documented
- [ ] digitalmodel cfd run pipeline.yaml orchestrates full FreeCAD → gmsh → OpenFOAM → results chain
- [ ] Cylinder drag validation: Cd within 10% of published value at Re=100
- [ ] Stage handoff validation: mesh quality gate between gmsh and OpenFOAM, field initialization check before solver start
- [ ] Error handling: diagnostic YAML report when any stage fails (geometry invalid, mesh quality below threshold, solver diverged)
- [ ] ParaView state file auto-generated for visualization of final results
- [ ] Pipeline runnable in dry-run mode (generates all files without solver execution) for testing on machines without OpenFOAM
- [ ] End-to-end integration test with box barge geometry
