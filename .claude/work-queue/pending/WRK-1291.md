## What

The standards transfer ledger (`data/document-index/standards-transfer-ledger.yaml`) has entries for only 4 of 13 domains in the research-literature domain map:

| Domain | Ledger entries |
|--------|---------------|
| structural | 97 |
| pipeline | 60 |
| cathodic-protection | 20 |
| drilling | 6 |
| geotechnical | 0 |
| hydrodynamics | 0 |
| bsee | 0 |
| metocean | 0 |
| subsea | 0 |
| naval_architecture | 0 |
| mooring | 0 |
| risers | 0 |
| economics | 0 |

This means `research-domain.py` returns 0 standards for most domains, reducing the usefulness of auto-generated research briefs.

## Why

The ledger is the canonical source for which standards map to which domain. Without entries, the research brief `applicable_standards` section is empty and agents must manually identify standards via WebSearch.

## Acceptance Criteria

1. Ledger has entries for all 10 Tier 1 domains (geotechnical, cathodic-protection, structural, hydrodynamics, drilling, pipeline, bsee, metocean, subsea, naval_architecture)
2. Each domain has at least 5 standards mapped with correct org, status, and domain fields
3. `research-domain.py --category <domain>` returns non-empty `applicable_standards` for all Tier 1 domains
