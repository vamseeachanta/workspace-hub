## Mission

Create a skill that systematizes research and literature gathering for engineering categories and subcategories. When implementing a new calculation module (e.g., geotechnical/pile_capacity), an agent needs to:

1. Identify the applicable standards and codes for that category/subcategory
2. Search the doc index (1M records) for relevant documents already in the library
3. Identify which documents need to be downloaded from external sources
4. Download or flag for manual download (standards behind paywalls)
5. Extract key equations, worked examples, and design data from gathered literature
6. Produce a structured research brief that feeds into TDD implementation

## What

The skill should:
- Accept a category (e.g., "geotechnical") and subcategory (e.g., "pile_capacity")
- Query the standards-transfer-ledger for mapped standards in that domain
- Query the doc index for relevant documents (by domain, keywords, org)
- Cross-reference with the capability map to identify what's implemented vs. gap
- Search for publicly available reference material (technical papers, code examples)
- Produce a structured output: applicable standards list, available docs, download tasks, key equations, worked examples
- Integrate with the calculation-report skill (provide inputs for YAML methodology section)

## Why

- Every calculation implementation starts with literature research
- Currently this is ad-hoc — each agent re-discovers the same research steps
- The doc index has 1M records but no systematic way to query by engineering need
- Geotechnical implementation (WRK-1179 Phase 2) is blocked by document downloads
- This skill compounds: every future calculation benefits from structured research

## Acceptance Criteria

1. Skill file at `.claude/skills/data/research-literature/SKILL.md`
2. Integrates with existing tools: query-ledger.py, capability map, doc index
3. Produces structured research brief (YAML or markdown) per category/subcategory
4. At least one worked example: geotechnical/pile_capacity research brief
5. Documents download workflow for paywalled vs. public standards
6. TDD: tests for research brief generation
