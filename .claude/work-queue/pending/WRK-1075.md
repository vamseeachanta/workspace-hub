## Mission
Auto-generate browsable API reference docs from docstrings so engineers can navigate
module interfaces without reading source code — making engineering WRK exploration fast.

## What
- Toolchain: MkDocs + mkdocstrings (Google-style docstring parser) per repo
- Config: `mkdocs.yml` in each tier-1 repo root; `docs/api/` auto-generated from src/
- Generation script: `scripts/docs/build-api-docs.sh [--repo <name>] [--serve]`
  - Runs `mkdocs build` per repo; outputs to `site/` (gitignored)
  - `--serve` flag launches `mkdocs serve` for live local browsing
- CI integration: build docs on every push to main; fail build if any autodoc
  import error occurs (catches broken docstrings early)
- Hub-level index: `config/docs/docs-manifest.yaml` — repo → local doc URL map;
  `/work status WRK-NNN` can link to relevant module docs

## Why
WRK-1058 will enforce docstring completeness. WRK-1059 will enforce reference doc
accuracy. Without a rendering pipeline those docstrings stay in source — useful but
not browsable. MkDocs + mkdocstrings converts them to searchable HTML in one command.

## Acceptance Criteria
- [ ] `mkdocs.yml` + `mkdocstrings` config for all 5 tier-1 repos
- [ ] `build-api-docs.sh` generates HTML for each repo without errors
- [ ] `--serve` launches local preview
- [ ] Build fails on broken docstring imports (catches API rot early)
- [ ] `docs-manifest.yaml` at hub level with local URL per repo
- [ ] Cross-review (Codex) passes
