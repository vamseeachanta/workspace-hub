## Mission

Re-run the 10-workbook POC using the updated xlsx-to-python skill — detect row patterns, collapse to Python loops, generate compact runnable code + HTML calc-reports.

## What

Using the same 10 XLSX files from WRK-1247, apply the full skill pipeline:

1. **Extract** — dual-pass openpyxl (values + formulas) [reuse WRK-1247 extraction]
2. **Detect patterns** — normalize formulas via `openpyxl.formula.translate.Translator`, group by canonical form
3. **Loop collapse** — repeated patterns (≥3 rows) → Python `for` loops or numpy vectorized ops
4. **Translate** — simple arithmetic → Python expressions; complex → `formulas` lib fallback
5. **Generate code** — one `.py` module per workbook with functions + loops (not 187K stubs)
6. **Generate tests** — baseline from Excel cached values + 10 parametric variations per calculation
7. **Generate HTML** — calc-report YAML → HTML via `generate-calc-report.py` (calculation-report skill)

### Key difference from WRK-1247

WRK-1247 extracted formulas as strings and generated 187K test stubs (one per cell).
This WRK produces **compact Python code** — the conductor workbook's 2,699 formulas
become ~132 functions with 64 loops. The output should look like engineer-written Python,
not a cell dump.

### Output per workbook

```
knowledge/dark-intelligence/xlsx-poc-v2/<stem>/
  patterns.yaml         # detected patterns + compression stats
  calculations.py       # Python module with functions + loops
  test_calculations.py  # baseline + parametric tests
  calc-report.yaml      # structured calc-report
  report.html           # rendered HTML via calculation-report skill
```

## Why

WRK-1247 proved extraction works but didn't produce usable Python code. The skill
now documents the pattern detection + loop collapse approach (researched: no existing
library does this). This POC validates the skill's methodology before scaling to 23K files.
