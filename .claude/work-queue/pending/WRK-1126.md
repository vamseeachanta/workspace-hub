## Mission

Establish maritime law as a first-class knowledge domain in workspace-hub by creating a dedicated skill, curating public-domain case law and liability data, and wiring it into the resource-intelligence and session-start pipelines.

## What

- Create `.claude/skills/domains/maritime-law/SKILL.md` covering: jurisdiction (UNCLOS, flag-state, port-state), liability regimes (CLC 1992, Bunker Convention 2001, HNS Convention, Athens Convention), standard forms (B/L, charter-party, P&I club), and dispute resolution (Lloyd's, LMAA, SCMA).
- Curate a `knowledge/seeds/maritime-law-cases.yaml` file with ≥10 landmark public cases (e.g. The MSC Flaminia, Prestige, Erika, MV Wakashio, Ever Given) including: case name, year, jurisdiction, key finding, liability outcome, and source URL (public record).
- Add `knowledge/seeds/maritime-liabilities.yaml` covering standard liability caps, conventions, and indemnity chains (ship-owner → P&I club → cargo interests → reinsurer).
- Index both YAML seeds into the knowledge base via `scripts/knowledge/build-knowledge-index.sh`.
- Add a `maritime-law` entry to `config/ai_agents/ai-tools-status.yaml` domain registry (if one exists) or equivalent config.

## Why

The workspace handles offshore engineering assets (OrcaFlex, ANSYS, CFD). Maritime law governs liability, insurance, and regulatory compliance for offshore structures and vessels. Having machine-readable case law and liability conventions allows agents to answer regulatory questions and flag compliance risks in WRK items.

## Acceptance Criteria

1. `SKILL.md` for maritime-law domain exists and passes skill-eval.
2. `maritime-law-cases.yaml` contains ≥10 unique landmark cases with full metadata.
3. `maritime-liabilities.yaml` covers ≥5 international liability conventions with cap values.
4. Both YAML seeds are indexed: `bash scripts/knowledge/build-knowledge-index.sh` exits 0.
5. `bash scripts/knowledge/query-knowledge.sh --category maritime-law` returns ≥5 results.
6. No client identifiers in any committed file (legal scan passes).
7. TDD: ≥3 unit tests validate YAML schema of both seed files.
