# chore(harness): review all settings.json and ecosystem configs for peak productivity

## Mission
**Audit every Claude Code and ecosystem settings file across the workspace-hub repo ecosystem, benchmark against known best-practice options, and apply improvements that measurably boost agent + human productivity — no scope creep into non-settings code.**

## What

With knowledge of the full repo ecosystem, audit and optimise every relevant settings file:

- `.claude/settings.json` (workspace-hub hub) — permissions, hooks, plugins, statusLine
- Per-repo `.claude/settings.json` (all submodules with their own)
- `CLAUDE.md` / `AGENTS.md` / `CODEX.md` / `GEMINI.md` across hub + tier-1 repos
- `.pre-commit-config.yaml` entries (legal scan, formatting)
- `pyproject.toml` tool sections (`[tool.pytest.ini_options]`, `[tool.ruff]`, etc.)
- `uv.toml` / `uv.lock` consistency
- Any VS Code / editor settings (`.vscode/settings.json`) that affect agent tooling
- Cron / nightly scripts config (`scripts/cron/`, `setup-cron.sh`)

## Why

The ecosystem has grown organically across 1000+ WRK items. Settings were added
piecemeal without a holistic review. Known productivity gaps include:

- Permissions `allow`/`deny` lists may be too narrow (blocking useful commands) or
  too broad (allowing risky ones)
- Hook overhead: some hooks may be slow or fire unnecessarily
- Plugin set may have gaps now that 9 official plugins are installed
- Agent harness files (CLAUDE.md etc.) may have drifted past 20-line limit
- `pyproject.toml` tool config may lack useful defaults (ruff, mypy, pytest flags)
- Comprehensive-learning nightly is the right place to codify best-practice
  recommendations into durable config, not just session notes

## Acceptance Criteria

- [ ] Inventory of all settings files across hub + tier-1 repos (table: file, repo, last-modified, size)
- [ ] Permissions audit: `allow`/`deny` lists reviewed; gaps and risks documented
- [ ] Hooks audit: latency measured per hook; slow hooks flagged (>2s threshold)
- [ ] Agent harness file compliance: CLAUDE.md/AGENTS.md/CODEX.md/GEMINI.md ≤20 lines each
- [ ] Plugin gap analysis: any official plugins from `anthropics/claude-plugins-official` not yet installed that would benefit this ecosystem
- [ ] `pyproject.toml` tool config reviewed across tier-1 repos; missing best-practice defaults proposed
- [ ] At least 5 concrete improvements applied (not just documented)
- [ ] Changes committed and INDEX regenerated
- [ ] Comprehensive-learning Phase 10 (or a new nightly section) captures the canonical settings baseline for future drift detection

## Agentic AI Horizon

Settings optimisation has outsized ROI — every agent session benefits from tighter
permissions, faster hooks, and richer plugin coverage. This is pure compound value;
the comprehensive-learning nightly can monitor for future drift automatically once
a baseline is established.
