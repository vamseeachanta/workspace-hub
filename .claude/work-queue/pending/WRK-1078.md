## Mission
Identify and fix workspace-hub WRK pipeline scripts that break on Windows
Git Bash (MINGW64) so acma-ansys05 can run routed work items without friction.

## What
Audit and patch the following known gaps:

1. **uv not available** — `verify-gate-evidence.py`, `generate-index.py`,
   `close-item.sh` all invoke `uv run`; add Windows fallback or installer hint
2. **`xdg-open` missing** — Stage 5/7/17 HTML review gates use `xdg-open`;
   replace with cross-platform opener (`start` on Windows, `open` on macOS)
3. **`sed -i` MINGW64 behaviour** — audit scripts using `sed -i` for
   in-place edit; MINGW64 requires `sed -i ''` or `sed -i.bak`
4. **`python3` bare calls** — audit remaining bare `python3` invocations
   (related to WRK-693); ensure `uv run --no-project python` or add fallback
5. **Crontab** — document that cron-based automation is ace-linux-1 only;
   add note to Windows setup guide

## Why
acma-ansys05 (orcaflex-license-machine) can run Claude Code and has all AI
CLIs installed, but hits friction on pipeline scripts due to Linux assumptions.
Fixing these gaps enables proper WRK routing to that machine.

## Acceptance Criteria
- [ ] `xdg-open` calls replaced with cross-platform opener helper
- [ ] `sed -i` usage audited; MINGW64-safe where needed
- [ ] `uv` install check added to `verify-setup.sh` Windows path with hint
- [ ] `dev-env-check.sh` runs without error on acma-ansys05
- [ ] Windows compatibility notes added to `.claude/docs/new-machine-setup.md`
- [ ] All changes pass existing test suite on ace-linux-1
