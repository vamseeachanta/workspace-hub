# arch(orchestrator): unify AI agent session logs in workspace-hub/logs/orchestrator/

## Objective
Ensure all AI agents (Claude, Codex, Gemini) have access to each other's session logs to enable cross-orchestrator assessment, analysis, and learning.

## Context
Currently, logs are scattered across different directories (e.g., `logs/`, `.hive-mind/logs/`, `.claude/state/`). To facilitate true multi-agent collaboration, these must be unified into a single, predictable location: `workspace-hub/logs/orchestrator/`.

## Plan
1. Create `workspace-hub/logs/orchestrator/` directory structure (e.g., partitioned by agent or date).
2. Update the CLI/logging configuration for Claude, Codex, and Gemini to output session transcripts and debug logs to this new unified directory.
3. Migrate or symlink existing active logs to the new directory.
4. Update `AGENTS.md` and any relevant `.claude/rules/` to reflect the new canonical log location.

## Acceptance Criteria
- [ ] `workspace-hub/logs/orchestrator/` exists.
- [ ] All three agents write their session logs to this directory.
- [ ] Agents can read logs from the other agents to perform cross-session analysis.
