# chore(onboarding): Add inotify Watch Limit Fix to New Machine Setup

## Mission

Add `fs.inotify.max_user_watches=524288` to new-machine-setup docs and workstations
skill so every Linux machine is pre-configured for Node.js dev servers.

## Why

Discovered running `npm run dev` for uigen course project on ace-linux-1: default
limit (65536) causes Turbopack to panic with "OS file watch limit reached". Will affect
every Linux machine running any modern JS dev server (Next.js, Vite, etc.).
Per 25% repetition rule — this WILL recur on every new Linux machine setup.

## What

1. Add to `.claude/docs/new-machine-setup.md` under "Linux System Tuning" section:
   ```bash
   echo fs.inotify.max_user_watches=524288 | sudo tee -a /etc/sysctl.conf && sudo sysctl -p
   ```
2. Add same step to `workstations/SKILL.md` § New Workstation Onboarding
3. Note: requires sudo — flag as one-time manual step

## Acceptance Criteria

- [ ] `new-machine-setup.md` includes inotify fix with context
- [ ] `workstations/SKILL.md` New Workstation Onboarding includes the step
- [ ] Both locations note it requires sudo and is a one-time fix
