# Fix invalid skill names over 64 chars

## What

Shorten the frontmatter `name:` values in skill files that exceed the loader's
64-character maximum so the skills load normally again.

## Why

The current skill loader skips 16 skills because their `name:` fields violate
the length constraint, reducing available local instructions and creating noisy
validation output.

## Acceptance Criteria

- [ ] All currently failing skill `name:` fields are reduced to 64 characters or less
- [ ] Skill validation/loading no longer reports these 16 length violations
