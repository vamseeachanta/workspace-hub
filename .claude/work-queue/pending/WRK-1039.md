# Harden gate verifier — 14 gaps from WRK-1035 session audit

## Mission
**Fix all 14 gate verifier gaps identified in the WRK-1035 session audit so that verify-gate-evidence.py catches the compliance failures that currently pass silently.**

## What

The WRK-1035 session audit (10 WRKs, 72h window) identified 14 gaps in
scripts/work-queue/verify-gate-evidence.py. Key gaps:
- Sentinel value detection (session_id: "unknown", confirmed_at: T00:00:00Z)
- Midnight UTC placeholder timestamp rejection
- Approval timestamp must postdate browser-open timestamp
- Publish timestamp must predate approval timestamp
- Codex provider identity verification (not just artifact presence)
- Per-stage stage: field check on approval artifacts
- Minimum elapsed time between browser-open and approval (>=60s)
- Claim artifact filename canonicalisation (claim-evidence.yaml not claim.yaml)

## Why

The gate verifier is the last automated defence before close. All 14 gaps have live
evidence from the audit — they are not hypothetical.
