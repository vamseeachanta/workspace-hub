## Mission
Protect the public interfaces of shared repos (assetutilities) from silent breakage
in their consumers (digitalmodel, worldenergydata) by encoding the contracts as
executable tests.

## What
- Contract test suite: `tests/contracts/` in digitalmodel and worldenergydata
  - Each file tests the exact public API surface of assetutilities that the consuming
    repo depends on: function signatures, return types, exception types, key constants
  - Tests use the real assetutilities installed from the local path — not mocks
  - Naming: `test_assetutilities_<module>_contract.py`
- Run integration: included in `run-all-tests.sh` as a separate step after unit tests
  (requires cross-repo PYTHONPATH setup already in place)
- Contract violation report: when a contract test fails, output includes the
  assetutilities version, the function signature that changed, and the consumer test
  that broke — giving a clear "what changed → what broke" signal
- Document the contract boundary: `docs/api-contracts.md` per consuming repo —
  lists which assetutilities symbols are under contract and the stability guarantee

## Why
assetutilities is the most depended-upon repo in the ecosystem. A public API change
(renamed function, changed return type, removed parameter) can silently break
digitalmodel or worldenergydata. No contract tests means breakage is only discovered
when those consumers' unit tests happen to exercise the changed path.

## Acceptance Criteria
- [ ] `tests/contracts/` with ≥3 contract test files in digitalmodel
- [ ] `tests/contracts/` with ≥2 contract test files in worldenergydata
- [ ] Contract tests run in `run-all-tests.sh` integration step
- [ ] Failed contract test output shows: changed symbol, assetutilities version, consumer test
- [ ] `docs/api-contracts.md` listing contracted symbols per consuming repo
- [ ] Cross-review (Codex) passes
