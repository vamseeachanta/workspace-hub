## Mission

Fix `infer_skill_type()` in eval-skills.py so guidance/reference skills are not scored
against workflow section requirements. Currently inflates `section_missing` to 2002 — most
are false positives from guidance skills evaluated as workflow type.

## What

1. Improve heuristics in `infer_skill_type()`:
   - Skills with `scripts_exempt: true` → guidance (already done)
   - Skills in business/, data/visualization/, data/documents/, science/ → guidance
   - Skills with no code blocks and <200 lines → reference
   - Skills with `## Quick Start` + code blocks → tool or workflow
2. Add `type:` field support in frontmatter for explicit override
3. Re-run eval and verify section_missing drops from 2002 to <300

## Why

- 2002 section_missing warnings make the eval report useless as a quality signal
- Most skills are guidance-type but scored as workflow (requires 7 sections)
- Fixing this makes the bottom quartile list actually meaningful

## Acceptance Criteria

- [ ] `infer_skill_type()` improved with better heuristics
- [ ] TDD: tests for type inference edge cases
- [ ] section_missing warnings reduced from 2002 to <300
- [ ] eval report top issues list reflects real quality problems, not structural noise
