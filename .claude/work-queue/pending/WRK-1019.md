# feat(skills): repo-portfolio-steering skill — balance harness vs technical work to hit GTM objectives

## Mission
**Create a `/repo-portfolio-steering` skill that gives a periodic read on (a) harness vs engineering work balance, (b) which technical capabilities are closest to GTM-ready, and (c) the next highest-leverage action to find projects and fund the repos — preventing harness over-investment at the expense of billable engineering output.**

## What

A skill that answers the recurring strategic question: *"Given where the queue is today, am I spending the right proportion of effort on harness vs engineering work, and what should I do next to move toward funded projects?"*

The skill is not a dashboard — it is an actionable steering report. It reads the current queue state and produces:

1. **Balance snapshot**: ratio of harness : engineering : other work in the active queue (pending + working)
2. **Harness saturation signal**: flag if harness items exceed a threshold (e.g. >30% of queue) — signals over-sharpening
3. **GTM-ready technical capabilities**: which engineering modules have enough maturity to be demoed/sold today (based on test coverage, documentation, brochure status)
4. **Next 3 actions to fund**: specific WRK items or module pairs that, if completed, unlock a project pitch or billable engagement
5. **Recommended harness budget**: how many harness items to run per 10 engineering items (a spend rate, not a ban)

## Why

The harness (skills, queue, workflow, hooks) is essential infrastructure but has no natural stopping point — it can absorb unlimited effort. Without a steering check, sessions drift toward harness refinement and the engineering repos (which generate revenue) stagnate.

GTM objectives require specific technical capabilities to exist and be demonstrable. This skill makes the connection explicit: here is what you need to build, here is what you need to market, here is what the queue should look like to get there.

## Skill Design

**Name**: `repo-portfolio-steering`
**Trigger**: `/repo-portfolio-steering` or during `/session-start` weekly mode
**Output**: Structured steering report (~1 page) with 5 sections above
**Inputs**: INDEX.md category view, `specs/data-sources/`, brochure status fields, WRK `category` fields

## Acceptance Criteria

- [ ] Skill file at `.claude/skills/workspace-hub/repo-portfolio-steering/SKILL.md`
- [ ] Balance snapshot: reads INDEX.md By Category and computes harness % vs engineering % of active queue
- [ ] Harness saturation threshold: configurable (default 30%), warns when exceeded
- [ ] GTM-readiness scan: reads brochure_status fields + test coverage proxies to rank modules by demo-readiness
- [ ] "Next 3 to fund" section: maps high-maturity engineering WRKs to GTM opportunities (engineering domain → client persona → project type)
- [ ] Recommended harness budget: outputs a spend-rate recommendation (e.g. "run 1 harness item per 5 engineering items this week")
- [ ] Integrates with `/session-start` as an optional weekly steering mode
- [ ] Skill description triggers on "project funding", "GTM", "balance harness", "portfolio steering", "what to build next"

## Agentic AI Horizon

- As the queue grows and engineering modules mature, this skill becomes the primary planning tool for deciding where to invest.
- The harness/engineering ratio signal is a leading indicator: if harness > 30%, the next 2 weeks of sessions should be pure engineering.
- GTM-ready modules generate project opportunities → funded work → more engineering investment → virtuous cycle.
