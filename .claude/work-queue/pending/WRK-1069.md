## Mission
Attribute AI token usage to specific WRK items so cost-per-WRK is visible and
ROI decisions on complex vs simple routes are data-driven.

## What
- Extend session logger to tag each token-usage event with the active WRK id
  (reads `.claude/state/active-wrk`); emit structured JSONL:
  `{"event":"token_usage","wrk_id":"WRK-NNN","provider":"claude","input_tokens":N,"output_tokens":N,"ts":"..."}`
- Aggregation script: `scripts/ai/wrk-cost-report.py [WRK-NNN]`
  - Reads session JSONL logs, groups by wrk_id
  - Reports: input/output tokens, estimated cost (using provider pricing table in
    `config/ai-tools/pricing.yaml`), provider breakdown
  - Summary table: all closed WRKs with cost + complexity — enables ROI comparison
- `pricing.yaml`: static file with per-model input/output cost per 1M tokens;
  updated manually when pricing changes
- Integrate into `close-item.sh`: print cost summary to terminal at WRK close

## Why
Currently AI spend is tracked at session level only. Without per-WRK attribution
we can't answer: "Was this Route C worth 3× the tokens vs Route B?" or "Which
subcategories consume the most AI budget?" Cost transparency drives better routing.

## Acceptance Criteria
- [ ] Session logger emits `token_usage` events tagged with active WRK id
- [ ] `wrk-cost-report.py WRK-NNN` prints token totals + estimated cost
- [ ] `pricing.yaml` covers claude-sonnet-4-6, claude-opus-4-6, gpt-5.4, gemini models
- [ ] Summary table (all WRKs) sortable by cost, complexity, subcategory
- [ ] `close-item.sh` prints cost summary at close
