# Research-to-WRK Synthesis Script

## Mission
**Write a script that takes a research paper URL, fetches and summarises key findings, maps them to automation opportunities, and drafts WRK items — replacing the manual fetch-analyse-create loop.**

## What
1. Accepts a URL as argument
2. Fetches and summarises content via Claude
3. Outputs structured YAML: `findings`, `automation_opportunities[]`, `suggested_wrk_titles[]`
4. `--draft` flag scaffolds WRK files in `.claude/work-queue/pending/` via `next-id.sh`

## Why
Each research synthesis re-derives the same steps. The 25% repetition rule mandates a script. Savings: avoids a full LLM analysis pass every repeat session.

## Acceptance Criteria
- [ ] Script `scripts/ai/research-to-wrk.sh <url>` fetches content and calls Claude API
- [ ] Outputs YAML: `source_url`, `summary`, `automation_opportunities[]`, `suggested_wrk_items[]`
- [ ] `--draft` flag scaffolds pending WRK files using next-id.sh
- [ ] Works offline gracefully (prints warning if URL unreachable)
- [ ] ≥2 TDD tests (mock fetch, verify YAML output structure)
