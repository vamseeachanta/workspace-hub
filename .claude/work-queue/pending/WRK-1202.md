## Mission
Convert document intelligence and extraction skills from LLM prose to deterministic, reusable scripts — enabling both local document parsing and online document fetching/parsing pipelines.

## What
Skill-to-script conversions (25% repetition rule enforcement):

### Tier 1 — HIGH priority (blocking active WRKs)
1. `manifest-to-archive.py` — convert doc-intelligence manifests to dark-intelligence YAML (unblocks WRK-1183)
2. `deduplicate-manifests.py` — merge/dedup batch extraction manifests
3. `fetch-queue-manager.py` — async URL fetch queue with rate limiting, resume, domain-aware throttling

### Tier 2 — Skill prose → scripts (engineering extraction)
4. `extract-engineering-constants.py` — deterministic Layer 1 content extraction (from `doc-extraction/SKILL.md`)
5. `normalize-units.py` — SI/imperial/field unit normalization (from `doc-extraction/SKILL.md` Layer 2)
6. `parse-standard-reference.py` — extract DNV/API/ISO section references to structured YAML
7. `validate-naval-architecture.py` — IMO criteria + hull coefficient range checks (from `naval-architecture/SKILL.md`)

### Tier 3 — Online document parsing infrastructure
8. `crawl-and-enqueue.py` — seed URL crawling with link extraction, robots.txt, domain filtering
9. `fetch-from-api.py` — REST API document fetcher (BSEE, EIA, IMO GISIS) with auth + pagination
10. `end-to-end-online-extraction.py` — orchestrate: fetch → extract → deduplicate → archive

### Tier 4 — Quality & audit
11. `audit-extractions.py` — extraction statistics report (counts, confidence, errors)
12. `validate-confidence.py` — filter low-confidence manifests, cross-validate against benchmarks
13. `generate-research-brief.py` — automate research-literature skill Steps 1-4 (ledger + index + capmap queries)

## Why
- Skills describe extraction logic as prose — LLM must re-derive each time (slow, non-deterministic)
- 25% repetition rule: these operations recur across every extraction/literature WRK
- Enables overnight batch runs without LLM involvement
- Unblocks WRK-1183 (dark intelligence) and WRK-1188 (Phase B extraction QA)

## Acceptance Criteria
1. ✅ Tier 1 scripts (3) implemented with TDD tests and CLI interfaces
2. ✅ Tier 2 scripts (4) implemented — engineering extraction heuristics are deterministic Python
3. ✅ Tier 3 scripts (3) implemented for online document parsing
4. ✅ Each script has `--help`, reads from stdin/file, writes YAML/JSONL output
5. ✅ All scripts follow `uv run --no-project` pattern
