## Mission
A pre-push git hook runs tests and quality checks automatically so regressions are
caught before they reach the remote — closing the feedback loop between engineering
WRK execution and quality gates.

## What
- Git pre-push hook: `scripts/hooks/pre-push.sh`
  - Detects which repos have staged changes (via `git diff --name-only origin/HEAD..HEAD`)
  - Runs `scripts/testing/run-all-tests.sh --repo <name>` for each changed repo
  - Runs `scripts/quality/check-all.sh --repo <name>` for each changed repo
  - Blocks push if any unexpected test failures or ruff errors (exit code ≠ 0)
  - Skip with `--no-verify` documented but discouraged (flag use logged)
- Install script: `scripts/hooks/install-hooks.sh` — symlinks pre-push into
  `.git/hooks/` for hub and each submodule
- Fast mode: `--changed-only` flag (default) runs only affected repos;
  `--all` runs full suite

## Why
Without a pre-push gate, regressions enter the remote and are discovered at cross-review
(Stage 13) — expensive late-stage rework. Blocking at push time costs seconds, not hours.

## Acceptance Criteria
- [ ] `scripts/hooks/pre-push.sh` runs changed-repo tests + quality checks
- [ ] Blocks push on unexpected test failure or ruff error
- [ ] `install-hooks.sh` symlinks hook for hub and all submodules
- [ ] `--changed-only` default; `--all` full suite option
- [ ] `--no-verify` usage logged to session log
- [ ] Hook completes in <60s for typical single-repo change
- [ ] Cross-review (Codex) passes
