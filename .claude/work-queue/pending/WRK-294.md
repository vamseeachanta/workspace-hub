# Standardize ace-linux-2 mount paths

## What

Add proper fstab entries for the two data HDDs on ace-linux-2 with clean, script-friendly
mount paths. Currently they auto-mount via the desktop environment to paths with spaces
and inconsistent names.

## Why

- `/media/vamsee/Local Analysis` has **spaces** — breaks unquoted shell scripts and tools
- `/media/vamsee/DDE` is a cryptic label — not self-documenting
- Neither is in fstab — not guaranteed to mount on headless boot or SSH-only sessions
- WRK-287 (network sharing) needs predictable, stable mount paths for SSHFS exports
- Consistent paths across machines (`/mnt/local-analysis` pattern) simplify cross-machine work

| /dev/sdc2 | C8F2A5DAF2A5CD4E | /media/vamsee/DDE | DDE |

## Target State

| Device | Mount Point | Notes |
|--------|-------------|
-------|
| /dev/sda | /mnt/local-analysis | 931 GB WDC — workspace-hub lives here |
| /dev/sdc | /mnt/dde | 2.7 TB Seagate — bulk data |

## Steps

1. Get UUIDs: `sudo blkid /dev/sda2 /dev/sdc2`
2. Create mount points: `sudo mkdir -p /mnt/local-analysis /mnt/dde`
3. Add fstab entries using UUID (not device name) with `defaults,nofail`
4. Unmount current desktop mounts, test `sudo mount -a`
5. Create symlink: `/mnt/workspace-hub -> /mnt/local-analysis/workspace-hub`
6. Update aceengineer-02.md with new mount paths
7. Reboot and verify mounts survive

## Acceptance Criteria

- [ ] Both HDDs have fstab entries using UUID
- [ ] Mount paths are `/mnt/local-analysis` and `/mnt/dde` (no spaces)
- [ ] `/mnt/workspace-hub` symlink points to workspace-hub directory
- [ ] Mounts persist across reboot
- [ ] aceengineer-02.md updated with standardized paths
