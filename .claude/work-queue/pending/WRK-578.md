# Audit Official MCP Registry for Engineering-Domain Servers

## What

Systematically query the MCP Official Registry
(registry.modelcontextprotocol.io) programmatic API for engineering-domain
servers. Filter by keywords: geospatial, scientific data, unit conversion,
weather, structural analysis, marine, energy, geology.

## Why

The registry holds ~2000 servers (as of Feb 2026). A targeted audit would
identify servers installable into workspace-hub with zero integration code,
accelerating agent capability without custom development.

## Deliverables

- Skill at `.claude/skills/workspace-hub/mcp-registry-audit/` that queries
  the registry API and filters by engineering keywords
- Output: priority-ranked install list with trust assessment per server

## Acceptance Criteria

- [ ] Skill queries registry API and returns filtered results
- [ ] Each result includes: name, description, relevance score, trust notes
- [ ] Top 10 candidates documented with install instructions
- [ ] Supply-chain security checklist applied to each candidate

## Research Findings (2026-03-15)

**Key finding: Engineering is deeply underserved in MCP registries.**

### Top 5 Candidates

| Server | Stars | Domain | Why |
|---|---|---|---|
| blender-mcp | 17,735 | CAD/3D | 3D visualization; already relevant to digitalmodel |
| mcp-google-map | 193 | GIS | Geocode, elevation, directions for site work |
| isaac-sim-mcp | 136 | Simulation | Physics sim / digital twins (NVIDIA) |
| math-mcp | 1 | Scientific | GPU-accelerated FFT, optimization, linear algebra |
| weather-mcp (dgahagan) | — | Marine | Marine conditions data for offshore engineering |

### Notable Gaps (opportunity for workspace-hub)
No MCP servers exist for: FEA, CFD, structural analysis, marine/offshore engineering, corrosion, cathodic protection, pipeline engineering, geology/subsurface, oil & gas, or energy market data.

### Trust Assessment
All engineering-domain servers except blender-mcp have weak trust signals (single-digit stars, single maintainers). The ecosystem is early-stage for engineering use cases.
