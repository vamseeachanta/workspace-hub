## Mission

Add a default interactive menu to the work-queue-workflow that appears at user-review stages, giving the user clear action choices instead of freeform input.

## What

Research existing session/stage interaction patterns and design a standard menu presented at every user-review stage (1, 5, 7, 17) in the work-queue-workflow. Preliminary menu options:

1. **Approve stage N** — mark the current stage as approved and proceed
2. **Go to previous stage** — revert to the prior stage for rework
3. **Pending as further research needed** — park the WRK with a research-needed status
4. **Chat about this** — enter a conversational mode to discuss the current stage before deciding

## Why

Currently, user-review hard gates require freeform input — the user must know what actions are available. A standard menu reduces cognitive load, ensures consistent gate responses, and makes the workflow more discoverable for new users.

## Acceptance Criteria

- [ ] Research existing stage interaction patterns in work-queue-workflow, session scripts, and related skills
- [ ] Design the menu structure (options, shortcuts, extensibility)
- [ ] Menu appears automatically at stages 1, 5, 7, 17 (human gate stages)
- [ ] Each menu option maps to a defined action (approve, revert, park, chat)
- [ ] Menu is extensible for future options without code changes
- [ ] Integration with `is-human-gate.sh` detection
- [ ] Tests for menu rendering and option routing
