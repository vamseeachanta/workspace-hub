## Background

The 6 official OrcaWave example models (L01–L06) in
`digitalmodel/docs/domains/orcawave/examples/` collectively cover the full
parameter space of OrcaWave's YAML configuration format. A systematic audit of
these files produces a definitive parameter reference for future work.

## Goal

Extract every unique parameter key from all 7 YAML files across L01–L06,
categorise them, and produce `PARAMETER_REFERENCE.md` in the examples folder.

## Scope

- Read all .yml files in `docs/domains/orcawave/examples/Lxx/`
- Categorise: Solver, Environment, Mesh quality, Output, QTF, Restart, Body,
  MorisonElement, Drawing
- Cross-reference parameter presence across examples
- Produce `docs/domains/orcawave/examples/PARAMETER_REFERENCE.md`

## Acceptance Criteria

- [ ] All unique parameter keys from L01–L06 listed with type and description
- [ ] Parameters that differ across examples highlighted (solver method, QTF, etc.)
- [ ] Reference committed to repo

## Output File

`digitalmodel/docs/domains/orcawave/examples/PARAMETER_REFERENCE.md`
