## Mission
Catch the security, complexity, and dead-code issues that ruff cannot — before they
reach cross-review or client-facing reports.

## What
- Extend `scripts/quality/check-all.sh` with three additional passes:
  1. **bandit** (`bandit -r src/ -ll`) — security: hardcoded passwords, shell injection,
     insecure hash functions, subprocess misuse; exit 1 on MEDIUM+ severity
  2. **radon cc** (`radon cc src/ -n C`) — cyclomatic complexity; flag functions
     scoring C or above (complexity ≥ 10); warn only initially, not blocking
  3. **vulture** (`vulture src/ --min-confidence 80`) — dead code: unused functions,
     classes, imports above 80% confidence; report as warning
- Config: `.bandit` allowlist per repo for known false positives; radon thresholds
  in `pyproject.toml`
- Baseline: `config/quality/static-analysis-baseline.yaml` captures existing
  violations; new violations above baseline block
- Pre-commit: bandit on staged files only (speed); radon + vulture in full-suite mode

## Why
ruff covers style and basic errors. It does not catch: `eval()` misuse,
`subprocess shell=True`, functions with complexity 20+, or 30% dead code accumulation.
bandit finds real vulnerabilities; radon flags unmaintainable code early;
vulture surfaces deletable dead weight.

## Acceptance Criteria
- [ ] `check-all.sh` runs bandit, radon, vulture per repo
- [ ] bandit: MEDIUM+ blocks; LOW warns
- [ ] radon: complexity C+ reported; non-blocking initially
- [ ] vulture: 80%+ confidence dead code reported as warning
- [ ] `.bandit` allowlist per repo for known false positives
- [ ] `static-analysis-baseline.yaml` captures existing violations
- [ ] bandit added to pre-commit hooks (staged files only)
- [ ] Cross-review (Codex) passes
