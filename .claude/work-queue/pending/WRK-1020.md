# feat(cron): portfolio-signals.yaml — daily agentic activity + capability research signal

## Mission
**Create and maintain `.claude/state/portfolio-signals.yaml` — a daily-refreshed signal
file that feeds the `repo-portfolio-steering` skill with per-provider agentic activity
data and AI capability research updates.**

## What

Two signal types written to `.claude/state/portfolio-signals.yaml`:

### Signal 1 — Per-provider agentic activity (Layer 2)
Parse archived WRK files (last 7 days) to compute per-provider (claude/codex/gemini)
work breakdown by category. Answers: "Is Claude drifting toward harness? Is Codex
doing engineering work?"

- Source: `.claude/work-queue/archive/YYYY-MM/*.md` frontmatter (`orchestrator:` + `category:`)
- Output: `provider_activity.{claude,codex,gemini}.{harness,engineering,data,...}` counts

### Signal 2 — Agentic capability research (Layer 3)
Daily lightweight research to surface new Claude/Codex/Gemini capabilities relevant
to engineering work. Keeps the steering report aware of tool improvements.

- Sources: Anthropic changelog, OpenAI platform changelog, Google AI changelog
- Method: `gemini -p "What new Claude/Codex/Gemini features were announced in the last 7 days
  relevant to engineering/scientific computation? List with date, provider, capability, impact."  -y`
- Filter: only high-confidence, official-source signals; max 5 entries per run
- Output: `capability_signals[]` list in portfolio-signals.yaml

## What to Build

1. **`scripts/cron/update-portfolio-signals.sh`**:
   - Parses last 7 days of archived WRK files for provider×category counts
   - Runs gemini capability research query
   - Writes `.claude/state/portfolio-signals.yaml` (overwrites)
   - Idempotent; safe to re-run
   - Dry-run mode: `--dry-run` prints without writing

2. **Integration into `comprehensive-learning-nightly.sh`**:
   - Add as Phase 0.5 (before Phase 1) OR append to Phase 1 post-processing
   - Runs on ace-linux-1 only (same guard as comprehensive-learning)

3. **Schema**: `.claude/state/portfolio-signals.yaml` (see WRK-1019 plan.md for schema)

## Acceptance Criteria

- [ ] `update-portfolio-signals.sh` generates valid portfolio-signals.yaml from archive
- [ ] Provider activity counts match manual count of last-7-day archive files
- [ ] Capability research: gemini query runs, ≤5 signals written, source URLs included
- [ ] `--dry-run` flag prints without writing file
- [ ] Script integrated into `comprehensive-learning-nightly.sh`
- [ ] `.claude/state/portfolio-signals.yaml` gitignored (state file, not committed)
- [ ] `repo-portfolio-steering` skill reads the file gracefully when missing (no crash)

## Agentic AI Horizon

- Provider activity signal closes the loop between "who's doing what" and "are we balanced"
- Capability research signal surfaces tool improvements that could unlock faster engineering —
  e.g. a new Claude reasoning mode might make complex structural analysis tractable
- Together they make the steering report self-updating rather than requiring manual research
