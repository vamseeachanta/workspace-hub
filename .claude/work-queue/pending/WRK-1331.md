## Mission

Write `scripts/knowledge/update-github-issue.py` with TDD.

Generates GitHub Issue body from WRK frontmatter + evidence files using the approved template (see issue #39).

### Template sections (collapsible)
1. Status line (always visible) — status, priority, category, repo, complexity, machine
2. Implementation Summary — files changed, commits
3. Final Plan — approved plan with decisions
4. Acceptance Criteria — checkboxes
5. TDD Results — test evidence
6. Evidence and Cost — tokens, cross-review, legal, future work
7. Stage Progress — 20-stage checklist

### Interface
```
update-github-issue.py WRK-NNN [--create | --update | --close]
```

### ACs
- [ ] Renders body from WRK frontmatter + evidence files
- [ ] Creates new issue via `gh issue create`
- [ ] Updates existing issue via `gh issue edit`
- [ ] Closes issue via `gh issue close`
- [ ] TDD: 8+ tests covering template rendering and edge cases
