# OpenFOAM Deep Solver Workflows

## Mission

Move OpenFOAM from "can generate case files" to "can run, monitor, and extract validated engineering results" — end-to-end solver workflows with automated convergence control and post-processing pipelines.

## What

WRK-047 built the case generation framework and WRK-343 audited the ecosystem. This item adds operational depth:

1. **Tutorial reproduction as validation suite** — reproduce cavity, damBreak, and ship hull resistance tutorials programmatically, compare results against published data, use as regression tests
2. **Marine solver templates** — pre-configured case templates for seakeeping (wave loading on fixed structures), green water (deck overtopping), and current drag (steady flow past subsea equipment)
3. **Automated convergence monitoring** — real-time residual parsing, Courant number tracking, early termination on divergence, automatic timestep adjustment recommendations
4. **Post-processing pipeline** — force/moment integration on patches → pandas DataFrame, free surface extraction from alpha.water → elevation time series, probe data aggregation, convergence plots

## Why

The digitalmodel OpenFOAM module can set up cases but cannot yet run them end-to-end with confidence. Marine engineers need validated templates they can trust, convergence monitoring that catches divergence early (saving hours of wasted compute), and post-processing that produces engineering-ready outputs (force RAOs, pressure distributions, free surface profiles).

## Acceptance Criteria

- [ ] Cavity tutorial: programmatic setup → run → residual convergence verified against OpenFOAM docs
- [ ] damBreak tutorial: VOF free surface → compare water column height at probes against published results
- [ ] Marine template: wave loading on box barge → force time series extraction → comparison with potential flow (Morison/AQWA)
- [ ] Convergence monitor script: parse log in real-time, detect divergence (residuals >1e6), report convergence rate
- [ ] Post-processing: force extraction → pandas, free surface → numpy array, convergence plot → PNG/HTML
- [ ] All workflows runnable via digitalmodel CLI or Python API
- [ ] Test suite with known-answer validation (cylinder drag Cd ≈ 1.0 at Re=100)
