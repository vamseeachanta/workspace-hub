## Mission

Establish a repeatable OrcaFlex Python scripting workflow on acma-ansys05 — from model
load through result extraction — so analyses can be automated rather than run manually.

## What

Set up OrcaFlex automation scripting foundations:
- Verify OrcaFlex Python API (`OrcFxAPI`) is importable on acma-ansys05
- Write a minimal working example: open a `.dat` model, run a static analysis, extract a
  key result (e.g., effective tension at a node), save to CSV
- Document the venv / uv setup needed on Windows Git Bash for OrcaFlex scripting
- Identify and link the Orcina automation training resource (LinkedIn post from 2026-03-11)

## Why

OrcaFlex is licensed only on acma-ansys05. Manual model runs don't scale across parametric
studies (load cases, metocean scatter, sensitivity sweeps). Scripted automation enables
batch runs, result aggregation, and integration with the broader workspace-hub analysis
pipeline.

## Acceptance Criteria

- [ ] `OrcFxAPI` imports successfully on acma-ansys05 in a scripted context (no GUI needed)
- [ ] Minimal script opens a model, runs static analysis, extracts tension, writes CSV
- [ ] Script is under `scripts/orcaflex/` in workspace-hub
- [ ] uv / Python environment setup documented in `docs/engineering/orcaflex-scripting.md`
- [ ] Follow-on WRKs identified for parametric studies and result aggregation
