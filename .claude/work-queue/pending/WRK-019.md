# Establish drilling, completion, and intervention cost data by region and environment

## What
Build a cost data layer on top of the field data pipelines that provides drilling, completion, and intervention cost estimates segmented by:
- **Environment**: offshore vs. onshore
- **Region**: Gulf of Mexico, North Sea, Brazil, Mexico, Texas (RRC), etc.
- **Activity type**: drilling (by well type/depth), completion (by type), intervention (by workover type)
- **Water depth / well depth bands** for offshore

Sources may include public cost indices, IHS/Rystad benchmarks (where available), BSEE/regulatory filings, and derived estimates from duration-based proxies (rig day rates × activity duration).

## Why
Cost data transforms activity analysis from descriptive to economically meaningful. Clients need cost context to evaluate field economics, benchmark operations, and make investment decisions. Without cost, activity counts alone don't convey operational burden.

## Acceptance Criteria
- [ ] Identify available public and derivable cost data sources
- [ ] Define cost categories: drilling cost, completion cost, intervention cost
- [ ] Segment costs by offshore vs. onshore, region, water depth, well depth
- [ ] Implement cost estimation methodology (direct data where available, proxy-based where not)
- [ ] Integrate cost layer with field data pipeline outputs
- [ ] Produce cost summaries per field, lease, or region query
- [ ] Cross-regional cost benchmarking capability
- [ ] Document data sources, methodology, assumptions, and confidence levels
- [ ] Validate against published industry cost benchmarks where available

### Phase 1 — Cost Data Source Identification and Schema

1. **Define cost data sources** — create `config/analysis/cost_data/cost_sources.yml`:
   - **Direct data**: BSEE/regulatory filings (limited public cost data)
   - **Proxy-based**: Duration x day rate (drilling days x rig rate, completion days x completion rate)
   - **Public indices**: Baker Hughes rig count (proxy for market tightness), EIA drilling productivity report
   - **Regional benchmarks**: Published industry averages by water depth band and region
   - **Existing config**: Extend `economic_assumptions.yml` pattern to multiple regions

2. **Define cost schema** — create `src/worldenergydata/bsee/analysis/cost/models.py`:
   ```
   @dataclass
   class CostEstimate:
       activity_type: str          # drilling, completion, intervention
       environment: str            # offshore, onshore
       region: str                 # gom, north_sea, permian, etc.
       water_depth_band: str       # shallow (<500ft), mid (500-5000ft), deep (>5000ft), ultra-deep (>7000ft)
       well_depth_band: str        # shallow (<10Kft), mid (10-20Kft), deep (>20Kft)
       cost_usd: float
       cost_per_day_usd: float
       duration_days: float
       confidence: str             # high (direct), medium (proxy), low (benchmark)
       source: str
       year: int
   ```

3. **Define cost category taxonomy**:
   - Drilling: tangible (casing, cement, mud) + intangible (rig time, services) + mobilization
   - Completion: perforating, stimulation, gravel pack, equipment
   - Intervention: workover rig, coiled tubing, wireline, plug-and-abandon

### Phase 2 — Cost Estimation Engine

1. **Create cost engine**: `src/worldenergydata/bsee/analysis/cost/cost_engine.py`
   - Extends `DrillingCompletionProcessor` from `financial/drilling_completion.py`
   - Adds region-aware cost selection
   - Adds water depth band classification (from BSEE well data)
   - Adds well depth band classification

2. **Day-rate database**: `config/analysis/cost_data/day_rates.yml`
   - Offshore GOM by water depth: shallow/jackup ($80-150K/day), mid-water/semisubmersible ($200-350K/day), deepwater/drillship ($400-600K/day)
   - Onshore by basin: Permian ($15-25K/day), Eagle Ford ($18-28K/day), Bakken ($20-30K/day)
   - Completion rates: conventional ($50-100K/day), HPHT ($150-250K/day)
   - Intervention rates: workover rig ($40-80K/day), coiled tubing ($30-60K/day), wireline ($10-20K/day)
   - Year-indexed (2020-2025) to capture market cycles

3. **Proxy calculation**: `cost_engine.py` methods:
   - `estimate_drilling_cost(well_data, region, year)` — matches well to day rate by depth/region, multiplies by BSEE-reported drilling days
   - `estimate_completion_cost(well_data, region, year)` — same approach for completion days
   - `estimate_intervention_cost(intervention_data, region, year)` — maps workover type to cost category
   - `estimate_with_confidence(...)` — returns `CostEstimate` with confidence level
