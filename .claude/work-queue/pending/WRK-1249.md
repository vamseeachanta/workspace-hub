# gmsh Deep Meshing Workflows

## Mission

Enable deep, production-grade gmsh meshing capabilities beyond the current reference skill — parametric convergence studies, OCC boolean operations on complex assemblies, adaptive boundary layer refinement, and automated mesh quality validation with pass/fail gates.

## What

The existing gmsh-meshing skill (WRK-139) and integration (WRK-140) cover standalone usage and solver export. This item adds the operational depth needed to run real engineering analyses:

1. **Parametric mesh convergence studies** — sweep element sizes, compare results, identify mesh-independent solutions
2. **Complex geometry via OCC kernel** — multi-body boolean operations (hull + appendages + moonpool), STEP assembly import and defeaturing
3. **Boundary layer mesh refinement** — field-based adaptive sizing for CFD boundary layers (y+ targeting), anisotropic refinement near free surfaces
4. **Automated mesh quality pipeline** — aspect ratio, skewness, Jacobian thresholds → pass/fail verdict with YAML report, integrated as pre-solver gate

## Why

Current skills tell you HOW to use gmsh. This item enables you to USE gmsh effectively on real geometries with engineering rigor — mesh convergence is mandatory for any credible CFD/FEA result, and quality gates prevent wasted solver time on bad meshes.

## Acceptance Criteria

- [ ] Mesh convergence study script: sweep element sizes, run solver (or dry-run), tabulate results, plot convergence curve
- [ ] OCC boolean workflow: import multi-body STEP assembly, boolean union/cut, export clean surface mesh
- [ ] Boundary layer field specification: y+ target → first cell height → geometric growth ratio → gmsh field definition
- [ ] Mesh quality gate script: configurable thresholds (aspect ratio <10, skewness <0.85, min Jacobian >0.3), YAML verdict output
- [ ] Integration tests with known-geometry test cases (box barge, cylinder with appendages)
- [ ] Update gmsh-meshing skill with deep workflow sections
