## Mission

Extract worked examples from USNA EN400 manifest, convert to parametrized pytest fixtures in digitalmodel, establishing TDD validation infrastructure for future naval architecture implementations (Holtrop-Mennen, GZ curves, Wageningen).

## What

- Parse the USNA EN400 manifest for 107 worked examples (inputs, method, expected answers)
- Create pytest parametrized test fixtures in digitalmodel/tests/naval_architecture/
- Each test encodes: problem statement, input parameters, expected output, source reference
- Tests initially xfail (no implementation yet) — they become green as modules are built

## Why

Compound value: every future naval architecture WRK (resistance, stability, propulsion) gets free validation from day one. TDD-first approach — tests exist before code.

## Acceptance Criteria

- [ ] AC1: Script extracts worked examples from EN400 manifest into structured YAML
- [ ] AC2: pytest fixtures created with parametrize decorators and source attribution
- [ ] AC3: Tests run (xfail) without errors in digitalmodel test suite
- [ ] AC4: At least 10 worked examples converted as prototype proof-of-concept
- [ ] AC5: Assessment documented — effort vs. value for remaining 97 examples
