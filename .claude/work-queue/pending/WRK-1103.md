# learn(harness): Claude Code in Action — Official Course + Learnings

## Mission

Complete Anthropic's official "Claude Code in Action" SkillJar course and extract actionable
improvements for this workspace-hub harness (skills, hooks, CLAUDE.md, workflow patterns).

Course URL: https://anthropic.skilljar.com/claude-code-in-action/301615

## Why

This is the canonical, Anthropic-authored reference for how Claude Code is intended to be used.
Any gap between how this harness operates and official best practice is a debt risk — mis-tuned
CLAUDE.md, sub-optimal hook design, missing keybindings, or unknown features we aren't using.

## What

### Phase 1: Complete the course (all modules)
- Watch / work through every module in order
- Take structured notes per module: new feature / gap in current harness / action item

### Phase 2: Gap analysis
- Compare course-recommended patterns against current:
  - `CLAUDE.md` (root + submodules)
  - Hook configuration (`.claude/hooks/`)
  - Active skills in `.claude/skills/`
  - Keybindings / settings
- Classify each gap: **adopt** (use as-is), **adapt** (modify for workspace patterns), **skip** (already superseded by a better local solution)

### Phase 3: Implement adopted improvements
- Each adopted item → targeted edit (one file, one concern)
- Each adapted item → brief design note in `specs/modules/` before touching code
- Skip items → document reason in `docs/course-learnings.md`

### Phase 4: Capture learnings
- Write `.claude/docs/course-learnings.md` with module-by-module notes + gap table
- Update MEMORY.md with any stable patterns that will apply across sessions
- Spin off WRK items for any improvements too large to include here

## Acceptance Criteria

- [ ] All course modules completed (progress visible in SkillJar)
- [ ] `.claude/docs/course-learnings.md` written with module notes + gap table
- [ ] Each gap classified: adopt / adapt / skip with rationale
- [ ] Adopted improvements implemented and committed
- [ ] Any new WRK items spun off for out-of-scope improvements
- [ ] MEMORY.md updated with durable cross-session patterns found

## TDD Approach

- For any script/hook changes: write tests in `tests/` before editing
- For CLAUDE.md / skill edits: validate with existing skill-eval or manual spot-check
- Learnings doc: peer-review via Gemini cross-review before closing

## Non-goals

- Rewriting existing skills wholesale — adopt improvements as targeted edits only
- Converting the course into internal training material for others
- Automating SkillJar progress tracking
