## Mission

One week after WRK-1035/1044 hardening, audit real session logs from Claude, Codex,
and Gemini to verify gate compliance improvements are holding in live (non-simulated) runs.

## What

Review all WRK sessions run between 2026-03-08 and 2026-03-15 and measure:

1. **Stage 5 hard gate adherence** — did agents stop and wait, or did they advance prematurely?
2. **Stage 7 hard gate adherence** — same check
3. **Stage 17 hard gate adherence** — same check
4. **L1 hook effectiveness** — did the write-backstop (WRK-1044 L1) block any future-stage writes?
5. **Gate verifier --json usage** — was CI integration adopted in any pipeline?
6. **Near-miss rate** — how many sessions had self-correcting near-misses vs clean runs?

## Why

Simulation results are synthetic. Real sessions expose edge cases, context-rot patterns,
and provider-specific failure modes that simulations miss. One-week holdback gives enough
sessions to draw statistical conclusions.

## Acceptance Criteria

- [ ] Parse session logs from `logs/orchestrator/claude/` + Codex/Gemini native stores for 2026-03-08 to 2026-03-15
- [ ] Score each WRK session: gates_respected / total_gates_encountered (target ≥90%)
- [ ] Identify any hard-gate violations (stage advance without approval artifact)
- [ ] Compare live scores vs simulation baseline (Claude=100, Codex=100, Gemini=95)
- [ ] Write audit report to `assets/WRK-1045/session-compliance-audit.md`
- [ ] Capture any new gaps as follow-up WRK items
