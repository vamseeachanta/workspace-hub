# Config drift — agent harness file sync and validation

## Mission

Ensure CLAUDE.md, AGENTS.md, CODEX.md, and GEMINI.md in all 6 repos stay valid, within line limits, and consistent with hub-level conventions.

## What

1. **Line limit enforcement** — all 4 harness files must be ≤20 lines (per coding-style.md); fail if exceeded
2. **Required sections check** — AGENTS.md must have YAML frontmatter (per WRK-1073 onboarding maps); validate schema
3. **Hub → submodule propagation check** — identify hub-level conventions that should appear in submodule files but are missing
4. **Drift report** — `logs/quality/config-drift-{date}.yaml` — which files are stale, overlimit, or missing sections
5. **Pre-push gate** — add config drift check to pre-push hook (fast, <5s)
6. **Auto-fix for line-limit violations** — suggest what to migrate to a skill or doc

## Why

WRK-1073 added AGENTS.md YAML frontmatter across repos. But there's no ongoing enforcement: files can grow past the 20-line limit or lose required sections silently. WRK-1013 fixed oversized skills once; without a ratchet it will happen again.

## Acceptance Criteria

- [ ] `scripts/quality/check_config_drift.py` — validates all 4 harness file types across 6 repos
- [ ] Line limit check: FAIL if any file >20 lines
- [ ] AGENTS.md schema validation (YAML frontmatter required fields)
- [ ] Drift report YAML
- [ ] Pre-push hook integration
- [ ] ≥5 TDD tests (line limit, schema, propagation check, report format, pre-push integration)
- [ ] Passes `check-all.sh` for the script itself
