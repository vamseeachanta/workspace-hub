# Promote github_issue_ref enforcement to Level 3 (hook)

## What

Currently `validate-wrk-frontmatter.sh` emits a non-blocking warning when
`github_issue_ref` is empty (Level 2 enforcement per patterns.md). Promote
this to a pre-commit hook or stop-hook that blocks commits of WRK files
in `working/` or `done/` that lack a `github_issue_ref`.

## Why

Level 2 (script) warnings can be missed. A hook ensures no WRK item
progresses past capture without a linked GitHub issue, keeping the
issue tracker in sync with the work queue.

## Acceptance Criteria

- [x] Hook fires on git commit when WRK files in working/done lack github_issue_ref
- [x] Hook does NOT block WRK files in pending/ (ref may not exist yet during multi-step capture)
- [x] Hook is skippable with documented escape hatch for offline/no-gh-auth scenarios
- [x] Existing tests continue to pass
