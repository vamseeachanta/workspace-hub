## Mission

Restructure aceengineer-admin into a consistent, navigable document taxonomy — the repo is the single source of truth for AceEngineer Inc business records.

## What

The repo has grown organically since 2013. Documents are scattered across inconsistent directories making tax prep, audits, and financial lookups slow.

### Current Problems

1. **Duplicate tax dirs**: `Tax/` (2014-2024, 79 files) vs `taxes/` (2025+, new) — merge or define boundary
2. **Sabitha/ is a catch-all** (509 files): monthly CC statements, expense spreadsheets, personal invoices — should be split by function
3. **Scattered bank/financial**: `CapitalOne/`, `USBank/`, `Deposits/`, `Sabitha/` each hold financial statements — consolidate under `financials/`
4. **Inconsistent naming**: PascalCase (`Tax/`, `Experience/`, `Reference/`), lowercase (`invoices/`, `admin/`)
5. **Root script sprawl**: 10+ one-off invoice generation scripts at repo root — consolidate into `scripts/invoicing/`
6. **Empty/stale dirs**: `coordination/` (0 files), `Kohls/` (1), `Deposits/` (1), `Superseded/` (3)
7. **No README per directory**: most dirs have no index explaining their contents
8. **Security**: Filing Taxes sheet in expense xlsx contains plaintext credentials

### Proposed Taxonomy

```
aceengineer-admin/
  financials/
    bank-statements/chase/YYYY/
    bank-statements/capital-one/
    expense-tracking/YYYY/
    deposits/
  invoices/                     # already well-organized by client
    acma/YYYY/
    seanation/
  taxes/YYYY/                   # merge Tax/ historical + taxes/ current
  admin/
    payroll/
    loans/
    incorporation/
    preferred-vendor/
  contracts/
  employees/
  src/
  scripts/invoicing/            # consolidate root scripts
  docs/
  config/
```

## Why

Tax prep (WRK-5085) revealed how hard it is to locate documents. Every annual tax cycle will repeat this pain. A clean taxonomy pays dividends across every future financial task.

## Acceptance Criteria

1. All financial documents accessible via consistent `financials/` hierarchy
2. Tax documents unified under `taxes/YYYY/` (historical + current)
3. No orphan scripts at repo root
4. README.md in each top-level directory explaining contents
5. No plaintext credentials in any committed file
6. Git history preserved (use `git mv`, not delete+create)
