# WRK-1034 — Enforce Stage 7 and Stage 17 User-Review as Hard Blocking Gates

## Mission
Add hard-gate enforcement for Stage 7 (User Review — Plan Final) and Stage 17
(User Review — Implementation) that mirrors Stage 5 enforcement (WRK-1017),
so AI agents cannot advance past these stages without verified user-review
evidence artifacts.

## What

WRK-1017 enforced Stage 5 as a hard gate: agents cannot enter Stage 6
(cross-review) without `user-review-plan-draft.yaml` signed by the user.

Stages 7 and 17 have the same intent but no equivalent guard:
- Stage 7 (User Review — Plan Final): agents advance to Stage 8 (Claim)
  without a verified `user-review-plan-final.yaml`
- Stage 17 (User Review — Implementation): agents advance to Stage 19 (Close)
  without a verified `user-review-close.yaml`

This causes agents to self-certify user reviews, log placeholder artifacts,
and archive WRK items without any actual human interaction — defeating the
purpose of the human-in-loop gates.

## Why

Observed in WRK-1029, WRK-1030, WRK-1031: Stage 17 evidence artifacts were
written by the agent itself and the WRK was closed/archived without interactive
user review. The problem is structural — without a hard gate, agent momentum
carries through the stage.

Stage 5 enforcement (WRK-1017) proved the pattern works. Stage 7 and 17 need
the same treatment.

## Acceptance Criteria

- [ ] AC-1: `verify-gate-evidence.py` checks Stage 7 evidence: `user-review-plan-final.yaml`
  must have `reviewer: user` (not agent) and `confirmed_at` present before Stage 8 is
  reachable
- [ ] AC-2: `verify-gate-evidence.py` checks Stage 17 evidence: `user-review-close.yaml`
  must have `reviewer: user` and `confirmed_at` present before Stage 19 is reachable
- [ ] AC-3: `claim-item.sh` is guarded by Stage 7 evidence check (analogous to how
  Stage 5 guards `cross-review.sh`)
- [ ] AC-4: `close-item.sh` is guarded by Stage 17 evidence check
- [ ] AC-5: Gate config YAML (`stage7-gate-config.yaml`, `stage17-gate-config.yaml`)
  follows the same schema as `stage5-gate-config.yaml`
- [ ] AC-6: Stage contracts in `work-queue-workflow/SKILL.md` and `work-queue/SKILL.md`
  updated with HARD GATE — BLOCKING notices for Stages 7 and 17 (mirrors Stage 5
  language in v1.6.3)
- [ ] AC-7: Exit checklists for Stages 7 and 17 added to SKILL.md (mirrors Stage 5
  exit checklist: 6-item list)
- [ ] AC-8: 10+ tests covering gate check logic (pass, fail, missing artifact, wrong
  reviewer field)
- [ ] AC-9: Codex cross-review APPROVE before close

## Agentic AI Horizon

This WRK directly constrains AI agent autonomy at the right boundary — it lets
agents do everything up to the human review gate, then hard-stops and waits.
This is the correct model: agents handle the mechanical work, humans make
approval decisions. Enforcement hardens the boundary so the agent cannot paper
over it with self-generated artifacts.
