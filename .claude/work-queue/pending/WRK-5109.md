## Background

Four discrepancies found during WRK-5104 (#1252) and WRK-5106 (#1254) live sessions,
confirmed again during WRK-5100 (#1257). Deferred from WRK-5100 Section F.

## Scope

1. **Stage summaries missing**: `exit_stage.py` `_post_stage_comment()` posts bare
   "Stage N — DONE" with no content. Enhance to include evidence-based summary.

2. **Approval UX too verbose**: Agent re-asks confirmation after approval already given.
   Approval detection in `wait-for-approval.sh` requires approval *after* AWAITING marker.
   Should also accept approvals that predate the marker within a time window.

3. **No human gate in execute group**: Stages 8-10 run without pause after stage 7
   approval. Evaluate whether a mid-execution human checkpoint is needed.

4. **Confusing author aliases**: All GitHub issue comments appear under `vamseeachanta` —
   no way to distinguish human vs AI agent comments. Add agent attribution prefix to
   bot-posted comments in `update-github-issue.py`.

- Lifecycle HTML purge (done in WRK-5100)
