# Local Data Strategy — Gitignore BSEE Bins, Keep Repo Lean

## What
Keep 2.1 GB of BSEE binary data available for local analysis while keeping the git repo lean. After WRK-098 cleaned git history to 177 MB, the parallel refresh script regenerated all 135 .bin files (2.1 GB) locally. These must not be committed to git.

## Why
Committing 2.1 GB of pickle files would undo the WRK-098 git cleanup work. The data is freely downloadable from data.bsee.gov and can be regenerated in ~8 minutes via `refresh_bsee_all.py`. The repo should track code and structure, not bulk data.

## Plan

1. **Gitignore all BSEE .bin files** — add `data/modules/bsee/bin/**/*.bin` to `.gitignore`, remove tracked .bin files from git index (`git rm --cached`)
2. **Keep directory structure** — add a `.gitkeep` per bin subdirectory so git tracks the 30 directories
3. **Canonical data population** — `scripts/refresh_bsee_all.py` is the single way to populate data; document in project README
4. **Loader graceful degradation** — update all BSEE bin loaders to detect missing data and print actionable message: "Run `python3 scripts/refresh_bsee_all.py` to download BSEE data"
5. **Setup target** — add a Makefile or `scripts/setup_data.sh` for first-time data population
6. **Document pattern** — write `docs/data/LOCAL_DATA_PATTERN.md` describing the local-data-only approach for future modules

## Acceptance Criteria
- [ ] Zero `.bin` files tracked in git (verified via `git ls-files '*.bin'`)
- [ ] All 30 bin subdirectories preserved via `.gitkeep`
- [ ] `refresh_bsee_all.py --dry-run` works on fresh clone
- [ ] Loaders return empty DataFrame + warning when bins missing (not crash)
- [ ] Setup target documented in README
- [ ] Pattern documented for reuse by other modules
