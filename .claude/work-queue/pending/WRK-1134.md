## Mission

Decide whether uninitialized git submodules in workspace-hub should be initialised or
removed; take the appropriate action so no ghost empty directories remain.

## What

Many directories in workspace-hub root show as size-0 (empty) because they are registered
in `.gitmodules` but have never been cloned via `git submodule update --init`.

Known empty submodules (from `.gitmodules`):
- CAD-DEVELOPMENTS (https://github.com/bakkiprasad5669/CAD-DEVELOPMENTS)
- OGManufacturing, aceengineer-admin, aceengineer-website, achantas-data, achantas-media
- acma-projects, assethold, assetutilities, client_projects, digitalmodel
- doris, frontierdeepwater, hobbies, investments, pdf-large-reader, pyproject-starter
- rock-oil-field, sabithaandkrishnaestates, saipem, sd-work, seanation
- teamresumes, worldenergydata

## Why

Empty submodule directories are confusing (appear as folders but contain nothing) and
may indicate stale references to repos that no longer exist or are irrelevant.

## Acceptance Criteria

- [ ] Each submodule classified: keep (init it) or remove (deinit + delete dir)
- [ ] All "keep" submodules initialised and non-empty
- [ ] All "remove" submodules deinited from `.gitmodules` and dirs gone
- [ ] `git status` clean after changes
- [ ] No size-0 mystery directories remain in workspace-hub root

## Plan

1. For each submodule in `.gitmodules`, check if the remote URL is accessible
2. Classify: active repos → `git submodule update --init <path>`; dead/irrelevant → `git submodule deinit <path> && git rm <path>`
3. Commit changes with `chore(WRK-1134): init or remove stale submodules`
4. Verify clean state
