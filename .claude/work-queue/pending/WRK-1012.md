# chore(plugins): install and verify official Claude Code plugins on all 3 workstations

## What

Install 9 official Claude Code plugins (from `anthropics/claude-plugins-official`) on all
3 workstations and verify they load correctly. The plugins are already declared in
`.claude/settings.json` (`enabledPlugins` + `extraKnownMarketplaces`) and installed on
`ace-linux-1`. Remaining work is pulling workspace-hub on ace-linux-2 and acma-ansys05,
running the install command for each plugin, and confirming `claude plugin list` shows all
9 active.

## Why

Plugins are committed to `.claude/settings.json` so the full team benefits from consistent
tooling across every workstation. Without installing on each machine, ace-linux-2 and
acma-ansys05 will see the `enabledPlugins` entries but fail to load them (missing cache).

## Plugins to install (same set on all workstations)

| Plugin | Purpose |
|--------|---------|
| `frontend-design` | Production-grade HTML/CSS — supports WRK-1011 workflow-html skill |
| `skill-creator` | Create, improve, and eval skills |
| `playground` | Interactive HTML prototype explorer |
| `code-review` | Multi-agent automated PR review with confidence scoring |
| `pr-review-toolkit` | Specialized PR review agents (tests, error handling, type design, simplification) |
| `feature-dev` | 7-phase guided feature workflow |
| `pyright-lsp` | Python type intelligence for digitalmodel / assetutilities / worldenergydata |
| `claude-md-management` | CLAUDE.md quality audit + session learning capture |
| `hookify` | Create workflow enforcement hooks |

## Install command (run on each workstation)

```bash
# 1. Pull latest workspace-hub (picks up enabledPlugins + extraKnownMarketplaces)
cd /path/to/workspace-hub && git pull

# 2. Install plugins
for p in frontend-design skill-creator code-review pr-review-toolkit feature-dev playground pyright-lsp claude-md-management hookify; do
  claude plugin install "$p"
done

# 3. Verify
claude plugin list
```
