## Mission

Migrate agent-written structured data files from Markdown to YAML or JSON where the data is tabular/structured and agents are the primary writers — reducing LLM corruption risk.

## What

Ecosystem audit identified these files as migration candidates:

| File | Current | Target | Reason |
|------|---------|--------|--------|
| `ac-test-matrix.md` (Stage 12) | Markdown table | **JSON** | Agents corrupt MD tables; JSON `passes` boolean is atomic |
| `test-results.md` | Markdown table | **YAML** (list) | PASS/FAIL per test is structured; YAML list is append-safe |
| `cross-review results/*.md` | Prose + counts | **YAML** with prose field | Verdict, severity counts should be structured; narrative stays as string |

Files that stay Markdown (prose-heavy, human-reviewed):
- Plan specs, WRK item bodies, review narratives, INDEX.md (auto-generated)

## Why

Anthropic's research: "JSON format preferred over Markdown — models less likely to corrupt it." Our ecosystem already uses YAML for most evidence files but has 3 structural holdouts in Markdown where agents write tabular data that gets silently reformatted, rows deleted, or columns misaligned.

## Acceptance Criteria

- [ ] ac-test-matrix migrated to JSON (ac-test-matrix.json) with backward-compat in Stage 12/14 scripts
- [ ] test-results migrated to YAML (test-results.yaml) with list-of-dicts schema
- [ ] cross-review results use YAML for structured fields (verdict, counts, deferred findings)
- [ ] Existing gate scripts (verify-gate-evidence.py) updated to read new formats
- [ ] Migration script for converting existing WRK assets (best-effort, non-destructive)
- [ ] Format decision rule added to coding-style.md or patterns.md
