## Mission

Investigate and resolve the mismatch between documented `/work` command behavior and the Codex adapter implementation so that Codex and Claude do not diverge on basic work-queue entrypoints.

## What

Current investigation confirms:

- [`CODEX.md`](../../../CODEX.md) routes Codex through [`scripts/agents/work.sh`](../../../scripts/agents/work.sh).
- [`scripts/agents/work.sh`](../../../scripts/agents/work.sh) currently supports only:
  - `run`
  - `list`
  - `approve-batch`
  - `next`
  - `status`
- Work-queue documentation and skill references still advertise broader `/work` behavior, including:
  - `/work add <desc>`
  - `/work status WRK-NNN`
  - `/work archive WRK-NNN`
  - `/work report`
  - `/work prioritize`

## Evidence

- [`.claude/work-queue/process.md`](../process.md) documents `/work add <desc>`.
- [`.claude/docs/command-registry.md`](../../docs/command-registry.md) registers `/work` as the work-item capture and processing pipeline.
- [`.claude/skills/coordination/workspace/work-queue/SKILL.md`](../../skills/coordination/workspace/work-queue/SKILL.md) shows `/work add <description>` in Quick Start.
- [`.claude/skills/coordination/workspace/work-queue/command-interface/SKILL.md`](../../skills/coordination/workspace/work-queue/command-interface/SKILL.md) lists `/work add`, `/work archive`, and `/work report`.
- [`scripts/agents/work.sh`](../../../scripts/agents/work.sh) rejects unsupported subcommands and lists only the reduced Codex surface.

## Expected Outcome

One of these must be made true:

1. Codex `/work` wrapper reaches feature parity with the documented `/work` command interface.
2. The docs and skills are narrowed so they describe the actual provider-specific behavior without ambiguity.

## Acceptance Criteria

- [ ] Root cause documented: why Claude accepts `/work add` while Codex does not.
- [ ] Decide whether parity or doc correction is the intended fix.
- [ ] If parity is intended, implement Codex support for the missing documented `/work` subcommands or route them to canonical scripts.
- [ ] If doc correction is intended, update all user-facing `/work` references to distinguish Claude-only behavior from shared wrapper behavior.
- [ ] Regenerate queue indexes after any queue mutation.
