# Catalog and Port ANSYS .inp Scripts — 1,070 Files

## What

1,070 ANSYS APDL `.inp` files indexed across `/mnt/ace/docs/disciplines/` and
`/mnt/remote/ace-linux-2/dde/documents/` implement structural FEA analyses.
Catalog them, document what each group computes, and port boundary conditions,
load cases, and result extraction logic to Python/digitalmodel.

## Phase 2 — Port: Casing FEA to OGManufacturing (42 files)

Key casing `.inp` files: `OUTERCASING01.INP`, `INNERCASING01.INP`, `FULLRISER.INP`
These compute burst/collapse/tension for drilling casing strings.

Port to: `OGManufacturing/src/ogmanufacturing/drilling/casing/fea_capacity.py`
- Extract boundary conditions, material properties, load cases from `.inp`
- Rewrite as parametric Python functions (not FEA — analytical capacity checks)
- Standards: API TR 5C3 (burst/collapse), ISO 10400 (equivalent)

## Phase 3 — Port: SCF Analysis (2 files)

`commandOBJ_SCF_WT1125_2.inp` — Stress Concentration Factor for wall thickness
transition. Implements DNVGL-RP-C203 Appendix C parametric SCF equations.

Port to: `digitalmodel/src/digitalmodel/fatigue/scf_weld.py`
- Parametric SCF for girth welds (misalignment, thickness change)
- Inputs: OD, WT_1, WT_2, weld type, joint classification
- Output: SCF_m (membrane), SCF_b (bending)

## Phase 4 — Port: Riser FEA Load Cases (key subset, ~20 files)

Extract canonical load case structure from `FULLRISER.INP` and parametric variants:
- Storm (100yr, 1yr); installation (HRT, lowering, deployment)
- Document as `specs/analysis/riser-fea-load-cases.yaml`
- Port load application to `digitalmodel.structural.riser_fea.load_cases`

## Phase 5 — Port: Pile/Foundation (76 files)

Pile lateral/axial capacity from the `Piles/` group.
Port to: `digitalmodel/src/digitalmodel/structural/foundations/pile_capacity.py`
Standard: API RP 2A-WSD Section 6 (pile axial + lateral capacity)
