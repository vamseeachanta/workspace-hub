# Set up remote desktop access on dev-secondary

## What
Install and configure a remote desktop solution on dev-secondary so the machine can be accessed via GUI without physical KVM presence. Candidate options: x11vnc (share live session), xrdp (RDP protocol), or NoMachine.

## Why
KVM display drops when switching away (WRK-307). Remote desktop provides an alternative access path and reduces dependency on physical KVM switch for GUI work on dev-secondary.

## Acceptance Criteria
- [x] Remote desktop solution installed and running on dev-secondary
- [x] Can connect from dev-primary (LAN) — SSH tunnel + xtigervncviewer verified
- [~] Connection works over Tailscale (10.1.0.2) — requires SSH tunnel (x11vnc is localhost-bound); direct bind not configured
- [x] Service auto-starts on boot — `~/.config/autostart/x11vnc.desktop` present
- [x] Connection method documented in `specs/modules/hardware-inventory/aceengineer-02.md`

## Notes
- dev-secondary already has an Xorg session on `:1` (confirmed) — x11vnc is lowest-friction option
- Tailscale already configured (10.1.0.2/32)
- Related: WRK-307 (KVM display loss — EDID dongle fix)

Verified live (SSH to dev-secondary):
- x11vnc running: PID 244665, `x11vnc -display :1 -auth /run/user/1000/gdm/Xauthority -forever -bg -nopw -listen localhost -rfbport 5900`
- Port 5900 confirmed LISTEN on 127.0.0.1 and [::] (localhost-bound)
- Autostart file present: `~/.config/autostart/x11vnc.desktop`

Note: x11vnc binds to localhost only (-listen localhost flag). Both LAN and
Tailscale access require an SSH tunnel. Direct Tailscale bind would require
changing the x11vnc startup flags to bind on the tailscale0 interface.
