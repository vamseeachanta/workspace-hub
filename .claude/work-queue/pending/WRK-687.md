# arch(comp-learning): per-workstation analysis with ace-linux-1 final compilation

## Problem

The current `comprehensive-learning` skill has a hard single-machine guard that limits
the full pipeline to ace-linux-1 only. With `logs/orchestrator/` now established as the
unified per-machine log directory (WRK-683), every workstation should run its own Phase 1
analysis against its local logs, commit derived state, and let ace-linux-1 do the final
cross-machine compilation.

## Design

Each workstation runs its own comprehensive-learning analysis:

```
ace-linux-1              ace-linux-2              acma-ansys05 (Windows)
logs/orchestrator/       logs/orchestrator/       logs/orchestrator/
├── claude/*.jsonl       ├── claude/*.jsonl       ├── claude/*.jsonl (if used)
├── codex/*.log          ├── codex/*.log          ├── codex/*.log
└── gemini/*.log         └── gemini/*.log         └── gemini/*.log
        │                        │                        │
        ▼ comp-learning          ▼ comp-learning          ▼ comp-learning
  session-signals/         session-signals/         session-signals/
  candidates/              candidates/              candidates/
  skill-scores.yaml        skill-scores.yaml        skill-scores.yaml
        │                        │                        │
        └────────── git commit ──┴────────────────────────┘
                          │
                git pull (ace-linux-1)
                final compilation pass
                cross-machine report
```

## Deliverables

### 1. `comprehensive-learning/SKILL.md` (EDIT)
- Remove hard `hostname != ace-linux-1` guard from Phase 1 (raw log analysis)
- Phase 1 sources: add `logs/orchestrator/` as a read path alongside `session-signals/`
- Phases 1–9: run on any machine against local data
- New Phase 10a (ace-linux-1 only): `git pull` → aggregate all machines' committed state
  → write cross-machine compilation report
- Keep existing Phase 10 (report) as the final step after compilation

### 2. Per-machine commit step (EDIT — in SKILL.md)
- After Phases 1–9, always commit derived state:
  `session-signals/`, `candidates/`, `corrections/`, `patterns/`, `skill-scores.yaml`
- Runs on all three machines (not ace-linux-1 only)

### 3. `workstations` SKILL.md routing table (EDIT)
- Update `cron_variants` for each machine to include `comprehensive-learning` as a
  per-machine nightly step
- acma-ansys05: note Windows Task Scheduler equivalent; logs/orchestrator/ path may
  differ from Linux convention

### 4. `logs/orchestrator/README.md` (EDIT)
- Add section documenting the per-machine analysis flow and compilation step
