## Mission

Sample the 426K text-extractable documents to measure deep extraction yield — what
fraction contain tables, charts, equations, constants, and calculation procedures?
This informs WRK-1245 Phase 2 prioritization and parallelization strategy.

## Why

We know 9 hand-picked standards yielded 664 tables + 1,153 charts. But we don't know
the yield rate for the broader corpus (project docs, misc standards, spreadsheets).
Without this data we're guessing whether batch deep extraction on 332K PDFs is worth
the ~92 compute hours.

## Plan

### Step 1: Stratified sample (500 docs)

Sample 100 docs from each source with text content:
- ace_standards PDFs (100)
- ace_project PDFs (100)
- ace_project XLSX/XLS (100)
- dde_project PDFs (100)
- dde_project DOCX/DOC (100)

### Step 2: Run deep extraction on sample

```bash
uv run --no-project python scripts/data/doc-intelligence/deep-extract.py \
    --input <path> --domain general --report
```

Collect per-doc: tables_count, charts_count, pages, file_size.

### Step 3: Compute yield metrics

For each source × extension:
- % of docs with ≥1 table
- % of docs with ≥5 tables (data-rich)
- % of docs with ≥1 chart/figure
- Mean tables per doc
- Correlation: file_size vs table_count (estimate yield without extraction)

### Step 4: Extrapolate to full corpus

Project total extractable tables/charts across 426K docs. Identify high-yield
segments (e.g., "ace_standards PDFs >1MB have 85% table hit rate") to prioritize
batch extraction.

### Step 5: Write assessment script

Create `scripts/data/document-index/assess-deep-extraction-yield.py`:
- Takes sample size, source filter, extension filter
- Runs deep-extract on sample
- Outputs YAML report with yield metrics
- Deterministic (runs once, captures baseline)

## Acceptance Criteria

## Results Summary

**700 docs sampled across 7 strata (30 min run)**

| Stratum | Pop | Tables% | Figures% | Machine% | OCR% |
|---------|-----|---------|----------|----------|------|
| ace_project_xls | 11,741 | 93% | 0% | 100% | 0% |
| ace_project_pdf | 158,557 | 80% | 9% | 64% | 30% |
| dde_project_pdf | 169,762 | 69% | 15% | 65% | 27% |
| ace_standards >1MB | 1,899 | 53% | 52% | 77% | 16% |
| dde_project_doc | 25,902 | 26% | 0% | 100% | 0% |
| ace_standards (all) | 54,655 | 1% | 1% | 98% | 1% |
| og_standards | 27,504 | 0% | 0% | 100%* | 0% |

*og_standards: 0/100 extracted OK — likely DRM-protected stubs

**Extrapolation**: ~2.88M tables, ~1.1M figures across full corpus
**Phase 2 OCR**: ~94K docs need OCR before text extraction (27-30% of project PDFs)
