# Complexity Tier → Model Routing

## Mission
**Extend `task_classifier.sh` to output a complexity tier (routine|standard|complex) so routine tasks auto-route to Haiku, reducing token costs without quality loss.**

## What
- Add `complexity_tier` output to `scripts/coordination/routing/lib/task_classifier.sh`
- `routine` = single-file edits, doc updates, config tweaks, data formatting
- Wire into `agent-usage-optimizer`: routine → Haiku, standard → Sonnet, complex → Opus
- Anthropic research shows 67% exposure for data-entry tasks; cheap models handle these equivalently

## Why
Routing routine tasks to Haiku cuts costs without quality loss. Direct application of Anthropic's finding that routine/entry-level work is the most AI-exposed.

## Acceptance Criteria
- [ ] `task_classifier.sh` outputs `complexity_tier=routine|standard|complex`
- [ ] Routine keywords: format, config, rename, copy, update-doc, scaffold
- [ ] Complex keywords: architecture, cross-repo, security, design
- [ ] `agent-usage-optimizer` SKILL.md updated with tier → model mapping table
- [ ] ≥3 TDD tests (routine match, complex match, default standard)
- [ ] Documented in `scripts/coordination/routing/README.md`
