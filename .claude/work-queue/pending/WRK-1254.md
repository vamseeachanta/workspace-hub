## Mission

Fix 27,504 og_standards index entries whose paths point to the removed
`/mnt/ace/0000 O&G/0000 Codes & Standards/` directory. WRK-1246 showed 0/100
extraction success for this source — all files missing.

## Why

The og_standards source contains 27K engineering standards (API, DNV, ASTM, BSI).
These are high-value documents for methodology extraction (Phase 3 of WRK-1245).
The directory was restructured but the index was never updated.

## Context

- Old path: `/mnt/ace/0000 O&G/0000 Codes & Standards/`
- Known new locations: `/mnt/ace/O&G-Standards/`, `/mnt/ace/docs/_standards/`
- `ace_standards` source (54K docs at `/mnt/ace/docs/_standards/`) works fine
- og_standards and ace_standards have 0 overlap by org/doc_number

## Plan

1. Scan the actual filesystem to find where og_standards files moved
2. Build a path mapping table (old → new)
3. Update `index.jsonl` with remapped paths using `phase-e2-remap.py` pattern
4. Verify ≥90% of remapped paths resolve to existing files
5. Re-run yield assessment on og_standards to confirm extraction works

## Acceptance Criteria

1. [ ] Path mapping built: old og_standards paths → current locations
2. [ ] index.jsonl updated with remapped paths
3. [ ] ≥90% of og_standards entries resolve to existing files
