# fix(claim-item): allow resume of already-claimed WRK via /work run WRK-NNN

## Mission
Fix `claim-item.sh` so that resuming an already-claimed WRK (in `working/`) via
`/work run WRK-NNN` succeeds instead of hard-failing with exit 1.

## What

**Bug observed 2026-03-10**: Running `/work run WRK-1102` fails because WRK-1102 is
already in `working/`. `claim-item.sh` has a hard guard added by WRK-1049 that exits 1
if the item is in `working/`, but this also blocks legitimate resume in a new session.

**Root cause (two parts)**:
1. `scripts/work-queue/claim-item.sh` — the `working/` guard is unconditional; no
   distinction between concurrent-collision (bad) and resume (good).
2. `.claude/skills/coordination/workspace/work-queue/actions/process.md` Step 4 — calls
   `claim-item.sh` even when resuming an explicitly named WRK-NNN already in `working/`.

## Why

`/work run WRK-NNN` is the standard resume path. Blocking it breaks every cross-session
WRK continuation and forces the user to manually bypass or understand internal queue state.

## Fix

**Fix 1 — `claim-item.sh`**: when item is already in `working/` AND
`evidence/activation.yaml` exists with a non-empty `session_id`, exit 0 with
`"⚠ WRK-NNN already claimed — resuming"` instead of exit 1.

**Fix 2 — `process.md` Step 4**: add a pre-check — if the explicit WRK-NNN argument is
already in `working/`, skip Step 4 (claim) and jump to resume (load checkpoint, continue
from last stage). Only invoke `claim-item.sh` for items in `pending/`.

## Acceptance Criteria

- [ ] `claim-item.sh WRK-NNN` exits 0 with resume message when item is in `working/`
      and `evidence/activation.yaml` contains a non-empty `session_id`
- [ ] `claim-item.sh WRK-NNN` still exits 1 when item is in `working/` but
      `activation.yaml` is missing or `session_id` is empty (collision guard preserved)
- [ ] `process.md` Step 4 documents the skip-if-working pre-check
- [ ] Running `/work run WRK-1102` no longer fails for an already-claimed item
- [ ] Existing `test-claim-collision.sh` tests still pass
