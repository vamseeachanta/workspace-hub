## Mission
Give every AI agent an instant orientation to any tier-1 repo through a single
machine-readable file — eliminating the 10-minute codebase exploration that opens
every engineering session.

## What
- File per repo: `AGENTS.md` at repo root (≤20 lines per agent-file rules) with:
  - Purpose (one sentence)
  - Primary entry points: key modules, classes, CLI commands
  - Test entry point: exact pytest command
  - Key dependencies (external + internal repo deps)
  - Link to deeper docs (if any)
- Companion: `config/onboarding/repo-map.yaml` at hub level — machine-readable
  aggregation of all repos with: name, path, purpose, test_command, primary_modules[],
  depends_on[], maturity (alpha/beta/stable)
- Generator script: `scripts/onboarding/generate-repo-map.py` — reads each AGENTS.md
  and pyproject.toml, writes repo-map.yaml; run after any structural change
- Session-start integration: load repo-map.yaml context at session start for any
  WRK whose target_repos field is set

## Why
The top time-sink at Stage 10 execution is codebase orientation. An agent spending
10 minutes reading directory trees and import chains before writing a single line of
code is a directly measurable inefficiency. A committed, curated map eliminates it.

## Acceptance Criteria
- [ ] `AGENTS.md` created for all 5 tier-1 repos (≤20 lines each)
- [ ] `config/onboarding/repo-map.yaml` generated and committed
- [ ] `generate-repo-map.py` produces valid yaml from AGENTS.md + pyproject.toml
- [ ] Session-start loads repo-map context when target_repos is set in active WRK
- [ ] Cross-review (Codex) passes
