## Mission
Build a data center electrical topology modeling module at
`digitalmodel/power/datacenter/topology.py` covering UPS, STS, and N+1/2N redundancy schemes.

## What
- RedundancyScheme enum: N, N_PLUS_1, TWO_N, TWO_N_PLUS_1
- UPSModule: rated kVA, battery runtime, efficiency curve, bypass logic
- STSModule: transfer time, preferred/alternate source, retransfer logic
- DataCenterPowerPath: cascaded power chain (utility → MV switchgear → transformer → UPS → STS → PDU → rack)
- TopologyValidator: verify redundancy level, single-point-of-failure detection, uptime tier classification (Tier I–IV per Uptime Institute)

## Why
Data center topology is a critical gap — "not mentioned anywhere" in the resume. This module
provides concrete evidence of familiarity with mission-critical power distribution architectures,
UPS/STS systems, and redundancy analysis — all explicit JD requirements.

## Acceptance Criteria
- [ ] RedundancyScheme + UPSModule + STSModule data models — ≥8 tests
- [ ] DataCenterPowerPath with cascaded power chain modeling — ≥8 tests
- [ ] TopologyValidator with SPOF detection and tier classification — ≥10 tests
- [ ] Example configurations for Tier II, III, IV topologies — ≥3 tests
- [ ] Module importable from `digitalmodel.power.datacenter`
