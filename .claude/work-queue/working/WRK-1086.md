## Mission
Generate a correctly structured, test-ready new module skeleton in any tier-1 repo
with one command — eliminating the copy-paste-and-adapt boilerplate that starts
every engineering module WRK.

## What
- Script: `scripts/scaffolding/new-module.sh <repo> <module-name> [--domain structural|marine|energy|generic]`
  - Creates: `src/<repo>/<module-name>/__init__.py`, `<module-name>.py` (class stub),
    `tests/test_<module-name>.py` (pytest skeleton with one failing test),
    `docs/<module-name>.md` (purpose stub)
  - Applies correct import structure per repo (PYTHONPATH, `from __future__ import annotations`)
  - Adds `py.typed` if repo has PEP 561 (after WRK-1079)
  - Domain flag pre-fills: structural → DNV/ABS code reference stubs;
    marine → hydrodynamic assumption stubs; energy → regulatory reference stubs
- Template source: `scripts/scaffolding/templates/<domain>/` — Jinja2-rendered
- Post-create: prints next steps (run test → see red → implement)

## Why
Every new engineering module (CP, structural, geotechnical, EIA parser) starts with
the same boilerplate: `__init__.py`, class with `__init__(self, cfg)`, pytest file,
docstring. Agents currently write this from scratch or copy-adapt from another module
— inconsistently. Scaffolding encodes the correct pattern once.

## Plan

Route A — simple. No external spec. Inline plan.

1. **Create `scripts/scaffolding/` directory** with `new-module.sh` (bash) and `templates/` subdirs (`generic/`, `structural/`, `marine/`, `energy/`)
2. **Write Python template renderer** (`scripts/scaffolding/render_template.py`) that replaces `{{MODULE}}`, `{{REPO}}`, `{{DOMAIN_STUBS}}` placeholders — avoids Jinja2 dep; plain `str.replace`
3. **Templates per domain**: `generic/module.py.tmpl` (class stub + `__init__(self, cfg)`), `module_init.py.tmpl`, `test_module.py.tmpl` (1 failing test), `docs_module.md.tmpl`; structural/marine/energy add domain-specific docstring stubs
4. **`new-module.sh`**: validates repo arg against 5 known repos, resolves correct `src/<repo>/` path, calls renderer, runs `ruff check --fix` + `mypy` on generated files, prints TDD next-steps banner
5. **TDD**: write test script `scripts/scaffolding/tests/test_new_module.sh` that calls `new-module.sh` in a temp dir and asserts 4 artefacts exist + `grep` the failing test marker; 5 cases (1 per domain + assetutilities default)

## Acceptance Criteria

- [ ] Generated code passes ruff and mypy immediately (no pre-existing violations)
