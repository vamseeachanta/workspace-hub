# Nomenclature canonicalisation — eliminate session/stage/phase confusion

## Mission
**Audit all work-queue scripts, skills, and artifact field names and replace non-canonical use of session/stage/phase/step with the definitions established in WRK-1035.**

## What

Canonical definitions (WRK-1035 thought 12):
- WRK session: single Claude conversation bounded by /clear or context reset
- WRK stage: one of the 20 numbered lifecycle stages
- Phase: sub-unit within a stage
- Step: numbered action within a phase
- Checkpoint: snapshot artifact written at end of a WRK session
- Resume: loading a checkpoint into a new session

Audit scope: scripts/work-queue/, work-queue/SKILL.md, work-queue-workflow/SKILL.md,
workflow-gatepass/SKILL.md, evidence artifact field names, slash commands.

## Why

Terminology confusion is a root cause of agent compliance failures.

- [ ] Artifact field names audited; breaking changes documented
