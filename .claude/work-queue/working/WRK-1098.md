## Mission

Create a single authoritative skill that makes repo ecosystem terminology correct and
consistent every time — for both human users and coding assistants (Claude, Codex,
Gemini).

## What

A `workspace-hub:ecosystem-terminology` skill (and matching AGENTS.md/CODEX.md/GEMINI.md
snippet) that defines:

- **Canonical repo names** and their short aliases (e.g. `assetutilities` not
  `asset_utilities`, `asset-utilities`, or `AssetUtilities`)
- **Module/package naming** within each repo
- **Relationship vocabulary** — what "tier-1 repo", "submodule", "hub", "harness"
  mean in this ecosystem
- **File/directory names** that are easily confused (e.g. `work-queue` vs
  `work_queue`, `SKILL.md` vs `skill.md`)
- **Acronyms and abbreviations** in consistent expanded form (e.g. WRK, GTM, CP, VIV)
- **People/machine names** — `ace-linux-1`, `ace-linux-2`, `acma-ansys05`; never
  `ace1`, `linux-1`, etc.
- **Do-not-use list** — deprecated names, client codenames, incorrect pluralisations

## Why

- Coding assistants regularly hallucinate repo/module names or use inconsistent
  capitalisation, leading to broken imports and confusing commits.
- Humans onboarding to the ecosystem have no single reference — terminology is
  scattered across CLAUDE.md, MEMORY.md, and various docs.
- A skill that is always loaded (or easily invokable) eliminates both problems at
  source.

## Acceptance Criteria

- [ ] `.claude/skills/workspace-hub/ecosystem-terminology/SKILL.md` created with
  canonical name tables, relationship vocabulary, and do-not-use list
- [ ] Skill registered in `.claude/commands/workspace-hub/` for `/ecosystem-terminology`
  slash command discoverability
- [ ] Cross-referenced from CLAUDE.md, AGENTS.md onboarding section
- [ ] Legal scan passes (no client identifiers in examples)
- [ ] At least one coding assistant (Codex or Gemini) cross-reviews the terminology
  table for completeness
