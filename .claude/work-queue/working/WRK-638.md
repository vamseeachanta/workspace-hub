# WRK-638 — Memory Quality Eval Harness

## Mission

Implement read-only eval script that measures memory quality metrics before/after compaction.

## Context

Spun off from WRK-637 (memory compaction/curation). compact-memory.py and curate-memory.py
are now in place. This WRK adds a standalone eval script to measure memory quality metrics
before and after compaction runs.

## Plan

- Implement `scripts/memory/eval-memory-quality.py` with 6 metrics: pct_done_wrk,
  pct_stale_paths, signal_density, memory_md_headroom, topic_file_headroom, dedup_candidates
- CLI: `--memory-root`, `--format json|md`, `--check-paths`, `--compare before.json after.json`
- Script is read-only — never modifies memory files
- Write `tests/memory/test_eval_memory_quality.py` with 10 TDD tests (all 6 metrics + compare + read-only guard)

## Acceptance Criteria

- eval-memory-quality.py --memory-root exits 0 and prints JSON report
- All 6 metrics present in output
- --format md flag emits markdown table
- Pre/post comparison: --compare before.json after.json shows delta per metric
- 5+ TDD tests covering each metric and comparison mode
- Script does NOT modify memory files (read-only)

## Out of Scope

- Nightly cron integration (WRK-635)
- MEMORY.md rewrite (future WRK)
