## Mission

Reclassify 101,471 "other" domain records in the 1M-record document index into proper engineering domains, validate accuracy, and update the registry.

## What

Resource intelligence (Stage 2) revealed that AC1 is already met — no null domains exist. The real gap is 101,471 records (9.8%) classified as "other" that likely belong to specific engineering domains. Work focuses on: improving classification rules in phase-e2-remap.py, potentially extending the taxonomy, validating accuracy on ≥50 samples, and updating registry.yaml.

## Why

101k "other" records hide documents from domain-specific extraction pipelines. Proper classification enables targeted document-intelligence workflows per discipline.

## Acceptance Criteria

- [x] AC1: All records in index.jsonl have a non-null `domain` field (ALREADY MET — 0 nulls)
- [x] AC2: Classification accuracy validated on 60 samples via validate-classification.py
- [x] AC3: Updated domain counts published to registry.yaml (14 domains)
- [x] AC4: project-management added to config.yaml taxonomy
- [x] AC5: phase-e2-remap.py serves as incremental reclassification script
- [x] AC6: "other" reduced from 101,471 (9.8%) to 44,705 (4.3%)
