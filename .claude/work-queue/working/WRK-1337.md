## Mission

Enhance the WRK-to-GitHub-Issue pipeline so that the `subcategory` frontmatter field is included as a GitHub label (e.g. `domain:structural-dynamics`, `domain:tax-preparation`). This enables finer-grained filtering on the GitHub project board beyond the coarse `cat:` labels.

## Scope

1. Update `scripts/work-queue/migrate-wrk-to-github.sh` to extract the `subcategory` field and add a `domain:<value>` label
2. Update `scripts/work-queue/archive-item.sh` follow-on issue creation to include `domain:` label
3. Backfill existing GitHub issues with `domain:` labels (batch script or one-time migration)
4. Ensure labels are auto-created on the repo if they don't exist (`gh label create` or `--force`)

## Acceptance Criteria

- [ ] New WRK items with a `subcategory` field get a `domain:<subcategory>` label on their GitHub issue
- [ ] Existing issues are backfilled with the correct domain label
- [ ] Items without a subcategory field skip the domain label gracefully
