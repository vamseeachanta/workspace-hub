# improve(workflow): Add Plan-Mode Ideation Stage to WRK Lifecycle

## What

Add a new Stage 4a ("Plan Mode Ideation") to the WRK 20-stage lifecycle. Before
writing any plan artifacts, the orchestrator enters `EnterPlanMode` to propose the
full plan as text-only. The user can iterate without file side-effects. Once
approved, `ExitPlanMode` is called and Stage 4b proceeds to write the spec + HTML.

## Why

Currently Stage 4 immediately writes `specs/wrk/WRK-NNN/plan.md` and generates
HTML. Iterating on a wrong approach means regenerating artifacts repeatedly. Plan
mode forces the orchestrator to think through the full scope before touching anything,
producing higher quality first-draft plans and reducing wasted artifact regeneration.

The pattern was identified during WRK-1005 execution: the user asked whether
`EnterPlanMode` should be used at the plan stage — this WRK implements the answer.

## Proposed Stage Split

| Sub-stage | Action | Tools |
|-----------|--------|-------|
| 4a: Plan Mode Ideation | `EnterPlanMode` → orchestrator drafts plan as text → `ExitPlanMode` | EnterPlanMode, ExitPlanMode |
| 4b: Plan Artifact Creation | Write `specs/wrk/WRK-NNN/plan.md` → generate plan-draft HTML | Write, Bash |
| 5: User Review - Plan (Draft) | Open HTML in browser, publish to origin | Bash |
