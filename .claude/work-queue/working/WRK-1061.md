## Mission
Auto-generate a structured cross-review input file from the WRK item and its git
diff, so agents stop hand-writing review bundles and the review is always complete.

## What
- Script: `scripts/review/generate-review-input.sh WRK-NNN [--phase N]`
- Reads: WRK frontmatter (mission, ACs, complexity, route), staged/unstaged git diff
  scoped to repos listed in `target_repos`, and last checkpoint summary
- Outputs: `scripts/review/results/wrk-NNN-phase-N-review-input.md` with sections:
  - WRK context (mission, ACs, route)
  - Changed files list + diff (truncated at 300 lines with summary if larger)
  - Test results snapshot (from last run-all-tests.sh output if available)
  - Review focus prompts derived from WRK complexity and subcategory
- After generation, caller can pipe straight into `cross-review.sh`:
  `scripts/review/cross-review.sh results/wrk-NNN-phase-N-review-input.md all`

## Why
Every WRK Route B/C item requires a cross-review input file. Currently agents write
it manually — inconsistent structure, missing context, and takes 5–10 minutes per
phase. Auto-generation makes cross-review fast, consistent, and complete.

## Acceptance Criteria
- [ ] `generate-review-input.sh WRK-NNN` produces a valid input file
- [ ] Output file includes: mission, ACs, diff (scoped), test snapshot, focus prompts
- [ ] Diff truncation works correctly for large diffs (>300 lines)
- [ ] `--phase N` flag included in output filename
- [ ] Integrates with `cross-review.sh` without modification
- [ ] Cross-review (Codex) passes

## Plan

### Step 1 — Write `scripts/review/generate-review-input.sh`
Single bash script, no external deps beyond standard git + coreutils.

**Logic:**
1. Parse args: `WRK-NNN` (required), `--phase N` (default: 1)
2. Find WRK item: check `pending/`, then `working/`; error if not found
3. Extract frontmatter fields via `awk` (between `---` markers):
   - `title`, mission text (first `## Mission` paragraph), ACs (all `- [ ]` / `- [x]` lines),
   - `complexity`, `route`, `subcategory`, `target_repos`
4. Build git diff:
   - If `target_repos` is non-empty: `git diff HEAD -- <repo1> <repo2> ...`
   - Else: `git diff HEAD` (full workspace diff)
   - Count lines; if >300 truncate and append summary: `## Diff truncated at 300/N lines`
5. Read checkpoint summary: `assets/WRK-NNN/checkpoint.yaml` → `context_summary` field
6. Read test snapshot: check `scripts/testing/results/` for most-recent markdown; best-effort
7. Generate focus prompts table based on `subcategory` + `complexity`
8. Write structured markdown to `scripts/review/results/wrk-NNN-phase-N-review-input.md`

**Output sections:**
```
# WRK-NNN Phase N Review Input
## WRK Context  (title, mission, ACs, route, complexity)
## Changed Files + Diff  (scoped diff, truncated if >300 lines)
## Test Snapshot  (last run-all-tests.sh output or "not available")
## Checkpoint Summary  (context_summary from checkpoint.yaml)
## Review Focus  (3-5 prompts tailored to subcategory + complexity)
## Verdict Request
```

### Step 2 — Tests: `tests/testing/test-generate-review-input.sh`
Using the pattern in `scripts/testing/tests/` with fixture files.

**Tests (≥5 cases):**
1. Basic invocation produces output file with correct name `wrk-NNN-phase-1-review-input.md`
2. Output contains all required sections (grep for headings)
3. `--phase 2` produces `wrk-NNN-phase-2-review-input.md`
4. Diff truncation: inject 400-line fake diff, assert output contains truncation notice
5. Missing WRK item: exits non-zero with clear error
6. Empty `target_repos`: falls back to full diff gracefully

### Test Strategy
- Fixtures: small synthetic WRK yaml + fake diff files
- Run: `bash tests/testing/test-generate-review-input.sh`
- Exit 0 = all pass; non-zero = failure with line indication
