# chore(harness): Stage-17 Gate-Evidence Green Checker

## Mission

Create a wrapper script for Stage 17 that calls `verify-gate-evidence.py` and
confirms ALL gates are OK (no MISSING) before allowing close — making the
Stage 17 pre-close check a single deterministic script call.

## What

Script: `scripts/work-queue/check-gates-green.sh WRK-NNN`

- Call `verify-gate-evidence.py WRK-NNN` (close phase)
- Parse output: any `MISSING` line → fail
- Exit 0 if all gates OK or WARN; exit 1 if any MISSING gate found
- Print a summary: `N gates OK, M warnings, P missing`

## Acceptance Criteria

- [ ] Script exits 0 when verify-gate-evidence.py has no MISSING gates
- [ ] Script exits 1 and lists MISSING gates when any present
- [ ] WARN gates do not cause failure (only MISSING does)
- [ ] Integrated into stage-17-user-review-implementation.md checklist

## Plan

- Write `scripts/work-queue/check-gates-green.sh WRK-NNN` (~35 lines):
  1. Call `uv run --no-project python scripts/work-queue/verify-gate-evidence.py "$WRK_ID"` (close phase)
  2. If verify-gate-evidence.py itself fails with exit 2 (infra error) → propagate as exit 2 with message
  3. Capture stdout; count lines matching `: OK `, `: WARN `, `: MISSING `
  4. If any MISSING count > 0 → print summary + list missing gate names → exit 1
  5. Print `N gates OK, M warnings, 0 missing` → exit 0
- TDD: write 3 tests using bash with mock verify-gate-evidence.py (all-OK, MISSING-present, WARN-only)
- Add `check-gates-green.sh WRK-NNN` as pre-close step in `stage-17-user-review-implementation.yaml`
- Note in close-item.sh header that check-gates-green.sh should be run before closing
