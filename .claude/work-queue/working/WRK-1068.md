## Mission
Parse all `blocked_by:` fields in the work queue and render a dependency graph so
the critical path through 490 items is visible at a glance.

## What
- Script: `scripts/work-queue/dep-graph.py` — reads all pending/working WRK
  frontmatter, builds a directed graph of `blocked_by` relationships
- Outputs:
  1. ASCII critical-path table: chain with most items from root → leaf, printed
     to terminal (fast, no dependencies)
  2. DOT file: `scripts/work-queue/dep-graph.dot` (renderable via `dot -Tsvg`)
  3. Unblocked leaf nodes: list of items with no blockers — these are ready to run
- Filters: `--category <name>` to show only one category's graph; `--critical-path`
  to show only the longest chain
- Integrate into `/work list` output as a footer: "N items unblocked, longest chain: X"

## Why
With 490 items and growing blocked_by relationships, sequencing decisions are made
by manually cross-referencing frontmatter. A dep graph surfaces the critical path
instantly and identifies which items to unblock first for maximum downstream throughput.

## Acceptance Criteria
- [ ] `dep-graph.py` reads all pending + working WRK items
- [ ] ASCII critical-path table output (no extra dependencies required)
- [ ] DOT file output for optional SVG rendering
- [ ] `--category` and `--critical-path` filter flags work
- [ ] `/work list` footer shows unblocked count + longest chain length
- [ ] Cross-review (Codex) passes

## Plan

1. **`scripts/work-queue/dep-graph.py`**: polyglot shebang; parse frontmatter from pending/ + working/ + blocked/; reuse `parse_frontmatter` + `normalize` from generate-index.py; build `blocked_by` dict; compute unblocked nodes (empty blocked_by AND status != "blocked"); treat archived blockers as satisfied by scanning BOTH `archive/` AND `archived/`; compute critical path via DP over `graphlib.TopologicalSorter` topological order (chain length = node count); handle `CycleError`; CLI flags: `--category` (cross-category blockers kept as opaque deps — subgraph view only), `--critical-path`, `--dot <path>` (no default file; require explicit flag), `--summary` (always prints, even on empty graph: "0 unblocked, longest chain: 0 nodes").
