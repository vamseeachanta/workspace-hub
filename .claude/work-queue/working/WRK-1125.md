## Mission

Refine `whats-next.sh` output to surface per-item context that helps the user maintain
focus and quickly triage items without re-explaining context each session.

## What

1. **Show `note:` field** for WORKING and IN-PROGRESS UNCLAIMED sections — display as
   a dim annotation line below each item (e.g., `  ↳ needs multi-workstation coordination`)
2. **Show `not_before:` date** for IN-PROGRESS items that have it — helps distinguish
   "punted" from "actively stuck"
3. **Suppress working items that have a `note:` starting with "punted"** from the
   WORKING section header count (they inflate active count misleadingly)
4. **Add a `PARKED` sub-section** under WORKING for items in `working/` that have
   `note:` set — separate from truly-in-flight items

## Why

Session learnings (2026-03-11):
- WRK-1109 showed as IN-PROGRESS UNCLAIMED but is punted to 2026-04-10; user had to
  explain this verbally. `not_before` field exists but is not surfaced.
- WRK-1022 showed as WORKING but is stalled on multi-workstation coordination;
  user had to clarify. `note:` field now added but not displayed.
- Without these signals, user must maintain mental context about why items are parked,
  which defeats the purpose of the dashboard.

## Acceptance Criteria

- [ ] AC1: WORKING section shows `note:` value as a dim indented line under each row
- [ ] AC2: IN-PROGRESS UNCLAIMED shows `not_before:` date when present
- [ ] AC3: Items with `note:` in working/ appear under a `⏸ PARKED` sub-header
- [ ] AC4: `bash scripts/work-queue/whats-next.sh` smoke-test passes with WRK-1022
  showing its note and WRK-1109 showing its not_before date
