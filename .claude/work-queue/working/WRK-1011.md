# feat(work-queue): workflow-html skill — mandatory consistent HTML review artifact

## What

Create a `workflow-html` skill at `.claude/skills/workspace-hub/workflow-html/SKILL.md`
that defines the canonical HTML template, CSS design system, and section structure for
**all** WRK review artifacts (draft plan, final plan, implementation review, close
review). The skill is added to the work-queue workflow harness and made mandatory — every
WRK item must produce review HTML that conforms to this template before progressing
through plan and close gates.

## Why

Current HTML review artifacts are generated ad-hoc. Some use the polished warm-parchment
design (WRK-624 governance-review, WRK-690 weekly summary) while others use a plain,
minimal style (WRK-624 workflow-final-review). There is no single canonical template, so
the user gets inconsistent presentation across hundreds of WRK reviews. A skill-backed
HTML template will:
- give every review the same look and feel regardless of which session generated it
- reduce the cognitive overhead of reviewing by making section positions predictable
- make it easy to diff reviews from different stages (draft vs final vs close)

## Design Reference

**Color/layout canonical source:** `.claude/work-queue/assets/WRK-624/workflow-governance-review.html`
— warm parchment design system (`--bg: #f3efe6`, Georgia serif, card with 18px radius,
teal/amber accent pair, hero gradient header).
