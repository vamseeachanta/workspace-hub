## Mission

Wire automatic browser opening of BOTH lifecycle HTML and plan HTML at human-interactive stages (5, 7, 17). Open once at stage entry; 30s auto-refresh in-browser handles updates thereafter.

## What

- `start_stage.py` generates HTML but never opens it in a browser
- Two HTML files per WRK (v2.3.0 two-file model): `WRK-NNN-lifecycle.html` + `WRK-NNN-plan.html`
- `log-user-review-browser-open.sh` exists and handles cross-platform opening + evidence logging, but is never called automatically
- Micro-skills (stage-05, stage-07, stage-17) say "open HTML" in prose but no script invocation happens

## Why

Users don't see lifecycle or plan HTML during `/work run` — the review gate is blind. Both HTML files auto-refresh every 30s in the browser, but they never get opened in the first place.

## Acceptance Criteria

- [ ] At stages 5, 7, 17 entry: both `WRK-NNN-lifecycle.html` and `WRK-NNN-plan.html` are opened in default browser
- [ ] HTML opens cross-platform: Linux (`xdg-open`), macOS (`open`), Windows (`cmd.exe start`)
- [ ] Evidence written to `assets/WRK-NNN/evidence/user-review-browser-open.yaml` for each file
- [ ] `generate-html-review.py --lifecycle` and `--plan` both run BEFORE the open calls
- [ ] Open once per stage entry only — 30s `<meta http-equiv="refresh">` handles subsequent updates
- [ ] No double-open if user manually opens the file before stage entry
