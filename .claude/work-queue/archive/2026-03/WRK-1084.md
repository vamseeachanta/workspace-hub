## Mission

Add https://skills.sh/ as a canonical reference in skill-learning documentation and wire it into every workflow that assesses, analyzes, learns, or prunes skills.

## What

- Browse https://skills.sh/ and identify skills that can replace, improve, or extend existing workspace-hub skills
- Adopt/adapt relevant skills from skills.sh directly into `.claude/skills/`
- Update skill-learner SKILL.md to list skills.sh as primary external reference during any skill lifecycle operation (assess, analyze, learn, prune, curate)
- MEMORY.md already updated

## Why

skills.sh is a curated skill library. Pulling from it avoids reinventing patterns already proven elsewhere, and using it as a standard reference during skill curation ensures we don't drift from best practices.

## Acceptance Criteria

- [ ] skills.sh browsed; relevant skills identified and noted
- [ ] At least one skill adopted or meaningfully improved from skills.sh patterns
- [ ] skill-learner SKILL.md (or equivalent) lists skills.sh as canonical external reference
- [ ] Any skill assessment/curation workflow (WRK-1009, WRK-1010, WRK-1053) notes skills.sh as input source
- [ ] MEMORY.md entry already written (done 2026-03-09)
