## Mission
Every session start should emit a structured log entry capturing the active model,
context window limit, and effort/thinking level for each provider in use. This makes
session logs self-contained — reviewable without cross-referencing config files.

## What
Extend `scripts/hooks/session-logger.sh` (or the session-start hook) to emit at session
start (and optionally at session end):

```jsonl
{"event":"session_params","provider":"claude","model":"claude-sonnet-4-6","context_k":200,"effort":"thinking=off","ts":"..."}
{"event":"session_params","provider":"codex","model":"gpt-5.4","context_k":128,"effort":"medium","ts":"..."}
```

Sources: same config reads as `ai-usage-summary.sh` Active Provider Config block.
Can reuse the Python logic as a shared helper script (`scripts/ai/session-params.py`).

## Why
When diagnosing why a session took long or produced poor results, the active model/context/
effort at that moment is critical context. Currently reconstructed from config files —
should be captured automatically at session time.
