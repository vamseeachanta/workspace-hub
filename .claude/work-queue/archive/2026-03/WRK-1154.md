# chore(harness): Stage-01 Frontmatter Validator Script

## Mission

Create a deterministic script that validates all required WRK frontmatter fields
are present and non-empty, enabling Stage 1 exit to be a binary check (Level 2
on the enforcement gradient) rather than LLM prose.

## What

Script: `scripts/work-queue/validate-wrk-frontmatter.sh WRK-NNN`

- Read `pending/WRK-NNN.md` or `working/WRK-NNN.md`
- Check all required fields from work-queue SKILL.md are present and non-empty:
  `id`, `title`, `status`, `priority`, `complexity`, `created_at`, `target_repos`,
  `computer`, `plan_workstations`, `execution_workstations`, `category`, `subcategory`
- Exit 0 if all present; exit 1 with missing field list on failure

## Acceptance Criteria

- [x] Script exits 0 for a complete WRK file
- [x] Script exits 1 and names missing fields for an incomplete file
- [x] Integrated into stage-01-capture.md checklist item
