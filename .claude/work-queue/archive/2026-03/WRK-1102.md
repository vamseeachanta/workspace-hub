# fix(comprehensive-learning): Repair the Modular Chain

## Design Principle

The architecture is correct: thin orchestrators call focused scripts, each phase is
independent, best-effort phases don't block mandatory ones. **Do not redesign — repair.**

## Actual Call Graph (confirmed by reading code)

```
2AM  nightly-cron.sh
       └── claude --skill comprehensive-learning   ← spawns LLM session (wrong)
             └── comprehensive-learning.sh
                   ├── Phase 1:  session-analysis.sh    ← naming bug (0 signals found)
                   ├── Phase 1b: detect-drift.sh
                   ├── Phase 2:  daily-reflect.sh        → SKIPPED (file missing)
                   ├── Phase 3:  knowledge-capture.sh    → SKIPPED (file missing)
                   ├── Phase 3:  pipeline.py phase 3
                   ├── Phase 4:  improve.sh              → SKIPPED (file missing)
                   └── Phase 5-9: pipeline.py (636 lines, all phases in one file)

3AM  session-analysis.sh (standalone cron)          ← DUPLICATE of Phase 1 above
```

## Fixes (ordered by value/effort ratio)

### Fix 1 — Signal naming mismatch (15 min, immediate unblock)

`session-analysis.sh` line 69 uses `find … -name "${ANALYSIS_DATE}-*.jsonl"`.
Hooks write `${ANALYSIS_DATE}.jsonl` (no timestamp suffix). 297 real signals/day,
0 detected.

```bash
# Change:
find "$SIGNALS_DIR" -name "${ANALYSIS_DATE}-*.jsonl" -print0
# To (matches both old timestamp-suffix and new consolidated format):
find "$SIGNALS_DIR" \( -name "${ANALYSIS_DATE}-*.jsonl" -o -name "${ANALYSIS_DATE}.jsonl" \) -print0
```

Verify: run `bash scripts/analysis/session-analysis.sh` and confirm `N_SESSIONS > 0`.

### Fix 2 — Remove duplicate 3AM cron (5 min)

`session-analysis.sh` runs at 2AM as Phase 1 of comprehensive-learning AND again at
3AM as a standalone cron. Remove the 3AM entry from crontab. The 2AM run is sufficient.

### Fix 3 — Replace `claude --skill` with direct script call in nightly-cron.sh (10 min)

`nightly-cron.sh` ends with `claude --skill comprehensive-learning` — spawning a full
LLM session with quota spend to run a deterministic shell+python pipeline.

```bash
# Change:
claude --skill comprehensive-learning || _nightly_exit=$?
# To:
bash scripts/learning/comprehensive-learning.sh || _nightly_exit=$?
```

`comprehensive-learning.sh` already exists as the shell implementation. Skills are for
interactive use; nightly cron should call scripts directly (patterns.md: scripts > LLM).

### Fix 4 — Write missing phase scripts as focused modules (medium)

Phases 2, 3, 4 always SKIP because their scripts don't exist. Create them:

| Phase | Script | What it does |
|-------|--------|-------------|
| 2 | `scripts/analysis/daily-reflect.sh` | Summarise yesterday's git commits across all repos; write to `state/reflect-history/YYYY-MM-DD.md` |
| 3 | `scripts/analysis/knowledge-capture.sh` | Extract new domain facts from signal files; append to `state/learned-patterns.json` |
| 4 | `scripts/analysis/improve.sh` | Apply learned patterns to MEMORY.md + skill-scores.yaml; log diffs to `state/improve-changelog.yaml` |

Each script: focused single responsibility, <100 lines, callable standalone, exits 0 on
success. comprehensive-learning.sh already has the call sites — just make the files exist.

### Fix 5 — Split pipeline.py into per-phase modules (medium)

`scripts/analysis/comprehensive_learning_pipeline.py` is 636 lines containing phases 5-9.
Split into focused modules:

```
scripts/analysis/
  phase5_correction_trends.py   (was: pipeline.py phase 5)
  phase6_wrk_feedback.py        (was: pipeline.py phase 6)
  phase7_action_candidates.py   (was: pipeline.py phase 7) ← absorbs WRK-1018
  phase8_report_review.py       (was: pipeline.py phase 8)
  phase9_coverage_audit.py      (was: pipeline.py phase 9)
```

Update `comprehensive-learning.sh` `run_py_phase()` to call each module directly.
Keep `pipeline.py` as a shim that delegates to modules during transition.

### Fix 6 — Extend Phase 1 inputs (small, absorbs WRK-1025 + WRK-636)

`session-analysis.sh` already reads signal files once Fix 1 is applied. Extend it to:

  emit `{"event":"stage_exit","wrk":…,"stage":N,"edits":…,"tool_calls":…}` at Stop time.
  Phase 1 already reads signals — no pipeline changes needed, just a new event type.

- **Codex session ingestion** (WRK-636): Add a pre-Phase-1 step in
  `comprehensive-learning.sh` that converts Codex JSONL to signal format:
  ```bash
  bash scripts/analysis/ingest-codex-sessions.sh  # new focused script, <60 lines
  ```

### Fix 7 — Replace `call_anthropic_api()` in classify.sh with deterministic routing (medium)

`scripts/improve/lib/classify.sh` is self-described as "10% shell, 90% API". It sends up
to 50 signal JSONL lines to the Anthropic REST API every nightly run to decide which file
to update. But signals are **structured** — `event` field determines routing deterministically:

| `event` value | Target file | Action |
|---------------|-------------|--------|
| `session_tool_summary` | `memory/MEMORY.md` | Append WRK pattern note if edits > threshold |
| `skill_invoked` | `state/skill-scores.yaml` | Increment usage count |
| `drift_counts` | `rules/` section | Append violation summary |
| `stage_exit` | `memory/MEMORY.md` | Append stage performance note |
| `context_reset` | `memory/MEMORY.md` | Append session pattern |
| Anti-patterns (string) | `rules/patterns.md` | Append warning |

Rewrite `classify.sh` as pure shell routing by `event` type:
```bash
phase_classify() {
    while IFS= read -r line; do
        event=$(echo "$line" | jq -r '.event // empty')
        case "$event" in
            session_tool_summary) route_to_memory "$line" ;;
            skill_invoked)        route_to_skill_scores "$line" ;;
            drift_counts)         route_to_rules "$line" ;;
            stage_exit)           route_to_memory "$line" ;;
            *)                    true ;;  # unknown events: drop (no LLM fallback)
        esac
    done < "$merged"
}
```

Remove `call_anthropic_api()` and the `curl` dependency entirely. The `--api` flag can
remain as an optional future hook, but the nightly default must be script-only.

**Impact**: eliminates one API call per nightly run, removes OAuth credential dependency
from the pipeline, makes Phase 4 testable without network.
