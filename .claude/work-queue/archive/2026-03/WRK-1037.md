# cross-review hardening: uv readiness tests + Codex-missing user approval gate

## Mission
**Fix two P1 Codex findings from WRK-1008 review: ensure uv is always ready for
renderer use (with background readiness tests), and surface Codex-CLI-missing as a
mandatory user-approval gate instead of a silent fallback.**

## What

**P1-A — uv readiness (`submit-to-codex.sh`)**
`run_renderer()` falls back from `uv` to `python3` only when `uv` is absent — not
when `uv` is present but failing (e.g. cache/permission issue). Fix: uv should always
be ready. Add a background uv readiness pre-check that verifies `uv run` actually
works before the renderer is called; if uv is broken, surface it and fix it rather
than silently falling back. Add tests covering: uv present+working, uv present+broken,
uv absent.

**P1-B — Codex CLI missing (`cross-review.sh`)**
`# Codex CLI not found` is currently `NO_OUTPUT` class → fallback-eligible in `all`
mode → 2-of-3 (Claude+Gemini) can pass the hard gate without Codex. Fix: Codex CLI
missing must surface to the user explicitly and require mandatory user approval to
proceed (not auto-fallback). The user must consciously decide to continue without
Codex, not have the gate silently weakened.

## Why
Cross-review is a hard gate in the WRK lifecycle. Silent fallbacks undermine the
guarantee that Codex has reviewed before merge. uv reliability must be verified, not
assumed. These findings were left unactioned from a Feb 27 Codex review of WRK-1008.

## Acceptance Criteria
- [ ] `run_renderer()` in `submit-to-codex.sh`: pre-check uv works before calling it;
      if uv present but broken → surface error and fix (not silent fallback to python3)
- [ ] Background uv readiness test added (e.g. `scripts/review/tests/test-uv-readiness.sh`)
      covering: uv working (exit 0), uv present but broken (surfaces error), uv absent
- [ ] `cross-review.sh`: Codex CLI not found → print explicit warning to user +
      require `--allow-no-codex` flag or interactive y/n prompt before proceeding in
      `all` mode; never auto-pass via 2-of-3 fallback when Codex is absent
- [ ] Existing tests still pass: `test-submit-to-codex-robustness.sh`,
      `test-codex-fallback.sh`, `test-cross-review-path-sanitization.sh`
- [ ] New test added: `all` mode with Codex CLI missing exits non-zero without
      explicit user approval flag

## Agentic AI Horizon
- Codex is a hard gate — any weakening of it compounds across every WRK execution
- uv readiness checks here become a model for other scripts that depend on uv
