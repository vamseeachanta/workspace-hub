## Mission

Update eval-skills.py to enforce the new 200-line hard limit and flag oversized skills
as warnings or critical failures.

## What

1. Add `check_line_count()` to eval-skills.py:
   - >200 lines → warning
   - >400 lines → critical
2. Update `audit-skill-violations.sh` word count threshold from 5000 to match
   (approximately 200 lines ≈ 1500 words)
3. Re-run eval to establish new baseline with line count check

## Why

- eval-skills.py currently only checks word count >5000 (from old 400-line era)
- The new 200-line hard limit is not enforced by any eval check
- Without this, oversized skills slip through undetected

## Acceptance Criteria

- [x] `check_line_count()` added to eval-skills.py
- [x] >200 lines = warning, >400 lines = critical
- [x] TDD: tests for line count check (6 tests, 35/35 total)
- [x] Baseline re-run shows current violation count (293/408 exceed limit)
