## Mission

Enforce one-WRK-per-session scope discipline via hookify rule and session state tracking — preventing the "agent does too much at once" failure mode.

## What

Anthropic identifies "the agent's tendency to do too much at once" as the core failure mode for long-running agents. The fix: confine work scope to a single feature per session.

Workspace-hub already tracks active WRK via `.claude/state/active-wrk`, but nothing prevents an agent from drifting to work on other items mid-session.

Implement:
1. Hookify rule: warn when agent attempts to set a second active-wrk without archiving/completing the first
2. Session state: track `wrk_started_at` timestamp in active-wrk state
3. Skill update: add "scope discipline" reminder to session-start output

## Why

Without enforcement, agents accumulate half-finished work across multiple WRKs in a single session, leaving the codebase in a broken intermediate state that the next session inherits.

## Acceptance Criteria

- [x] Scope guard in set-active-wrk.sh fires SCOPE_GUARD_WARNING on second WRK activation without first WRK completion
- [x] Session-start displays active WRK reminder if one exists (step 3d)
- [x] Documentation added to work-queue SKILL.md referencing scope discipline
