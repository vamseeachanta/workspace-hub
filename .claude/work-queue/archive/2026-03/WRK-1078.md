## Mission
Identify and fix workspace-hub WRK pipeline scripts that break on Windows
Git Bash (MINGW64) so licensed-win-1 can run routed work items without friction.

## What
Audit and patch the following known gaps:

1. **uv not available** — `verify-gate-evidence.py`, `generate-index.py`,
   `close-item.sh` all invoke `uv run`; add Windows fallback or installer hint
2. **`xdg-open` missing** — Stage 5/7/17 HTML review gates use `xdg-open`;
   replace with cross-platform opener (`start` on Windows, `open` on macOS)
3. **`sed -i` MINGW64 behaviour** — audit scripts using `sed -i` for
   in-place edit; MINGW64 requires `sed -i ''` or `sed -i.bak`
4. **`python3` bare calls** — audit remaining bare `python3` invocations
   (related to WRK-693); ensure `uv run --no-project python` or add fallback
5. **Crontab** — document that cron-based automation is dev-primary only;
   add note to Windows setup guide

## Why
licensed-win-1 (orcaflex-license-machine) can run Claude Code and has all AI
CLIs installed, but hits friction on pipeline scripts due to Linux assumptions.
Fixing these gaps enables proper WRK routing to that machine.

## Acceptance Criteria
- [ ] `xdg-open` calls replaced with cross-platform opener helper
- [ ] `sed -i` usage audited; MINGW64-safe where needed
- [ ] `uv` install check added to `verify-setup.sh` Windows path with hint
- [ ] `dev-env-check.sh` runs without error on licensed-win-1
- [ ] Windows compatibility notes added to `.claude/docs/new-machine-setup.md`
- [ ] All changes pass existing test suite on dev-primary

## Plan

### Phase 1 — Cross-platform browser opener (xdg-open)
- **File**: `scripts/work-queue/log-user-review-browser-open.sh`
- Replace the `xdg-open "$HTML_PATH"` call with a `_open_browser()` helper
  that detects OS via `uname`/`OSTYPE` and routes to `xdg-open` (Linux),
  `open` (macOS), or `start ""` (Windows/MINGW64)
- Test: `bash scripts/work-queue/log-user-review-browser-open.sh WRK-TEST --stage plan_draft --html /path/to/test.html --no-open` still works

### Phase 2 — archive-item.sh python3 → uv
- **File**: `.claude/work-queue/scripts/archive-item.sh`
- Line 59: `python3 <<EOF` → `uv run --no-project python <<EOF`
- Line 79: `python3 "${QUEUE_DIR}/scripts/generate-index.py"` → `uv run --no-project python ...`
- Test: run archive on a dummy item; confirm index regenerates

### Phase 3 — verify-setup.sh Windows uv hint
- **File**: `scripts/setup/verify-setup.sh`
- In the `_fail "python not found"` branch, add a Windows-specific hint:
  `If on Windows/MINGW64: $USERPROFILE/.local/bin/uv or run the uv installer`
- Test: `bash scripts/setup/verify-setup.sh` still produces clean output on Linux

### Phase 4 — Fix Python Unicode encoding on Windows console
- **File**: `scripts/work-queue/start_stage.py` and `exit_stage.py`
- Add `import sys; sys.stdout.reconfigure(encoding='utf-8', errors='replace')`
  near top of `_main()` in both files (after arg parse, before any print)
- This ensures box-drawing characters in checkpoint_writer don't crash cp1252
- Test: run both scripts without `PYTHONIOENCODING` env var on Windows; verify output

### Phase 5 — Crontab/new-machine-setup.md callout
- **File**: `.claude/docs/new-machine-setup.md` §5 Cron Jobs
- Add a callout note: _"Windows machines (licensed-win-1, licensed-win-2) use Windows
  Task Scheduler for contribute-minimal tasks. Full cron automation (nightly
  pipeline, comprehensive-learning) runs on dev-primary only."_

### Test Strategy

- Run `bash scripts/work-queue/log-user-review-browser-open.sh` with `--no-open` on MINGW64
