# feat(git-hooks): Post-Commit Auto-Push + Trunk-Based Routing Script

## Mission

Encode the trunk-based solo/multi-workstation git policy in scripts and hooks,
not prose. Rules that matter must be enforced mechanically.

## What

### 1. `scripts/hooks/post-commit` — auto-push after every commit

Mirror the existing `post-merge` / `post-checkout` / `post-rewrite` hook pattern.

```bash
#!/usr/bin/env bash
# post-commit: auto-push to remote after every commit (WRK-1141)
# Guards: skip if no remote, detached HEAD, CI environment, or dry-run flag
```

Guards required:
- `SKIP_PUSH=1` env var → skip (lets scripts opt out)
- `CI=true` / `GITHUB_ACTIONS` → skip
- Detached HEAD → skip
- No upstream remote configured → skip with warning
- Submodule context → skip (hub-level push only)

### 2. `scripts/work-queue/start-wrk.sh WRK-NNN` — trunk vs branch routing

Reads WRK frontmatter (`complexity`, `compound`, estimated session count) and:
- **simple / single-session** → prints "Commit to main directly"
- **medium / multi-session** → `git checkout -b <prefix>/WRK-NNN-<slug>`
- **complex / compound** → same as medium, with worktree option

Removes the human decision point from session start.

### 3. Wire into `scripts/setup/install-all-hooks.sh`

Add `post-commit` to the canonical hooks install list so it propagates to
all machines via `post-merge` re-install.

## Why

- Doc-only rules are context-rot — a `post-commit` hook fires every time, docs don't
- `start-wrk.sh` removes the branching decision from LLM judgment → deterministic
- Consistent across all workstations once `install-all-hooks.sh` runs

## Acceptance Criteria

- [ ] `scripts/hooks/post-commit` created; pushes on every commit with all guards
- [ ] `SKIP_PUSH=1 git commit -m "..."` skips push (tested)
- [ ] `CI=true git commit` skips push (tested)
- [ ] `scripts/work-queue/start-wrk.sh WRK-NNN` routes correctly for simple/medium/complex

- [ ] `post-merge` re-install picks up `post-commit` on pull (verified)
