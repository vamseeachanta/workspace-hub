# WRK-5108: Prune /work workflow — keep lean and on target

## Description

After ~7 days (~2026-03-28), review session logs for the `/work` workflow and prune any unused code, scripts, skills, or dead paths to keep the system lean.

## Scope

1. Review session logs for /work usage patterns across all 3 providers (claude, codex, gemini) — identify which scripts/skills are actually invoked
2. Check GitHub issues/comments left by the user for discrepancies and feedback
3. Identify unused or redundant code in `scripts/work-queue/`, `.claude/skills/coordination/workspace/work-queue/`
4. Prune dead code, consolidate overlapping scripts
5. Verify remaining scripts still work after cleanup

## Workflow

- **User** posts discrepancy items / feedback as GH issue comments
- **Agents** review session logs (claude, codex, gemini) for usage patterns and dead code
- This pattern applies to all WRK items, not just this one

## Inputs

- Session logs (state/session-signals/) — all 3 providers
- GitHub issue comments ([#1256](https://github.com/vamseeachanta/workspace-hub/issues/1256))
- User feedback left in GH over the next week

## Acceptance Criteria

- No unused scripts remain in work-queue paths
- All retained scripts pass basic smoke tests
- /work workflow still functions end-to-end after pruning
