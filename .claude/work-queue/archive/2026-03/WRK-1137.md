## Mission

Codify the rule that all non-licensed engineering work must run on Ace Linux machines, and that large features must be decomposed into Linux-executable chunks to minimise use of shared licensed machines.

## What

- Add or update workstation routing documentation (e.g. `config/work-queue/workstation-routing.md` or the work-queue SKILL.md routing table) to state explicitly:
  - Non-licensed work → Ace Linux machines (dev-primary, dev-secondary) only
  - Licensed machines (licensed-win-1, etc.) are shared / budget-constrained — avoid unless a license is strictly required
  - Big features must be decomposed so the majority of stages run on Linux

## Why

Licensed machines (licensed-win-1) are shared by multiple users and operate under a limited budget.  Over-use crowds out colleagues and drives up cost.  Maximising Linux utilisation is both economical and gives agents a larger resource pool.

## Acceptance Criteria

1. A clear written policy exists (doc or config) with the Linux-first rule and the decomposition requirement
2. The work-queue SKILL.md workstation routing table references or links to this policy
3. Any existing WRKs wrongly assigned to licensed machines for non-licensed work are flagged for reassignment
