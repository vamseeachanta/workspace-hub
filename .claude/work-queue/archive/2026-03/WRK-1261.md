## Mission

Encode and quantify the skill quality hierarchy into the eval tooling and curation
workflow. The hierarchy (best to worst):

1. **Scripts** — deterministic, testable, auditable (Level 2-3 in patterns.md)
2. **LLM text with references** — prose backed by script calls, file pointers, or
   worked examples (Level 1 micro-skill)
3. **Single focused skill file** — self-contained, < 200 lines, clear trigger
4. **Large monolithic skill file** — > 500 words, multiple concerns, degrades with
   context window pressure (Level 0 prose)

## What

1. Define a `quality_tier` scoring rubric in `config/skills/quality-tiers.yaml`:
   - Tier A: skill has ≥1 script + frontmatter `scripts:` field
   - Tier B: skill references scripts/files in body (`bash scripts/`, file paths)
   - Tier C: skill is < 200 lines, focused, no script refs needed
   - Tier D: skill is > 500 words with no script refs (candidate for decomposition)
2. Add `quality_tier` computation to `eval-skills.py` — output tier per skill
3. Add tier distribution to `ecosystem-eval-report.sh` collated output
4. Create `scripts/skills/skill-tier-report.py` — ranks all skills by tier,
   flags Tier D for curation (decompose into script + micro-skill)
5. Document the hierarchy in skill-creator SKILL.md and comprehensive-learning
   Phase 9 so future curation follows the pattern

## Why

- patterns.md already defines the enforcement gradient (prose → micro-skill → script → hook)
  but skill eval doesn't measure where each skill sits on that gradient
- Without quantification, curation is subjective — "is this skill good enough?" has
  no measurable answer
- The 25% repetition rule means any repeated LLM reasoning should become a script;
  this WRK makes that measurable per-skill

## Acceptance Criteria

- [ ] `config/skills/quality-tiers.yaml` defines 4 tiers with scoring criteria
- [ ] `eval-skills.py` computes and outputs `quality_tier` per skill
- [ ] `ecosystem-eval-report.sh` includes tier distribution in collated YAML
- [ ] `skill-tier-report.py` produces ranked list with Tier D flagged for curation
- [ ] Tier hierarchy documented in skill-creator and comprehensive-learning
- [ ] Current ecosystem tier distribution recorded as baseline
