## Mission
Give mypy in digitalmodel and worldenergydata full visibility into assetutilities
types so cross-repo type errors are caught statically rather than at runtime.

## What
- Approach: inline types (preferred over stubs) — add type annotations directly
  to assetutilities public API modules (the contracted symbols from WRK-1074)
- Scope: the symbols under contract in WRK-1074:
  `SaveData`, `ReadData`, `AttributeDict`, `Transform`, `WorkingWithYAML`,
  `TrackedQuantity`, `unit_checked`, `UnitMismatchError`, `FileManagement`,
  `update_deep_dictionary`
- Add `py.typed` marker file to `assetutilities/src/assetutilities/` (PEP 561)
  so consumers' mypy picks up inline types without stub configuration
- mypy config in consumers: enable `assetutilities` in `mypy.ini` / `pyproject.toml`
  `[tool.mypy]` section; verify zero new errors on existing consumer code
- Verify: `uv run mypy src/ --ignore-missing-imports` passes in digitalmodel
  and worldenergydata after annotations added

## Why
WRK-1056 adds mypy to consumers but mypy currently ignores assetutilities calls
(missing stubs). A type error in how digitalmodel calls `SaveData` — wrong argument
type, missing field — is invisible until runtime. PEP 561 inline types fix this at
zero ongoing maintenance cost once annotated.

## Acceptance Criteria
- [ ] All 10 contracted assetutilities symbols have complete type annotations
- [ ] `py.typed` marker added to assetutilities package
- [ ] mypy passes in assetutilities on annotated modules (--strict or equivalent)

- [ ] mypy passes in worldenergydata with assetutilities types visible
- [ ] Cross-review (Codex) passes

## Plan

**Route B — 3 phases:**

**Pre-knowledge (from resource intelligence):**
- `TrackedQuantity`, `UnitMismatchError` already fully typed — no changes needed
- `unit_checked` needs minor tightening only (`Callable` → `Callable[..., Any]`, explicit `-> Any` on wrapper)
- Heavy work: `SaveData`, `ReadData`, `Transform` (all in `common/data.py`, 1200 lines)
- `WorkingWithYAML` partially typed (1/15 methods) — annotate remaining 14
- `FileManagement`: cfg is attribute-accessed — type as `Any` (pragmatic, avoids Protocol scope-creep)
- `AttributeDict`: canonical in `update_deep.py` — annotate `__init__` + type as `dict[str, Any]` subclass
- `update_deep_dictionary`: single function — `(MutableMapping[str, Any], Mapping[str, Any]) -> MutableMapping[str, Any]`
- `disallow_untyped_defs = true` already set — must annotate completely or mypy fails
- data.py has latent bug: `DataFrame_To_xlsx_xlsxwriter` missing `self` — flag, do not fix (out of scope)

**Phase 1 — Infrastructure + units/ (low risk):**
1. Create `assetutilities/src/assetutilities/py.typed` (empty, PEP 561)
2. Add `from __future__ import annotations` to ALL files being annotated:
   `data.py`, `file_management.py`, `update_deep.py`, `yml_utilities.py`
3. Tighten `unit_checked` in `units/computation.py`: `Callable` → `Callable[..., Any]`,
   explicit `-> Any` on wrapper; import `ParamSpec` from `typing_extensions` (confirmed in lock)
   only if needed to preserve decorator signature — otherwise `Callable[..., Any]` suffices
4. Annotate `update_deep_dictionary` in `common/update_deep.py`
5. Annotate canonical `AttributeDict.__init__` in `common/update_deep.py`;
   **also** annotate duplicate `AttributeDict.__init__` in `data.py` (required since
   `disallow_untyped_defs = true` applies to entire file)
6. Run: `uv run mypy src/assetutilities/units/ src/assetutilities/common/update_deep.py` → 0 errors; commit

**Phase 2 — WorkingWithYAML + FileManagement:**
7. Annotate `WorkingWithYAML` remaining 14 methods in `common/yml_utilities.py`
   (cfg as `dict[str, Any]`; `from __future__ import annotations` added in Phase 1 Step 2)
8. Annotate `FileManagement` public methods in `common/file_management.py`
   (cfg typed as `Any` — attribute-access pattern is too dynamic for Protocol; documented in
   module docstring: "cfg uses AttributeDict attribute access; typed Any by design")
9. Run: `uv run mypy src/assetutilities/common/yml_utilities.py src/assetutilities/common/file_management.py` → 0 errors; commit

**Phase 3 — SaveData, ReadData, Transform (data.py surgery):**
10. Add annotations to `ReadData` public methods (cfg as `dict[str, Any]`, df as `pd.DataFrame`)
11. Add annotations to `SaveData` public methods; flag `DataFrame_To_xlsx_xlsxwriter` missing-self bug
12. Add annotations to `Transform` public methods (union returns as `Union[str, dict[str, Any]]`)
13. Run: `uv run mypy src/assetutilities/common/data.py` → 0 errors; commit
14. Verify consumers: `PYTHONPATH=src uv run mypy src/` in digitalmodel and worldenergydata → 0 new errors

**Test strategy:**
- No unit tests for type annotations themselves — mypy IS the test
- Gate: `uv run mypy` exit code 0 in all 3 repos after each phase
- ac-test-matrix.md will list each mypy invocation as a PASS/FAIL row
