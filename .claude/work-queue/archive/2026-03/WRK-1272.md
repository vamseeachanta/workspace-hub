## Mission

Bring all 293 SKILL.md files exceeding the new 200-line hard limit into compliance
by splitting oversized skills into focused sub-skills.

## What

1. Create `scripts/skills/find-oversized-skills.py` — list all skills >200 lines with
   line count, category, and suggested split points
2. Decompose into child WRKs by category batch:
   - 1200+ lines (40 skills) — highest priority, 6-10x over limit
   - 801-1200 lines (27 skills)
   - 401-800 lines (125 skills)
   - 201-400 lines (101 skills) — lowest priority, minor trims
3. For each batch: split by responsibility (reference tables, code examples, API docs
   become sub-skills; core guidance stays in main skill)

## Why

- coding-style.md hard limit changed from 400 to 200 lines
- 72% of skills (293/408) now violate the hard limit
- Oversized skills waste agent context window and degrade triggering accuracy

## Acceptance Criteria

- [ ] Script created to identify and prioritize oversized skills
- [ ] Feature plan with child WRK decomposition by category batch
- [ ] All 1200+ line skills (40) split to <200 lines
- [ ] All 801-1200 line skills (27) split to <200 lines
- [ ] All 401-800 line skills (125) split to <200 lines
- [ ] All 201-400 line skills (101) trimmed to <200 lines
- [ ] eval-skills.py re-run shows 0 skills over 200 lines
