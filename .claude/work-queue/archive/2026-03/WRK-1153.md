## Mission
Build a protocol simulation and data-mapping toolkit at `digitalmodel/power/protocols/scada_integration.py`
covering Modbus TCP/RTU, DNP3 point mapping, OPC UA node modeling, and EPMS data model with I/O list export.

## What
- ModbusRegisterMap: CSV import, register lookup, raw-to-engineering scaling, consistency validation
- DNP3PointList: analog/binary inputs, Excel import, SEL RTAC-compatible config export
- OPCUANodeModel: device tree → namespace builder, XML export (Ignition/Wonderware compatible)
- EPMSDataModel: generator/switchgear/BMS points, template compliance, Excel-ready I/O list generation

## Why
Closes the hardest gap (communication protocols) at lowest effort. I/O list generation is a literal
JD deliverable. Demonstrates Modbus, DNP3, OPC UA, IEC 61850, EPMS/BMS integration architecture.

## Acceptance Criteria
- [ ] ModbusRegisterMap with CSV import, scaling, validation — ≥10 tests
- [ ] DNP3PointList with Excel import and config export — ≥10 tests
- [ ] OPCUANodeModel with namespace builder and XML export — ≥8 tests
- [ ] EPMSDataModel with template compliance and I/O list export — ≥10 tests
- [ ] All classes have docstrings referencing applicable standards
- [ ] Module importable from `digitalmodel.power.protocols`
