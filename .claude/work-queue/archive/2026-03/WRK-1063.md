## Mission
Provide domain-specific Route C spec templates so engineers don't write specs from
scratch — halving spec authoring time for the 112 engineering WRKs.

## What
Templates in `specs/templates/`:
- `route-c-structural.md` — structural/FEA: sections for load cases, element types,
  material properties, verification standards (DNV, ABS, API)
- `route-c-marine.md` — naval architecture/offshore: RAOs, fatigue, code references
  (IACS, DNV-RP, ABS), hydrodynamic assumptions
- `route-c-energy.md` — oil & gas / energy: reservoir params, production profiles,
  economic assumptions, BSEE/regulatory hooks
- `route-c-generic.md` — existing base template (consolidate from current ad-hoc)
- Each template: pre-filled Mission/What/Why/AC skeleton + domain-specific checklist
  + links to relevant rules and standards docs in `.claude/docs/`
- Update `/work add` capture to suggest relevant template based on WRK category keywords

## Why
Route C spec writing is the longest pre-implementation step. Domain templates encode
the "what not to forget" knowledge (load cases, code compliance, test strategy) that
agents currently reconstruct from training memory.

## Acceptance Criteria
- [ ] 4 templates created in `specs/templates/` (structural, marine, energy, generic)
- [ ] Each includes: mission skeleton, domain checklist, standard references
- [ ] Templates reference real `.claude/docs/` or `specs/modules/` artefacts where available
- [ ] `scripts/work-queue/new-spec.sh WRK-NNN [domain]` copies the right template
- [ ] Cross-review (Codex) passes

## Plan
Route A inline plan:
1. Create `specs/templates/route-c-generic.md` — base WRK spec skeleton (Mission/What/Why/AC + meta fields)
2. Create `specs/templates/route-c-structural.md` — extends generic with FEA checklist: load cases, element types, material props, DNV/ABS/API references
3. Create `specs/templates/route-c-marine.md` — extends generic with offshore/naval checklist: RAOs, fatigue, IACS/DNV-RP/ABS code refs, hydrodynamic assumptions
4. Create `specs/templates/route-c-energy.md` — extends generic with O&G checklist: reservoir params, production profiles, economic assumptions, BSEE/regulatory hooks
5. Create `scripts/work-queue/new-spec.sh` — copies right template based on `[domain]` arg (structural|marine|energy → generic fallback); creates `specs/wrk/WRK-NNN/spec.md`
6. Codex cross-review of all 5 files
