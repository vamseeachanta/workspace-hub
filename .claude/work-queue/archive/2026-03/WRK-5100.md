## Background

Stage micro-skills and YAML contracts still reference `WRK-NNN-lifecycle.html` — an artifact
from the old local-file workflow. GitHub issues are now the human-facing review surface
(WRK-5104). Two live sessions (WRK-5104 #1252, WRK-5106 #1254) exposed 4 discrepancies.

Found during WRK-1389 verification + WRK-5104/WRK-5106 live sessions.

## Scope

### A — Purge dead lifecycle HTML references (stages 2-20 micro-skills)

Stage 1 already fixed (commit 3780f948). Remaining 18 stage micro-skills still reference
`WRK-NNN-lifecycle.html` as entry reads, checklist items, or exit artifacts.

Files: `.claude/skills/workspace-hub/stages/stage-{02..20}-*.md`

- Stage 11 (artifact generation): rewrite — entire checklist is HTML-era; repurpose as

or `entry_reads`.

Files: `scripts/work-queue/stages/stage-{04,05,06,07,10,12,13,17}-*.yaml`

`scripts/work-queue/templates/archive-tooling-template.yaml` has `html_verification_ref`

match the gate.

### E — Update ecosystem terminology

`Lifecycle HTML` as canonical. Update to reflect GitHub issue as the review surface.

### F — Address session discrepancies (from WRK-5104 #1252 + WRK-5106 #1254)

1. **Stage summaries missing**: stage completion comments post bare "Stage N — DONE" with
   no summary. `exit_stage.py` should include a brief summary from evidence files.
2. **Approval UX too verbose**: agent re-asks confirmation after approval already given.
   Approval detection should accept natural language without re-prompting.
3. **No human gate in execute group**: stages 8-9-10 run without pause after stage 7
   approval. Evaluate whether a mid-execution checkpoint is needed.
4. **Confusing author aliases**: all comments appear under `vamseeachanta` — no way to
   distinguish human vs AI agent comments. Add agent attribution prefix.
