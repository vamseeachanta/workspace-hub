# feat(doc-intel): promote-to-code.py — promote all 8 manifest content types

## Mission
Create scripts/data/doc-intelligence/promote-to-code.py that promotes constants to Python, tables to CSV, equations to Python functions, worked_examples to pytest, curves to CSV+interpolation, procedures to YAML skills, requirements to docstrings, definitions to glossary YAML. TDD per type.

## Why
Promotion is the final step — turning extracted knowledge into executable, testable code artifacts. This closes the loop from PDF to production code.

## Acceptance Criteria
- [ ] constants promoted to Python module with typed variables
- [ ] tables promoted to CSV files in data/standards/promoted/
- [ ] equations promoted to Python functions with docstrings citing source
- [ ] worked_examples promoted to pytest parametrized test cases
- [ ] curves promoted to CSV + scipy interpolation wrapper
- [ ] procedures promoted to YAML skill files
- [ ] requirements promoted to docstring annotations
- [ ] definitions promoted to glossary YAML
- [ ] Each promotion type has dedicated TDD tests
- [ ] --dry-run flag shows what would be promoted without writing

## TDD Approach
Test each of the 8 promotion types independently with fixture data. Test dry-run mode. Test idempotent re-promotion.
