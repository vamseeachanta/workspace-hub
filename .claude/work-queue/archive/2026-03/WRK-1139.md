# chore(cc-adoption): Adopt Claude Code 2.1.72–2.1.74 Ecosystem Improvements

## Mission

Adopt three actionable Claude Code 2.1.72–2.1.74 improvements that reduce
context usage, improve hook reliability, and integrate with our existing
memory architecture.

## What

Three targeted changes triggered by CC release notes analysis:

### 1. HTML Comments in CLAUDE.md (2.1.72)
`Changed CLAUDE.md HTML comments (<!-- ... -->) to be hidden from Claude when
auto-injected. Comments remain visible when Read tool is used.`

Audit workspace-hub CLAUDE.md (and ecosystem repo CLAUDE.md/AGENTS.md files)
and move verbose non-essential notes, TODOs, and metadata behind HTML comment
blocks (`<!-- ... -->`). This reduces auto-injected context tokens without
losing documentation.

### 2. `autoMemoryDirectory` Configuration (2.1.74)
`Added autoMemoryDirectory setting to configure a custom directory for
auto-memory storage.`

Configure `autoMemoryDirectory` in `.claude/settings.json` to point to
`/home/vamsee/.claude/projects/-mnt-local-analysis-workspace-hub/memory/`
so Claude auto-memory aligns with our existing MEMORY.md-indexed structure.

### 3. `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` Tuning (2.1.74)
`Fixed SessionEnd hooks being killed after 1.5 s on exit regardless of
hook.timeout — now configurable via CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS.`

Check session-end hook scripts (scripts/hooks/ or .claude/settings.json
SessionEnd hooks) for expected runtime. If > 1.5s, set
`CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS` appropriately in the environment
or settings.

## Why

- Reduces per-session token overhead from CLAUDE.md auto-injection
- Aligns auto-memory writes with existing indexed memory system
- Prevents session-end hooks from being silently killed mid-execution

## Acceptance Criteria

- [x] CLAUDE.md HTML comment audit done — already 17 lines, nothing to hide (compliant)
- [x] `.claude/settings.json` has `autoMemoryDirectory` set to our memory path
- [x] Session-end hook runtimes checked (0.89s total) — no timeout tuning needed
- [x] No functional regression — settings.json validates as valid JSON

## Plan

   (verbose notes, TODOs, historical context not needed for task dispatch)
2. Edit CLAUDE.md — wrap candidates in `<!-- -->` blocks; verify injected size drops

4. Check `scripts/hooks/` for SessionEnd hooks; measure or estimate runtime;
