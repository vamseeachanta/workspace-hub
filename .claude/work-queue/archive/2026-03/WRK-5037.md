# feat(doc-intel): extract-document.py + document parse layer

## Mission
Create scripts/data/doc-intelligence/extract-document.py with pdfplumber PDF parsing, python-docx DOCX parsing, openpyxl XLSX parsing, producing extraction manifests.

## Why
The parse layer is the foundation of the pipeline — it converts raw documents into structured extraction manifests that downstream tools (indexer, query, promote) consume.

## Acceptance Criteria
- [ ] extract-document.py accepts file path and outputs extraction manifest YAML
- [ ] PDF parsing via pdfplumber extracts text, tables, and figures
- [ ] DOCX parsing via python-docx extracts text, tables, and embedded objects
- [ ] XLSX parsing via openpyxl extracts tabular data with sheet metadata
- [ ] Manifest schema validated against design spec
- [ ] CLI interface with --input, --output, --domain flags
- [ ] Error handling for corrupt/unsupported files

## TDD Approach
Test each parser independently with fixture files (PDF, DOCX, XLSX). Test manifest schema validation. Test CLI argument handling.
