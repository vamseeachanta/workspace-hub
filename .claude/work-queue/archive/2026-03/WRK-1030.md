# Add /resume slash command for WRK checkpoint resumption

## Mission
**Create a `/resume WRK-NNN` slash command that reads `checkpoint.yaml` and loads the `entry_reads` files into context, so a new session can pick up exactly where the previous one left off — without manual file hunting.**

> Read this at the start of every execution stage. If a task, file change, or decision is
> not directly in service of this mission, stop and capture it as a new WRK item instead.

## What
Add a `.claude/commands/workspace-hub/resume.md` slash command that:
1. Reads `.claude/work-queue/assets/WRK-NNN/checkpoint.yaml`
2. Prints stage, `next_action`, and `context_summary` to orient the agent
3. Lists `entry_reads` files so they can be @-mentioned or read immediately

## Why
Currently resuming a WRK item is a manual process — find checkpoint.yaml, read it, read entry_reads files, remember the next_action. A `/resume WRK-NNN` command makes this a single step and reduces context-restore friction between sessions.

## Acceptance Criteria
- [ ] `.claude/commands/workspace-hub/resume.md` exists and loads via `/resume WRK-NNN`
- [ ] Command prints current stage, next_action, and context_summary from checkpoint.yaml
- [ ] Command reads and displays content of each file in entry_reads
- [ ] Graceful error when checkpoint.yaml does not exist (suggests running checkpoint.sh first)
- [ ] Command documented in checkpoint.sh header comment

## Agentic AI Horizon

- Directly boosts session-resumption speed for any agent or human picking up an in-progress WRK
- Low agentic-ceiling risk — this is a simple file-read command with no external dependencies
- No reason to wait; straightforward and high daily-use value
