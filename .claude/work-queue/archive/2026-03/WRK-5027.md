# audit(patterns): Recurring LLM-Prose Operations → Scripts (25% Repetition Rule)

## Mission

Audit skill prose, CLAUDE.md, and rules files for recurring operations being performed
via LLM reasoning that should instead be deterministic scripts, per the updated
25% Repetition Rule in `patterns.md`.

## Why

`patterns.md` was updated with a hard rule: if an operation has ≥25% chance of recurring,
write a script instead of reasoning inline. LLM prose is slow, non-deterministic, and
not auditable. This audit finds existing violations and converts them to scripts.

## What

### Phase 1: Scan for prose-based recurring operations

Search across:
- `.claude/skills/**/*.md` — skill steps that say "check X by reading Y" or "count Z"
- `.claude/rules/*.md` — rules referencing manual LLM checks
- `.claude/docs/*.md` — process docs with inline reasoning steps
- `CLAUDE.md` files across all repos

Flag any step that:
- Parses, counts, filters, or transforms data inline
- Re-derives facts that a grep/script could compute
- Generates YAML/JSON by hand that a script could produce
- Would need to be repeated across repos or files

### Phase 2: Triage flagged items

For each flagged operation:
- Does a script already exist? → Update prose to call the script
- Is it a one-liner? → Write inline script in the docs
- Is it a reusable utility? → Write to `scripts/quality/` or `scripts/work-queue/`

### Phase 3: Implement top-priority conversions

Convert the top 5 highest-frequency prose operations to scripts.
Each conversion: write script → add tests → update prose to call script.

## Acceptance Criteria

- [ ] Audit report written: `docs/patterns-audit-25pct-rule.md` with flagged items table
- [ ] Each item classified: existing-script / new-one-liner / new-utility
- [ ] Top 5 conversions implemented with tests
- [ ] Prose in skills/rules updated to call scripts instead of reasoning inline
- [ ] Cross-review (Gemini) on the audit report before closing

## TDD Approach

New scripts: write failing test first, then implement, following existing `tests/` patterns.
