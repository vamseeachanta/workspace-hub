# Investigate verbose output effect on session logs and log learning to skill

## Mission
**Determine definitively whether Claude Code's "Verbose Output" config setting reduces session
log content (which feeds comprehensive-learning), and persist verified finding to the appropriate
skill so the question is never re-investigated.**

> Read this at the start of every execution stage. If a task, file change, or decision is
> not directly in service of this mission, stop and capture it as a new WRK item instead.

## What
Ad-hoc investigation triggered by user question: does `Verbose Output = false` in Claude Code
config reduce session logs and thereby degrade comprehensive-learning signal quality?

## Why
Risk: if verbose=false silently suppressed log content, nightly learning pipeline would
degrade without obvious signal. Verified answer prevents mis-configuration.

## Acceptance Criteria
- [x] Verbose setting behaviour confirmed via agent investigation (settings files + log structure)
- [x] Live test run via `submit-to-codex.sh` — before/after log counts compared
- [x] Finding logged to `.claude/skills/workspace-hub/file-taxonomy/SKILL.md`
- [x] Ecosystem health confirmed: `submit-to-codex.sh` working correctly

## Plan
- Agent 1: inspect settings files + session log structure + comprehensive-learning Phase 1
- Agent 2: search Claude Code docs for verbose flag behaviour
- Live test: run `submit-to-codex.sh` with simple prompt, compare log counts before/after
- Write finding to file-taxonomy/SKILL.md under AI Agent Log Locations section

## Findings
- `--verbose` / "Verbose Output" is **terminal display only** — no `verbose` field in settings.json
- `logs/orchestrator/claude/session_YYYYMMDD.jsonl` written by `session-logger.sh` hook on every
  pre/post tool call — completely independent of verbose setting
- `~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl` (~296K/session) written by codex CLI — also
  independent of Claude Code verbose
- Comprehensive-learning signal quality is **unaffected** by verbose=false
- `submit-to-codex.sh` confirmed working: logs grew 694→700 lines (Claude), 3→4 codex sessions

## Agentic AI Horizon
- This is a one-off harness verification — no future leverage concern.
