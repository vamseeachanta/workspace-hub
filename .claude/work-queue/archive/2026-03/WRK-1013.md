# chore(skills): fix Anthropic guide violations — README.md in skill folders + oversized SKILL.md files

## Background

After reading the Anthropic official guide (`references/anthropic-complete-guide-building-skills.pdf`
inside `skill-creator`), two categories of violations were found across the 499-skill ecosystem:

**Violation 1: README.md inside skill folders** (109 violations)
The guide states: *"Don't include README.md inside your skill folder. All documentation goes in
SKILL.md or references/."* Frontmatter appears in Claude's system prompt; README.md does not.

**Violation 2: SKILL.md over 5,000 words** (6 violations)
The guide recommends keeping SKILL.md under 5,000 words. Oversized files increase context load
and degrade performance. Detailed docs should move to references/ and be linked.

## Violation Inventory

### README.md in skill folders (109 files)
Examples:
- `ai/agent-usage-optimizer/README.md`
- `ai/prompting/prompt-engineering/README.md`
- `business/communication/calendly-api/README.md`
- `coordination/workspace/skill-learner/README.md`
... (full list: `find .claude/skills -name "README.md" | grep -v "^.claude/skills/README.md"`)

**Fix**: For each skill folder README.md:
1. Check if it has content beyond the SKILL.md
2. If yes → migrate unique content to SKILL.md or `references/README.md`
3. Delete the README.md from the skill folder

### SKILL.md over 5,000 words (6 files)
| Skill | Words | Action |
|-------|-------|--------|
| `workspace-hub/comprehensive-learning` | 5,928 | Move pipeline detail to references/ |
| `data/energy/production-forecaster` | 5,828 | Move examples to references/ |
| `_diverged/digitalmodel/ai-prompting/pandasai` | 5,687 | _diverged — lower priority |
| `ai/prompting/pandasai` | 5,685 | Move API reference to references/ |
| `_internal/meta/module-based-refactor` | 5,666 | Move patterns to references/ |
| `engineering/marine-offshore/structural-analysis` | 5,490 | Move standards refs to references/ |

## What

1. Batch-delete README.md files from all skill folders (after content audit)
2. For skills with unique README content, migrate to `references/` subdirectory
3. Refactor the 6 oversized SKILL.md files to extract bulk content into references/
4. Verify no skill breaks (trigger test on each refactored skill)
