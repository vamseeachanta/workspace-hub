# MISSION: Self-healing and self-improving repo ecosystem

## Mission Statement

**Build an ecosystem where every cycle — code commit, session, nightly cron — automatically makes the entire system better.** Code quality cannot regress. Dependencies stay healthy. Type safety grows. Cross-repo breaks surface before they reach main. Docs stay current. Agent sessions produce skills, not manual work. The whole system heals and improves without manual nudging.

Scope revised 2026-03-09: expanded from agent/session layer to the full repo ecosystem across all 6 repos.

## Why This Parent Exists

Child items target individual pillars. Without a parent holding the single mission, agentic execution risks:
- **Contradictions** — two items solving the same problem differently
- **Drift** — a pillar optimises locally but breaks another pillar
- **Gaps** — integration points between pillars go unowned
- **Misaligned priorities** — effort goes to interesting work rather than critical-path

Every child item should be tested against: *does this make the ecosystem more self-healing and self-improving?*

## Alignment Rules for Agentic Execution

When executing any child item, the executing agent must:

1. **Read this mission first** — understand the single objective before touching any file
2. **Test each decision against the ecosystem loop** — does this output slot into the loop? Which pillar?
3. **Check for contradictions** — does this duplicate or conflict with an adjacent item? Stop and surface to user
4. **Prefer scripts over LLM judgment** — encode gates in scripts; skill prose rots
5. **Prefer integration over invention** — extend existing hooks/scripts before creating new ones
6. **One file at a time** — changes are atomic; each file edit is independently reviewable
7. **Surface, don't decide** — when uncertain between two approaches, surface both to user
