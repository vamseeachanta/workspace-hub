# feat(session): Execution-Stage Analytics

> **ABSORBED**: Scope completed under WRK-1102 (comprehensive-learning modular chain repair),
> commit 29435808. WRK-1102 archived 2026-03-12.

## What

Instrument Stage 10 (Work Execution) with structured signals emitted during execution,
then analyze them in comprehensive-learning Phases 5-6.

Mirrors the WRK-691 Option C split for the execution stage:
- **Prevention**: scope guards (warn if files touched outside `target_repos`)
- **Analytics**: per-WRK execution signals → `session-signals/` → nightly trend analysis

## Why

WRK-691 adds post-session drift detection at the session level. But during execution,
agents make file changes, run tools, and may exceed WRK scope without any structured
signal being emitted. Without execution-phase signals, comprehensive-learning can only
infer behavior from git history — not from real-time tool usage.

## Scope

### Signals to Emit During Stage 10
- Tool call count per WRK (via Stop hook or execution wrapper)
- File paths touched — check against `target_repos`
- WRK scope adherence: files outside target_repos = scope drift event
- Execution duration (start/end timestamps)

### Output
- `session-signals/execution-<YYYYMMDD>.jsonl` — per-execution events
- `comprehensive-learning` Phase 5 or new Phase 1c reads this for trend analysis

### Files
- `scripts/agents/execute.sh` (EDIT) — emit execution signals at start/end
- `scripts/session/detect-scope-drift.sh` (CREATE) — check files vs target_repos
- `comprehensive-learning/SKILL.md` (EDIT) — document execution analytics phase
