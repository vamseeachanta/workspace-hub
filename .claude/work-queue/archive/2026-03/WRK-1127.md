## Mission

Upgrade the work-queue / WRK lifecycle workflow so that large efforts are planned as features first, then broken into correctly-sized AI-agent chunks before any items enter the queue.

## What

Three concrete improvements to project management practice:

1. **AI-agent chunk sizing** — define a heuristic (e.g. max context budget, max file changes, max stage count) for splitting work into units that fit inside a single agent context window.
2. **Feature → work-item decomposition** — introduce a "feature" grouping layer above individual WRK items so related items are planned and tracked together (e.g. a `feature/` prefix, a feature spec, and a parent WRK referencing children).
3. **Pre-queue feature planning** — require a feature-level plan (spec + acceptance criteria + decomposition) to be approved before child WRK items are created and queued.

## Why

Currently large pieces of work are captured as single WRK items or as ad-hoc batches, which leads to:
- Context overflows mid-execution (agent loses coherence)
- Partial implementations when scope is underestimated
- Replanning during execution rather than before it

## Acceptance Criteria

- [ ] A documented chunk-sizing heuristic exists (in a skill or rule file)
- [ ] A feature-planning template exists (spec + decomposition section)
- [ ] work-queue SKILL.md or work-queue-workflow SKILL.md references the feature layer
- [ ] At least one example feature is walked through the new flow end-to-end
- [ ] No existing WRK items or scripts are broken
