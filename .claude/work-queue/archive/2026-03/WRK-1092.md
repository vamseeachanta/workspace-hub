# Type coverage ratchet — mypy error count ratchet

## Mission

Type safety can only improve over time — mypy error counts per repo must never increase.

## What

Modelled on the coverage ratchet (WRK-1067):

1. **Baseline capture** — run `mypy` per repo, record error count in `config/quality/mypy-baseline.yaml`
2. **Ratchet check script** — `scripts/quality/check_mypy_ratchet.py`: reads baseline, runs mypy, fails if error count increased
3. **Pre-push gate integration** — add mypy ratchet check to existing pre-push hook (alongside coverage ratchet)
4. **Auto-baseline update** — when error count decreases, update baseline automatically (ratchet moves only forward)
5. **`check-all.sh` integration** — add `--mypy-ratchet` flag

## Why

`check-all.sh --mypy-only` runs mypy but doesn't enforce a ratchet. Type errors can accumulate silently. The ratchet makes type improvement a one-way gate.

## Acceptance Criteria

- [ ] `config/quality/mypy-baseline.yaml` — error counts per repo
- [ ] `scripts/quality/check_mypy_ratchet.py` — ratchet logic (same pattern as `check_coverage_ratchet.py`)
- [ ] Pre-push hook extended with mypy ratchet check
- [ ] `check-all.sh --mypy-ratchet` flag
- [ ] ≥5 TDD tests (baseline read/write, error count comparison, ratchet pass/fail, auto-update)
- [ ] Passes `check-all.sh` for the script itself
