## Mission
Eliminate the single-point-of-failure risk of dev-primary holding all workspace
skills, memory, queue state, and configs by establishing an automated off-machine
backup that runs without human intervention.

## What
- Identify what to back up (everything not in remote git):
  - `~/.claude/projects/` — auto-memory across all projects
  - `.claude/work-queue/assets/` — WRK evidence packages (large; not pushed to remote)
  - `.claude/state/` — active WRK, session locks
  - `config/` — machine configs, pricing, env-parity
  - `logs/` — session and orchestrator logs
- Backup script: `scripts/operations/backup-workspace.sh`
  - rsync to dev-secondary (`/mnt/workspace-hub-backup/`) over SSH
  - Incremental (rsync `--link-dest` for space efficiency)
  - Exclude: `.git/`, `__pycache__/`, `*.pyc`, `site/` (generated)
  - Log: append completion + size delta to `logs/backup/backup.log`
- Schedule: daily cron at 04:00, dev-primary → dev-secondary
- Recovery doc: `.claude/docs/workspace-recovery.md` — step-by-step to restore
  from dev-secondary backup onto a fresh machine
- Test restore: dry-run restore to `/tmp/workspace-restore-test/` to verify backup
  integrity after first run

## Why
dev-primary is the primary engineering workstation. A disk failure would destroy:
500+ WRK evidence packages, 200+ auto-memory entries, session logs used by
comprehensive-learning, and all workspace config. The git remote holds code but
not operational state. rsync to dev-secondary (already SSH-accessible) costs
nothing and takes <2 minutes daily.

## Plan

Route A (simple) — inline plan. Updated after Codex cross-review (MINOR findings addressed).

1. **Write `scripts/operations/backup-workspace.sh`**
   - Timestamped snapshots: `dev-secondary:/mnt/workspace-hub-backup/snapshots/YYYY-MM-DD_HHMMSS/`
   - `--link-dest` pointing at last successful snapshot (space-efficient incremental)
   - rsync into a `_in-progress/` staging dir; rename to final timestamp only on success
   - Lock file (`/tmp/backup-workspace.lock`) to prevent overlapping cron runs
   - Excludes: `.git/`, `__pycache__/`, `*.pyc`, `site/`, `*.pyc`
   - Logs size delta + timestamp + exit status to `logs/backup/backup.log`
   - Retention: keep 14 daily + 8 weekly + 6 monthly; prune after each run
   - `--dry-run` flag for restore verification without rsync

2. **Add daily cron entry** at 04:00 on dev-primary (verified: no conflict with existing jobs)

3. **Write `.claude/docs/workspace-recovery.md`**
   - Bootstrap prerequisites (SSH trust, packages)
   - Restore ordering + ownership/permission verification
   - Dry-run validation + post-restore smoke test

4. **Run backup** — verify first run completes; dry-run restore to `/tmp/workspace-restore-test/`

5. **Codex cross-review** of the script; fix any findings

Test strategy: `--dry-run` first, then live run, then dry-run restore.
