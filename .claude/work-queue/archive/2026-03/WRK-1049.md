# work-queue: prevent concurrent session claim collision

## Mission
**Prevent two terminals from processing the same pending WRK item simultaneously.**

## What

**Bug observed 2026-03-08**: Two Claude sessions both picked up WRK-1037.
- Session 1 (this session): running stages 1–4 in the 20-stage lifecycle (item in pending/)
- Session 2 (second terminal): ran `/work run` → old `process.md` action → `mv pending/ → working/`
- Result: WRK-1037 in BOTH pending/ and working/ simultaneously (working/ had status: pending)

**Root causes**:
1. **`process.md` skill action is stale** — has its own `Auto-Claim` (`mv pending/ → working/`)
   that predates the 20-stage lifecycle. Any `/work run` invocation triggers it.
2. **No pre-claim session lock** — stages 1–7 run in pending/ with no ownership marker.
   A second terminal sees the same item as unclaimed.
3. **`active-wrk` not checked** — `.claude/state/active-wrk` isn't validated before
   starting new work.

## Fix

**P1 — Retire `process.md` auto-claim block**: Remove or replace Step 4 (`Auto-Claim`)
in `.claude/skills/coordination/workspace/work-queue/actions/process.md` with a hard
redirect to `claim-item.sh`. The old mv logic must not run in parallel with the
20-stage lifecycle.

**P2 — Early session lock in assets/WRK-NNN/**: Stage 1 (`start_stage.py`) writes a
`assets/WRK-NNN/session-lock.yaml` with `session_pid`, `started_at`, `hostname`.
`claim-item.sh` checks for this file before proceeding — if it exists and the process
is still alive, block claim with a warning.

**P3 — `claim-item.sh` working/ pre-check**: Before claiming, check if WRK-NNN.md
already exists in working/ → exit 1 with "already claimed by another session".

## Acceptance Criteria
- [ ] `process.md` Step 4 replaced with: "Run `claim-item.sh WRK-NNN` — do not mv manually"
- [ ] `start_stage.py` Stage 1 writes `assets/WRK-NNN/session-lock.yaml`
- [ ] `claim-item.sh` checks working/ pre-existence before mv; exits 1 if found
- [ ] Test: two concurrent claim-item.sh calls → only first succeeds, second exits 1
