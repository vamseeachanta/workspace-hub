# Documentation currency — drift detector

## Mission

Detect when documentation has fallen out of sync with the code it describes, and surface those gaps automatically as WRK items.

## What

1. **Public API snapshot** — for each repo, capture the set of public symbols (modules, classes, functions) using the AST symbol index (WRK-1085)
2. **Docs coverage check** — for each public symbol, check if it appears in docs/ or README.md
3. **Drift score** — `(undocumented_public_symbols / total_public_symbols)` per repo; track over time
4. **Staleness detection** — if a symbol's docstring or docs entry hasn't changed but the implementation has (via git blame diff), flag as potentially stale
5. **Nightly report** — `logs/quality/doc-drift-{date}.yaml`; drift score increase → auto-WRK
6. **Integration with `check-all.sh --docs`** — extend existing docs flag with drift score output

## Why

WRK-1058 added docs quality checks (ruff D rules, README section check). WRK-1075 built MkDocs pipeline. But neither detects drift between code changes and existing docs. Public APIs change; docs don't always follow.

## Acceptance Criteria

- [ ] `scripts/quality/check_doc_drift.py` — computes drift score per repo
- [ ] Reads symbol index from WRK-1085 (`config/search/symbol-index.jsonl`)
- [ ] `config/quality/doc-drift-baseline.yaml` — baseline drift scores per repo
- [ ] Drift score increase → auto-WRK capture
- [ ] Nightly cron entry in `crontab-template.sh`
- [ ] ≥5 TDD tests (symbol extraction, coverage check, staleness detection, drift score, auto-WRK)
- [ ] Passes `check-all.sh` for the script itself

## Plan

### Phase 1: TDD — Write tests first
- `tests/quality/test_check_doc_drift.py` with ≥8 tests (cross-review-expanded):
  1. `test_load_symbol_index_reads_jsonl` — reads symbol-index.jsonl, returns list of dicts
  2. `test_load_symbol_index_missing_file_exits_cleanly` — absent file → empty list + warning (no traceback)
  3. `test_build_doc_mention_set_whole_word_only` — substring matches NOT counted (false-positive guard)
  4. `test_compute_drift_score_all_documented` — score = 0.0 when all symbols in docs
  5. `test_compute_drift_score_none_documented` — score = 1.0 when no symbols in docs
  6. `test_batch_git_modified_files_returns_set` — single git log call, returns set of file paths
  7. `test_detect_staleness_uses_batch_not_per_file` — uses pre-batched set; no subprocess per symbol
  8. `test_format_drift_candidates_returns_strings_not_auto_capture` — returns list[str] for human review; never writes to work-queue
- All tests RED before implementation

### Phase 2: Implement `scripts/quality/check_doc_drift.py`
- `load_symbol_index(path)` → list of symbol dicts; empty list + warning if file absent
- `build_doc_mention_set(repo_path)` → set of names (whole-word grep, token-boundary, over docs/ + README.md)
- `compute_drift_score(symbols, doc_mentions, repo)` → float (undocumented_ratio: 0.0=good)
- `batch_git_modified_files(repo_path)` → set[str] via single `git log --diff-filter=M --name-only`; cached per repo
- `detect_staleness(file_path, modified_files, docs_path)` → bool using pre-batched set
- `run_drift_check(repos, symbol_index_path, baseline_path)` → report dict
- `format_drift_candidates(report, baseline)` → list[str] for human review (copy-paste into `/work add`)
- CLI: `uv run --no-project python scripts/quality/check_doc_drift.py [--repo <name>] [--update-baseline]`

### Phase 3: Baseline + integration
- `config/quality/doc-drift-baseline.yaml` — run script once, save initial scores
- Extend `check-all.sh`: add `--drift` flag → calls `check_doc_drift.py`; warn-only (non-blocking)
- `logs/quality/` directory (gitignored except `.gitkeep`)

### Phase 4: Cron
- Add nightly entry to `scripts/cron/crontab-template.sh` (02:30 dev-primary)
- Nightly output: `logs/quality/doc-drift-{date}.yaml` + candidate WRK strings printed to stdout for human triage

### Test Strategy
- Unit tests use fixture `symbol-index.jsonl` (3-5 entries) + temp README.md
- Staleness test uses `tmp_path` + `git init` + dummy commit history
