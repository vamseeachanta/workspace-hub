## Mission
A single command shows the health of every workspace-hub repo at a glance:
git state, branch, last commit, and last test result — so engineering work starts
with a clear situational picture.

## What
- Script: `scripts/repo-health.sh` — iterates all submodule repos + hub
- Per repo row: branch | ahead/behind | dirty (Y/N) | last commit date | last
  test result (pass/fail/unknown, sourced from last run-all-tests.sh output)
- Highlights: red = dirty or test fail, yellow = unknown test state, green = clean
- Output: colour-coded table to terminal + optional `--json` for machine consumption
- Integrate into `/today` report as a collapsible section

## Why
Engineering sessions frequently start by manually running `git status` across 6+
repos. A health dashboard surfaces stale branches, uncommitted changes, and test
regressions instantly — preventing wasted context on a broken baseline.

## Acceptance Criteria
- [ ] `scripts/repo-health.sh` covers all submodule repos and hub
- [ ] Shows: branch, dirty flag, ahead/behind origin, last commit, last test result
- [ ] Colour output (red/yellow/green) in terminal; `--json` flag for scripting
- [ ] Integrates last test result from WRK-1054 output (graceful if absent)
- [ ] Hook or alias makes it easy to run at session start
- [ ] Cross-review (Codex) passes
