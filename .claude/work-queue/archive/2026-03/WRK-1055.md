## Mission
Identify, install, and configure MCP servers that give AI agents direct access to
engineering reference data — standards, literature, material properties, codes — so
agents stop relying on training-time knowledge for domain facts.

## What
- Audit WRK-578 findings (MCP registry scan for engineering-domain servers)
- Install Semantic Scholar MCP (WRK-579) as first server
- Identify additional high-value servers: materials databases, structural codes,
  regulatory APIs (e.g. BSEE, IMO), geospatial/GIS data
- Add each to `config/ai-tools/mcp-servers.yaml` with: name, install command,
  use-case, status (active|pending|evaluated)
- Wire active servers into Claude/Codex configs on dev-primary

## Why
Engineering WRKs frequently need domain facts (material properties, code clause
references, well data). Agents currently use training knowledge — unreliable and
unverifiable. MCP servers provide authoritative, citeable sources.

## Acceptance Criteria
- [ ] `config/ai-tools/mcp-servers.yaml` created with all evaluated servers
- [ ] Semantic Scholar MCP installed and reachable from Claude on dev-primary
- [ ] At least one additional engineering-domain MCP identified and evaluated
- [ ] Usage pattern documented in `.claude/docs/mcp-servers.md`
- [ ] Cross-review (Codex) passes
