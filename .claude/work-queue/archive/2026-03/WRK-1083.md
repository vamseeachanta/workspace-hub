## Mission

Identify which work-queue-workflow stages benefit from EnterPlanMode, define activation contracts, and add a planning skill to automate plan-mode entry at those stages.

## What

Analyze the 20-stage WRK lifecycle and determine which stages should automatically invoke `EnterPlanMode` before generating artifacts. Create a `planning` skill (or extend an existing skill) that codifies when and how plan mode is entered, and update stage contracts / the `work-queue-workflow` skill to reference it.

## Why

Plan mode prevents file writes during deliberative stages (e.g., Stage 4 plan draft, Stage 6 cross-review synthesis), ensuring the agent thinks through the full approach before committing anything to disk. Currently, activation is ad-hoc and depends on agent discipline — automating it raises plan quality consistency.

## Acceptance Criteria

- [ ] Stages that benefit from plan mode are enumerated with rationale
- [ ] A planning skill (`superpowers:writing-plans` extension or new `work-queue:plan-mode` skill) is created or updated with explicit triggers for those stages
- [ ] `work-queue-workflow/SKILL.md` references the planning skill in the relevant stage contracts
- [ ] At least one stage contract YAML (`stage-04-*.yaml` or similar) has `plan_mode: required` annotation
- [ ] Existing `superpowers:writing-plans` skill is reviewed; if adequate, simply wire it in rather than creating a new skill
