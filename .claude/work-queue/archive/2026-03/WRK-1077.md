## Mission
Set up acma-ansys05 (orcaflex-license-machine) so it passes workspace-hub
readiness checks and can participate in the work queue.

## What
Manually execute the workspace-hub setup steps on the Windows/Git Bash machine:
1. Pull latest repo (includes the new `acma-ansys05.yml` manifest)
2. `bash scripts/setup/install-all-hooks.sh`
3. Add bashrc-snippets to `~/.bash_profile`
4. `claude config set statusBarEnabled true`
5. Copy `config/claude/keybindings.json` → `~/.claude/keybindings.json`
6. `bash scripts/setup/verify-setup.sh` — resolve any FAIL items
7. `bash scripts/operations/system/dev-env-check.sh` — confirm manifest validates

## Why
Machine has claude, codex, gemini installed (all confirmed) but no workspace-hub
harness setup. Needed before any WRK items can be routed to it.

## Acceptance Criteria
- [ ] `verify-setup.sh` reports 0 FAIL (WARNs acceptable for cron/SSH)
- [ ] `dev-env-check.sh` finds and reads `acma-ansys05.yml` manifest
- [ ] Git hooks installed (pre-commit, post-merge, post-rewrite)
- [ ] Shell aliases (`ws`, `wrk`, `wh-verify`) available in Git Bash

## Plan

Route A — manual execution runbook for acma-ansys05 (Windows Git Bash).
Run all commands from Git Bash. Each command is self-contained (includes `cd`).

1. **Pull latest**:
   ```bash
   cd /c/workspace-hub && git pull
   ```
   Ensures `acma-ansys05.yml` manifest is present.

2. **Install hooks**:
   ```bash
   cd /c/workspace-hub && bash scripts/setup/install-all-hooks.sh
   ```
   Copies pre-commit / post-merge / post-rewrite to `.git/hooks/`.

3. **Add bashrc snippet** (one-time, skip if already present):
   ```bash
   grep -qF "bashrc-snippets" ~/.bash_profile 2>/dev/null \
     || echo 'source /c/workspace-hub/config/shell/bashrc-snippets.sh' >> ~/.bash_profile
   source ~/.bash_profile
   type ws wrk wh-verify   # must resolve — confirms aliases active
   ```
   Uses `~/.bash_profile` (Git Bash on Windows, not `.bashrc`).

4. **Claude statusline**:
   ```bash
   claude config set statusBarEnabled true
   ```

5. **Keybindings**:
   ```bash
   cp /c/workspace-hub/config/claude/keybindings.json ~/.claude/keybindings.json
   ```

6. **Verify**:
   ```bash
   cd /c/workspace-hub && bash scripts/setup/verify-setup.sh
   ```
   Target: 0 FAIL. Cron/SSH WARNs are expected on Windows — acceptable.

7. **Manifest check**:
   ```bash
   cd /c/workspace-hub && bash scripts/operations/system/dev-env-check.sh
   ```
   `acma-ansys05.yml` must be found/read. Hostname must match exactly
   (lowercase `acma-ansys05`) — if `hostname` returns uppercase, run
   `hostname | tr '[:upper:]' '[:lower:]'` and verify manifest filename matches.

No code changes needed. All scripts and manifest already exist in repo.

## Notes
- No SSH access — must be done physically on the machine
- Windows Git Bash (MINGW64) — use `bash`, not PowerShell
- Workspace path: `C:\workspace-hub` (or wherever cloned)
- Cron WARN in verify-setup is expected on Windows — ignore
