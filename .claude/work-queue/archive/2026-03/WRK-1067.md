## Mission
Make the 80% coverage minimum in testing.md verifiable and enforced, so coverage
regressions are caught at push time rather than discovered in code review.

## What
- Extend `scripts/testing/run-all-tests.sh` with `--coverage` flag: adds
  `--cov=src --cov-report=term-missing --cov-fail-under=80` per repo
- Coverage report: per-repo summary written to
  `scripts/testing/coverage-reports/WRK-NNN-coverage-YYYYMMDD.txt`
- Baseline capture: run once to establish current coverage per repo; store in
  `config/testing/coverage-baseline.yaml` (repo → current %)
- Pre-push hook (WRK-1064): add `--coverage` to gate; block if any repo drops
  below its baseline by >2% (ratchet — baseline only moves up)
- Known-low modules: allow per-module exemptions in `coverage-baseline.yaml`
  with mandatory reason comment (e.g. generated code, legacy untested)

## Why
testing.md mandates 80% coverage but no tooling verifies it. New WRK implementations
can silently regress coverage. A ratchet gate means coverage trends upward over time
as engineering WRKs complete TDD cycles.

## Acceptance Criteria
- [ ] `run-all-tests.sh --coverage` generates per-repo coverage report
- [ ] `coverage-baseline.yaml` populated with current coverage per repo
- [ ] Pre-push hook blocks if repo drops >2% below baseline
- [ ] Exemptions supported in baseline yaml with required reason field
- [ ] Coverage reports stored with WRK reference for audit trail
- [ ] Cross-review (Codex) passes
