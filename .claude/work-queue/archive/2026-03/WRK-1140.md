# feat(automation): Nightly Cron Job — Release-Notes Scan and WRK Capture

## Mission

Run a nightly cron job that checks for new Claude Code (and optionally Codex /
Gemini) releases, analyses release notes for ecosystem-applicable improvements,
and auto-captures WRK items for actionable findings.

## What

- A nightly cron (or `/loop`-based) job that:
  1. Fetches latest CC release notes (`claude /release-notes` or equivalent)
  2. Compares against last-seen version stored in `config/ai-tools/`
  3. If new versions exist, runs analysis (via Claude) against ecosystem context
  4. Auto-creates WRK pending items for actionable improvements found
  5. Updates last-seen version marker
- Optionally extends to Codex and Gemini release feeds in later iterations

## Why

- Removes manual `/work add <release notes>` step from user workflow
- Ensures improvements are captured promptly rather than accumulating
- Consistent analysis quality — same prompt/context every run

## Acceptance Criteria

- [ ] `scripts/automation/nightly-release-scan.sh` created and executable
- [ ] Last-seen version persisted in `config/ai-tools/release-scan-state.yaml`
- [ ] New WRK items auto-created in `pending/` when new versions detected
- [ ] Cron entry documented in `config/automation/cron-schedule.md` (or equivalent)
- [ ] Dry-run mode (`--dry-run`) prints findings without writing WRK files
- [ ] Script idempotent — re-running on same version produces no duplicate items
- [ ] At least one successful nightly run verified in logs

## Plan

1. Design `config/ai-tools/release-scan-state.yaml` schema (last_seen_version per provider)
2. Write `scripts/automation/nightly-release-scan.sh`:
   - Fetch release notes → diff against last-seen
   - Pipe new content to Claude with ecosystem-analysis prompt
   - Parse output → write WRK pending file(s) via `next-id.sh` + template
   - Update state file
3. Add cron entry (or `/loop` invocation) for nightly execution
4. Write tests: idempotency, dry-run, no-new-versions path
5. Document in `config/automation/`
