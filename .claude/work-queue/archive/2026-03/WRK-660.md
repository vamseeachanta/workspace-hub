# analysis(assetutilities): Test Coverage Gap Audit — 5 Untested Modules

## What

Five assetutilities modules have coverage gaps (confirmed by baseline run):
- `calculations`: most files at 0% (casing, riser, wellhead, pipeline, etc.); scr_fatigue at 39%
- `devtools`: 0% across all 5 files (cli, modernize_deps, organize_structure, propagate_commands)
- `base_configs`: 0% (nested modules: csv, excel, yaml, web, etc.)
- `tools`: 0% (git/, pdf/ subdirs)
- `units`: already 82–100% covered via tests/unit/ — focus on residual gaps only

Use Gemini to read source + coverage gaps and produce targeted, executable test stubs.

## Plan (v2 — post cross-review)

### Phase 0 — Coverage baseline (deterministic gap discovery)

```bash
cd assetutilities
uv run python -m pytest --cov=assetutilities --cov-report=term-missing \
  2>&1 | grep -E "base_configs|calculations|devtools|tools|units" \
  > specs/wrk/WRK-660/coverage-baseline.txt
```

Baseline already captured: devtools=0%, most calculations=0%, units=82-100%.

### Phase 1 — Gemini test plan from gap list

```bash
{
  echo "=== COVERAGE GAPS ==="
  cat specs/wrk/WRK-660/coverage-baseline.txt
  echo "=== SOURCE FILES ==="
  find src/assetutilities/calculations src/assetutilities/devtools \
       src/assetutilities/base_configs src/assetutilities/tools \
       -name "*.py" | sort | xargs cat
  echo "=== EXISTING TEST PATTERN REFERENCE ==="
  find tests/unit/ -name "test_*.py" | head -3 | xargs cat
} | gemini -p "
For each module with coverage gaps (calculations, devtools, base_configs, tools):
1. List every public function/class at 0% coverage that needs a test
2. For each, write a complete pytest test function (not a stub — must have at least one assert):
   def test_<function>_<scenario>_<expected>():
3. Include at least one happy path, one edge case, and one error case per function
4. Output as ready-to-paste pytest code blocks, one section per module
5. Use the reference test pattern for imports and fixture style" -y \
> specs/wrk/WRK-660/test-coverage-gap-plan.md
```

### Phase 2 — Overlap validation

```bash
grep -rhoP '(?<=def )test_\w+' tests/ --include="*.py" | sort > /tmp/existing_tests.txt
grep -hoP '(?<=def )test_\w+' specs/wrk/WRK-660/test-coverage-gap-plan.md | \
  sort > /tmp/proposed_tests.txt
comm -13 /tmp/existing_tests.txt /tmp/proposed_tests.txt
# Output = proposed-only names. Empty output = no collisions.
```

### Phase 3 — Scaffold test files and verify execution

Flat test files (matching existing tests/ pattern):
```
tests/modules/calculations/test_calculations_coverage.py
tests/modules/devtools/test_devtools_coverage.py
tests/modules/base_configs/test_base_configs_coverage.py
tests/modules/tools/test_tools_coverage.py
```

```bash
uv run python -m pytest tests/modules/calculations/ tests/modules/devtools/ \
  tests/modules/base_configs/ tests/modules/tools/ -v
# Must exit 0. All tests must have real assertions, not `pass`.
```

Re-run coverage after scaffold — each module must show increase vs baseline.
