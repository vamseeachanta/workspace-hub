## Mission
Surface async failures (nightly cron, CI gate, benchmarks) at session start so they
are never silently missed — engineering work should not begin on a broken baseline.

## What
- Notification log: `logs/notifications/YYYY-MM-DD.jsonl` — each async job writes
  a completion event:
  `{"source":"cron|ci|benchmark","job":"<name>","status":"pass|fail","ts":"...","details":"..."}`
- Writer shim: `scripts/notify.sh <source> <job> <status> [details]` — appended by
  cron scripts, pre-push hook, and benchmark runner on completion
- Session-start reader: `scripts/hooks/session-start-notify.sh` — reads last 24h
  of notification log; prints a summary banner if any failures:
  ```
  ⚠  2 async failures since last session:
     FAIL  cron/nightly-tests  2026-03-09 02:14  assethold: 3 new failures
     FAIL  benchmark           2026-03-09 03:01  digitalmodel: +35% regression
  ```
- Retention: notifications older than 7 days purged automatically
- Integration points: nightly test cron, pre-push CI hook (WRK-1064), benchmark
  runner (WRK-1071) each call `notify.sh` on exit

## Why
Cron failures and benchmark regressions currently go unnoticed until someone
manually checks logs. An agent starting an engineering session on a broken baseline
produces incorrect results and wastes the entire session. A 3-line session-start
banner prevents this.

## Plan

Route A — simple execution.

1. **`scripts/notify.sh`** — writer shim
   - Args: `<source> <job> <status> [details]`
   - Appends `{"source":...,"job":...,"status":...,"ts":"<ISO>","details":"..."}` to `logs/notifications/$(date +%Y-%m-%d).jsonl`
   - Creates `logs/notifications/` if absent
   - Exit 0 always (non-blocking side-effect)

2. **`scripts/hooks/session-start-notify.sh`** — reader
   - Reads all `logs/notifications/` files from last 24h
   - Counts FAIL entries; if >0 prints banner to stderr listing each failure
   - Completely silent (no output) when zero failures

3. **Wire `notify.sh` into nightly cron** (`comprehensive-learning-nightly.sh`)
   - Append `bash scripts/notify.sh cron nightly-learning "$exit_status"` before final exit

4. **7-day retention** — add purge line to `crontab-template.sh`
   - `find logs/notifications/ -name "*.jsonl" -mtime +7 -delete`

5. **Placeholder comments** in `scripts/notify.sh` header for WRK-1064/1071 wiring

6. **Tests** (`scripts/hooks/tests/test-session-start-notify.sh`)
   - Test 1: no notifications dir → silent
   - Test 2: only PASS entries → silent
   - Test 3: one FAIL entry → banner printed to stderr
   - Test 4: mixed PASS/FAIL → banner lists only FAILs

## Acceptance Criteria
- [ ] `notify.sh` appends structured JSONL to notification log
- [ ] `session-start-notify.sh` reads last 24h and prints failure banner if any
- [ ] Banner suppressed (no output) when all jobs pass — zero noise on clean runs
- [ ] Nightly test cron wired to call `notify.sh` on completion
- [ ] Benchmark runner (WRK-1071) wired to call `notify.sh` on regression
- [ ] Pre-push hook (WRK-1064) wired to call `notify.sh` on block
- [ ] 7-day retention purge runs in nightly cron
