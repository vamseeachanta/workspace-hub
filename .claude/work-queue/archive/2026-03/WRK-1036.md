# Agent teams lifecycle — audit logs, session-start availability, session-exit tidy

## Mission
**Audit stale agent teams, make teams readily available at session-start, and add session-exit tidy-up so ghost teams and orphaned task dirs never accumulate again.**

## What

Three phases:

**Phase 1 — Audit (one-time)**
- 3 stale named teams (`doc-intel-wave2`, `engineering-standards`, `wrk-272-b401`, last active Feb 20–27 2026)
- 76 orphaned UUID task dirs under `~/.claude/tasks/`
- Read inboxes + task files to reconstruct what each team attempted/completed/abandoned
- Capture any unrecorded follow-up as new WRK items; then clean up

**Phase 2 — Session-start availability**
- Add an `agent-teams` section to `session-start` SKILL.md so every new session knows:
  - Teams feature is active (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`, `MAX_TEAMMATES=10`)
  - How to spawn a team for the current WRK (one-liner recipe)
  - Naming convention: `wrk-NNN-<slug>` so teams are traceable
- Add a helper script `scripts/work-queue/spawn-team.sh WRK-NNN <slug>` that creates the team, sets the name, and prints the spawn recipe — zero-friction team creation

**Phase 3 — Session-exit tidy**
- Extend the existing Stop hook (`emit-session-quality-signals.sh` or a new `tidy-agent-teams.sh`) to:
  1. Detect any teams whose WRK is now archived/closed → `TeamDelete` them
  2. Detect task UUID dirs older than 7 days with no active team reference → purge
  3. Emit a one-line summary signal: `agent_teams_tidied: N_deleted, M_tasks_purged`
- Keep the hook fast (<5s): use shell+jq only, no Python

## Why
Agent teams are wired up but effectively invisible at session start — leading to underuse and ghost accumulation. A lightweight lifecycle (spawn recipe at start + auto-tidy at exit) closes the loop and makes teams a first-class execution tool instead of an afterthought.
