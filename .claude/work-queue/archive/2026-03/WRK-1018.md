# feat(comprehensive-learning): Phase 6 surfaces unqueued future-work.yaml recommendations as WRK candidates

## Mission
**Scan all recently-archived WRK assets for `future-work.yaml` entries with `captured: false` (or `captured: true` but no corresponding WRK file) and add them to Phase 7 ActionCandidates so they enter the queue on the next nightly run.**

## What

`future-work.yaml` (Stage 15) captures follow-up ideas discovered during WRK execution. Currently nothing reads these files after archiving — items sit dormant forever. This feature wires them into the nightly `comprehensive-learning` Phase 6 (WRKFeedback) so unqueued recommendations automatically become WRK candidates.

## Why

Stage 15 is only useful if its outputs feed back into the queue. Without this loop, future-work.yaml is a write-only artifact — effort spent capturing insights that never become action.

## What to Build

1. **Phase 6 enhancement** in `comprehensive-learning/SKILL.md`:
   - After existing WRKFeedback analysis, scan `assets/*/evidence/future-work.yaml` files
   - Filter to items where `captured: false` OR (`captured: true` but no WRK file exists with matching title)
   - Add qualifying items to the Phase 7 ActionCandidates list with source attribution

2. **Script** (optional): `scripts/work-queue/scan-future-work.py` — standalone script that prints unqueued future-work items; can be called from Phase 6 or run manually

3. **Lookback window**: only scan `future-work.yaml` files modified in the last 30 days (avoids re-surfacing stale items repeatedly)

## Acceptance Criteria

- [ ] `comprehensive-learning/SKILL.md` Phase 6 step added: scan `assets/*/evidence/future-work.yaml` for unqueued items
- [ ] Items with `captured: false` are surfaced as Phase 7 ActionCandidates on the next nightly run
- [ ] Items already queued (WRK file exists) are not re-surfaced (deduplication)
- [ ] 30-day lookback window applied
- [ ] `scan-future-work.py` script (or inline logic) handles missing/malformed yaml gracefully

## Agentic AI Horizon

- Closes the Stage 15 → queue feedback loop that WRK-1015 opened.
- As more WRKs are archived with future-work.yaml, the nightly surfacing compounds — the queue self-feeds from its own execution history.
