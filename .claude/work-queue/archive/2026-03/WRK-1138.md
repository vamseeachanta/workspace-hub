# fix(whats-next): Guard Pending Scan Against Archived Ghost Items

## Mission

Prevent archived WRK items with stale `pending/` copies from surfacing in
`whats-next.sh` and `claim-item.sh`.

## What

9 WRK items (WRK-691, WRK-1019, WRK-207, WRK-208, WRK-230, WRK-259, WRK-310,
WRK-570, WRK-624) appeared as "HIGH PRIORITY — ready to start" in `/whats-next`
despite being fully archived. Root cause: `archive-item.sh` removes the
`working/` copy but a separate `pending/` copy can exist (e.g. from an earlier
capture before the item was claimed). Neither `whats-next.sh` nor `claim-item.sh`
cross-checks `archive/` before surfacing an item.

## Why

Ghost items pollute the work queue view and trigger confusing errors when
`/work run NNN` is called on an already-completed item.

## Acceptance Criteria

- [x] `archive-item.sh` sweeps all queue dirs (working/pending/blocked/done) and
      removes every copy — prevents ghost creation at the source
- [x] `whats-next.sh` `process_file()` skips any `pending/` item whose ID is
      already in `archive/` (defensive layer)
- [x] `claim-item.sh` early-exits with a clear "already archived" message if the
      WRK is in `archive/` (defensive layer)
- [x] `scripts/work-queue/scan-ghost-pending.sh` created — reports (and with
      `--fix` removes) pending files already in archive; exits 1 if any found
- [x] `tests/scripts/test_ghost_pending.sh` passes 9/9
- [x] Manual run of `scan-ghost-pending.sh` on current queue returns 0 ghosts

## Plan

1. Add `is_archived` guard in `whats-next.sh` `process_file()` — 3-line change
2. Add early-exit in `claim-item.sh` before Stage 5 check — 5-line change
3. Create `scripts/work-queue/scan-ghost-pending.sh` (detect + `--fix` mode)
4. Write `tests/unit/test_ghost_pending.sh`
