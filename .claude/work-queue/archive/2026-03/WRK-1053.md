# chore(skills): convert deterministic skill loops to scripts

## Mission

**Replace LLM-prose deterministic loops in skills with callable scripts, moving
from non-deterministic AI judgment to repeatable, verifiable execution.**

## Background

WRK-1013 established the structural principle: *skills trim → references/ picks up*.
The next layer of the same principle is: *skills orchestrate → scripts execute*.

The `patterns.md` hard rule already states:
> "When a script exists for a decision, gate check, or enforcement action → call the script.
> Never substitute LLM reasoning where a verifiable, repeatable script can run."

WRK-1013 applied this to *documentation* (prose → references/). This WRK applies it to
*execution* (LLM loops → shell/python scripts).

## What

Audit all workspace-hub skills for deterministic loops — places where a skill instructs
Claude to iterate over files, scan directories, count matches, compare values, or enforce
rules — and replace with scripts that Claude calls and reads results from.

### Candidate patterns (discovered during WRK-1013 session)

| Skill | Deterministic loop | Candidate script |
|-------|--------------------|-----------------|
| `work-queue` | Queue dir scanning for item counts, category tallies | `scripts/work-queue/queue-status.sh` (exists — verify skill calls it) |
| `comprehensive-learning` | Phase 9 skill coverage audit — scanning .claude/skills/ | `scripts/skills/skill-coverage-audit.sh` (new) |
| `session-start` | Drift-risk files, active-wrk state, harness readiness | `scripts/session/detect-drift.sh` (exists), `nightly-readiness.sh` (exists) — verify skill calls these |
| `resource-intelligence` | Resource pack YAML field validation | `scripts/work-queue/validate-resource-pack.sh` (exists — verify skill calls it) |
| `verify:skill-eval` | README.md presence + word count scanning | `scripts/skills/audit-skill-violations.sh` (new) |
| `workflow-gatepass` | Gate evidence checking across 30 fields | `scripts/work-queue/verify-gate-evidence.py` (exists — skill must call script, never prose) |
| `comprehensive-learning` Phase 6 | Ecosystem health — grepping logs, counting signals | `scripts/cron/ecosystem-health-check.sh` (exists — verify Phase 6 calls it) |

### New scripts to create

1. **`scripts/skills/audit-skill-violations.sh`** — replaces ad-hoc find/wc commands;
   outputs structured YAML of violations (README.md present, SKILL.md >5000w, description
   >1024 chars, XML tags in body); exit 0 = clean, exit 1 = violations found; callable
   from `verify:skill-eval`, WRK-1013-style audits, and CI.

2. **`scripts/skills/skill-coverage-audit.sh`** — implements comprehensive-learning Phase 9;
   maps WRK item categories to available skills; outputs coverage gaps as YAML;
   cron-safe (silent when clean, emits gaps to candidates/).

### Existing scripts to wire (verify and update skill references)

3. `scripts/session/detect-drift.sh` — session-start Step 0 calls this, not prose re-derivation
4. `scripts/work-queue/validate-resource-pack.sh` — resource-intelligence skill calls at Stage 2 entry
5. `scripts/work-queue/verify-gate-evidence.py` — workflow-gatepass references as single gate truth

## Why

- **Repeatability**: script runs identically every time; LLM loop varies by session
- **Speed**: `find + wc` = milliseconds; Claude scanning 500 skill files = tokens + time
- **Verifiability**: exit codes are machine-readable; LLM prose is not
- **Context efficiency**: script output is compact; inline scanning fills context
- **Self-improvement**: once scripted, violations trigger CI/cron alerts automatically

## Agentic AI Horizon

As Claude capability grows, deterministic scripts become more valuable — they provide
ground truth the model cannot hallucinate. Skills-as-orchestrators + scripts-as-executors
is the workspace-hub analog of tool-use in agentic systems.

## Plan

### Phase 1 — Create new scripts (Route B execution depth)

**1a. `scripts/skills/audit-skill-violations.sh`**
- Input: optional `--skill-dir` (default `.claude/skills/`)
- Checks per SKILL.md: README.md present, word count ≤5000, description ≤1024 chars, no XML tags in body
- Output: YAML to stdout; exit 0 = clean, exit 1 = violations
- Tests: happy path (clean skill dir), violation detected (oversized), missing input dir

**1b. `scripts/skills/skill-coverage-audit.sh`**
- Input: optional `--skills-dir`, `--queue-dir`
- Maps WRK category values → skill subdirs; counts how many have a script call vs. prose-only
- Output: per-skill coverage YAML + gap list; cron-safe (silent when clean)
- Tests: full coverage, partial coverage (gap found), empty skills dir

### Phase 2 — Wire existing scripts into skills

**2a. session-start** → call `detect-drift.sh --log <session-log>` at Step 0 (replace prose drift check)
**2b. work-queue** → verify `queue-status.sh` is called (add reference where missing)
**2c. resource-intelligence** → fix hub-root path alias for `validate-resource-pack.sh`
**2d. skill-eval** → add calls to `audit-skill-violations.sh` + `validate-skills.sh`
**2e. comprehensive-learning** → define Phase 9 calling `identify-script-candidates.sh` + `skill-coverage-audit.sh`

### Phase 3 — TDD / Eval

- Run each new script's 3 tests (happy path, violation, missing input)
- Verify wired skills emit correct script calls (grep-check SKILL.md for script names)
- Run `scripts/work-queue/verify-gate-evidence.py WRK-1053`

### Test Strategy

| Script | Tests | Method |
|--------|-------|--------|
| `audit-skill-violations.sh` | 3 | Bash test with fixture dirs |
| `skill-coverage-audit.sh` | 3 | Bash test with fixture queue/skills |
| Skill wiring | Grep-check | `grep -l <script>` in updated SKILL.md |
