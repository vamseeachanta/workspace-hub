# O*NET Task Tagging for WRK Capture

## Mission
**Add an optional `onet_category` field to WRK frontmatter and a tagging script so we can benchmark our automation rates against Anthropic's O*NET occupational exposure data.**

## What
- Add `onet_category` optional field to WRK frontmatter schema (e.g. `15-1251` CS programmers, `43-9021` data entry)
- Script `scripts/ai/tag-onet.sh <WRK-NNN>` suggests top-3 O*NET codes given WRK title/mission
- `observed-exposure-report.py` (WRK-1128) groups by `onet_category` when present
- Produces category distribution comparable to Anthropic's occupation exposure table

## Why
Tagging with O*NET lets us answer: "For tasks in category X, what % are we actually automating vs what the Anthropic research says is theoretically possible?" Closes the loop on WRK-1128.

## Acceptance Criteria
- [ ] `onet_category` field documented in work-queue SKILL.md (optional, O*NET code + label)
- [ ] `scripts/ai/tag-onet.sh <WRK-NNN>` reads WRK mission and suggests top-3 O*NET codes
- [ ] `observed-exposure-report.py` groups by `onet_category` when field is present
- [ ] ≥2 TDD tests (code format validation, report grouping)
- [ ] Blocked by WRK-1128
