## Mission

Use historical skill execution data to weight skill routing decisions.

## What

Build `scripts/skills/skill-preference-weights.py` that:
1. Reads `skill-executions.jsonl`
2. Computes success rate per skill-task-pattern pair
3. Outputs `skill-preferences.yaml` with weighted scores
4. Integrates with `task_classifier.sh` to prefer higher-success skills

## Why

From WRK-5085 research (cognee pattern): historical success rate per skill-task-pattern pair informs better routing. Skills that consistently succeed on certain task types should be preferred over those that don't.

## Acceptance Criteria

- [x] Script computes frequency-based weights from execution history
- [x] Output: skill-preferences.yaml with (skill, invocation_count, unique_sessions, avg_duration_s, weight)
- [x] Minimum sample threshold (n >= 5) before weighting applies
- [x] TDD tests — 12 passing with synthetic execution data
