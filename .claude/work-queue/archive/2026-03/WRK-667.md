# feat(work-queue): Strengthen Resource Intelligence Skill with Measurable Quality Impact

## Mission

Extend the existing resource-intelligence skill with measurable quality metrics, an HTML
summary block, validator checks for required RI refs, and at least 3 before/after
comparison examples — demonstrating that RI increases execution strength, not ceremony.

## What

Harden the `resource-intelligence` stage into a repeatable, skill-driven, validator-checkable process with explicit quality metrics and evidence outputs.

## Why

User gap review on WRK-624 requested revision on Resource Intelligence: confirm skill presence and demonstrate that it increases planning/execution strength instead of adding ceremony.

## Acceptance Criteria

- [ ] Confirm and document the canonical `resource-intelligence` skill usage path in work-queue flow docs
- [ ] Define measurable impact metrics (minimum):
  - [ ] plan rework reduction
  - [ ] missing-artifact reduction at claim/close
  - [ ] cross-review blocker reduction attributable to resource gaps
- [ ] Add a standard per-WRK artifact schema for resource intelligence outputs
- [ ] Add validator checks for required resource intelligence refs on new/touched WRKs
- [ ] Add HTML summary block showing resource intelligence findings and confidence
- [ ] Produce at least 3 comparison examples (before/after) demonstrating quality lift
- [ ] Cross-review with Claude, Codex, Gemini and synthesize verdict

## Plan

### Phase 1 — Document Canonical Usage Path (AC 1)
1. Add `## Resource Intelligence Usage` section to work-queue SKILL.md (or a companion doc)
   mapping the flow: Stage 2 runs RI → evidence/resource-intelligence.yaml → verifier gate
2. Update `stage-02-resource-intelligence.yaml` entry_reads/exit_artifacts references
   to reflect the current SKILL.md v1.1.0 expectations

### Phase 2 — Quality Metrics Schema (AC 2 + 3)
3. Extend `resource-intelligence.yaml` SKILL.md template with `quality_signals` block:
   ```yaml
   quality_signals:
     plan_changes_after_ri: 0        # plan rework count post-Stage-2
     artifacts_gaps_prevented: []    # artifacts that would have been missing
     cross_review_blockers_prevented: []  # blocker labels resolved early via RI
     confidence: high | medium | low
   ```
4. Add `quality_signals` to `templates/resource-intelligence-template.yaml` (optional field)

### Phase 3 — Validator Checks (AC 4)
5. Extend `validate-resource-pack.sh` to check:
   - `evidence/resource-intelligence.yaml` exists with `completion_status` set
   - `skills.core_used` list has ≥ 3 entries
   - `resource_pack_ref` is set in WRK frontmatter (warn if missing)
6. Add `check_resource_pack_ref_gate` to `verify-gate-evidence.py` (warn-only first)

### Phase 4 — HTML Summary Block (AC 5)
7. Add RI summary section to `generate-html-review.py --lifecycle`:
   - Show: completion_status, core skills used, top_p1 gaps, top_p2 gaps, confidence
   - Use green/amber/red chip for completion_status
8. Update `WRK-667-lifecycle.html` to include RI section

### Phase 5 — Comparison Examples (AC 6)
9. Select 3 pre-RI WRKs (e.g. WRK-610, WRK-617, WRK-600) and 3 post-RI WRKs
   (e.g. WRK-690, WRK-1003, WRK-1028)
10. Write `evidence/ri-comparison-examples.md` for each pair:
    - Before (no RI): cross-review blockers, plan rework events, missing artifacts at close
    - After (with RI): same metrics, plus RI evidence quality_signals

### Phase 6 — Cross-Review (AC 7)
11. Write `scripts/review/results/wrk-667-phase-1-review-input.md`
12. Submit: `scripts/review/cross-review.sh <file> all`
13. Synthesize verdict; fix MAJOR findings
