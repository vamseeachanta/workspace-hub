# chore(harness): Micro-Skill → Script Extraction Audit

## Mission

After WRK-1142 fleshes out the 20 stage micro-skills, audit every checklist item across
all stage micro-skills and identify which items can be promoted to deterministic scripts —
moving rules from Level 1 (prose in micro-skill) to Level 2 (script) on the enforcement
gradient defined in `.claude/rules/patterns.md`.

## Context

The enforcement gradient (`patterns.md`) defines 4 levels:
- Level 0: prose in monolithic SKILL.md (context-rot prone)
- Level 1: micro-skill auto-loaded at stage entry (guaranteed context, LLM interprets)
- Level 2: script called from micro-skill or CI (binary, auditable)
- Level 3: hook (fires automatically, cannot be silently skipped)

WRK-1142 gets us from Level 0 → Level 1. This WRK identifies candidates for Level 2/3.

## What

### 1. Audit script (`scripts/work-queue/audit-micro-skill-scripts.py`)

Read all `stages/stage-NN-*.md` files, extract checklist items (lines starting with `- [ ]`
or numbered steps), and classify each as:
- `scriptable`: binary check expressible as exit 0/1 (e.g., "verify gate evidence exists")
- `judgment`: requires LLM interpretation (e.g., "assess if plan is complete")
- `already-scripted`: step already calls an existing script

Output: YAML report at `.claude/work-queue/assets/WRK-1144/micro-skill-script-candidates.yaml`

### 2. Review and prioritize candidates

For each `scriptable` item:
- Estimate script complexity (trivial/small/medium)
- Check if an existing script already covers it
- Mark high-priority candidates (appear in 3+ stages, or are in hard-gate stages 1/5/7/17)

### 3. Create WRK items for top candidates

For each high-priority script candidate not already covered:
- Create a WRK item to implement the script
- Set `blocked_by: [WRK-1144]`

### 4. Promote hook candidates

Items that should fire on every session exit or commit → flag for hook promotion (WRK-1141
already covers post-commit; new items go into a new WRK or WRK-1141 extension).

## Acceptance Criteria

- [ ] `audit-micro-skill-scripts.py` runs and produces YAML report
- [ ] Report classifies all checklist items across all 20 stage micro-skills
- [ ] ≥3 high-priority script candidates identified and documented
- [ ] WRK items created for top candidates
- [ ] Report committed to assets/

## Plan

1. Write `audit-micro-skill-scripts.py` (TDD: test with synthetic micro-skill fixture)
2. Run against all 20 stage micro-skills post WRK-1142
3. Review report, identify top candidates
4. Create child WRK items for scripts to build
5. Commit report + WRK items
