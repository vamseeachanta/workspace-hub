### Run 2: dev-secondary (2026-03-17)

- Switched from pdfplumber to `pdftotext` (poppler) — 37x faster per file
- pdfplumber in multiprocessing hung in D (disk sleep) on NTFS I/O; SIGALRM cannot
  interrupt kernel-level I/O. `subprocess.run(timeout=30)` with pdftotext solved this.
- 8-worker multiprocessing pool, chunksize=50, ~49 files/sec sustained
- Path remapping: `/mnt/remote/dev-secondary/dde/` → `/mnt/dde/` (local NTFS),
  `/mnt/ace/` → `/mnt/remote/dev-primary/ace/` (NFS to dev-primary)
- Total runtime: 1.7h for 296,626 PDFs
- Index copied back to dev-primary via scp, verified matching
