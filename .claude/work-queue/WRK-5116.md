## Problem

ace-linux-2 uses SSH (`ssh ace-linux-1 "cd ... && git ..."`) for all workspace-hub operations because `/mnt/workspace-hub` is an SSHFS mount where git commands hang. This adds latency and fragility.

## Goal

Wire up the default location on ace-linux-2 so all operations (git, file edits, tool calls) work directly without SSH indirection. All operations should be direct.

## Possible Approaches

- Use the local NTFS copy at `/mnt/local-analysis/workspace-hub` as the primary workspace on ace-linux-2
- Set up a local git clone with ace-linux-1 as remote
- Configure git to work reliably over SSHFS (unlikely but worth investigating)
- Bidirectional sync (rsync/unison) between local copy and ace-linux-1
