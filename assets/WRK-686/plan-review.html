standardization(ux): define and enforce canonical HTML templates for plan and review artifacts

Objective

Standardize the HTML format for plan reviews and final implementation summaries to ensure a consistent experience for end users across all AI agents (Claude, Codex, Gemini).

Context

Currently, different agents and tasks produce HTML artifacts with varying styles, headers, and structures. This lack of uniformity makes it harder for users to quickly scan and approve work. We need a single, "workspace-hub" branded CSS and structure that all agents must adopt.

Plan

  1. Inventory Styles: Collect existing HTML artifacts from WRK-669, WRK-670, WRK-671, and WRK-682.
  2. Define CSS Baseline: Create a shared CSS file (e.g., assets/shared/orchestrator-standard.css) with a professional, consistent look (fonts, colors, spacing).
  3. Create Template Generator: Update or consolidate scripts/work-queue/generate-html-review.py to use a single Jinja2 or similar template that enforces the standard layout.
  4. Refactor Agents: Update agent instructions in AGENTS.md and .claude/rules/ to mandate the use of the standardized generator or template.

Acceptance Criteria

  • [ ] A canonical CSS file exists and is used by all new HTML artifacts.
  • [ ] Plan reviews for Claude, Codex, and Gemini look identical in structure and styling.
  • [ ] Implementation summaries (Review Synthesis) follow a standard, scannable format.
